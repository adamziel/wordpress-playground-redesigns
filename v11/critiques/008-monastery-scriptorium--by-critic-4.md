# Critique — 008-monastery-scriptorium — by Critic 4

## Verdict
Major revisions. The idiom is gorgeous on paper, the Atkinson-Hyperlegible/Cormorant-Garamond pairing is correct, and the gilt restraint is real — but the desktop screenshot shows the rail capitals are barely active-distinguishable, the quill flourish is too faint to register, several annotation overlays sit on top of the iframe headline, and a footstrip below the iframe carries explainer-text helpers that the brief expressly bans.

## Hard violations of v11 taste constraints
- [ ] **Annotation overlays are everywhere.** `.anno` is an absolutely-positioned chip with a gilt 5×5 square and a `border:1px solid var(--gilt-edge)`, and three of them appear over Panel A alone — "capsule · 6 affordances", "gilt only on Saved-pill + active M's fill", "active capital · quill flourish curling around the awake letter". The third overlaps the iframe content's main headline in the desktop screenshot. These violate constraint #10 (simplicity), and the gilt borders on the annotation chips violate constraint #7 (color proliferation — gilt was supposed to live in *two* places, not on every annotation chip). Remove all `.anno` elements from inside `.desktop`; if you need a caption, put it in the existing `panel-cap` below the frame.
- [ ] **Explainer chrome in the footstrip.** `.footstrip` at the bottom of the iframe area reads `Press F for focus mode · ⌘K for command palette` and `Quiet rail · 5 docks · 1 swap pane`. The first half is a "tip / shortcut" floater explicitly banned by the §"Hard taste constraints" checklist; the second half is the README leaking onto the chrome. Both go.
- [ ] **Tab strip inside the Manager pane.** `<div class="tabs">Settings | Files | Blueprint | Database | Logs</div>` (panel C, panel D, etc.) re-creates the 5-dock navigation as in-pane tabs. The locked model says each Site Manager surface **is its own rail dock**; you've added a secondary nav layer inside the pane that selects between the same five things. This is two navigation models for one set of destinations — drop the tabs. The active rail dock already says you are in Manager > Settings; the pane head should just show "Manager · Settings" (which it does), no tab bar needed.
- [ ] `IBM Plex Mono` and `Source Code Pro` are referenced in `.code-editor` font-family but neither is loaded via `<link>`. The browser will fall back to system mono — functionally fine because mono is allowed in `.code-editor`, but the design loses its declared face. Either add `IBM Plex Mono` to the Google Fonts URL, or accept the system fallback and remove the unloaded family names from the stack.
- [ ] `font-family:inherit` on the `.apply-block .actions .cancel` / `.confirm` buttons (which inherit from `.apply-block` which inherits `Atkinson Hyperlegible` from `body`) is fine, but the typed-row `<input>` is set to `Cormorant Garamond italic` for a destructive-confirmation field. That's antique-instrument flavour bleeding into a security-critical input — the user types "reset coffee shop" and the field renders it in italic display serif. Switch the input to Atkinson Hyperlegible to keep the confirmation visually honest.

## Annotation overlays / explainer chrome
- [ ] Six `.anno` instances across panels A, B, C — every one is designer commentary inside the chrome. Delete the class and its three references, or move them to the existing `panel-cap` row underneath each desktop.
- [ ] The `.footstrip` strip is itself an explainer surface (last-save time + plugin count + keyboard shortcut + design label). The corner widget already carries site identity; the keyboard shortcut and the "Quiet rail · 5 docks · 1 swap pane" are redundant. Either kill the footstrip entirely or reduce it to one line ("last save 14:32:11") and drop the shortcut hint.

## Rail button distinctness (Forest-Botanical test)
- The five capitals S, M, B, D, L are set in Cormorant Garamond 28 px — visually distinct as letterforms, but their silhouettes are similar (all roughly square serifed glyphs of comparable optical mass). The Forest-Botanical reference uses *different shape vocabularies* per dock (woodblock vs. monogram vs. caption number). A column of five serif letters reads as "five items of the same kind" — the user remembers which is active by *colour* (gilt vs. ink-secondary), not by silhouette. That's still valid (you do have the quill flourish, and the README owns this tradeoff), but say so in the README more honestly: the rail is a typographic index, not a five-silhouettes-distinct treatment.
- (a) **Is the active capital truly gilt?** Yes — verified in CSS (`.dock.active .cap{color:var(--gilt)}` where `--gilt: #B3893D`) and visible in the screenshot as warm amber-gold, distinct from the ink-secondary `#7C7058` of the inactive capitals. Good.
- (b) **Is the quill flourish visible at this scale?** Barely. The SVG strokes are `stroke-width:0.7` at `opacity:.85` on a 54×54 cell with `inset:-3px`. In the screenshot the flourish reads as faint tickmarks at the top-left and bottom-right of the active M, not as a *curl wrapping the letter*. Either thicken to `stroke-width:1.2` and bring opacity to 1, or extend the curl arcs further around the letter so the wrap reads. As is, the README's "single quill flourish curls around the awake letter" oversells what the screen delivers.

## Iframe primacy
- The iframe is sharp and the active gilt M is the loudest chromatic event — that part works. But the `panel-label` chip in Cormorant Garamond italic at the top-left of the desktop, plus the two `.anno` chips with gilt borders sitting over the chrome, plus the gilt 5×5 squares inside each `.sec-head` (and the apply-block `h4` and the corner widget dot), pull the gilt count well past the "twice" the README claims. Gilt currently appears in: active capital, save-pill, anno-chip squares, sec-head squares (5+ in any pane), apply-block heading square, corner widget dot, save-pill saved-state, library card "num", category active underline, mobile rail dot. That is at least 10 gilt placements, not two. Cut all the `.sec-head::before` squares and `.apply-block h4::before` squares — let the type hierarchy carry the section heads. Drop the corner-widget dot. That brings gilt down to two/three meaningful uses (active capital + saved-pill + maybe the destructive Apply CTA).

## Typography sanity
- (c) **Atkinson Hyperlegible loaded and rendering?** Yes — verified in `<link>` (Google Fonts URL includes Atkinson Hyperlegible 400/700). Body text in the screenshot ("A hand-illuminated manuscript at candlelight…") shows the characteristic open apertures and high-x-height of Atkinson — it is rendering correctly, not falling back to serif. Good.
- Cormorant Garamond italic at 14 px (`.panel-label`, `.corner .nm`, `.bp-mini .num`, etc.) is on the small side for a display serif and goes slightly fuzzy on Mac sub-pixel rendering, but legible. Acceptable.
- (d) **Annotation overlay severity** — see above; the `.anno` chips are the biggest typography-adjacent visual noise on the chrome.

## Clarity & focus test
- Fails in current form. The hero (Cormorant Garamond `M`onastery Scriptorium drop cap) competes with the chrome's `M`anager / `S`ites display drops; the annotation chips intercept the eye before the iframe; the footstrip adds a fourth horizontal band beneath the iframe. After the cleanup above, the test would pass: rail (five capitals), capsule, single pane, iframe — four objects, clearly located.

## Suggested concrete fixes (Round 2)
1. Delete the `.anno` class and all six instances. Move their content into the existing `panel-cap` beneath each desktop.
2. Delete `.footstrip` entirely (or reduce to a single neutral line). Keyboard hints belong in a `⌘K` palette, not painted under the iframe.
3. Remove the in-pane `.tabs` strip from Site Manager panels — rely on the rail dock + pane title to communicate location.
4. Strip gilt from `.sec-head::before` and `.apply-block h4::before`. Drop the gilt dot on the corner widget. Keep gilt on: active rail capital, Saved save-pill, primary Apply CTA. Three uses, period.
5. Thicken the quill flourish (`stroke-width:1.2`, opacity 1) and extend the curl arcs further around the letter so "flourish wraps awake letter" actually reads at 54 × 54.
6. Switch the typed-confirmation input font from Cormorant italic to Atkinson Hyperlegible — destructive inputs should not be decorative.
7. Either add IBM Plex Mono to the `<link>` or remove unloaded family names from the editor `font-family` stack.
