# 077 — Block-editor styled (Gutenberg-inside-Gutenberg)

## Thesis
The Playground's own chrome IS the block editor. The running WordPress iframe is treated as a single "Site Block" being edited inside Gutenberg: it has a selection outline, a floating block toolbar (refresh, address, transform, more), an "Inserter" slide-out on the left that holds every way to start a new playground (categories + search, exactly like the block inserter pattern), and a tabbed Inspector panel on the right that exposes Settings / Files / Blueprint / Database / Logs the way Gutenberg's sidebar exposes Block / Document / Styles. The top bar mirrors the editor's: app icon, undo/redo replaced by Refresh, document title (the playground name), view options, save status, and the Publish-equivalent (Save). The intent: a WordPress user who has ever used Gutenberg knows where every Playground feature lives within ten seconds.

## What was remixed, merged, removed
- **Merged** the address bar, refresh and the additional-actions menu into a single contextual Block Toolbar that floats above the selected Site Block — same pattern as the rich-text block toolbar.
- **Merged** the "Saved Playgrounds" launcher and the full 43-blueprint gallery into one Inserter panel with a sticky search, category chips (All / Featured / Website / Personal / Content / Themes / Gutenberg / Experiments / WooCommerce / News), a "Blocks" tab for blueprints, a "Patterns" tab for the six "Start a new Playground" sources (Vanilla, WordPress PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip), and a "Saved" tab for "Your Playgrounds" — same structure Gutenberg uses for blocks/patterns/media.
- **Merged** the quick settings popover and the Site Manager Settings tab into the Inspector's "Settings" panel (the destructive Apply & Reset gets a real warning, not a small banner) — the popover trigger in the toolbar simply scrolls/opens the Inspector to that panel, so there is one source of truth.
- **Removed** the duplicate "Save site locally" CTA inside the Settings tab — the Save status badge in the top bar (idle / unsaved / saving / saved) replaces it, the way Gutenberg's "Saved" / "Save draft" indicator works.
- **Kept** every feature in §2: WP Admin / Homepage jump buttons live as "Block Navigation" pills above the canvas; File browser, Blueprint editor, Database, Logs are Inspector tabs; Export to GitHub PR and Download .zip live in the "More options" kebab on the Block Toolbar, just like a block's More menu.

## Trade-offs
This metaphor will delight Gutenberg-fluent users and disorient anyone who has never seen the block editor — it leans hard on a single mental model. The floating block toolbar costs vertical space above the iframe; on mobile it collapses into a single overflow button, which means the address bar requires one extra tap. Theming the chrome to look like Gutenberg means visual updates to the real Gutenberg in the future may require us to follow along to keep the illusion. And because a real block editor has only one selected block at a time, multi-playground workflows (open two side-by-side) are not expressible — but the current Playground doesn't support that anyway, so the metaphor doesn't actually cost a feature.
