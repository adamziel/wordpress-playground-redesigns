# Playground Workspace — Notion-like

## Thesis
A WordPress Playground reimagined as a **Notion-style workspace**. The chrome is not a toolbar — it's a **nested page tree** on the left. Each playground is a top-level page; under it sit sub-pages for *Settings*, *Files*, *Blueprint*, *Database*, *Logs*. Below the workspace are library pages: *New Site* (the launcher), *Blueprints* (gallery), and *Imports*. The center pane is the page body. There is no floating toolbar — every action is a block inside the page (a title with inline editing, a callout, a property strip, a slash-menu, breadcrumbs with overflow actions). The iframe itself becomes a "live preview" block embedded in the *Overview* page of a site.

## What I remixed, merged, and removed
**Merged**: The top toolbar dissolved entirely. Refresh, address bar, save, and the playground name now live as inline blocks at the top of the *Overview* page (a renameable H1, a URL property, a save-state callout, a refresh button on the embedded iframe block). Settings collapsed from three entry points into a single *Settings* sub-page (the popover is gone; the Site-Manager Settings tab and the inline "Save site locally" notice are the same page now). The Site Manager sidebar is replaced by the always-visible page tree. The Saved-Playgrounds drawer is replaced by the *New Site* and *Your Playgrounds* tree section. **Remixed**: The blueprint gallery is a database view (Notion's gallery view) with a filter pill bar and search input at the top of the page. Files + Blueprint are sibling sub-pages so they no longer fight. The "Additional actions" menu becomes a `···` overflow next to the page title like Notion. **Removed**: nothing — every §2 feature has a home in the tree. Even mobile preserves it via a slide-in tree drawer and a bottom "+" sheet.

## Trade-offs
The page-tree-as-IA is incredible for power users (everything reachable in two clicks, deep linking, keyboard navigable) but the first-touch user lands on a UI that looks more like a productivity app than a sandbox — a small "Live preview" hint helps. Putting actions inside the page means destructive Apply-and-Reset lives below the fold of a long Settings page; I added an inline danger callout to compensate. The browser-tab-like top is gone, so switching between sites is one tree click instead of one tab click — same speed but the affordance is less obvious to non-Notion users. Mobile loses the always-visible tree (slides over the content) which is a tradeoff every Notion-shaped app makes.
