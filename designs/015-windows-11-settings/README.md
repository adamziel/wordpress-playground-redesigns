# Design #015 — Windows 11 Settings

## Thesis
Playground re-imagined as the Windows 11 Settings.app. The whole chrome adopts Fluent / Mica visual language: a soft Mica gradient backdrop, frosted rounded cards, pill-shaped category and filter buttons, breadcrumbs at the top of every page, oversized whitespace, and a left rail of categories with little tinted square glyphs. The thesis is that "managing a Playground" is conceptually the same as "managing a Windows session" — fast switching between Live Site, Settings, Files, Blueprint, Database and Logs categories, each with a quiet, scrolling content pane. The Refresh and Address bar live in a single pill-shaped URL toolbar (like the Windows 11 search/command bar), the Save action is a colored pill, and the quick-settings popover is a Fluent flyout in the top-right.

## What I remixed / merged / removed
The Site Manager's five tabs are dissolved into the Settings.app sidebar categories (Settings, Files, Blueprint, Database, Logs) — there is no separate "open / close Site Manager" because the manager *is* the app. The "Live Site" category holds the iframe along with the URL bar, quick-version chips, and quick-settings popover, so toolbar and viewport become one breathing page. The "Saved Playgrounds" launcher is promoted to a full-screen Mica overlay (like Windows 11's "All apps") containing the three required sections — Start a new Playground (Vanilla / WP PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip), the 43-blueprint gallery with the full pill-shaped category filter row plus search, and Your Playgrounds. The Additional actions menu (Export to GitHub PR, Download .zip) lives at the bottom of the sidebar as its own row, treated like any other Settings entry. WP Admin and Homepage become icon-only buttons in the URL pill, since they are just navigation shortcuts. A dedicated "Boot status" pane demonstrates the loading state in Fluent style.

## Trade-offs
Going all-in on the Settings.app metaphor means the chrome is calmer than the current Playground UI but also more horizontally split — the live iframe shares vertical space with the URL pill and a quick-stats strip, which is great for orientation but eats pixels you might prefer to give to WordPress. Promoting the launcher to a modal overlay (rather than a side drawer) hides the iframe momentarily, but earns room to show the full 43-blueprint gallery and Your Playgrounds together. The Mica/glass look needs a backdrop-filter-capable browser to feel right; on browsers without it the gradient still reads but loses some of the depth. Finally, the sidebar's nine categories are great on desktop but get collapsed into a horizontal pill scroller on mobile — every feature is reachable, but two-step navigation (open launcher, then pick) is more taps than the current drawer.
