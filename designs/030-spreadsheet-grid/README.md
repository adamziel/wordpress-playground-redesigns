# 030 — Excel / spreadsheet

## Thesis
WordPress Playground is reframed as a data-grid application. Every blueprint and every saved playground is a **row**; their configuration (WP version, PHP, language, network, multisite, plugins, themes, tags, category, last-opened, status) are **columns**. The "address bar," "settings popover," and "saved playgrounds launcher" all collapse into the universal spreadsheet idioms users already know: a formula bar at the top (the URL/path), a ribbon of column/sort/filter/group controls, a tabbed status bar at the bottom (Settings · Files · Blueprint · Database · Logs), and a **preview cell** on the right that hosts the live WordPress iframe for the selected row. Selecting a row swaps the iframe; double-clicking boots it; right-clicking gives the additional-actions menu (Export to GitHub PR, Download .zip, Import .zip, etc.). Filtering and grouping make the 43-blueprint catalog feel as queryable as a real dataset instead of a sea of cards.

## What I remixed / merged / removed
- **Merged** the launcher, blueprint gallery, and "Your Playgrounds" into a single sheet (`Sheet1: Playgrounds`) where the **Source** column distinguishes a saved site from a featured blueprint from a "WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip" entry — the "start new" actions become the bottom **+ New row** affordance (a small inline picker), exactly like inserting a row.
- **Merged** the three settings entry points (popover, sidebar tab, info-notice button) into a single **Format → Cell** sidebar that opens when you click the WP/PHP/Language/Network/Multisite column header or use the ribbon's "Format" button. Apply + Reset live in a destructive footer with a confirm.
- **Merged** the Site Manager tabs into the **status bar tabs** at the bottom of the workbook (Settings · Files · Blueprint · Database · Logs) — same metaphor as Excel sheet tabs.
- **Removed** redundant chrome: the right-side launcher drawer, the floating popover, the duplicated "Save" affordance. Save becomes Ctrl+S on a row (and a ribbon button). The marketing landing page inside the iframe is replaced by an empty-state spreadsheet welcome.
- **Remixed** category filters into native **column filters** (click the funnel in the Category header) and **groups** (Group by → Category, WP version, PHP, etc., yielding collapsible group rows).

## Trade-offs
A spreadsheet metaphor is dense and unapologetically pro-user. New WordPress users who land on `playground.wordpress.net` to "just try WordPress" might be intimidated by a grid of 43 rows — but the empty preview cell on first paint still shows a friendly "Pick a row to launch" callout and the **Quick Start** column (a single 🚀 button per row) gives one-click boot, so they don't have to learn the grid to use it. Power users — the bulk of Playground's audience (devs testing PRs, blueprint authors, theme/plugin demoers) — get sortable/filterable/groupable access to every property at once, plus genuine keyboard navigation. On mobile, the grid collapses to a card list with the "preview cell" becoming a full-screen iframe pane; sort/filter still work but groups flatten. The biggest cost is novelty: nothing else on the web treats a WP sandbox as a spreadsheet, so the first 10 seconds need to read as inviting rather than corporate. The visible iframe preview on the right (with row hover-to-preview) does that heavy lifting.
