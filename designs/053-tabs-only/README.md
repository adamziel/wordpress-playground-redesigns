# 053 — Tabs-only

## Thesis
WordPress Playground has too many places to live: a top toolbar, a left Site Manager sidebar with its own tabs, a right Saved Playgrounds drawer, a settings popover, and modal-ish things in between. This design collapses *every* panel onto a single horizontal tab strip directly above the iframe — Site, Settings, Files, Blueprint, Database, Logs, plus dynamic tabs for the Launcher and Blueprint Gallery. There are no sidebars and no drawers. Tabs are the only navigation primitive. Open as many as you want, close them, drag them; the active tab fills the entire content area. The mental model is a browser, applied recursively to a tool that already runs inside a browser.

## What I remixed / merged / removed and why
The Site Manager's tabbed sidebar is the seed of the idea, so I promoted those tabs to first-class status and merged them with the *top* toolbar that previously sat above them — one strip now does both jobs. The "Saved Playgrounds" right-side drawer becomes a Launcher tab that you open by clicking "+" on the tab strip; the Blueprint Gallery opens as its own sibling tab so search/filter has room to breathe. The Edit Settings popover is gone — there's already a Settings tab, three doors to the same room was the original sin. The address bar and Refresh button live on the Site tab's secondary bar (per-tab chrome, like a browser), while playground-wide actions (Save, Additional actions menu with Export PR / Download .zip, Site Manager toggle which is now a no-op because the tabs ARE the site manager) live on a slim global bar above the tab strip. WP Admin / Homepage are quick-jumps on the Site tab.

## Trade-offs
Tab strips get crowded fast — if you open Launcher + Gallery + Files + Blueprint + Logs you've burned half your horizontal pixels. I mitigate this with icon-only tabs at narrow widths and an overflow chevron, but a 6th-or-7th-tab user is going to feel it. Mobile collapses the strip into a horizontal-scrolling chip rail, which works but loses the spatial parallelism that makes the desktop layout legible. And by killing the right-side launcher drawer I trade glanceability ("see the iframe and your saved sites at once") for focus ("one panel at a time, fullscreen"). For a tool whose entire job is *running WordPress in a window*, full-bleed beats peripheral vision — but power users juggling a blueprint edit and the live preview will miss the split.
