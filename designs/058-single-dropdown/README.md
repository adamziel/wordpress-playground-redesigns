# 058 — Single dropdown

## Thesis
The browser already has chrome. WordPress already has chrome. Playground adds a third layer of chrome and that's the problem. This design takes the brief literally: there is no toolbar, no sidebar, no drawer, no popover, no toast region docked anywhere. There is the iframe, and there is one black square in the top-right corner labelled `MENU`. Click it and the entire feature surface unfurls as a single flat document — a brutalist directory of every action Playground can perform, written in Helvetica and IBM Plex Mono on cream paper, with 1px black borders and zero ornament. Every panel that used to be a separate room (Settings, Files, Blueprint, DB, Logs, Saved Playgrounds, Site Manager) is now a section heading inside that one document. You scroll the menu. There is nothing else to learn.

## What I remixed / merged / removed
**Removed:** the top toolbar, the right drawer, the left sidebar, the settings popover, the floating toast region as a separate UI, all icons, all colour, all border radius, all shadow, all hover animation. **Merged:** Site Manager's five tabs + the Saved Playgrounds launcher + the Settings popover + the Additional Actions menu + the toast region into one scrollable sheet. Categories in the blueprint browser became a sticky 1-row monospace filter strip; cards became a dense table because brutalism doesn't decorate, it lists. **Remixed:** the address bar lives at the very top of the dropdown (so navigation is a menu item like everything else); the destructive "Apply Settings & Reset" is rendered as a `[DESTRUCTIVE]` tag in caps rather than a red banner — typography does the warning. The "Unsaved Playground / Save" affordance, which used to be a small text button, is now a 64px-tall row at the top of the menu that you cannot miss because it owns the whole width.

## Trade-offs
One menu means one mental model — beginners love it, power users will resent the round-trip to the menu for every action. There are no shortcuts visible (a real ship would add `?` keymap; I left it out to stay honest to brutalism). The iframe gets 100% of the canvas, which is the goal, but means there's no persistent indicator of which playground you're in once the menu closes — I compensated with a permanent 1px label-strip along the very top edge showing `<NAME> · <URL>`, which is the minimum status I could justify. Mobile drops the strip entirely and the menu becomes the whole screen. Brutalism in 2026 is itself a fashion — it isn't neutral — but the brief asked for it.
