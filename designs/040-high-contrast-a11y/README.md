# 040 — High-contrast accessibility-first

## Thesis
Playground rebuilt as if WCAG AAA were the brief, not an afterthought. Every interactive element is at least 56×56 px, every label has a real label (no icon-only buttons), every primary action carries a visible keyboard shortcut in a bracketed badge, and focus rings are 4 px solid amber that never fade. The palette is pure black on near-white (#0B0B0B on #FAFAFA) with one accent (#1645AD blue) and one alarm (#A40000 red) — both checked at AAA against their backgrounds. Type is Atkinson Hyperlegible because it was literally designed for low-vision reading. Nothing relies on color alone: every state carries an icon, a text label, and a pattern.

## What I remixed / merged / removed
The three settings entry points collapse into one place: a tabbed Site Manager that *is* the chrome — the popover quick-settings is gone because a fully-keyboard-reachable tab list with persistent shortcut hints is faster than a hover popover. The toolbar uses real text buttons ("Refresh", "Save site", "Sites") rather than icon-only chrome — icons sit alongside text. The "Saved Playgrounds" launcher and the Blueprint gallery merge into one full-screen *Library* surface with a left-rail nav (New / Library / Your Playgrounds / Import) so a screen reader sees one landmark, not nested drawers. Destructive "Apply & Reset" gets a confirmation dialog with a typed-out warning and a separate hold-to-confirm pattern. The status of the running playground (loading / running / error) is always announced via a visible status strip *and* aria-live.

## Trade-offs
Visual density is intentionally low — large hit targets and persistent keyboard hint badges mean fewer features per pixel than a Figma-style canvas. Designers who want a polished, brand-forward marketing surface will find this stark. The single-accent palette restricts decorative differentiation between blueprints (no per-category color washes), so categories rely on labels and icons. The persistent kbd hints add ~24 px to every primary control; on mobile they collapse into a Shortcuts sheet to recover space. This is a tool, not a showpiece — but everyone can use it.
