# 073 — Restaurant-menu style

## Thesis
WordPress Playground reframed as a neighborhood bistro. The blueprint gallery becomes a printed menu — *Apéritifs* (quick starts), *Entrées* (themed sites), *Plats du Chef* (featured), *Sides* (small experiments), *Desserts* (fun extras), *Digestifs* (PR previews, beta). Every blueprint earns a short prose description like a real menu listing, and a **Sommelier** (a tiny rule-based helper, not AI) pairs blueprints to what the visitor came hungry for. The chrome around the iframe is the dining room: a maître d' header bar, a tableside Site Manager docket, and toasts arrive like the server announcing a dish. Warm cream paper, deep claret, gold accents — Playfair Display for menu copy, IBM Plex Mono for the technical "kitchen" surfaces (logs, blueprint JSON, file tree).

## Remixed / merged / removed
The three settings entry points (popover, Site Manager tab, info-notice) are unified into one "Edit my order" popover that the Site Manager Settings tab references rather than duplicates. The "Saved Playgrounds" launcher, the blueprint browser, and the six "start a new playground" entry points (Vanilla, WP PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip) all live inside the **Menu** overlay so visitors discover them together — these six are reframed as the *Apéritifs* section of the menu. "Your Playgrounds" becomes the *Regular's Tab* at the bottom of the menu. The Site Manager turns into a slim right-side **Bill / Docket** tile with tabs (Settings, Pantry = files, Recipe = blueprint, Cellar = database, Service Log = logs). The destructive "Apply Settings & Reset" gets a real confirmation language: "Send back to the kitchen — your dish will be remade from scratch." Mobile collapses the menu into a folded card and the docket into a bottom sheet.

## Trade-offs
The flavor metaphor risks feeling cute past the first visit — to mitigate, the menu copy is the *only* place the metaphor goes hard; technical surfaces (file browser, blueprint JSON editor, logs) stay flat, monospaced, and functional. Power users who want to skim 43 blueprints quickly may chafe at prose-heavy listings, so the menu also includes a compact search bar and category chips that match the brief's required filters. The "sommelier" is deliberately rule-based, not LLM-driven, so it stays inside the brief's "no new features" rule — it just filters the same 43 blueprints by tags inferred from three questions. Finally, the warm paper aesthetic is opinionated and might clash with WordPress's own admin chrome inside the iframe; the iframe is given a clean white inner mat to keep the boundary readable.
