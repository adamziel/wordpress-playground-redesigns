# 089 — AR/HUD military aesthetic

## Thesis
WordPress Playground reframed as a tactical heads-up display: the iframe is the "view through the canopy" and every piece of chrome around it is HUD glass — phosphor-green vector overlays, monospaced telemetry, targeting reticles in each corner, and bracket marks at the edge of the operating viewport. The four corners are permanent status readouts (WP version top-left, PHP/Engine top-right, mission name & save state bottom-left, live log tail bottom-right). Every panel that opens does so as a translucent over-glass module with bracket framing and a callsign in the header. The visual language is borrowed from aircraft MFDs and AR overlays: thin 1px lines, scanline texture, contained color palette (phosphor green / amber warning / red destructive / cool grey), all controls labeled with militarized callsigns ("BLUEPRINT ACQUISITION", "SITE OPS", "SYS PARAMETERS"). The result feels like operating WordPress from a cockpit.

## What I remixed / merged / removed
The three settings entry points are unified into a single SYS PARAMETERS module that opens over-glass — the popover, sidebar tab, and info-notice are the same surface invoked from different chips. The Site Manager's five tabs become a vertical "WEAPONS BAY" rail of tactical modules (SETTINGS / FILES / BLUEPRINT / DB / LOGS) that swap the right-hand panel, while the launcher is reframed as the BLUEPRINT ACQUISITION targeting view with featured tiles, full 43-strong gallery, category filter chips, and the "new mission" sources (Vanilla, WP PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip) collapsed into a "DEPLOY VECTORS" strip at the top of the same screen. The Logs corner readout is a live tail that mirrors the full Logs module — same data, two fidelities. The marketing welcome page inside the iframe is muted by the HUD overlay, which gives returning users a working dashboard immediately. Nothing was removed; the Additional Actions menu (Export to GitHub PR, Download .zip) lives as an "ORDNANCE" chip on the SITE OPS bar.

## Trade-offs
The aesthetic is divisive — playful for some, oppressive for others — and the green-on-black palette has accessibility concerns at low contrast that would need a "daylight" reskin in production. Monospace everywhere costs horizontal density, so long blueprint titles truncate aggressively. Militarized labels are fun but raise a translation/onboarding cost: a tooltip discipline would be needed. The persistent corner telemetry is great information density but adds visual noise around the iframe; on mobile the corners collapse to a bottom status bar and a hamburger drawer, sacrificing some of the cockpit feel for thumb reach. Finally, the scanline/grid textures are decorative and would need to be opt-out for users with motion sensitivity.
