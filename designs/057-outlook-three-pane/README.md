# 057 — Three-pane Outlook/Mail

## Thesis
Playground is reframed as an "inbox of things you can do." A Microsoft Outlook / Apple Mail style three-pane shell organises every Playground surface into the language of mail: a left **folder rail** ("Your Playgrounds", "Start new", "Blueprints", "Site Manager: Files / Blueprint / Database / Logs"), a center **list pane** that scrolls a uniform feed of items (saved sites, starters, the 43 blueprints, file rows, log lines, settings cards), and a right **reading pane** that previews whatever the user selected — most of the time, that's the live WordPress iframe with a tiny action ribbon above it. Selecting a log entry, blueprint, or file swaps the reading pane the same way reading a mail does. Everything is one verb away.

## What I remixed / merged / removed
The biggest merger: the **launcher drawer**, the **Site Manager sidebar**, and the **settings popover** all collapse into one folder list. "Start a new Playground" becomes a folder containing six conversation-style items (Vanilla, WordPress PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip). The blueprint gallery becomes a filterable mail-list with category chips that act like Outlook's Focused / Other tabs. The Site Manager's five tabs become five folders (Settings, Files, Blueprint, Database, Logs) — each opens a list of items rather than a custom UI. The **address bar / refresh / save / playground name / WP Admin / Homepage / Additional Actions** all live in a slim message-header ribbon directly above the reading pane (the equivalent of mail action buttons). The **settings popover** is preserved as a quick "compose"-style flyout on the toolbar for users who don't want to navigate to the Settings folder. Toasts dock to the bottom-right of the reading pane, like Outlook desktop notifications.

## Trade-offs
Forcing every surface into a list/reading-pane metaphor means some screens lose bespoke chrome — the Files folder uses a flat path-prefixed list rather than a true tree, and the Database tab becomes a list of tables you click to inspect. Power users may miss the always-visible Site Manager tab strip; here it's one click deeper (folder rail → folder). Conversely, anonymous first-touch users get a clearer mental model — "this is mail, click around" — than today's three-entrypoint UI. Mobile drops the reading pane to a stack: list pushes the iframe down into a sheet you can drag up, similar to Outlook iOS.
