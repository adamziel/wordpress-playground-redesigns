# 005 — Browser-tab metaphor

## Thesis
Playground stops pretending to be an app and starts behaving like a browser. Every saved Playground is a real tab in a Chrome-style tab strip; "New Tab" is "New Playground". The URL bar *is* the WordPress address bar (it routes inside the iframe, not to the public web). The bookmarks bar holds blueprints — five featured ones pinned, plus a chevron to the full 43-card gallery. The "New Tab" page is the launcher. DevTools-style bottom panels carry files, blueprint JSON, the SQLite DB browser and logs. Settings live behind a small chip near the URL bar, the way browsers tuck site settings into the padlock area. The whole design assumes the user already knows what a browser is, and uses that knowledge to make every Playground feature feel like a familiar control they already know how to operate.

## What was remixed, merged, removed
The three settings entry-points (toolbar popover, sidebar tab, info notice) collapse into **one** popover that drops out of a settings chip on the URL bar — the inline "save site locally" notice now lives at the top of that popover. The left-side Site Manager is dissolved: its WP-Admin / Homepage shortcuts become URL-bar quick-jumps, its Additional Actions menu becomes the browser's overflow `⋮` menu (Export to GitHub PR, Download .zip), and its file / blueprint / DB / logs tabs become a bottom DevTools drawer toggled with `F12`. The Saved Playgrounds drawer disappears: saved sites are just tabs; closed tabs and "Your Playgrounds" merge into a recently-closed-tabs menu plus a "Reopen Playground" tile on the New Tab page. The blueprint gallery is now the New Tab page's main grid, with category chips on the left and a search input that doubles as the omnibox in "blueprint search" mode.

## Trade-offs
The browser-inside-browser is the obvious hazard: real Chrome already has tabs, and a user could mistake Playground's strip for their own. We mitigate with a clearly different chrome tint, a Playground-branded address bar, and a dotted top-edge to read as "not your real browser". Power users may miss the dedicated left sidebar — the DevTools drawer is hidden by default and discoverable only via `F12` or the overflow menu, mirroring how nobody finds Chrome DevTools on day one. Tabs scale poorly past ~10 Playgrounds, so the strip overflows into a "Tabs" search dropdown (Chrome's tab-search popover) once it runs out of room. Mobile abandons the strip metaphor entirely and surfaces tabs as a fullscreen "tab switcher" sheet, the way mobile Safari does — accepting that the metaphor only fully holds on desktop.
