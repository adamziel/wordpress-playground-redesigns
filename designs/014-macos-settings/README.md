# 014 — macOS Settings.app (Ventura style)

## Thesis
WordPress Playground is reframed as a System Settings window, in the visual idiom of macOS Ventura's `Settings.app`. A search-topped sidebar enumerates every Playground surface as a category row — Live Preview, Saved Playgrounds, Blueprint Library, Address & Navigation, WordPress Version, PHP & Runtime, Files, Blueprint, Database, Logs, Export & Share, Import. Selecting a category swaps the right pane, the way `Network` or `Bluetooth` swaps when you click them in macOS. The iframe is no longer the entire UI — it's just one category: "Live Preview". This subtly demotes the marketing landing-page experience, and elevates the *configuration* of a Playground to first-class peer status with running it.

## What I remixed / merged / removed
The three overlapping settings surfaces in the current app (popover, sidebar tab, info-notice) collapse into a single canonical category — but split semantically into **WordPress Version** and **PHP & Runtime** rows so each setting has one home. The "Saved Playgrounds" launcher is decomposed: the saved sites list becomes its own row ("Saved Playgrounds"), the new-from-X options collapse into the "New Playground" toolbar pill at the top of that pane, and the 43-blueprint gallery becomes its own row ("Blueprint Library") with the macOS-style search and segmented category chips. File Browser, Blueprint editor, Database, and Logs each become rows that mirror the existing Site Manager tabs, but accessible without a modal. Export-to-GitHub-PR and Download-.zip live in a dedicated "Export & Share" row, so the "additional actions" menu disappears. The top window chrome keeps the traffic lights, name label, refresh and an address bar — exactly like a Finder window — so the toolbar is preserved but feels native.

## Trade-offs
Treating every surface as a settings row trades a bit of immediacy for legibility: getting to the live site is now one click on "Live Preview" instead of "the whole screen *is* the live site". I lean into that by making Live Preview the default selected row and putting the iframe right there at full pane width — but power users who lived in the iframe pay a small tax. The sidebar becomes long (12 rows); I mitigate that with grouped section headers ("Playground / Authoring / Data") and a working filter. On mobile, the metaphor inverts to iOS Settings.app: a stack of rows; tapping pushes a detail view (CSS-only). The Ventura grey/blue aesthetic limits brand expression, but it leans on a UI language users already trust for "this is configuration".
