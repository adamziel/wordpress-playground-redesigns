# Design #022 — Music DAW / Ableton

## Thesis
WordPress Playground reimagined as a digital audio workstation. A blueprint is a **song**: a sequence of *clips* (install plugin, activate theme, run wp-cli, set option, login as editor) arranged on horizontal *track lanes* against a timeline. The WordPress iframe becomes the **master output bus** — the audible mix of every step layered above it. You hit Spacebar to *play* the blueprint, see clips light up as they execute, and the master channel renders the resulting site in real time. Saved playgrounds are *projects* in the Live Browser on the left; the 43 blueprints are *presets* and *packs*. Site Manager tabs (Settings, Files, Blueprint, DB, Logs) collapse into a bottom *device chain* / *detail view* — the same metaphor Ableton uses for editing a selected clip or device. Authoring a blueprint stops being JSON-poking and becomes drag-clip-onto-lane composition.

## What I remixed / merged / removed
The classic three-region DAW shell (Browser left, Arrangement center, Detail bottom) absorbs almost every existing surface. **Merged**: the launcher panel, the saved-playgrounds list, and the 43-blueprint gallery all live inside one unified **Live Browser** (categories tree + previews) — Ableton's pattern of "one browser, many content types". **Merged**: the Edit Settings popover, the Site Manager Settings tab, and the inline "Save site locally" notice all consolidate into a single **Master Channel strip** on the far right, where WP/PHP/language/network/multisite are visible as faders and the Save button is the giant red **Arm/Record** dot — you can't miss whether your session is being recorded to OPFS. **Merged**: File browser and Blueprint editor share the bottom Detail pane via a Files/Code/Database/Logs tab strip (Ableton's Clip vs Device view). **Remixed**: the top toolbar becomes a *Transport bar* — Refresh = Rewind, Run Blueprint = Play, Save = Record, address bar = the song's time-position field; the Saved Playgrounds dropdown is the "Project" name plate. **Remixed**: WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip are **Sources** at the top of the browser (Ableton's "Places" — drop targets you drag from). The destructive "Apply Settings & Reset" gets the bright yellow "Reload Project — Unsaved changes will be lost" treatment Live uses for sample-rate changes. **Removed**: nothing functional — every §2 feature is present.

## Trade-offs
The DAW metaphor is gorgeous for **authoring** a blueprint (drag clips, reorder steps, see timing) and brutally honest about destructive resets (you can *see* the playhead about to wipe state). But it overpromises for the 90% of users who just want to click "Coffee Shop" and look at WordPress — they don't need a timeline, they need a thumbnail. I mitigated that with a prominent "Session view" toggle in the top-right that flips the arrangement into a clean clip-grid of featured blueprints (Ableton's Session view literally exists for this reason), but it's still a metaphor tax. Second trade-off: the visual density and dark chrome will feel intimidating versus the current friendly white UI; on mobile the lanes collapse to a vertical step-list which works but loses the parallelism. Third: power users will absolutely want to MIDI-map keys to clips (Play/Stop, jump to clip 3), which the design hints at but doesn't fully spec — a deliberate "this could grow here" affordance, not a new feature.
