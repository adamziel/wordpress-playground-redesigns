# 063 — Reader-first

## Thesis
The WordPress site is the product. Chrome is overhead. In Reader-first, the live iframe gets a full 75% of the viewport on desktop and the entire viewport on mobile, while every existing control collapses into a single 56px vertical icon rail on the left edge. Each rail icon has a sharp tooltip on hover and blooms into a 360px slide-in panel on click — never a modal, never an overlay over the canvas. The reader keeps reading; the controls come to them only when summoned.

## What I remixed / merged / removed
The horizontal top toolbar is gone; its seven controls (refresh, address, name, save, launcher, site-manager, settings popover) become the top half of the rail. The Site Manager's five tabs (Settings / Files / Blueprint / Database / Logs) are no longer tabs inside a sidebar — they are five first-class rail icons, each opening its own slim panel. This deliberately merges the Site Manager into the rail itself and resolves the "three settings entry points" pain by making Settings a single panel that the popover, info-notice, and tab all reduce to. The "Saved Playgrounds" launcher and full 43-blueprint gallery share one panel that switches between the **Start new** / **Blueprints** / **Your Playgrounds** sub-views via segmented control — preserving every entry point (Vanilla, WordPress PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip). Additional actions (Export to GitHub PR, Download .zip) live in the playground-name menu at the rail's bottom, alongside WP Admin / Homepage shortcuts.

## Trade-offs
The 75% canvas is gorgeous but every action is one extra click behind a rail icon — power users who lived in the old sidebar will feel the indirection. Tooltips do heavy labeling work, so this design depends on the user noticing iconography; we mitigate with a "Show labels" toggle that pins names beside icons (78px rail). Mobile cannot preserve 75% canvas + rail, so the rail flips to a bottom dock with the same icons and panels become bottom sheets — visually a different idiom, but the same vocabulary. Apply-and-reset is still destructive; we make it a red filled button at the bottom of the Settings panel rather than rely on a small banner.
