# #034 — Sketch / Excalidraw hand-drawn

## Thesis
WordPress Playground reimagined as a designer's sketchbook page. Every panel is a hand-pinned scrap of paper on graph-ruled stock: wonky borders, Caveat/Kalam/Patrick-Hand handwriting, sticky-note saved playgrounds, highlighter-yellow featured tape, and red marker stamps for destructive actions. The Playground's nature — a disposable, scratchpad WordPress you can scribble on and throw away — is mirrored in the chrome itself, which looks like the napkin you sketched the idea on.

## What I remixed / merged / removed
The three duplicated settings entry points (toolbar popover, Site Manager Settings tab, info-notice button) all stay, but I unify their visual language: the popover is literally a torn-off corner of the Site Manager card with a tape stub pointing back. "Your Playgrounds" becomes a stack of color-coded sticky notes — the only spot in the UI with that pure cardstock yellow/pink/blue — so the saved/unsaved distinction is instantly readable (saved = solid stickies, unsaved = dashed dotted-line ghost). The blueprint gallery, often a dense grid in other redesigns, is shown as a corkboard of doodled cards with thumb-pin annotations and feature-tape labels. Logs lose their "unstyled text" curse by becoming a handwritten log line with colored marker for level (red ERR / orange WARN / blue INFO / green OK), and the DB browser and Logs share one workbench panel via a small inked toggle — file browser keeps its own page because authoring blueprints alongside files is a deliberate dual-pane workflow. Mobile collapses everything into a hamburger + bottom sheet on a hand-drawn phone frame, preserving "Launcher" access via a pull-up sheet.

## Trade-offs
Hand-drawn aesthetics carry obvious costs: the readability hit of cursive fonts at small sizes is real (I lean on Patrick Hand and Kalam for body text rather than pure Caveat, but it's still less crisp than Inter), and dense data like the database table or the long blueprint JSON sits awkwardly inside a "doodle" frame — purists will say a code editor should never use a handwriting font, and they're right, so the JSON is rendered in Kalam but laid out monospace-style with syntax color, a deliberate compromise. The "wonky rotation" gimmick is tasteful at ±0.5° but would feel oppressive if every element wobbled — most actually sit straight, only stickers, stickies, and tape rotate. Finally, accessibility: hand-drawn fonts score worse on dyslexia-friendliness; in a real ship I'd add a "level the page" toggle that swaps to the same layout with system-ui at the same weights. Internationalization is the other casualty — Caveat doesn't cover CJK and most Cyrillic glyphs, so non-Latin locales would fall back to system handwriting fonts and lose half the charm.
