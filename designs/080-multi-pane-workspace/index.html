Mobile collapses the quad

On phones the four panes turn into a single active surface with a circular pane switcher. Every desktop pane content is reachable: Preview, Blueprint, Files, Logs, Launcher (which is where Database, Settings, Saved Playgrounds, Export, and Import .zip live as scrollable sections). The status bar shrinks to the version pills and the command palette.

Swiping left/right between panes mirrors the desktop "maximize one quadrant" gesture — on mobile, every quadrant is always maximized.

Every §2 feature is in this design. Refresh (app bar + preview head), URL bar (preview pane), Save (app bar primary), playground name label (app bar pill), Saved Playgrounds launcher (rail Launcher pane + app bar trigger), Site Manager (collapsed into pane contents — Files / Blueprint / Database / Logs / Settings), Settings popover (gear → popover or maximize a Settings pane), Additional actions menu (Export button in app bar — Export to GitHub PR + Download .zip), Import .zip (Launcher → Start New → Import .zip), WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Vanilla WordPress (Launcher Start New tiles), Featured + full 43 blueprints with categories + search (Launcher Gallery mode), Your Playgrounds (Launcher Your Playgrounds list), WP Admin + Homepage shortcuts (pills in Preview head), Apply & Reset (Settings popover footer + dialog), inline "Save site locally" notice (status bar dot + Save button), toast region (bottom of preview pane, not shown).
