# 080 — Multi-pane workspace

## Thesis
Playground is reframed as a **single workspace of four equal quadrants**: top-left Preview (the WordPress iframe), top-right Blueprint editor, bottom-left File browser, bottom-right Logs. None of the four "owns" the screen — they are peers around a central crosshair. Any quadrant can be **maximized** (it expands to 100% and the other three minimize to tabs along an edge), **swapped** with another quadrant, or **retargeted** to a different pane type (Database, Saved Playgrounds, Settings, Blueprint gallery — every Site Manager surface is just another pane content). The whole IA becomes "pick what each window shows," not "navigate from one screen to another." Debugging a blueprint? Maximize Logs, keep Preview as a peek. Authoring? Blueprint editor full, files docked. Browsing? All four are Launcher tiles. The chrome is a thin app-bar at the top and a vertical command rail on the left — everything else is panes.

## What I remixed, merged, and removed
- **Merged** the Site Manager's five tabs (Settings, Files, Blueprint, Database, Logs) into **pane content types**. Each quadrant's header has a small pane-picker that lets you turn that quadrant into any of those surfaces, plus Preview, Launcher, and Saved Playgrounds. The Site Manager as a separate sidebar is gone — its tabs are now a menu of pane contents.
- **Merged** the three settings entry points: the top-toolbar gear, the Site Manager Settings tab, and the "Save site locally" info notice. They are all the same Settings pane content, surfaced wherever you docked it. The destructive Apply+Reset action gets a real confirmation dialog instead of a banner.
- **Merged** the saved-playgrounds drawer and the new-playground "Start a new Playground" list into a single **Launcher pane** with three rails: Start New (Vanilla, WordPress PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip), Featured Blueprints, and Your Playgrounds. Clicking "View all 43 blueprints" switches the Launcher pane into Gallery mode with search + category chips.
- **Remixed** the address bar into the Preview pane's own header — it's a contextual property of the Preview pane, not the global app, so you can have two Preview panes pointed at different URLs of the same site.
- **Remixed** the additional-actions menu (Export to GitHub PR, Download .zip) into a workspace-level "Workspace" menu on the top app-bar, beside the playground name.
- **Added (as a remix, not a new feature)** named **layouts**: the four quadrants and their pane assignments can be saved as a layout per playground — so a "Debug" layout (Preview + Logs + DB + Files) and an "Author" layout (Preview + Blueprint + Files + Launcher) live alongside the default "Quad" layout. This is just a save of which-pane-shows-what, no new functionality.
- **Removed** the modal/drawer pattern entirely. There are no overlays except the destructive confirm dialog and the settings popover (kept as a quick variant on mobile).

## Trade-offs
The four-quadrant idea is exhilarating on a 1440 desktop and useless on a 390 phone — so on mobile the layout collapses to a single active pane with a swipe-able pane switcher at the bottom (the same pane-picker that's in each quadrant header on desktop). Power users will love being able to see logs as they type a blueprint; novices will see four panes and wonder where to look — we mitigate with a labeled "Try one of these layouts" callout on the empty default and by making the Preview quadrant the visually dominant one in the default layout (it carries a subtle highlight ring). Saved layouts add a tiny bit of state to learn. The maximize-then-three-tabs interaction is the trickiest moment; we use a familiar window-control vocabulary (minimize / restore / maximize) and animate to the destination so the spatial mapping is obvious.
