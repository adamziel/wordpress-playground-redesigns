# 019 — Slack layout

## Thesis

WordPress Playground is essentially a long conversation between a developer and their throwaway WordPress instance — boot logs, PHP notices, SQL queries, blueprint steps. Slack already has the perfect grammar for that conversation: a workspace rail for switching contexts, a channel sidebar for jumping inside one context, and a threaded right pane for following one running stream of messages. This redesign maps Playground onto that grammar 1-to-1. Each saved playground is a *workspace* (rail icons with unread badges); each Site-Manager tool — Settings, Files, Blueprint, Database, Logs — is a *channel*; the PHP error log is a permanent *thread* in the right pane with reactions, replies, and slash commands. The live iframe becomes the active channel's "canvas" and the address bar lives in the top global search slot, so navigating WordPress feels like jumping into a Slack channel.

## What I remixed / merged / removed

I **merged** the three Slack-iest features of the existing app: the workspace switcher (left rail) replaces "Saved Playgrounds" — every saved site is a colored squircle with hover name, unread count, and an active ring; the channel list replaces the Site Manager tabs *and* surfaces "Start new from…" as a third group of DM-like entries (Vanilla WP, WordPress PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip), so all entry points sit in one place; the right thread pane permanently houses the Logs channel as a real chat stream (avatars per log level, day separators, stack traces in a quoted block, reactions, "5 replies in thread"). The big launcher modal is a Slack-style "huddle/Add channels" sheet that hosts the full blueprint gallery, category filters, search, the 43 cards, and Your Playgrounds. The destructive "Apply & Reset" warning gets the Slack-banner treatment in both the popover and the Settings channel. The "Unsaved Playground" affordance becomes a yellow workspace status card with "Save site locally" — same prominence as a Slack "Set a status" prompt. I removed all duplicate triggers (no separate Site-Manager-vs-popover settings — the channel and the popover are visually identical) and the marketing "Hello from WordPress Playground" hero, replacing it with a real WP-rendered demo page.

## Trade-offs

Slack's chat metaphor is *strong* — too strong if you read it literally ("can I @mention a PHP fatal?"). I kept the chat affordances (reactions, threads, compose box with slash commands) because they map cleanly to real Playground actions (`/save`, `/zip`, `/pr`, filter logs by level), but a first-time visitor may briefly wonder where the iframe went on a narrow screen. The four-column desktop layout (rail + sidebar + stage + thread = 70 + 260 + flex + 320) eats horizontal space; on smaller laptops the thread can be collapsed via its X button. Mobile drops both side rails and the thread for a Slack-mobile-style bottom tab bar (Live, Files, +, Logs, Settings) and the launcher / blueprint gallery becomes a full-sheet modal. Finally, the visual aubergine reads as "Slack" to a fault — I retuned it toward WordPress blue for the active states so the brand still feels like Playground, not a Slack skin.
