# 066 - TweetDeck multi-column

## Thesis
WordPress Playground reimagined as a TweetDeck-style **horizontally scrolling deck of narrow columns**. Every concern - a saved playground, a blueprint category, the live logs feed, the file browser, the blueprint editor, the launcher - is a 320px-wide column that lives side-by-side with all the others. The user **curates their deck**: pin the columns you care about, scroll horizontally to reveal more, drag to reorder. The WordPress iframe itself is a special wide "Live" column (700px) that the deck navigates around. This converts Playground's modal/tabbed UI into a single flat scrollable surface where every panel is always visible at a glance - the way an ops dashboard, a Slack-with-many-channels, or TweetDeck has always worked.

## What I remixed / merged / removed
- **Saved Playgrounds drawer + Site Manager tabs collapse into the deck itself.** Each saved site is a column (with its mini "WP Admin / Home / Refresh" header). The Site Manager's 5 tabs (Settings, Files, Blueprint, DB, Logs) are 5 columns you can add per playground - so a power user can have Logs, Files, and the live iframe of the *same* playground visible simultaneously, which the original tabbed UI made impossible.
- **One unified "Add Column" picker** replaces the launcher, settings popover, and blueprint gallery. It's the New Column flow from TweetDeck: pick a column type (Playground, Blueprint category, Logs, Files, Blueprint editor, DB, Settings, Launcher), then configure. Featured blueprints, "WordPress PR", "Gutenberg PR", "From GitHub", "Blueprint URL", "Import .zip" are all column types under "New session".
- **Blueprint gallery → one column per category.** The 9 categories (Featured, Website, Personal, Content, Themes, Gutenberg, Experiments, WooCommerce, News) each get their own narrow column with searchable card list. Search is per-column, scoped.
- **Three settings entry points → one Settings column** pinned to a playground. The popover trigger in the toolbar opens a *temporary* Settings column that auto-pins to whatever playground is "Live".
- **Additional actions menu** (Export GitHub PR, Download .zip) lives in each Playground column's overflow `...` menu - one tap from every column header, not buried in a sidebar.
- **Toasts** move to the top of the deck as a persistent strip so they're visible regardless of horizontal scroll position.
- **Removed**: the marketing iframe landing page (replaced by a deck-empty-state "drag a blueprint to a column"), the destructive-banner confusion (Apply+Reset now requires a typed confirm in the Settings column, like deleting a GitHub repo).

## Trade-offs
TweetDeck's strength is density - and Playground's pain is exactly that: many things buried in tabs. Trade-offs: (1) horizontal scroll is unfamiliar to some users; I mitigate with a "column minimap" at the bottom and keyboard `←/→` arrows. (2) The iframe is naturally wider than a 320px column, so the "Live" column is a special-cased 700px wide; mobile reverts to a single column with a left/right pager. (3) Curation cost: you have to add columns before they're useful. Defaults solve this - a fresh deck ships with Launcher, Live, Logs, and Featured Blueprints already pinned. (4) The metaphor strains if you only have one playground - it then looks like a fancy sidebar - but the moment you have two, the value (simultaneous visibility of two playgrounds' logs side by side) becomes obvious.
