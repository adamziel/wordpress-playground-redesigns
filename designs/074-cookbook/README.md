# 074 — Cookbook

## Thesis
WordPress Playground becomes "The WordPress Cookbook" — a warm, tactile recipe book where every blueprint is a recipe. The home page is the cover of a numbered edition; the launcher is a table of contents; each blueprint card is plated like a magazine recipe with **ingredients** (plugins, themes, content), **prep time** (PHP/WP versions, language, network), and a **method** (the blueprint steps as numbered instructions). When you "Cook" a recipe, the iframe is the dining plate. This reframes Playground's most intimidating surface — the 43-blueprint gallery and the blueprint JSON editor — as something inviting, browse-able, and human.

## What I remixed / merged / removed
The blueprint JSON tab and the blueprint gallery are merged: every recipe in the gallery shows the same data the JSON editor exposes, just plated as ingredients + method. The Site Manager keeps its five tabs but they are renamed as the "Kitchen" (Settings = Pantry, Files = Larder, Blueprint = Recipe, Database = Cellar, Logs = Service Log). The top toolbar becomes the "Counter" with chef's apron labels. The launcher's "Start a new Playground" is a Quick Bites strip (Vanilla, WP PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip). "Your Playgrounds" is the shelf of "Saved Dishes." The settings popover is a small index card. The destructive Apply & Reset is intentionally framed as "Clear the table" with a typographic warning, because the recipe metaphor makes destruction legible. Additional actions (Export to GitHub PR, Download .zip) are plated as "Plate up & take away."

## Trade-offs
A skeuomorphic recipe metaphor risks feeling twee for power users who want speed; I keep the chrome small and the typography legible to compensate, and the toolbar is still one row. The serif display font costs a Google Fonts request. The metaphor leans hardest on the blueprint gallery; surfaces with no natural food analogue (Logs, DB browser) get a lighter touch so they still feel like tools. Mobile collapses the cookbook spread into a single column "phone-recipe" view and hides the Kitchen behind a bottom sheet. Long-term, terms like "Cellar" for the DB might add to the learning curve — the design surfaces the plain label as a subtitle on every panel so power users aren't lost.
