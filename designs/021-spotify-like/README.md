# 021 — Spotify-like

## Thesis
This redesign reframes WordPress Playground as a music player. A playground is a "track," a blueprint is an "album," the running WordPress iframe is the "now playing" record on the turntable, and the toolbar is the player's transport bar. The chrome the user touches most often (refresh, navigate, save, settings, more-actions) collapses into the bottom playback bar that's always within thumb-reach. The left rail is "Your Library" of saved playgrounds plus pinned blueprints and quick-starts. The center pane is a Spotify home page of "Made for you" blueprint shelves, browse-by-category color tiles, and a "Your Playgrounds" shelf. The right rail is the Site Manager reborn as a "Now Queue" with five tabs (Settings, Files, Blueprint, DB, Logs). Loud, dark, green-accented, deliberately consumer-facing — Playground as something you put on, not something you set up.

## What I remixed / merged / removed
The three settings entry points (top popover, Site Manager Settings tab, in-notice save button) all still exist, but each now plays its native role: the popover is a quick transport-bar tweak, the Site Manager tab is the deep dial-in, and the unsaved-banner with its save button lives inside the Site Manager Settings pane as in the original. "Save site locally" also lives as a heart icon next to the now-playing track in the bottom bar and the player-actions row — adopting Spotify's "Save to your library" pattern so users learn it instantly. The launcher drawer is dissolved: its three sections (Start new / Start from blueprint / Your Playgrounds) become three home-page shelves so they're discoverable without a click, and the full 43-blueprint gallery still opens as a modal with the original category chips and search. Additional actions (Export GitHub PR, Download .zip) live in a menu opened from the toolbar Share button and from the player-actions "..." button; WP-Admin / Homepage are both quick buttons and an address-bar path. The address bar is the URL pill in the toolbar; the refresh button is also the giant green Play disc — they're the same action wearing different clothes.

## Trade-offs
Pushing Playground into a music-player metaphor risks looking unprofessional to power users who associate WordPress with workstation-grade tooling; the deep tabs (Files, Blueprint, DB, Logs) stay serious to compensate. Calling the active session "Now playing" is friendly but slightly misleading — a playground isn't time-bounded — and the scrub bar is decorative (it shows session age, not progress). Sites being treated as albums in shelves trades the Site Manager's left-sidebar primacy for visual richness; the manager pane is still always-visible on desktop but collapses entirely under 900px (replaced by a bottom playback bar plus a one-tap Site-Manager toggle). The bottom transport bar holding refresh/back/forward/save/settings creates redundancy with the toolbar — by design, because in Spotify the transport always wins as the muscle-memory surface.
