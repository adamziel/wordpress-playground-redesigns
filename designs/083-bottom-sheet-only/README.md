# 083 — Bottom sheet only

## Thesis
The iframe is the wallpaper. There is no top toolbar, no left sidebar, no right drawer — every Playground control lives inside a single swipe-up bottom sheet that sits over the live WordPress site. The sheet has three detents: a peek (showing the playground name, URL, refresh, and quick-actions), a half-sheet (the launcher: new playgrounds, your saved sites, featured blueprints, settings), and a full-sheet (the site manager: file browser, blueprint editor, database, logs, full 43-blueprint gallery). Desktop is not a different layout — it's just the same sheet, the same gestures, and a slightly wider grab handle. WordPress fills the rest of the screen at all sizes.

## What I remixed / merged / removed
The top toolbar, the Saved Playgrounds drawer, and the Site Manager all collapse into one surface — the sheet — distinguished only by how far you drag it. The three settings entry points (popover, sidebar tab, info notice) merge into one Settings card inside the launcher detent that's always one tap away. The "Saved Playgrounds" launcher's three sections (Start new, From Blueprint, Your Playgrounds) become horizontal scroll-snap rails on the half-sheet, with Import .zip / WP PR / Gutenberg PR / From GitHub / Blueprint URL as chips on the "Start new" rail. The full blueprint gallery, file browser, blueprint JSON editor, database browser, and logs become tab-segments inside the full-sheet — same surface, different content, no extra modals. The address bar lives in the peek so it's always reachable without expanding. Toasts dock to the top so they never collide with the sheet handle.

## Trade-offs
Hiding everything behind a sheet means a power-user task like "compare logs and edit a file side-by-side" needs a vertical swipe between segments rather than two visible panes — the design optimizes for focus over density. The wallpaper iframe is dominant, which makes the running site feel like the product (good for newcomers) but pushes chrome a tap further away (a cost for engineers debugging). I also remove the visual distinction between "launcher" and "site manager" — they're the same sheet at different heights — so users learn one gesture instead of two surfaces, but lose the explicit "I'm in admin mode" cue. Mobile and desktop share one mental model, which is the whole point.
