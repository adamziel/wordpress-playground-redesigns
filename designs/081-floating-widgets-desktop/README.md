## Thesis

Playground OS treats the entire WordPress Playground UI as a personal *desktop environment*. The wallpaper is the canvas; every existing feature — the WordPress iframe, the address bar, the saved playgrounds launcher, the blueprint gallery, the file browser, the blueprint editor, the database browser, the logs, the settings — becomes a window/widget you can drag, resize, snap, minimize and dock. The chrome that used to be modes, tabs and drawers is now an arrangement of co-visible panels. The "layout" itself is a property of the playground: each saved site remembers which widgets you had open and where they were. Power users get a tiled IDE; new users get a clean default with just the WordPress window centered and a dock at the bottom for everything else.

## Remixed, merged, removed

The top toolbar, Site Manager sidebar, Settings popover and Saved Playgrounds drawer were *merged into* a single concept: each is just a widget on the desktop. The triple-entry "Settings" problem disappears because there is exactly one Settings widget — you either have it on screen or you launch it from the dock. The Site Manager tabs (Settings / Files / Blueprint / Database / Logs) were *un-grouped* into five independent widgets, so you can have Logs and File Browser open side-by-side without losing the WordPress window. The "Saved Playgrounds" drawer was split into a *Launcher* widget (new playground sources) and a *Your Playgrounds* widget (dock-pinned). The Blueprint Gallery is its own large widget with search + category chips. Additional Actions (Export PR, Download .zip) live in the WordPress window's title-bar menu where the rest of the iframe-level actions already are. A bottom *dock* is the persistent launcher for any widget that's been closed.

## Trade-offs

A draggable canvas is harder to learn than a fixed app shell — first-time users may not realize widgets are movable, so the default layout is pre-arranged and the dock is always visible with labeled icons. Floating windows on mobile would be hostile, so the mobile breakpoint collapses the desktop into a stacked column with the dock at the bottom and widgets as full-width cards. Saving layout per-playground is great for power users but means switching sites also rearranges your workspace; the design accepts that as a feature — your "writing on the train" playground can look different from your "WooCommerce debugging" playground. Finally, ceding screen real estate to chrome (title bars, dock) costs some iframe area; the snap-to-edge and "focus mode" double-click on the WordPress widget reclaim it.
