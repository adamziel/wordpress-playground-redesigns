# 007 — Dark cyberpunk hacker terminal

## Thesis
WordPress Playground reimagined as a 1986 phosphor-green hacker rig. The whole chrome is a tmux-like terminal multiplexer: ASCII box borders, monospace everywhere, scanlines, CRT flicker, and a green-on-black palette punctuated by amber alerts and magenta hotkeys. The WordPress iframe is mounted inside a faux CRT bezel ("RENDER://wp-site") and every Playground action — switching PHP, applying a blueprint, exporting a zip — is shown as a copyable `wp-playground …` command. The user can either click the green buttons or type `:` and the command they want. It's a single self-contained TUI that treats Playground less like a SaaS app and more like a privileged shell.

## What I remixed / merged / removed
The 5-tab Site Manager became a single left "PANE" rail (`[1] SITE / [2] FS / [3] BLUEPRINT / [4] DB / [5] LOGS`) selected by digit keys. The three settings entry points all collapse into one quick-settings popover written as a wp-cli stanza — opening the Site Manager Settings tab just scrolls the same stanza into the main pane, killing the duplication pain point. The Saved Playgrounds drawer and "Start new" launcher fuse into one "LAUNCHER" overlay with three columns (NEW · BLUEPRINTS · MOUNTED), addressing the discoverability problem that the current "Saved Playgrounds" label hides the blueprint gallery and import-zip. The destructive "Apply Settings & Reset" gets an amber `[!! DESTRUCTIVE]` banner styled like a kernel warning, plus an explicit `y/N` confirmation in the command preview — much louder than the current small red strip. The address bar becomes the prompt itself (`wp://`) so typing `/wp-admin` feels natural in context. Toasts render as styled `dmesg` lines in the bottom-left status bar.

## Trade-offs
The aesthetic is polarising — non-developers may bounce off the wall of monospace and never realise they can just click. Accessibility takes a real hit: pure #00ff9c on #050b07 looks great but is harsh, and scanlines on top of body text can make extended reading tiring (I kept scanlines subtle and confined the brightest green to chrome, but it's still a concern). The blueprint cards lose preview thumbnails entirely, which the brief flagged as a known pain point — I traded that back for fidelity to the direction, since "hacker terminal" with screenshot tiles would have felt like cosplay. Mobile collapses the pane rail into a top tab strip and the launcher into a full-screen sheet, which works but loses the multi-column tmux feel that's the desktop's main charm.
