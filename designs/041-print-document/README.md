# Design #041 — Print-document inspired

## Thesis
The WordPress Playground is presented as a typeset manuscript: a stack of A4 pages bound into a journal called *The Playground Quarto*. The iframe is the central "Plate" (Figure I) of the running issue; every control around it borrows from print conventions — running headers, folios, drop caps, marginalia, footnotes, signature lines, a card-catalog index. Settings are a *colophon*. Logs are an *errata*. Blueprints are an *index of plates*. The result feels like reading and editing a living document rather than operating a SaaS dashboard, which suits Playground's experimental, ephemeral character: a session is a draft, saving is signing the page, exporting is binding the manuscript.

## What was remixed / merged / removed
The toolbar is dissolved into a printed *running head*: refresh becomes a rubric symbol (the pilcrow), the URL bar is a typeset "page reference" line, save is a literal signature on a dotted rule. The Site Manager's five tabs (Settings / Files / Blueprint / DB / Logs) are merged into the document itself as named sections — *Colophon, Manuscript Sources, Recipe, Datastore, Errata* — reachable via a left-edge thumb-index, eliminating the dual-sidebar problem. The "Saved Playgrounds" launcher becomes the *Table of Contents* page that opens on demand and contains all six new-session entry points (Vanilla / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip) typeset as bibliography entries, plus the full 43-entry blueprint index laid out as a card catalog with search. The destructive Apply+Reset confirmation lives at the bottom of the colophon page as a footnote-style warning, finally given the weight it deserves. Marginalia on every page replaces in-product tooltips with friendly typeset tips.

## Trade-offs
Serif type and generous leading sacrifice information density — the same screen real estate holds fewer affordances than a conventional toolbar. Power users may find the playful chrome (folios, footnotes) slow on first encounter, though once the page metaphor clicks, navigation collapses into thumb-index taps. The print metaphor strains slightly on the Database and Logs surfaces, which we lean into by styling them as *errata sheets* and *catalogue cards* respectively. Mobile keeps the cover-page aesthetic but flattens marginalia into a bottom footnote rail to preserve readability at 390px.
