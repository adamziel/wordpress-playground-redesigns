§ III — The Recipe (Blueprint)

Recipe A blueprint is a typeset recipe — change a step, then Run to reprint the plate.

Why "errata"? Print errata are corrections bound after the fact. Logs serve the same purpose for a running plate.
  1. Five tabs, one document. The old Site Manager had Settings / Files / Blueprint / Database / Logs as parallel screens. We bind them as five sections of one printed source.
  2. Apply & Reset. A destructive action; printed at the foot of the Colophon, ringed in red ink, so it is never mistaken for routine.
  3. Datastore. Early access. Schemas may change between editions; please report errata.
  4. Errata. Limit 1,000 lines; older entries are sloughed off as new ones arrive.
