# Design #049 — Maps

## Thesis
Playground as a cartographic atlas. Every WordPress version, PHP version and blueprint is a place — a harbor, a peak, a town, a province — on a hand-drawn world map you pan, zoom and pin. Booting a playground is choosing a destination: you set your coordinates on the version-compass, drop a pin on a blueprint, and "sail" there. The iframe becomes the porthole window of the ship: you're always somewhere, with a coastline visible just behind you.

## What I remixed / merged / removed
The launcher panel, the blueprint gallery, the Site Manager Settings popover, the "From GitHub / Blueprint URL / PR" entries and "Your Playgrounds" all collapse into one continuous map. Blueprint categories are *islands* and *coasts* (Themes Archipelago, Gutenberg Volcanic Range, WooCommerce Coast, News Cape, Personal Bay, Content Mainland, Experiments Outer Reach). WP/PHP versions become a compass-rose dial; latitude=WP, longitude=PHP. Saved playgrounds are golden bookmarks on the map. The Site Manager moves into a "Captain's Log" drawer that swings out from the right (tabs become log sections: Settings, Files, Blueprint, Database, Logs). Top toolbar shrinks to a single "Heading" bar with: refresh-compass, address-bar wrapped as a scroll, save-as-anchor, additional-actions as a "More routes" sextant. The marketing landing page is replaced by the map itself — there is no "empty" state; you always have a world. Mobile turns the map into a vertical "spyglass" stack: the compass becomes a single bottom-sheet, the iframe stays full-bleed, the map slides over it as a sheet.

## Trade-offs
A map is unfamiliar for a dev tool — first-time users may not immediately read "pin = blueprint." I mitigate with explicit pin labels and category headings on each island, but it's a learning curve cost in exchange for memorability. The map metaphor also flattens hierarchy: a 43-pin gallery sprawls wide and demands panning, while a grid sorts faster. Search and category filters stay (they highlight matching pins and dim the rest) which softens this, but at small zoom levels the labels collide. Lastly the visual richness adds painting weight; I kept the map as one SVG and a CSS gradient ocean so it renders snappily.
