# Design #075 — Catalog / IKEA

## Thesis
WordPress Playground is reframed as a flat-pack furniture catalog. Every blueprint is a numbered article — a six-digit SKU like **PLG.204.871** — with cool product photography, a price-tag style spec block, an in-stock badge for WP/PHP versions, and a numbered "assembly instructions" panel that doubles as the blueprint JSON. The shop is the launcher; the iframe is the "showroom"; the Site Manager is "Customer Service" with five clearly labelled service counters. The interaction model is *browse → flip the tag → assemble → bring home*: pick an article from the showroom, read its tag, choose Assembly Instructions to inspect the blueprint, and "Bring Home" boots it. Saved playgrounds live in "Your Cart" — three sites you've already brought home, each with the same tag-card treatment.

## What I remixed, merged, removed and why
The 11 launcher entries (6 new-start types + 5 featured + gallery + your-playgrounds) collapse into one coherent **showroom**: a top "Featured Articles" rail, a "Departments" filter strip (the BRIEF categories), a numbered SKU grid for all 43 blueprints, and a sidebar with the six **Special Orders** (Vanilla, WordPress PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip) presented as service-desk forms. The settings popover, Site Manager Settings tab, and inline Save-locally notice all funnel into a single **Tag** (Article Details) panel that flips open from any article — so the three settings entry points become one *price-tag flip*. The Site Manager's five tabs become five **Service Counters** ("Specifications", "Stockroom" = file browser, "Assembly" = blueprint editor, "Inventory" = DB, "Service Log"). The top toolbar is a thin **store header** (refresh = "restock", address bar = "aisle finder", save = "bring home", saved-playgrounds = "your cart", site manager = "customer service", quick settings = the article tag). Additional actions (Export to GitHub PR, Download .zip) live as **Returns & Exchanges** in customer service.

## Trade-offs
Leaning into the catalog metaphor takes screen real-estate: the SKU cards are deliberately larger than a tight admin UI would be, because the whole point is to make 43 blueprints feel *browsable* rather than scrollable. Power users who want a dense list will find this generous. The IKEA-yellow header is high-contrast on purpose — it's a brand commitment, not a neutral chrome — and may clash with WordPress blue inside the iframe. The "Special Orders" desk treats PR/GitHub/URL boots as side-rail forms rather than headline tiles, which de-emphasizes them versus the current launcher (intentional: those are power-user paths). The mobile layout collapses the showroom into a single column and turns the customer-service tabs into a bottom sheet, which means deep service tasks (file editing, DB browsing) get a "best viewed on desktop" hint rather than a full mobile editor.
