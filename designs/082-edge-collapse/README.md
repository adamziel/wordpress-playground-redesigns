# 082 — Single-line top toolbar, everything else collapses

## Thesis
The Playground is, fundamentally, an iframe. So the chrome should get out of its way. This redesign reduces all persistent UI to a single 44px toolbar at the top — refresh, URL bar, playground name, Save, and four edge-direction handles — and pushes every other panel off-screen until summoned. The four screen edges each own one role: **Top** = quick Settings (because it sits next to the toolbar), **Left** = Site Manager (because it's a "sidebar mental model"), **Right** = Launcher + 43 blueprints + Your Playgrounds (because it's a wide content surface), **Bottom** = Logs / Database / File browser / Blueprint editor (because they're terminal-shaped, scrolling tools). Tiny tab-shaped handles peek out of each edge so the affordance is always discoverable; keyboard `[`, `]`, `,`, `\` and `Esc` make it muscle-memorable.

## What I remixed / merged / removed
The two redundant settings entry points (popover + Site Manager Settings tab) are kept but visually unified — the popover is a small slide-down version of the same settings rows, and the Site Manager Settings tab is the canonical, full version. The File browser, Blueprint editor, Database and Logs are merged into a single bottom drawer with tabs, because they're all "look-at-data while keeping the site visible" surfaces; clicking those tabs in the Site Manager simply redirects to the bottom drawer (one source of truth instead of four). The "Saved Playgrounds" drawer is merged into the launcher panel: New from… cards, Your Playgrounds list, and the full 43-blueprint gallery with categories and search live in one right-edge panel with a left rail of categories. Additional actions ("Export to GitHub PR", "Download .zip") are a floating menu reachable from both the toolbar name area and the Site Manager. The marketing iframe landing page is left intact (it's inside the iframe, out of scope).

## Trade-offs
Collapsing everything to edges costs first-time discoverability — a brand-new user sees a near-empty page and might miss the four edge handles. I mitigate that with a subtle pulse on the left handle, visible toolbar icons mirroring each edge, an onboarding toast that names the shortcuts, and labels on every handle. The single toolbar gets tight on narrow viewports — the playground-name chip drops on mobile and panels become full-width sheets. Concurrent panels are allowed on desktop (e.g. logs at bottom + launcher on right) but the scrim makes them feel modal; mobile forces one at a time. Finally, by routing the non-Settings Site Manager tabs into the bottom drawer, I've consciously broken the current "everything-in-one-sidebar" mental model — that's the cost of being decisive about which edge owns which information.
