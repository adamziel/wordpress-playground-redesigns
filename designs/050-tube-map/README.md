# 050 — Tube / metro line map

## Thesis
WordPress Playground reimagined as a transit network. The WP version numbers are stops along a red **WP Line**, PHP versions stops along a blue **PHP Line**, and blueprints become **interchange stations** where the two lines meet specific destinations — Art Gallery, Coffee Shop, Photography Portfolio. To boot a Playground, you literally pick two stations and "board". The chrome around the iframe is a transit display: ticker for logs, a route summary header, a service-information sidebar for the Site Manager. Every operation borrows from the language of travel — Save = save your ticket, Reset = end of the line, Apply Settings = change of platform — without slipping into pastiche.

## What I remixed, merged, removed
The three settings entry points collapse into a single Map view: choosing a WP station and a PHP station IS the settings popover IS the Settings tab IS the Edit Playground popover. "Apply Settings & Reset" becomes a "Board train" confirmation that openly admits it's destructive (you leave the current line). The Saved Playgrounds drawer and the Blueprint gallery merge into one full-screen Map; "Your Playgrounds" are saved routes pinned to the network with little train icons. The Site Manager sidebar becomes a "Service Information" panel with tabs (Settings, Files, Blueprint, Database, Logs) styled like a station information board. The PHP/SQLite emulation, network, multisite, language toggles become station "amenities". The Additional Actions menu (GitHub PR export, .zip download) and Import .zip and start-from-GitHub / Blueprint URL / WordPress PR / Gutenberg PR become the **"Terminus" stations at the ends of a third "External" line** — entry points from outside the network.

## Trade-offs
The metaphor is louder than a typical productivity UI; first-time users have to learn that stations = versions. I lean into that by always showing the literal labels (WP 6.5, PHP 8.2) next to the station dots. The map is dense at 1440px but collapses on mobile into a vertical timetable view — same data, no SVG panning required. Logs become a marquee ticker which is charming but harder to scan than a normal log viewer, so the Logs tab still opens a full panel beneath the map. The strongest critique: a real transit map enforces geographic relationships that don't exist here (PHP 8.2 isn't "near" WP 6.5). I treat that as licence — the lines are conceptual, like a London Tube diagram which famously distorts geography for clarity.
