# 006 — Mobile-first

## Thesis
Playground is treated as a phone app first. The entire UI is built around a 390×844 frame with a four-tab bottom bar (Site, Files, Blueprint, More), a single top action row, and bottom sheets that slide up for everything else. Desktop is not a separate design — it is the same column, centered, with two optional "side rails" that expose the same sheets as always-visible panes. This forces every feature to earn its place by being reachable with one thumb, and it dissolves the popover/sidebar/tab triple-entry-point problem of the current UI into a single sheet metaphor.

## What was remixed, merged, removed
The five Site-Manager tabs collapse onto the bottom tab bar (Settings folds into "More" with the additional actions, export, import, and the popover settings — they are literally the same surface now). The "Saved Playgrounds" launcher becomes the Sites sheet reachable from the title pill at the top; "Start new", "Blueprints", "Your Playgrounds" are three segmented chips inside that sheet. The 43-blueprint gallery is a full-screen sheet with sticky search and horizontally scrollable category chips. The URL bar shrinks into the title pill and expands inline when tapped. The destructive "Apply & Reset" action gets a dedicated confirm sheet rather than a small red banner. Three states are shown in-page: boot/loading, an active session with toast, and the Blueprint gallery sheet open. A second viewport on the right renders the same HTML as desktop, demonstrating that "desktop = wider phone".

## Trade-offs
Power users on desktop lose simultaneous visibility of, say, the file tree and the blueprint JSON — they must switch tabs (mitigated by the optional side rails on wide screens, which show the most recent sheet pinned). The code editor and DB browser have less horizontal room; long log lines wrap or scroll horizontally. The single-column constraint means dense screens (Database tab, Files tab) feel busier than they would on a desktop-first design. In exchange, the UI is identical at every breakpoint, accessibility on small screens is dramatically better, and a new user can find every feature by tapping one of four icons.
