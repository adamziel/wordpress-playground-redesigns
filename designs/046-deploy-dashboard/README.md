# 046 — Vercel/Netlify Dashboard

## Thesis
WordPress Playground is reframed as a deployment platform. Each saved playground is a "project" with a deployment history; each unsaved session is a "preview deployment". The familiar developer-dashboard grammar — Overview / Deployments / Logs / Settings, a sticky build-status bar, sliding build-logs drawer, monospaced commit-like IDs and timestamps — is mapped exactly onto Playground concepts. "Promote to Production" doesn't exist here; instead, **Save** is the promotion: it persists a preview into your local OPFS storage and gives it a stable name. The mental model that ships with modern web infra is borrowed wholesale, so a developer who already lives in Vercel/Netlify recognizes the surface in three seconds.

## Remixed / merged / removed
The three settings entry points are collapsed into a single **Settings tab** inside the project page, with the toolbar popover keeping only the most common quick-edit (WP & PHP version). The "Saved Playgrounds" launcher and the Site Manager become two halves of the same dashboard: the launcher is the **All Playgrounds index** (cards with status pills), and the Site Manager is the **single project view** (tabs: Overview, Files, Blueprint, Database, Logs, Settings). The blueprint gallery becomes a **"New Project" templates page** with category chips, search, and 43 templates listed Vercel-style with framework-style tags. New-project entry points (Vanilla, WP PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip) become deployment "Git providers" in the New Project flow. The Additional Actions menu becomes a deployment-context "..." menu with **Export to GitHub PR** and **Download .zip**. Refresh, address bar, save, and a build-status pill all live in a sticky deployment topbar above the iframe (the "preview frame").

## Trade-offs
The deploy metaphor is a stretch — Playground doesn't actually build or deploy anything, so words like "Deployment", "Preview", "Production" are aspirational. To stop confusion, status pills always reflect Playground reality ("Booting", "Ready", "Saved"). The dashboard chrome eats some pixels that previously gave the iframe more room; on narrow desktops the activity rail collapses. On mobile, the multi-pane dashboard collapses into a single stack with a tab bar, which loses the "scan a fleet of deployments" pleasure that's the whole point of the metaphor.
