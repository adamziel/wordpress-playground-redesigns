# 070 — Onboarding-driven

## Thesis

The first time anyone lands on Playground, they're asked exactly one question: **"Why are you here?"** — Theme author, Plugin author, Site owner, or Learner. That single answer becomes the spine of the IA: which Site Manager tab opens first, which blueprints get promoted to "For you", which keyboard shortcuts are live, which capabilities (networking, autologin, error logging) default on, and how loud the destructive "Apply Settings & Reset" warning is. The persona is persisted to localStorage, surfaced as a coloured avatar pill in the toolbar, and can be swapped at any time — but **no feature is hidden behind it**. Persona only *re-ranks and re-defaults*; the full feature inventory (refresh, address bar, save, library, site manager, settings popover, 43-blueprint gallery, file browser, blueprint editor, DB, logs, GitHub PR export, .zip download, WordPress PR, Gutenberg PR, From GitHub, Blueprint URL, import .zip) remains one click away in every persona.

## What I remixed / merged / removed

I **merged** the "Saved Playgrounds" drawer and the "Start new" menu into one persona-sorted *Library* with a "For you / All / Start new / Mine" segmented switcher — same surface, four lenses. The 6 "Start a new Playground" tiles live at the top of every Library state but are re-ranked by persona (Vanilla + Gutenberg PR float for Plugin, From GitHub floats for Theme, etc.). The three settings entry points (popover, sidebar tab, info-notice button) stay, but the tab and popover are explicitly marked as the *same* settings, with persona-aware defaults annotated inline so users can see *why* PHP 8.3 is starred and Networking is pre-on. I **remixed** the blueprint gallery into a single grid with a leading "For you" chip that filters to ~12 persona-matched cards, while preserving the full 11-category filter bar and the 43-card "All" view. The Site Manager keeps all 5 tabs but its default-open tab and pinned right-rail tab change per persona (Plugin → Files + Logs; Theme → Files + Blueprint; Owner → Settings only; Learner → Settings with tooltip overlays). Nothing was removed — pain points (3 settings entry points, opaque destructive Apply, ugly unstyled logs, no-preview blueprints, easy-to-miss Save) are all addressed by *defaults* the persona picks, plus a styled log viewer with level filter, plus gradient blueprint previews, plus a stronger Save button.

## Trade-offs

Asking a question on first run adds friction — some power users will resent the modal and just hit "Skip — give me everything", which I made an explicit, no-shame escape hatch (it falls through to "Plugin author" because that surfaces the most tools). The persona pill also adds visual weight to an already busy toolbar. There's a deeper risk: persona-tailored IA can hide useful surface area from people who picked the "wrong" persona — I mitigate this by never *removing* features, only re-ranking, and by surfacing the chosen persona prominently in the toolbar so it's obvious how to switch. Saved playgrounds are deliberately *not* persona-scoped: a site saved by a Plugin author opens fine for a Site owner because persona is just a view, not a data partition.
