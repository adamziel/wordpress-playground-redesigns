# Card stack (Tinder-style) — WordPress Playground

## Thesis
WordPress Playground becomes a deck of cards you swipe through. The "launcher" is no longer a drawer of tiny tiles — it is the home screen. A single tilted card dominates the viewport showing one blueprint's hero, title, description, WP/PHP target, and tags. Swipe right (or press →) to **launch** the blueprint into a live iframe; left to **skip**; up to peek the **blueprint JSON & settings**; down to **save it for later**. Everything else in Playground — your saved sites, WordPress-PR / Gutenberg-PR / From-GitHub / Blueprint-URL / Import-.zip launches, the Site Manager (Files, Blueprint, DB, Logs), the settings popover, the address bar, refresh, save, export — is reachable through a tab bar at the bottom that mirrors a dating app (Discover, Matches, Live, Tools, Me). The mobile-native feel scales up unchanged on desktop, where the stack sits center-stage and the iframe expands to full chrome the instant you "match".

## What I remixed / merged / removed
- **Merged the launcher's Featured + Full Gallery + Start-a-new-Playground rail into one deck.** The first cards in the deck are the six "new playground" launches (Vanilla, WordPress PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip) styled as black "wildcard" cards; then the 43 real blueprints follow, filtered by category chips that work like Tinder's distance/age sliders. "Featured" is just the default order.
- **"Your Playgrounds" becomes Matches.** Saved sites are conversations you can return to. Matches view also surfaces the WP Admin / Homepage / Export-to-PR / Download-.zip actions per saved playground, plus the Site Manager's Files / Blueprint / DB / Logs as inline drawers on each match — the Site Manager dissolves into per-site contextual sheets.
- **The settings popover collapses with the Site Manager Settings tab into a single bottom-sheet "Profile preferences"** opened from the Me tab; the redundant inline "Save site locally" notice becomes a persistent ribbon on Unsaved matches.
- **Top toolbar disappears.** Refresh + address bar + name + save live on the live-session screen as a thin floating bar over the iframe (with a big visible Save heart when unsaved). The "Saved Playgrounds" trigger becomes the Matches tab. Open/close Site Manager becomes a swipe-up gesture on the live card.
- **Logs become a chat-stream feed** under each match (it already reads like a conversation).

## Trade-offs
The swipe metaphor is delightful for discovery but lousy for precision: power users who already know exactly which blueprint they want would rather click than swipe through 43 cards, so I kept a search field above the deck and a "jump to" grid behind the small grid icon. The card-per-blueprint frame also limits how much metadata fits per screen; I solved this with the swipe-up "peek" that shows the full blueprint JSON, target WP/PHP versions, language, network/multisite flags, and Apply+Reset — but it's still one card at a time. Finally, dating-app vocabulary ("Matches", "Likes", "Super Save") is intentionally cheeky and might feel unserious for an engineering tool; the visual design uses muted WordPress-blue gradients rather than hot pink to keep it credible, and every gesture has a button equivalent so the app is fully usable without ever swiping.
