# 008 — Vellum Sage

## Thesis

Vellum Sage reads the v7 #010 model through the eyes of a botanist's field journal or a quiet university press: cream vellum paper (#F5F0E3), sage-green accents (#9CAF88), and ink-green text (#2D3A2D) set in hand-cut Cardo and EB Garamond. The chrome whispers; the iframe is the only object that speaks. Hairlines are 1px of ink-green with small dot or leaf end-caps where dividers terminate. The corner site widget is rendered as a small wax-seal stamp pressed into the paper. The Library overlay borrows the look of a card-catalogue drawer.

## What is locked vs. explored

Locked (carried verbatim from v7 #010): the left rail of five quiet docks, the single swap-content pane, the slim 36px top capsule (refresh · address · name · save-pill · more · fullscreen), the corner site widget in the iframe's bottom-left, the modal Library overlay reachable from `⌘K`, and the mobile mirror with a bottom tab-bar rail. Explored: the visual idiom — every dock label is a small hand-set serif initial rather than an icon; dividers terminate in a small leaf or filled dot; the wax seal replaces the rounded chip; the pane has a subtle SVG vellum noise texture; the destructive Apply+Reset uses a typed-phrase confirmation laid out as a manuscript caption with a diff sheet and an auto-snapshot note. Pane arrangement is the 5-dock variant — Sites, Site Manager, Blueprint, Database, Logs each open their own pane.

## Trade-offs

The serif system is intentionally quiet — power users may want denser monospace stacks. We mitigated by keeping the address bar, file tree, blueprint JSON and log lines in JetBrains Mono so technical content stays scannable. Color contrast on sage hairlines was tuned against WCAG AA on the cream substrate; the active-rail accent uses a deeper ink-green wash rather than a louder hue so it never competes with the WordPress iframe. The wax-seal corner widget is decorative-looking but functional; its click target is sized to 44×44.
