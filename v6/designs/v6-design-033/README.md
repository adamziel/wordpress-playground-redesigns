# V6 Design 033 - Paper Consequence Rail

## Thesis
Paper Consequence Rail uses the assigned TypeUI **paper** style as the product model: Montserrat headings, Roboto body text, PT Mono labels, low 4-8px radii, ink rules, tactile paper sheets, stamped labels, and consequence-led state language. The app idles as a usable WordPress Playground with a real iframe, a slim address slip, and a quiet black paper spine. The spine opens one louder casefile surface, while the seam gauge teaches site focus, balanced, and work focus without adding a manual drag handle.

## Remix / Merge / Remove
I kept the quiet-to-loud IA from V5 #018, but reduced the primary edge strip to Sites, Manager, and a Settings deep link into Manager instead of a separate blueprint or export destination. I kept V5 #023's ratio/focus idea through a visible detent gauge and focus modes, plus an explicit full preview mode that hides all chrome. I borrowed V5 #035 only as styling: elevation, paper shadow, and modal depth clarify hierarchy but are not a navigation model. I used V5 #036's Sites-widget lesson by making Sites own saved identity, all six launch routes, import, resume, featured blueprints, and the full 43-card gallery. I avoided V5 #030's persistent two-row top bar; identity/path/save live in one address slip and destinations live on the edge strip.

## Consequences And Final States
The assigned variation is finished in the reset flow. Manager -> Settings shows the destructive reset warning, opens a confirmation sheet with current and after identity, provides Cancel, gates Confirm behind typing `RESET`, shows reset progress over the iframe, and finishes with the active name changed to `Reset Playground - WP 7.1` plus a completion toast. Save/resume also shows before/after identity: Unsaved Playground becomes `Client demo - local`, with the saved state dot changing from warning to success and the site appearing as resumable local storage.

## Trade-Offs
The address slip plus edge strip is less literal than the current single horizontal toolbar, but it preserves the same controls while avoiding a permanent two-row header. The panel is intentionally one surface, so Sites and Manager cannot be open at once; this keeps ownership clear but means power users switch tabs inside the casefile. The paper visual system risks feeling dense, so mobile turns the casefile into a bottom sheet and keeps the iframe visible behind it instead of making a separate mobile IA. The WordPress surface is still an actual iframe, using an inline page document so GitHub Pages and local file rendering show a credible site instead of a blank cross-origin frame.

## Self-Critique / Revision Note
After the V6 insights pass, I removed a separate blueprint door and kept blueprint discovery inside Sites, made Settings a Manager deep link rather than a third primary destination, and kept export/download inside Manager actions. After the style-system pass, I pushed TypeUI Paper beyond color: typography, mono labels, low-radius borders, paper texture, ink rules, receipt-like before/after state, and document-style modal confirmations all come from the paper skill. Residual risk: the top address slip still has several controls, but the main destinations stay on the quiet edge strip and the full preview mode restores a pure iframe surface.
