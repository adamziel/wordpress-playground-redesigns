v7 · #030 — Depth-Decay Tabs

TAB + 3LD · DEV · MOB · single 48 px chrome row · iframe primacy

A single browser-style tab strip is the only persistent chrome. Each tab is a saved Playground. The active tab is sharp at z0 (0 px blur); tabs one slot away recede to z1 (4 px); further tabs to z2 (9 px); hibernated tabs and the “+ New” door sit at z3 (16 px). Depth is purely decoration — every tab stays click-target sharp — but the eye instantly knows where it is. The URL field lives inside the active tab (browser-of-browsers metaphor) so there is no second row of chrome. The Site Manager slides up from the bottom as the single secondary surface.

Composition. TAB is the only IA primitive: a single browser-style tab strip is the entire chrome. 3LD provides the visual hierarchy inside that strip — adjacent tabs at z1, two-away at z2, hibernated/+new at z3 — so the eye finds the active site instantly. DEV gives the bar its 48 px slim profile and embeds the URL pill inside the active tab rather than as a second row. MOB keeps the same primitives on 390 px width with adapted sizing, never a new pattern. The Site Manager is the one secondary surface that swaps content between Settings / Files / Blueprint / Database / Logs. Apply & Reset uses typed-phrase + diff + auto-snapshot, never a red banner. No two-row chrome. Blur never becomes a click target. No terminal echo.
