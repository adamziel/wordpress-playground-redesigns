K · Affordance audit

Counted live from frame A: WP mark · refresh · URL bar · status pill · Site Manager · Fullscreen · More kebab · Sites widget = 7 idle affordances (Save adds 1 when unsaved, =8). Within budget.

Idle affordances · frame A

  1. 1WP mark (also: refresh hit target)
  2. 2Refresh icon
  3. 3URL / address bar
  4. 4Status pill (status only)
  5. 5Site Manager toggle
  6. 6Fullscreen toggle
  7. 7More menu (kebab)
  8. 8Sites widget (corner — the only door to launcher / library)
Save materializes only when unsaved (frame J). The 5 Site Manager tabs and 6 launcher sources belong to their surface (Site Manager pane / Sites widget), not the idle chrome — secondary content, not always-visible affordances.

One door per feature

  1. ·Sites · Launcher · Library → Sites widget (only)
  2. ·Settings · Files · Blueprint · DB · Logs → Site Manager pane (only)
  3. ·Save → Save button (materializes when unsaved)
  4. ·Fullscreen → top-bar Fullscreen + F key
  5. ·Export PR · Download .zip · Rename · Delete → More kebab
  6. ·Apply & Reset → Settings → typed-phrase modal
  7. ·WP Admin / Homepage → Site Manager header (not duplicated to chrome)

One secondary surface

  1. ·The 520 px Site Manager pane is the only thing that ever opens beside the iframe.
  2. ·The Library is a momentary overlay, not a persistent surface.
  3. ·The Sites widget is anchored over the iframe — it's not a second pane; it's a corner widget that blooms inward.
No second sidebar. No bottom rail. No two-row top bar.
L · Mobile (390×844)

The CWS widget moves to the bottom (thumb-zone), Site Manager becomes a bottom sheet, Library becomes a full-screen overlay. Same IA — wider phone is desktop, narrower desktop is mobile.
