# Review of v7 #006 — Smart-resize × Epic recede

**Direction:** "Two-pane smart-resize where the secondary pane IS the cinematic Epic launcher at 50/50 when idle, then auto-recedes to 25 % the instant a site boots."

**Composed primitives (claimed → verified):**
- **SR (smart resize)** — claimed → **verified**. Each frame uses `--left / --right` flex ratios (`65fr/35fr`, `75fr/25fr`, `35fr/65fr`, `60fr/40fr`, `40fr/60fr`, `50fr/50fr`, `100fr/0fr`), and a `.seam` + `.gauge` lives at the boundary in every desktop frame. The gauge contains a ratio numeral, a vertical fill column (height keyed to the ratio), a directional arrow, and a snap-marker list (`35·50·60·75`). Seven distinct demonstrated ratios.
- **EPC (Epic launcher)** — claimed → **verified, with the cleanest implementation of the cinematic hero in the round**. Frame A's right pane carries: a "Featured · Today" tag, a 32 px heading ("Coffee Shop, WooCommerce."), one decisive `LAUNCH ⏎` button (the verb made literal), a six-tile sources grid (Vanilla / WP PR / GB PR / GitHub / Blueprint URL / Import .zip), 8 featured blueprint tiles, and a Your Playgrounds list of 3 saved sites. The launcher *is* the pane; nothing else lives there.
- **3LD (visual recede only)** — claimed → **verified, exactly as the spec allows**. The `.epic.receded` class applies `filter: blur(2px) saturate(.7); opacity: 0.5; transform: scale(0.98)`. Frame B shows the launcher mid-recede with that class applied. Depth is the transition, never the click target.
- **DEV (slim top bar)** — claimed → **verified**. Single 44 px row with refresh, urlbar, save-state pill, conditional Save dark button, Fullscreen, More. Save-state pill takes the HTTPS-equivalent slot in the address bar.
- **CWS (light reuse)** — claimed → **partially verified**. The README says "Sites tab carries a live status dot + saved-count" — that's CWS-as-status, but it lives *inside* the compact pane's tab strip, not in a corner of the iframe. README is candid: "I kept the widget inside the pane … the spirit of CWS (status-doubling-as-door) is preserved without spawning a second persistent chrome element." That's an acceptable interpretation — the design preserves the *idea* even if not the *anchor*.

**Affordance count at idle (Frame A):**
The HTML has actual numbered annotations on the canvas labeling all 8: refresh (1), urlbar (2), save-state pill (3), Save button (4, conditional), Fullscreen (5), More (6), Seam gauge (7, passive), LAUNCH verb (8). That's a real-on-canvas count, not just a README claim. 8 exactly, drops to 7 when saved.

**Scores (out of 10):**
- Composition: 10 — SR + EPC is the headline composition the meta line requested, and the launcher *physically* recedes when the seam glides. The two primitives integrate at the boot moment.
- Banned patterns avoided: 10 — single-row top bar; 3LD used strictly as recede transition (never clickable); no terminal echo.
- Restraint: 9 — exactly 8 idle, annotated on the canvas. The pane's 6-tab strip (Sites · Settings · Files · Blueprint · DB · Logs) is treated as one cluster — same edge-of-VEN issue as designs 003/004/005.
- Iframe primacy: 10 — iframe is always the left pane. Frame I shows 100/0 fullscreen with a corner pill and Esc hint. The launcher *never* steals the entire viewport; even at 50/50 the iframe waits with a "The iframe is waiting. Choose a blueprint…" message.
- One-door / one-secondary: 10 — Library/Blueprints are mode-disjoint (launcher pre-boot XOR Sites tab post-boot) — the cleanest "one door" reasoning I've seen. Save = top-bar pill only; the info-notice inline button is explicitly deleted.
- Direction adherence: 10 — "secondary pane IS the cinematic Epic launcher at 50/50 when idle" (Frame A, verbatim). "Auto-recedes to 25 % the instant a site boots" (Frame B mid-glide → Frame C settled 75/25). Every clause is materialized.
- Visual fidelity: 9 — Lucide icons (referenced as `<i class="lucide lucide-*">` though I notice no script tag invoking lucide.js — the icons may render as empty if the CDN script isn't loaded). Inter + system stack. Real blueprint JSON in Frame G with `installTheme`/`installPlugin`/`importWxr`/`setSiteOptions`/`runPHP` steps. Realistic PHP log lines in Frame F. functions.php editor in Frame E. SQLite tables in Frame H. 16+ blueprint cards in Frame D. 3 saved playgrounds.
- Synthesis quality: 10 — the recede *is* the synthesis point: SR animates the seam, EPC animates its own opacity/scale/blur, both happen on the same event (boot). One animation, two primitives. That's v7's thesis made concrete.

## What works

- The "one IA spine, two pieces of clothing on it" framing is sharp and load-bearing: the pane's content (launcher vs 6-tab strip) is *mode-disjoint* with the running state, which solves the "no triple doors to Library" problem cleanly.
- Frame B's mid-glide is the only frame in the round that captures an actual *animation moment* in stills: seam at 65/35 (mid-snap from 50/50 to 75/25), launcher with `.receded` class applied, live boot-step log visible in the iframe. The other designs show end-states; this one shows a transition.
- 8 idle affordances are annotated *on the canvas* with numbered badges and leader lines — the critic doesn't have to count; the designer counted in-frame.
- The compact-pane Sites tab has the "live dot + saved-count" CWS treatment that the README claims, so the door is at least signposted even though the widget isn't corner-anchored.
- The save flow has three explicit safety layers (typed-phrase, auto-snapshot, ⌘Z within 30 s) per the README's Frame C — more conservative than designs 003/004/005.
- Lucide icons referenced consistently throughout. JSON in Frame G is genuinely real schema-shaped, not pseudo-JSON.

## What's weak or missing

- The Lucide CDN script tag is not present in the file (I searched for `unpkg.com/lucide` and `lucide.umd` and they don't appear — only `<i class="lucide lucide-*">` markers). If Lucide isn't loaded, those tags render as empty inline elements. Check whether the design relies on `<i class="lucide …">` rendering correctly without a script. (Designs 003 and 004 explicitly include `<script src="https://unpkg.com/lucide@latest/dist/umd/lucide.js"></script>`.)
- The CWS claim is the weakest link. "Sites tab inside the pane with a live dot" is not what v5 #036 specified ("a small frosted widget in a corner that … expands inward as a panel anchored to that corner"). The README candidly admits the deviation — if you can't actually meet the spec, dropping the claim is cleaner than restating it.
- Frame B's seam is at 65/35 but the README says "seam glides 50/50 → 75/25 mid-transition" — 65/35 is on the path but it's an in-between point, not a snap detent. Critic-level question: does SR allow continuous values between snaps, or does it skip directly from 50/50 to 75/25? The gauge column fill at 50 % (matching the ratio numeral 50/50) contradicts the actual flex ratio (65/35) — small inconsistency.
- 6 pane tabs again. Same VEN drift as designs 003/004/005. Sites + Settings/Files/Blueprint/DB/Logs is two different scopes in one segmented control.
- The Mobile section (frames Mobile A/B/C) is described in the README but I haven't verified it has the same recede + content-swap. Quick HTML scan shows the markup is present (lines 1896+), but the recede CSS class needs to actually apply to the mobile launcher pane.

## How to push harder

- Add the Lucide script tag or replace `<i class="lucide …">` references with inline SVG so icons render reliably in stills. (Same fidelity reason designs 003/004 include the CDN.)
- Drop the CWS claim from §2 — the design is strong with SR + EPC + 3LD-visual + DEV. Naming a primitive you don't actually anchor weakens the composition story.
- Resolve the Frame B inconsistency: either set the gauge fill column to 65 % (matching the flex ratio) or set the ratio numeral to "65 / 35" if you mean continuous values, or keep "50 / 50 →" + label as "snapping to 75/25" if you mean a snap event.
- Collapse the 6 pane tabs to 5 (drop "Sites" from the running-state tab strip since it's already mode-disjoint as the launcher). This both respects VEN and reinforces the one-IA-spine thesis.

## Lesson for the next v7 designer

When you show a transition (mid-recede, mid-snap), wire every visible number to the same source value — the gauge ratio numeral, the column fill height, and the actual flex ratio should all agree, or the critic catches the rhetoric.
