# Review of v7 #010 — Quiet Rail · Depth Tiers

**Direction:** "Smart-resize is the layout backbone … Inside the secondary pane lives a 48 px quiet rail … Sites lives as a single frosted corner widget over the iframe." (Note: 010's meta.json reads "Composition of QR + 3LD" rather than the smart-resize-headed line — this designer reframed away from SR toward QR + 3LD. The brief allows it, but it is worth flagging.)

**Composed primitives (claimed → verified):**
- **QR (quiet-rail-to-loud IA)** — claimed → **verified**. A 64 px left rail with 5 docks (Sites, Site Manager, Blueprint, Database, Logs) is present in every desktop panel (A, B, C, D, F, G, H). Clicking a dock opens one pane between rail and iframe. The IA pattern is intact.
- **3LD (three-layer-depth visual)** — claimed → **verified, with care**. CSS custom props `--z0 0px / --z1 8px / --z2 18px / --z3 36px` exist and are actually applied via `backdrop-filter:blur(var(--z1/2/3))` on rail, pane, and library overlay respectively (lines 95, 112, 189). Depth is decorative, never interactive — meets the brief's restriction.
- **CWS (corner-widget Sites)** — claimed "light form" → **verified, but minimal**. A 218 px frosted widget at bottom-left of the iframe shows "Coffee Shop · WP 6.7 / PHP 8.2 · Running" and is wired as a door to the Sites pane. It's small and only appears in panel A; in B/C/D/E/G/H it's gone because the pane took its slot. The README is honest about this duplication with the rail dock.
- **DEV (slim top capsule)** — claimed → **verified**. Single 36 px capsule with refresh, address bar, page title, save-pill, ⋮, and an explicit "Fullscreen F" button. No two-row chrome.
- **VEN (≤ 5 items per pane)** — claimed → **verified**. Site Manager has exactly 5 tabs; Sites segmented control has exactly 3; Library sidebar has exactly 10 categories.
- **MOB (mobile = wider mobile desktop)** — claimed → **verified**. The 390×844 mobile in panel I rotates the rail to a bottom tab bar with the same 5 docks, the pane becomes a bottom sheet with z2 blur, and the CWS becomes a horizontal capsule above the rail. Same IA.
- **SR (smart resize)** — NOT claimed by 010 (the design dropped its assigned SR backbone and uses QR + 3LD instead). The deviation from meta.json is conscious but is a notable miss from the assigned direction.

**Affordance count at idle (panel A):**
By strict count of the HTML in the idle panel A: 1 refresh + 1 address bar + 1 page-title pill + 1 save-pill + 1 More menu + 1 Fullscreen button + 5 rail docks + 1 CWS widget = **11**. If "rail as one Dock-like surface" is granted (Apple precedent), drop the 5 docks to 1 = **7**. The save-pill is arguably status not a verb; if discounted, **10 / 6**. README admits "9 by strict reading." Real number is closer to 11 than to 6. Over budget either way; documented honestly.

**Scores (out of 10):**
- Composition: 9 — six primitives (QR + 3LD + CWS + DEV + VEN + MOB) actually wired through, not just name-dropped.
- Banned patterns avoided: 9 — single-row capsule, depth used as hierarchy not control, no terminal echo. Save-pill replaces the red banner.
- Restraint: 6 — eleven discrete idle controls is over the ≤ 8 budget. The "rail counts as one" trick is plausible but doesn't survive a strict count.
- Iframe primacy: 8 — iframe is the only z0 sharp surface in panel A and dominates the canvas; explicit "F" fullscreen exists. But the iframe-as-wallpaper still has a top WordPress bar collage that competes for attention.
- One-door / one-secondary: 8 — Sites is reachable via rail dock AND CWS widget AND ⌘K; the README calls that "symmetry" but it's a duplication. One pane swaps content cleanly otherwise.
- Direction adherence: 5 — the assigned direction line led with smart resize; this design dropped SR entirely. The depth-tier thesis is strong but it's not what was commissioned.
- Visual fidelity: 9 — real PHP log lines incl. timestamps and ms timings, real blueprint JSON syntax-highlighted with line numbers, 20 library cards, four save-states (unsaved/saving/saved/booting), typed-phrase confirm modal with diff. Inter + JetBrains Mono via CDN.
- Synthesis quality: 8 — the 3LD-quantifies-QR-distance idea is the integration thesis and it lands. The CWS is the weakest link because it's tiny and only visible in one panel.

## What works

- The quantified z-tier table is not just rhetoric: `blur(var(--z1))`, `blur(var(--z2))`, `blur(var(--z3))` are actually applied to rail / pane / library overlay. The user can see the rail at 8 px and the library overlay at 36 px and feel the depth hierarchy.
- Panel D's Apply+Reset escalation is the cleanest in the round so far: visible diff (PHP 8.2 → 8.3, WP 6.7 → 6.8-RC1), pre-flight auto-snapshot banner, and typed-phrase ("Coffee Shop") confirm with the confirm button at 0.55 opacity until the phrase matches.
- Real PHP log content: `strftime() is deprecated`, `include() failed`, `200 (212 ms · 41 queries · 18 MB peak)`, and a SAVEPOINT line tied to the snapshot in panel D.
- The blueprint JSON in panel G is syntactically real, uses the right resource verbs (`installPlugin`, `installTheme`, `runWpInstallationWizard`, `importWxr`, `setSiteOptions`), and is wired to a tree where `attached-files/products.json` is the actual VFS target of `importWxr`.
- 8 visible desktop states (A–H) plus a mobile pair plus a tier-legend column. That's serious coverage, not just three states.

## What's weak or missing

- **The design dropped its commissioned primitive.** meta.json says "Smart-resize is the layout backbone." There is no smart-resize, no momentum gauge straddling a seam, no 35/65 snaps. The 56 px / 64 px rail + 380–780 px pane is fixed-width per panel. If the brief said SR is "the entire primitive is gold — reuse it" and the meta line names SR explicitly, dropping it for QR + 3LD costs direction-adherence points.
- **Idle affordance count is ≥ 10 unless you redefine "affordance."** The README is candid but the rubric is binary: the rail has five visible click-targets, each with its own hover state and tooltip, each going to a different content surface. That's five affordances by any honest count.
- The CWS widget is only present in panel A (idle). The moment the pane opens it vanishes. The whole point of CWS is it's *persistent* live status — disappearing when the pane opens defeats the corner widget's job and gives back the symmetry the README claims.
- The "save-pill" in the top capsule is the right pattern, but the page-title `☕ Coffee Shop` immediately to its left in the same capsule duplicates the CWS widget's main label. Two things are simultaneously showing "this is Coffee Shop."

## How to push harder

- Either reintroduce smart-resize as the layout primitive (with momentum gauge) or rename the design to be honest about composing QR + 3LD only — the meta.json line is misleading future readers.
- Keep the CWS *visible* even when the pane is open — shift it to anchor at the bottom-right corner of the iframe so the live-status promise survives content swapping. Right now it's a feature that flickers off.
- Cut one of the duplicates: either the page-title in the capsule OR the CWS widget — they say the same thing. Buying back one affordance gets you to the budget.
- Tighten the affordance count: collapse "Database" and "Logs" docks into Site Manager (the README hints they're convenience shortcuts to the same pane), leaving 3 rail docks total. That gives you a defensible 6-affordance idle count.

## Lesson for the next v7 designer

If your meta.json names a primitive as the backbone, your index.html must *show* that primitive on the seam — don't quietly swap your assigned primitive for the one you find easier to draw, because the critic will read the meta line and grep the HTML for it.
