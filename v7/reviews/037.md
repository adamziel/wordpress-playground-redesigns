# Review of v7 #037 — Sites widget · Smart-resize tiles everywhere else

**Direction:** "The Sites widget is the only 'chrome chrome' you ever see. Everything else — address, save, Site Manager — lives as a smart-resize tileset in a single living mosaic. The iframe is just the largest tile by default."

**Composed primitives (claimed → verified):**
- **CWS (corner-widget Sites)** — claimed → **verified, and is the load-bearing primitive**. The `.cws` is a 300×62 frosted glass capsule at top-left, persistent across Panels A/B/C/D/E/F/G/H/I/J/K/M/N. Shows live status (`Coffee Shop` + pulse + `WP 6.7 · PHP 8.2 · running · 142 MB OPFS`) AND is the canonical door to Sites (`⌘K`). When expanded (Panel B) it grows in place to a Yours/New/Featured 3-segment panel anchored to its corner with a pin glyph. The widget *survives* fullscreen by retreating to a 18×18 frosted dot (Panel L) — never disappears.
- **SR (Smart Resize)** — claimed → **verified, with the most rigorously documented ratio table in v7**. Panel A shows two seams with explicit `.gauge` readouts: `SR 70 / 30 → iframe major` (vertical seam) and `SR 78 / 22 ↑ iframe vertical` (horizontal seam under the manager-glance). Panel M's audit page contains a full SR snap-ratio table: idle 70/30 + 78/22, iframe 75/25, pane 35/65 / 40/60, parity 50/50, boot 60/40, fullscreen 100/0. Different panels render different ratios (Panel E = Files at 35/65, Panel F = Blueprint at 40/60, Panel I = Settings pane60). The "touch tile → tile grows" behaviour is asserted in tile-head microcopy (`SR · touch to grow ⌘L`).
- **3LD-visual** — claimed → **verified as quantified tokens**. `--z0 0px / --z1 6px / --z2 14px / --z3 30px` are used as receded-tile blur (z1), CWS frosted (z2), Library scrim (z3). The "receded" class is applied to non-focused tiles in Panels E/F/G/H/I — visible as the blur cue.
- **TAB (pane-internal tabs)** — claimed → **verified**. When a Site Manager tile is grown, its head carries the 5-tab strip (Settings · Files · Blueprint · Database · Logs) with per-domain accent dots. The tabs only exist *inside* the grown pane — never as idle chrome.
- **MOB** — claimed → **verified in Panel O**. CWS rotates to a top-pill on mobile; the SR axis rotates 90° (iframe top, bottom sheet smart-resizes through 4 detents: 16/50/75/100). Same 5-tab strip in the sheet head.

**No top bar.** The brief's banned pattern (two-row top bar) is dodged by *removing the top bar entirely* — everything that would have lived in a top bar (refresh, URL, save-pill, fullscreen, more) lives inside an SR-grow address tile at the top-right. This is the most aggressive "chrome reduction" in v7.

**Affordance count at idle (Panel A):** README claims 8 and enumerates them; the in-panel `aff-badge` reads `8 affordances at idle ✓`. Verified: 1) CWS, 2) Refresh, 3) URL, 4) Save-pill (display), 5) Fullscreen, 6) More, 7) Manager-glance strip (one tile, 5 cells), 8) iframe tile. The Manager-glance counted as 1 is the contestable item (strict count = 12 if you count each glance cell separately); README is explicit about this.

**Scores (out of 10):**
- Composition: 10 — CWS + SR is the headline composition and *both* primitives carry equal weight in the IA. CWS is the sole persistent chrome; SR is the entire layout system for everything else. The composition's asymmetry ("one heavy widget, one light mosaic") is named in the README §1 and renders in Panel A. 3LD/TAB/MOB are scaffolding but each has a named CSS role.
- Banned patterns avoided: 10 — *no top bar at all* (not just no two-row top bar). 3LD is purely visual hierarchy (receded-tile blur, library scrim). No terminal echo (Logs is a one-way reader in Panel G).
- Restraint: 9 — exactly 8 idle affordances. The Manager-glance-as-1-tile convention is documented honestly; the strict reading (12) is acknowledged. The Save button is *display only* at saved state and only becomes interactive when there are unsaved changes — affordance count adapts to state.
- Iframe primacy: 10 — iframe is the largest tile by default (about 76% of pixels at idle per the README), and the mosaic collapses to a single iframe tile in fullscreen (Panel L). CWS retreats to a 18×18 frosted dot in fullscreen, so the path back is always visible. Fullscreen has one canonical anchor (address-tile Fullscreen button + F + Esc).
- One-door / one-secondary: 9 — Launcher/Library = only CWS. Sites = only CWS. Settings/Files/Blueprint/DB/Logs = each one glance cell → SR-grow → pane tab. Save = address tile button + ⌘S (pill is display). Fullscreen = address tile + F. README §3 documents that the only intentional overlap is "Blueprint tab's Download bundle" (Blueprint artifact) vs. "More menu's Download .zip" (whole playground) — different artifacts, justified.
- Direction adherence: 10 — the direction is taken to the letter: "the Sites widget is the only 'chrome chrome' you ever see." No top bar, no rail, no drawer. Every panel demonstrates the composition. The README quotes the direction line verbatim in §3.
- Visual fidelity: 9 — Espresso Bar coffee shop mock with three product cards (Single-origin El Salvador / House Espresso Blend / Cold-brew concentrate) and WooCommerce 9.6.0 hint, 20 of 43 blueprint library cards with real names (Art Gallery / Coffee Shop / Feed Reader / Gaming News / Non-profit / Personal Blog / Personal Resume / Photography / Skincare / University / Create Block Theme / etc.), 10 categories with correct counts (All 43 / Featured 5 / Website 10 / Personal 7 / Content 6 / Themes 6 / Gutenberg 4 / Experiments 3 / WooCommerce 4 / News 2 — matches the brief), real PHP `Fatal error: Call to undefined function wc_get_low_stock_amount()` log line with SAVEPOINT rollback in Panel G, real `coffee-shop.json` blueprint with `runWpInstallationWizard / installTheme / installPlugin / setSiteOptions / importWxr / updateUserMeta / login` steps in Panel F, real Apply+Reset diff with `php 8.2 → 8.3, wp 6.7 → 6.8-RC1, locale en_US → ja`.
- Synthesis quality: 10 — Panel M is a dedicated audit/map panel that lists the 8 affordances, the SR snap-ratio table, the composition tags (CWS sole chrome / SR everywhere / 3LD visual / TAB pane-internal). This is the most rigorous "show your work" frame in v7 so far. The composition is not just claimed; it's diagrammed.

## What works

- **15 panels is the most thorough state coverage in v7 so far.** A · idle, B · CWS expanded Sites Yours, C · CWS New, D · Library, E · Files SR-grow, F · Blueprint SR-grow, G · Logs (with real Fatal error), H · Database (13 tables, mixed post types), I · Settings + Apply+Reset, J · Booting Gaming News, K · Unsaved → Saved transition, L · Fullscreen with CWS-as-dot, M · audit/map, N · More menu, O · Mobile trio. All hard-requirement states are demonstrated.
- **The CWS is the persistent spatial anchor across every state.** It survives the Library overlay (Panel D), the booting state (Panel J), the destructive Apply+Reset modal (Panel I), and fullscreen (Panel L, where it shrinks to a dot but stays present). This is the strongest CWS execution in v7 — most designs hide the widget when a modal is open.
- **The SR ratio table in Panel M is the most rigorously documented SR vocabulary in v7.** Seven distinct ratios with semantic labels (idle / iframe / pane / parity / boot / fullscreen) and a brief description of when each applies. A critic can audit any panel's seam against this table.
- **The mosaic is rendered with three distinct tile types at idle**: iframe tile (largest), address tile (top-right), manager-glance strip (bottom row). Each has its own SR seam. Panel A's two visible `.gauge` readouts show the *two* ratios simultaneously (vertical 70/30 + horizontal 78/22) — the mosaic's bidirectional smart-resize is legible in stills.
- **The Manager-glance strip is a clever fusion.** At idle, each of the 5 cells shows a 1-2-line live summary (Settings: WP/PHP version + locale + network/multisite state; Files: 218 files + unsaved functions.php; Blueprint: 7 steps + first 4 verb names; Database: 13 tables + row counts; Logs: 2 warnings + most recent line). This is dashboard-as-IA — the user reads status at a glance, then SR-grows into the cell they need.
- **Apply+Reset escalation is the most thorough in v7**: diff of changes (php/wp/locale), named auto-snapshot (`pg_snap_20250521_1518_coffee-shop · 142 MB · restorable for 7 days`), typed-phrase confirm (`reset coffee-shop`) with live tick. All three guards together, no small red banner.
- **The fullscreen state (Panel L) is the most committed in v7**: the mosaic literally collapses to a single tile, every seam disappears, and the CWS retreats to a 18×18 frosted dot at top-left with the green running pulse still visible. A single exit pill at top-right confirms the mode. This is the cleanest "iframe primacy taken to extremity" in the cohort.
- **The mobile axis-rotation (Panel O) acknowledges the trade-off** that "the multi-tile mosaic compresses to a single horizontal seam — the tiles-everywhere thesis loses some of its multi-tile-ness at 390 px by physical necessity." Honest.

## What's weak or missing

- **Discoverability of "touch tile → tile grows" is the SR primitive's known cost** and is the design's biggest risk. README §4 acknowledges this and mitigates four ways (microcopy / gauges / blur / per-domain dots), but the user's first-touch moment is still "this looks like a static mosaic — how do I open something?" The microcopy `SR · touch to grow ⌘L` in tile heads helps but the tile content itself doesn't visually invite a click the way a button does.
- **The 5-cell Manager-glance counted as 1 affordance is contestable.** The cells are visually 5 separate items with distinct accent dots, distinct content, and distinct `⌘1`–`⌘5` shortcuts. A strict critic will count them as 5; the README is honest about this but a future designer reading this should know that "one tile, five fold-outs" is a *defended* convention, not an obvious one.
- **The address tile is also a *grow* target.** Touching it grows it (per the `SR · touch to grow ⌘L` label). But there's no rendered panel showing the address tile *grown* — what does the address tile look like at, say, 60/40 with focus? The README doesn't say, and no panel demonstrates. This is a gap in the SR vocabulary.
- **The Library overlay (Panel D) is a z3 modal scrim**, not an SR-grow of a tile. README §4 calls it "mutually exclusive with corner panels" but it breaks the "everything is SR tiles" rule. A strict reading of the direction would want even the Library to be an SR-grow of the CWS panel into the iframe tile. The current implementation uses a modal because of the 43-card content density — defensible but a *partial* exception.
- **The Blueprint Download bundle vs. More menu Download .zip overlap is acknowledged** but the visual presentation makes them look like the same affordance (both `Download .zip` icons, both buttons). A future user might not register that they're different artifacts. README §4 calls this out; it's documented but not resolved.
- **The "no top bar at all" thesis is bold but discoverability-costly.** Users coming from any browser-style app expect a chrome bar at the top. The address-tile-in-top-right is a creative response but it's not where the user's eye trains itself to look. Mitigation is the ⌘L shortcut for focus.
- **The CWS at top-left occludes ~ 2.3% of the iframe** — README acknowledges this and notes that in fullscreen it collapses to a dot. Defensible.
- **The "manager-glance is the only path to the 5 SM domains" claim is strong but means there's no global ⌘1–5 shortcut affordance reminder visible in the chrome.** The shortcuts are documented in the tile-head's `grow-info` label (`⌘1-5`) but a user who closes the glance tile (?) loses the visual hint.

## How to push harder

- Demonstrate the address tile grown by SR. Currently the address tile is rendered only at idle (300 px wide top-right). What does it look like when the user grows it for a long URL or for autocomplete? A Panel showing address-tile at 60/40 with bookmark / autocomplete / shortcut shelf would close the SR vocabulary loop.
- Consider whether the Library overlay should be an SR-grow of the CWS instead of a z3 modal. If the CWS expanded panel at the New segment shows 3 featured cards + a "View all 43" door, and clicking that grows the panel further to a full 43-card library (still anchored to the CWS corner pin), the modal scrim disappears and the design stays pure to the "everything is SR" thesis.
- Add a first-touch onboarding hint. The biggest SR cost is "how do I open something?" — a one-time animation or tooltip on first launch showing the gauge pill sliding toward the iframe tile would teach the gesture without permanently adding chrome.
- Resolve the Blueprint Download bundle vs. .zip overlap by renaming one. "Download blueprint bundle" vs. "Export playground (.zip)" — make the labels explicit. The icons are not enough.
- Document why TAB stays pane-internal instead of being collapsed into the glance strip. If the 5-cell glance is the *door* and each cell SR-grows to a *pane with the same 5 tabs* — does the user always switch tabs *inside* a grown pane, or does clicking a different glance cell switch tabs? The current rendering shows the tabs inside the pane (Panels E/F/G/H/I), but the relationship between "click glance cell" and "click tab in grown pane" needs to be one of "swap, not stack."

## Lesson for the next v7 designer

If your composition's headline is "the only chrome is X, everything else is Y," your fullscreen panel must show *both* X surviving (collapsed but present) AND Y collapsing to nothing — that's the proof-by-extremity that distinguishes a composed thesis from a layout choice.
