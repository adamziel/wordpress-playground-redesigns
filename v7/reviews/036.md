# Review of v7 #036 — Pocket DevTools · MOB + DEV

**Direction:** "Mobile-first DevTools: a single 44 px URL pill at the top and a bottom drawer that swaps Sites / Site Manager / Logs / Blueprint / DB / Settings at three detents (peek / half / full). The mobile form factor is the canonical one; desktop is the same layout stretched wider, proving 'Desktop = wider phone.'"

**Note:** This design has **no README.md**. The `index.html` is 575 lines but only contains a stylesheet header and a single `<div class="doc-head">` with the title and meta-row. **There are zero rendered frames, zero canvas containers, zero phone containers.** The artifact is essentially a stylesheet without markup.

**Composed primitives (claimed → verified):**
- **MOB** — claimed in meta.json → **not rendered**. No 390 × 844 phone container exists. The CSS defines `--bar-h-mob: 40px` and drawer detent tokens (`--drawer-peek/half/full`), but no markup uses them.
- **DEV (slim 44 px URL pill)** — claimed → **CSS-only**. `--bar-h: 44px` is declared as a token but no `.canvas` / `.frame` element instantiates the bar.
- **VEN (≤ 5 per pane)** — claimed implicitly (6 drawer surfaces named: Sites / Site Manager / Blueprint / Database / Logs / Settings — *over* the cap of 5) → not rendered.
- **CWS / SR / 3LD / GLS / TAB** — none mentioned in meta.json, none rendered.

**Affordance count at idle:** N/A — there is no idle state rendered.

## What works

- The CSS token system is internally consistent: dark-mode tokens (`--bg / --page / --panel / --ink / --ink-2 / --ink-3 / --accent / --green / --amber / --red / --pink / --violet`), DEV bar height tokens (`--bar-h: 44px / --bar-h-mob: 40px`), drawer detent tokens (`--drawer-peek: 64px / --drawer-half: 48% / --drawer-full: 88%`). The tokens are sensibly named and would support a real design if any markup existed.
- The heading "the mobile form is the canonical form" is the right thesis for a MOB + DEV composition.
- Inter + JetBrains Mono via Google Fonts is the correct typography choice for a DevTools-style design.
- Lucide icon script is loaded at the top — the design intended to use real iconography.

## What's weak or missing

- **The design is not implemented.** The HTML body contains only a `<div class="doc-head">` with the page heading. There are no panels, no frames, no canvas containers, no phone containers, no SVGs, no iframe mocks, no drawer, no URL pill — nothing.
- **No README.md.** The brief mandates a README with thesis + primitives + remix + trade-offs + self-critique. None exists.
- **The meta.json `summary` claims "13 demonstrated states" and "43 blueprints loaded"** — neither exists in the file. The meta-row chip strip in the doc-head includes "13 demonstrated states · 43 blueprints loaded" as a label, which is false advertising.
- **None of the hard requirements are met:**
  - Five Site Manager tabs reachable with visible content — not rendered.
  - Six "Start a new Playground" sources — not rendered.
  - Full 43-blueprint browser with search + 10 categories — not rendered.
  - Export PR + Download .zip — not rendered.
  - Save state transition — not rendered.
  - Three demonstrated states minimum — not rendered.
  - Mock data — not rendered.
  - Both 1440 × 900 and 390 × 844 — neither rendered.
  - Destructive Apply+Reset escalation — not rendered.
- **The drawer is claimed to swap 6 surfaces (Sites · Site Manager · Blueprint · Database · Logs · Settings) — over the ≤ 5 VEN cap.** If the design were to be completed as stated, it would already be over budget on the per-pane item count.
- **The meta.json title "Pocket DevTools · MOB + DEV"** suggests a composition the artifact does not even attempt.

## How to push harder

- **Render the design.** Add a `.canvas` for 1440 × 900 desktop and a `.phone` for 390 × 844 mobile. Instantiate the 44-px URL pill at the top. Add the bottom drawer at the three detents (peek / half / full).
- **Write the README.** State the thesis (mobile-first DevTools), name the primitives (MOB + DEV), explain how they compose (the URL pill is the DEV chrome at every width; the drawer is the MOB sheet at every width).
- **Resolve the 6-surface drawer count.** The brief's ≤ 5 cap means either Sites moves to the URL pill (e.g., as a leading swatch) or one of the SM tabs merges with another (Blueprint + Files? Database + Logs?). Or the drawer becomes a 5-tab strip with the 6th surface (Settings) reached via the More menu.
- **Demonstrate the three detents at both widths.** A panel at peek (64 px), at half (48% of viewport), and at full (88%). The detent vocabulary is the design's headline; one still per detent is the minimum to prove the IA.
- **Add the hard-requirement states**: working session / booting / launcher / Library / Apply+Reset / save transition / fullscreen.

## Lesson for the next v7 designer

A CSS token file without markup is not a design — primitives must be *rendered* to be composed; declaring `--drawer-half: 48%` in :root is not the same as showing a drawer at 48% in a frame.
