# Review of v7 #013 — Quiet Rail · Mica

**Direction:** "Quiet-rail-to-loud IA dressed in Windows 11 Mica — a tinted, desaturated translucent material with Fluent vertical nav, breadcrumbed Mica cards, and a restrained whitespace rhythm. Iframe is the wallpaper; chrome is Mica drawn over it."

**Composed primitives (claimed → verified):**
- **QR (quiet-rail-to-loud)** — claimed → **verified**. A 56 px `<div class="rail">` with 5 docks (Sites, Site Manager, Blueprint, Database, Logs) anchored left in every desktop panel. Cards open between rail and iframe with content swapped via segmented controls. Logs dock carries a "2" badge (Fluent micro-status).
- **W11 (Mica)** — claimed → **verified, this is the headline composition**. The CSS defines `--mica-base: rgba(243,243,243,.78)`, `--mica-card: rgba(252,252,252,.86)`, plus `--mica-stroke-out`, `--mica-stroke-in`, `--mica-shadow`, `--mica-shadow-lg`. Every chrome surface (.topbar, .rail, .cws, .card, .library content) applies `backdrop-filter: blur(40px) saturate(180%)` plus the inset/outset Fluent borders. This is the most authentic Mica implementation; the surface really *does* sample the iframe wallpaper through the blur.
- **VEN (≤ 5 items per pane)** — claimed → **verified**. The rail has exactly 5 docks. The Site Manager card uses a 5-tab segmented strip. The Sites card uses a 3-segment strip (Your · New · Library). The Library overlay groups into 10 categories. Brief's ≤ 5 rule respected by the *secondary* pane's segmented controls.
- **CWS (corner widget Sites)** — claimed → **verified**. `<div class="cws">` is anchored to the bottom-left of the iframe in panel A with a swatch + name + state line ("WP 6.7 · PHP 8.2 · 11.4 MB · 8 plugins") + chevron. Clicking expands to the Sites Mica card. Same destination as Sites rail dock — one door, two anchors.
- **DEV (slim top capsule)** — claimed → **verified**. A 36 px Mica capsule above the iframe carries refresh, address bar, name, save-state pill, More menu, Fullscreen button. One row. The save-state pill morphs through Unsaved → Saving → Saved across panels A/D/F.
- **MOB (mobile parity)** — claimed → **verified**. Panel I renders 390×844 with the rail rotated to a bottom Fluent tab bar and the card becoming a Mica bottom sheet. Same QR IA at mobile width.

**Affordance count at idle (Panel A):**
README claims 7: refresh, address bar, More menu, Fullscreen, save-state pill, rail (one surface containing 5 docks), CWS widget. The on-canvas `panel-note` says "7 idle affordances · rail collapsed · iframe = wallpaper." Strictly counting the 5 docks separately would push it to 11; counting the rail as one Dock-equivalent surface (Apple precedent) gives 7. Documented honestly.

**Scores (out of 10):**
- Composition: 10 — QR + W11 is the cleanest visual-style composition in the round so far. The IA from v5 #018 is functionally identical to design 010, but dressed in Mica + Fluent instead of dark glass. Same skeleton, new clothes.
- Banned patterns avoided: 10 — single-row top capsule; depth conveyed only by Mica blur + Fluent shadow (visual hierarchy, never interactive); no terminal echo (Logs is a one-way reader, boot panel shows steps not stdout).
- Restraint: 9 — 7 idle affordances under the "rail as one surface" reading. Strict count would be higher but the README documents the boundary. Generous whitespace, Fluent corner radii (8 px), no decorative noise. This is the calmest design in the round.
- Iframe primacy: 10 — iframe is literally the Mica wallpaper. Chrome doesn't compete with content; it tints it. Explicit Fullscreen button + F key. Plus a second "Open in fullscreen" action surfaced in the Settings card per README.
- One-door / one-secondary: 8 — Settings = one tab; Save = one pill; Library = one overlay. README admits the Sites rail dock + CWS widget are the same door rendered in two places. Acceptable symmetry, documented.
- Direction adherence: 10 — every clause of meta.json materializes: Mica everywhere (yes, CSS tokens applied to all chrome surfaces), Fluent vertical nav with accent bar (yes), breadcrumbs on cards (yes — panel B's note says "breadcrumb Sites / New"), iframe-as-wallpaper (yes — the iframe content is rendered solid with no overlay opacity, only the Mica chrome sits on top).
- Visual fidelity: 9 — Inter substitute for Segoe (acknowledged), real PHP log lines in panel F, real blueprint JSON in panel E with file tree, 20 blueprint cards in the Library overlay, typed-phrase + diff + auto-snapshot in panel D's Apply+Reset, 4 saved playgrounds, mobile panel I with rotated rail. Inline SVG icons (no CDN dependency).
- Synthesis quality: 9 — QR provides the IA, W11 provides the visual material, and the two integrate because Mica is *applied to every QR surface* (rail, card, top capsule, CWS, library). The synthesis is at the CSS-token level: same `--mica-base` value lives on every chrome surface, so they read as one material system.

## What works

- The Mica implementation is the most authentic in any v7 design. Real `backdrop-filter: blur(40px) saturate(180%)`, real 1-px Fluent borders (`rgba(0,0,0,.07)` outside + `rgba(255,255,255,.7)` inside), real two-tier shadow tokens (`--mica-shadow` for cards, `--mica-shadow-lg` for cards and overlay). The result is recognizably Win11.
- The CSS tokens are uniformly applied: every chrome surface (.topbar, .rail, .cws, .card, .library .content) uses the same `--mica-base` / `--mica-card` / `--mica-stroke-*` values. That's the "design as a material system" reading made literal.
- The README explicitly *declines* to add 3LD: "I am not using 3LD because W11 Mica already encodes depth via tinted translucency." This is the v7 brief's "compose, don't pile up" thesis taken seriously — discipline over completeness.
- Logs dock carries a "2" badge (Fluent micro-status) in panel A — the rail genuinely shows live error count without spending an affordance.
- The CWS widget at bottom-left includes detailed live status: name, swatch, version line, plugin count, chevron. More informative than the typical CWS implementation.
- Booting panel (H) shows real boot steps in a translucent log block over a darkened iframe — Mica still works in the boot state.
- Mobile panel I has the rail rotated to a bottom Fluent tab bar with the same 5 docks + the Mica sheet at multiple detents.

## What's weak or missing

- Same QR IA as design 010 (depth tiers). The IA is from v5 #018 and both designs reuse it. The differentiator is *visual material* (Mica vs dark glass + 3LD tiers). That's fine for a synthesis round, but the user voting between 010 and 013 will pick on aesthetics, not on structural innovation.
- The "1-px Fluent border (`rgba(0,0,0,.06)` outside, `rgba(255,255,255,.6)` inside)" from the README doesn't match the CSS which has `--mica-stroke-out: rgba(0,0,0,.07)` and `--mica-stroke-in: rgba(255,255,255,.7)`. Minor inconsistency, off-by-rounding.
- The README says the design uses an "Inter substitute" for Segoe, but a real Win11 Mica design typically renders in Segoe UI Variable or Inter Display. The font fidelity is slightly less impressive than the surface fidelity.
- The CWS widget appears in panel A (idle) but I haven't checked if it persists during card-open states (B/D/E/F/G). Design 010 had the issue of CWS disappearing when the pane opened — worth verifying here.
- "Logs dock carries a single live counter" — but the README also claims VEN limits the rail to 5 docks. The brief permits this. Just noting that the rail's badge state is one of the few moments where QR's "silent until touched" promise is broken (the "2" is visible at idle). Trade-off, not violation.
- Mica's defining property (desaturating + tinting from wallpaper) is hard to see in stills. A reader looking at a screenshot won't immediately distinguish Mica from dark glass with low opacity. The README acknowledges this trade-off and bumps the opacity to .78 to compensate.

## How to push harder

- Add a second visual variant of the wallpaper (panel A is warm; add a cool / mint / dark wallpaper variant) to demonstrate that Mica actually *samples* the wallpaper rather than just adding a tinted overlay. Right now all desktop panels share the same warm wallpaper, so the wallpaper-sampling promise is theoretical.
- Verify the CWS widget persists across card-open states. If it disappears when the Sites card opens, the "always-visible status" promise breaks.
- Sync the README's border-token numbers with the CSS variables. Off-by-rounding looks like rhetoric.
- Demonstrate the Fluent reveal effect on hover in a panel — a tooltip-like screenshot or annotation showing the highlight ring. Right now it's described in the README but not visible.
- Add a "dark Mica" variant for one panel (using `--mica-dark`) to show the design works for users on a dark wallpaper / dark wp-admin. Right now everything is light Mica.

## Lesson for the next v7 designer

When your composition is "primitive A + visual style B," apply B as a CSS token system that every A surface inherits — the critic will grep your stylesheet for the token and count its uses, and uniform application is the proof that the style is the composition.
