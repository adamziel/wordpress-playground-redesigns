# Review of v7 #024 — Atlas · Corner door home

**Direction:** "SPT + CWS. Idle is a pan/zoom Figma-style atlas of saved playgrounds. Focus a tile and the iframe takes the canvas; a frosted corner widget at bottom-left becomes the canonical door back to the atlas — it shows the current site's live status AND a tiny minimap of the atlas with the focused site highlighted."

**Composed primitives (claimed → verified):**
- **SPT (spatial canvas)** — claimed → **verified**. `.atlas` has a real graph-paper grid background; `.tile` elements carry real x/y coordinates (Coffee Shop at x:180,y:140; Art Gallery at x:480,y:210; Gaming News at x:780,y:155; etc.). The README enumerates 8 sites with specific coordinates. Connector edges between related tiles. Pan/zoom is the navigation.
- **CWS (corner widget Sites)** — claimed → **verified, with the strongest minimap implementation in the round**. The CWS at bottom-left contains a *tiny minimap of the atlas with the focused site highlighted*. README §5 explicitly states "the minimap is the synthesis with SPT" — the corner widget embeds the spatial layout, so the door home is also a coordinate-aware viewport indicator. Two primitives fused into one element.
- **DEV (slim top capsule)** — claimed → **verified**. 36 px frosted capsule centered above canvas. Refresh + URL + name/save-state pill. One row.
- **3LD (visual only)** — claimed → **verified**. `--z1: 8px / --z2: 18px / --z3: 36px` tokens applied to scrims and backgrounds, never interactive surfaces.
- **MOB** — claimed → **verified**. Same atlas at 390×844 with finger pan/zoom, CWS docks at bottom with pull-handle.

**Conscious non-uses:** SR, QR, GLS, EPC, VEN, TAB, iPAD, W11 all explicitly excluded with reasons. Same restraint discipline as designs 015/023/025.

**Affordance count at idle (Panel A):**
README claims 8 exactly, enumerated in a count-strip overlay: refresh (1), URL (2), status pill (3), Library button (4), New-playground button (5), Zoom cluster (6), Fullscreen (7), CWS widget (8). Atlas tiles are content, not chrome (per the established Apple precedent).

**Scores (out of 10):**
- Composition: 10 — SPT + CWS with the minimap as the synthesis point. The minimap embedded in the corner widget *is* the spatial layout in miniature — same coordinates as the full atlas. The corner widget is therefore simultaneously the "door home" and "viewport indicator." Two primitives fused, with the minimap as the load-bearing element.
- Banned patterns avoided: 10 — single 36 px capsule; depth tokens applied to scrims only (verified — blur never on interactive surfaces); no terminal echo (boot logs in panel C are not duplicated by a visual progress UI).
- Restraint: 10 — exactly 8 affordances, enumerated on-canvas. Atlas tiles classed as content per Apple Dock precedent. README cites this convention.
- Iframe primacy: 10 — site mode: iframe is the entire 1440×900 canvas, only topcap + CWS overlay. Explicit Fullscreen affordance present in both modes (zoom cluster in atlas, labeled pill in site mode). Mobile has dedicated button.
- One-door / one-secondary: 9 — Sites = CWS only. Site Manager = "Manage" sub-door. Settings = Manage → Settings only. Library has TWO anchors (top-right Library button + "View all 43" inside Start-new) — README defends as "both go to the same panel sub-tab." New-playground has two anchors (top-right button + CWS panel) — same pattern. Save = pill only.
- Direction adherence: 10 — every clause materializes: atlas of saved playgrounds (yes), pan/zoom (yes), frosted corner widget at bottom-left (yes), live status (yes), tiny minimap with focused site highlighted (yes). The synthesis described in the meta line is realized in the HTML.
- Visual fidelity: 9 — 8 saved playgrounds with named coordinates, real PHP log lines, real blueprint JSON, real wp_options table, 8+ desktop panels + 3 mobile panels, the minimap visible in every site-mode panel showing the focused site highlighted, Apply+Reset with diff + auto-snapshot + typed-phrase trio, 20 cards in Library overlay.
- Synthesis quality: 10 — the minimap is the single load-bearing synthesis element. README explicitly cites the v7 #010 review lesson ("if your meta names a primitive, your HTML must show it on the seam"), the v7 #011 review lesson ("every state change should ripple through both primitives"), and demonstrates them: the minimap dot position derives from the focused-site state, and the CWS swatch + name + pip + minimap-focus-dot all change together.

## What works

- Multiple v7-internal review lessons applied explicitly. README §5 cites #010, #011, and the §1 DROP rules by reference and demonstrates compliance. The v7 brief's "designs reviewed by prior critics" thesis taken seriously.
- The minimap is the cleanest CWS-as-spatial-indicator implementation in the round. Design 023 has a similar mini-canvas-map but treats it as a "you are here" cue; this one makes the minimap itself the door (clicking it returns to atlas).
- "Mode-disjoint" framing: atlas mode and site mode share the same CWS but the CWS plays different roles in each mode. In atlas mode it's a status pill; in site mode it grows 4 sub-doors (Manage / Library / Logs / Atlas). The widget's expand-in-place panel anchors to the same corner.
- 8 saved playgrounds with real named x/y coordinates — most coordinated SPT implementation in the round.
- Atlas tiles classed as content (not chrome) — defensible by Apple precedent. README is explicit about this counting rule.
- README §2 explicitly lists 8 declined primitives (SR, QR, GLS, EPC, VEN, TAB, iPAD, W11) with reasons — most disciplined non-use section.
- Connector edges between related tiles (Coffee↔Art, Art↔NP, Gaming↔Feed) suggest spatial clustering — a small but meaningful detail that other SPT designs miss.
- README §5 explicitly enumerates lessons applied from prior reviews (v5 #036, v7 #010, v7 #011). This is the v7 brief's "compose by learning from critics" thesis taken to its natural conclusion.

## What's weak or missing

- The Library has two doors (top-right Library button + "View all 43" inside Start-new). README defends as "both go to the same panel sub-tab" — defensible but flagged.
- The New-playground similarly has two anchors (top-right button + CWS panel Start-new). Same defense pattern.
- The minimap at 88 px tall with 8 dots is genuinely small. A user can see the spatial relationship but probably can't *navigate* via the minimap alone. README acknowledges this.
- The spatial canvas is the design's strongest thesis but also its weakest mass-appeal — README admits "users who don't lay out sites in 2D will see a list with extra coordinates noise." Mitigation: panel D shows the same data as a flat list with coordinates as quiet metadata.
- Mobile has only 2 sheet detents vs desktop's pan/zoom. Same compromise as design 015. Acceptable but acknowledged.
- Atlas tiles' opacity-based archival fade (Skincare at .55) is mentioned in README but the rule for what makes a tile "archived" is implicit (recency? user choice?). A small documentation gap.

## How to push harder

- Make the minimap interactive: clicking on the minimap should pan/zoom the atlas to that location. Currently the minimap shows you where you are but doesn't help you navigate. The README admits "the user can't *navigate* via minimap" — promote it to a real navigation primitive.
- Resolve the Library two-anchors by making the top-right button *the same* button rendered in two contexts (a single hover state, same destination, but different visual placement based on mode). This would make the discoverability story tighter.
- Document the archival rule for atlas tile opacity — e.g., "tiles fade after 30 days unopened" — so the visual encoding is verifiable.
- Show the connector edges in action: when the user focuses Coffee Shop, the Art Gallery edge brightens (suggesting "related sites"). Currently the edges are static.
- Add a "Reset atlas layout" action somewhere in the CWS panel, since the spatial layout is user-customized and may need to revert to default.

## Lesson for the next v7 designer

If your composition is "spatial layout + corner widget," embed the spatial layout *inside* the widget at scale — the widget becomes both the door home and the live-state indicator of the spatial dimension, so the synthesis is one element that carries both primitives' essence.
