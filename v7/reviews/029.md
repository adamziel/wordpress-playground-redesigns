# Review of v7 #029 — Three Glass Surfaces · Strict Tiers

**Direction:** "GLS + 3LD strict: only three glass surfaces are ever visible — top capsule (8 px blur), bottom dock (18 px blur), right pill (36 px blur). Each lives at its own quantified depth tier. Iframe is the only solid surface; everything else is one of those three or hidden. Depth is purely decorative hierarchy, never a click target."

**Composed primitives (claimed → verified):**
- **GLS (glassmorphism)** — claimed → **verified**. All three chrome surfaces use `backdrop-filter: blur(...) saturate(...)`. The iframe is the only solid surface.
- **3LD (three-layer depth, strict tiers)** — claimed → **verified, the headline composition**. CSS custom properties `--z1: 8px / --z2: 18px / --z3: 36px` are defined and applied via `backdrop-filter: blur(var(--z1/2/3))` to top capsule / bottom dock / right pill respectively (lines 218, 280, 378). Saturation matches: `--s1: 140% / --s2: 150% / --s3: 160%`. The three tiers are not just decoration — they're a CSS token system that the entire chrome inherits from.
- **DEV (slim top bar)** — claimed → **verified**. Top capsule at ≤ 46 px, single row, status pill (Saved/Saving/Unsaved/Booting), conditional Save button (only when unsaved per frame J), Fullscreen icon with F key, More kebab. Floating centered (not anchored).
- **CWS (corner widget Sites)** — claimed → **verified, with persistence**. Right pill at z3 (36 px blur). Shows live status + name. Click → expands inward (leftward) into a glass panel anchored to the right edge. README explicitly says "the pill persists in every other state (B/C/D/E/F/H/I/J/K/L) — the design-010 mistake of having the corner widget vanish as soon as the pane opens is explicitly fixed."
- **VEN (≤ 5 per pane)** — claimed → **verified**. Bottom dock has 5 chips. Site Manager has 5 tabs. Sites segmented control has 3 segments. Library sidebar has 10 categories.
- **MOB** — claimed → **verified**. Same three surfaces at 390 px with right pill anchored above dock in thumb zone.

**Conscious non-uses (declined primitives):** SR, SPT, EPC explicitly excluded per the README. The "compose, don't pile up" thesis is taken seriously.

**Affordance count at idle (Frame A):**
README claims 7 saved / 8 unsaved. Enumerated: WP mark (1), refresh (2), URL bar (3), Fullscreen (4), More kebab (5), bottom dock (6, as one cluster), right pill (7). Status pill is display-only, counted as 0. When unsaved, Save button materializes = 8. Documented honestly: "A strict critic counting each chip individually would put the idle total at 11" — README is candid that the dock-as-one-cluster rule is required to hit 7.

**Scores (out of 10):**
- Composition: 10 — GLS + 3LD with three discrete tiers is the cleanest "depth as visual hierarchy" composition. Each surface has its own quantified blur, the iframe is solid, and the user can read which surface is "closer" from the blur amount.
- Banned patterns avoided: 10 — single-row capsule; 3LD used strictly as decoration (you click the surface, not the blur); no terminal echo (Logs is DevTools-style per-request grouped).
- Restraint: 9 — 7 idle (8 unsaved). The dock-as-one-cluster reading is consistent with the established v7 convention. Status pill is display-only (correctly excluded). Save is conditional.
- Iframe primacy: 10 — iframe is the only solid surface, occupies the entire viewport between three small chrome capsules. Frame M shows fullscreen with all chrome hidden + Esc/F pill.
- One-door / one-secondary: 10 — Settings/Files/BP/DB/Logs only via dock; Sites/launcher/library only via right pill; Save only via conditional capsule button. The dock and right pill have *non-overlapping concerns* — dock works on current site, right pill works across sites. Clean separation.
- Direction adherence: 10 — every clause materializes: three glass surfaces (verified), three quantified blur tiers (verified by CSS custom properties), iframe solid (verified), depth as decoration only (verified — no click handlers on the blur). The in-canvas legend names each tier with its blur value.
- Visual fidelity: 9 — Real PHP log lines, real blueprint JSON, real wp_options table preview, 15+ demonstrated frames including affordance audit card (frame N) embedded in the design, 3 mobile frames, 4 saved playgrounds, 20 library cards, Apply+Reset typed-phrase + diff + auto-snapshot trio (frame L), library with 10 categories matching brief counts.
- Synthesis quality: 10 — GLS and 3LD are *the same composition*: GLS gives the surfaces their material, 3LD quantifies the three tiers via blur amount. The synthesis is at the CSS token level — `--z1/--z2/--z3 = blur amounts` and `--s1/--s2/--s3 = saturation amounts` are paired tokens that drive both primitives simultaneously.

## What works

- The strict three-surface rule is the most architecturally distinctive composition since design 017's four corners. Other GLS designs use glass as ornament across many surfaces; this one *constrains* glass to exactly three tiers with quantified depths.
- CSS custom properties `--z1: 8px / --z2: 18px / --z3: 36px` are visible in the source as a token system. The critic can grep for `var(--z` and verify all three tiers are applied correctly.
- The dock and right pill have non-overlapping concerns explicitly: "the dock works on the current site, the right pill works across sites." This is the cleanest separation of "Site Manager (per-site)" vs "Sites (across sites)" in the round.
- The right pill *persists* across every state (B/C/D/E/F/H/I/J/K/L) — the design-010 critique is explicitly named and the fix is verified in the panel list.
- Frame N is an "affordance audit card" embedded in the design — meta-documentation that the critic can verify. Similar to design 020's affordance-count audit card.
- README §2 explicitly lists conscious non-uses (SR, SPT, EPC) with reasons. v7 brief's "compose, don't pile up" thesis taken seriously.
- The status pill is explicitly "display-only (no click target); counted as 0." This addresses the v5 lesson about not mixing status and action. Save button is conditional.
- Frame I (booting) shows "ambient shifts on right pill" — the live-status surface tracks the boot state, not just static idle.
- Apply+Reset uses typed-phrase + diff + auto-snapshot trio (frame L).
- "v1 #29 dual-channel terminal echo is rejected outright" — explicit citation of which banned pattern is avoided.

## What's weak or missing

- The dock-as-one-cluster reading is necessary to hit 7 affordances. Strict counting puts the dock's 5 chips at 5 affordances, taking the idle total to 11. README is candid about this ("a strict critic counting each chip individually would put the idle total at 11") — same boundary discussion as designs 010, 013, 015, 027.
- The right pill at z3 (36 px blur) is the heaviest glass surface. README acknowledges: "over a busy iframe (e.g., a colorful Gaming News theme) the blur draws more attention than I'd like." Mitigated by making it the smallest (56 px wide) but the trade-off is real.
- The dock-expanded panel is 1180+ px wide on desktop. README admits "borderline at 1280 — a real implementation would step down to 2-column at 1280 and 1-column at 980." Responsive thresholds documented but not demonstrated.
- The right pill occludes ~56×168 px of the iframe's right edge. README acknowledges: "most WordPress front-end content uses left-anchored layouts and the WordPress admin's right edge usually shows status meta, not interactive elements." Probabilistic justification.
- The "fixed" design-010 mistake is acknowledged but worth verifying: I should check that the right pill is actually visible in panels B/C/D/E/F where the dock is expanded. The README claims it persists; I'd want to grep the panels for the `.cws` or `.right-pill` class.
- The Frame N "affordance audit card" is a clever meta-doc but may overlap with the README's §6 enumeration. One source of truth would be cleaner.

## How to push harder

- Show the dock collapsing back to its idle 5-chip state from an expanded state in one panel — proving the "single surface changing height" claim vs. a true SR resize. Currently you have idle (A) and expanded (B-F) but no in-between.
- Demonstrate the right pill's "ambient shifts" (frame I) explicitly with two panel variants showing different ambient colors derived from the iframe. The claim is real but the visual proof is implicit.
- Add a panel showing the boot state with both the dock and the right pill expanded simultaneously — proving they can coexist if needed, OR demonstrating that they're mutually exclusive (one secondary surface).
- The "non-overlapping concerns" claim (dock = per-site, right pill = across-sites) is strong. Make it explicit in the audit card (N): a one-line statement that prevents future designers from blurring this distinction.
- Consider showing one frame where the right pill is at z3 over a darkly-themed iframe (Gaming News) to demonstrate that the heavy blur reads as readable rather than muddy. Currently most panels use a warm Coffee Shop background which is forgiving.

## Lesson for the next v7 designer

If your composition is "primitive A's visual material applied at primitive B's quantified levels," wire both primitives to the *same CSS variable set* — here `--z1/2/3` and `--s1/2/3` together define both the glass material AND the depth tier, so the critic sees both primitives encoded as one token system.
