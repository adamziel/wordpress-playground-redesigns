# Review of v7 #003 — Smart-resize · Glass

**Direction:** "Smart-resize IA where the secondary pane is a single frosted-glass surface tinted from the iframe's ambient color; the iframe is the only solid surface. The momentum gauge is etched into the glass at the seam. SR + GLS composed; no other persistent chrome surface."

**Composed primitives (claimed → verified):**
- **SR (smart resize)** — claimed → **verified, beautifully**. The body is a CSS grid driven by `--iframe / --pane` custom properties, ratios are set per state (50/50, 75/25, 35/65, 40/60, 30/70, 60/40, 100/0), and each state carries a real gauge component (pill with the ratio numerals, four snap dots, and `F / Esc / ⌘1-6` kbd hints) anchored at `left: var(--iframe)`. Different `data-ratio` attrs (`rest / iframe-focus / pane-focus / fullscreen`) trigger small saturate filters on the yielding pane. This is the clearest SR implementation I've seen.
- **GLS (glassmorphism)** — claimed → **verified, with discipline**. Six ambient presets (`--amb-coffee / -art / -gaming / -nonprofit / -ink / -neutral`) are defined and applied via `class="frame desktop amb-coffee"` etc. The pane background is `rgba(var(--amb), 0.34)` over `backdrop-filter` — so the glass *is* genuinely tinted by the iframe's ambient color, exactly as claimed. Each state demonstrates a different tint. Inset highlights + hairline borders address the glass-over-white legibility risk that the README anticipates.
- **DEV (slim top bar)** — claimed → **verified**. A single ~44 px topbar holds refresh icon, URL bar, name-pill (status), conditional Save button (visible only when name-pill reads "Unsaved Playground" in panel A; absent on panels D/F2 where state is "Saved"), Fullscreen icon, More icon. One row. No two-row chrome.
- **CWS** — explicitly NOT claimed; the README justifies the omission ("the pane already is the live-status canonical door"). I accept that — using CWS just because would be cargo-culting.
- **3LD** — explicitly NOT claimed except as the visual texture of glass. Brief permits depth as decoration but not as a control; this design respects the line.

**Affordance count at idle (panel A):**
By strict count of the topbar + the segmented control: refresh (1) + urlbar (2) + name-pill (3) + Save button (4, conditional) + Fullscreen (5) + More (6) + pane segmented control as one cluster (7). The seam gauge is passive (no click target). README claims 7 at idle, 8 when unsaved — that count holds.

**Scores (out of 10):**
- Composition: 10 — SR is the IA, GLS is the surface, DEV is the bar. Each primitive does exactly one job; none of them fight.
- Banned patterns avoided: 10 — single-row top bar; depth used only as glass texture, never as control; no terminal echoing the UI.
- Restraint: 9 — 7 affordances at idle, 8 when there's unsaved work. The conditional Save button is a clean idea (only visible when actually needed). Pane segmented control with 6 tabs is borderline against the "≤ 5 items per pane" VEN guideline, but the README doesn't claim VEN, so the rule doesn't apply.
- Iframe primacy: 10 — iframe is literally the only solid surface in the whole file. Panel H shows 100/0 fullscreen with all chrome gone except a single Esc-pill. The "F" key is documented in the gauge.
- One-door / one-secondary: 10 — exactly one pane with six tabs; settings + WP Admin + Homepage all fold into pane chips; Export PR and Download .zip live only in the kebab. The README explicitly traces each door.
- Direction adherence: 10 — every clause of meta.json is present: smart-resize IA (yes), frosted pane tinted from iframe ambient color (yes, six ambient classes), etched gauge across the seam (yes, in every panel), no other persistent chrome (yes — only the slim DEV bar).
- Visual fidelity: 10 — Lucide icons via CDN, Inter + JetBrains Mono, real PHP log lines including `strftime()` deprecation / 401 / `Allowed memory size of 134217728 bytes exhausted` / cron TLS handshake timeout, syntax-highlighted blueprint JSON, 18-card library grid with 10 categories matching the brief, four save states, six ambient WP page mocks (Coffee Shop, Art Gallery, Gaming News, Non-profit, boot, neutral).
- Synthesis quality: 10 — SR + GLS + DEV is a three-primitive composition where the three layers integrate cleanly: SR governs *where* surfaces sit, GLS governs *how they look*, DEV is the minimum chrome bar. The etched gauge is literally the synthesis point — a SR primitive rendered in the GLS visual language.

## What works

- The ambient-tint-from-iframe-color promise is *real*, not just rhetoric. Each panel uses a different `amb-*` class and the iframe's `wp-*` background sets a matching gradient — Coffee Shop's iframe is warm amber, the pane's `backdrop-filter` picks up that warm amber, and the topbar borrows it. The chrome demonstrably belongs to the site.
- Panel H (fullscreen 100/0) is the cleanest fullscreen exit affordance in any v7 design: chrome is entirely gone, a single Esc / F pill sits over the iframe corner.
- The seam gauge isn't just a label: it's a pill with the ratio numerals, four snap dots indicating which detent you're on, and a kbd hint row. When the ratio is 50/50 the middle dot is on; 35/65 lights the leftmost; 75/25 lights the rightmost. Watching the gauge across panels A→B→C reads like an animation in stills.
- The conditional Save button (visible only when "Unsaved Playground", absent when "Saved") is a real instance of "one door per feature" — the verb appears only when there's work to do, the status pill carries the rest.
- 12 distinct demonstrated states (A through J + F2 + I mobile), each with a different ratio and a different ambient. This is the densest synthesis-with-discipline in the round so far.

## What's weak or missing

- The pane segmented control has 6 tabs (Sites · Settings · Files · Blueprint · DB · Logs). The brief's VEN guideline says ≤ 5 items per pane. The design doesn't claim VEN so it isn't strictly bound, but six tabs is on the edge and the "Sites" tab arguably belongs at a different level (Sites is the launcher; Settings/Files/Blueprint/DB/Logs are *inside a particular playground*). Folding Sites into a separate door could buy back a slot.
- Mobile is only one state (I), and the SR seam rotates 90° to become a sheet detent. The README is honest about this compromise, but it means SR's "click left → grow" gesture doesn't survive intact on mobile — it becomes "drag the sheet up." Different gesture, same primitive name. Worth flagging.
- The booting state (G) at 60/40 has a sharp iframe with "Cancel boot" content but the iframe surface is still rendered as a solid Gaming News background. In a real boot, the iframe would be black/spinner. Minor fidelity gap.
- "Sites" appears both in the pane segmented control AND in the topbar's More kebab (the kebab lists "Rename · Delete" which are per-site). Not a duplication of door but a small confusion about scope.

## How to push harder

- Either drop the Sites tab to a separate launcher door OR rename the segmented control's "Sites" to something disambiguating like "Library / Switch" so it doesn't read as a sibling of Settings/Files (which operate on the *current* playground).
- Show the gauge actually transitioning between snap detents in one panel — e.g., a directional arrow + a faint trail showing the previous position. Right now each panel is a static ratio; an inter-state arrow would prove the SR primitive is animated.
- Use the ambient color tint on the topbar as well (currently it sits in a dark hairline at the top of the frame). Tinting the topbar from the iframe would make the "chrome belongs to the site" thesis even more visible.
- Replace one of the six pane tabs with a contextual chip in the pane head — e.g., move Logs to the topbar's More kebab as "View Logs ⌘L". That gets you to 5 tabs and respects VEN even though you don't claim it.

## Lesson for the next v7 designer

When a primitive says "two panes always visible," prove it by setting both pane widths via the *same* CSS custom property in every panel — make the gauge a derived label of `var(--iframe)` so the critic sees the data binding, not the rhetoric.
