# Review of v7 #012 — Quiet Rail, Ventura Pane

**Direction:** "A 76 px rail of Ventura icon-and-label pills (not bare icons) is the only persistent chrome on the left edge; when a pill is tapped, a single 380 px Ventura-style pane expands beside it with generous whitespace and grouped settings rows. The iframe holds the rest."

**Composed primitives (claimed → verified):**
- **QR (quiet-rail-to-loud IA)** — claimed → **verified**. A `<nav class="rail">` on the left edge with 4 pills (Sites / Manager / Blueprints / Settings). One pane opens at 380 px (widens to 420 px for Files only). Five Manager tabs ride a segmented control inside the pane, not on the rail.
- **VEN (Ventura sidebar)** — claimed → **verified, with the most Ventura-faithful pill design in the round**. Each `.pill` contains a `.glyph` (tinted 16×16 SVG inside a 30×30 rounded gradient square) plus a `.label` (10.5 px). `data-tint` attribute switches the gradient color per pill (sites=blue, manager=purple, blueprints=orange, settings=graphite) — exactly the macOS Settings.app idiom. Pane uses 22 px gutters and grouped card-wrapped rows per the README.
- **DEV (slim top bar)** — claimed → **verified**. 44 px single row with traffic lights, URL pill (Back + Refresh + scheme + path + Go in one capsule), save-state pill (name + status dot + inline Save), Fullscreen, More kebab. README explicitly says no separate Refresh or Save button — they embed inside the relevant pill.
- **3LD** — README claims "visual only, when destructive Apply+Reset is in progress, iframe gets a soft 2px blur." That's a one-line use; not a load-bearing primitive. Honest scoping.

**Affordance count at idle (Panel A):**
README claims exactly 8: URL pill (1), save-state pill (2), Fullscreen (3), More menu (4), Rail Sites pill (5), Rail Manager pill (6), Rail Blueprints pill (7), Rail Settings pill (8). The Help pill is rendered at `opacity:.45;pointer-events:none` — an *idle visual hint*, not an affordance, which is documented. Strict count = 8. Drops to 7 when saved (Save button hidden inside the pill).

**Scores (out of 10):**
- Composition: 9 — QR provides the IA, VEN provides the visual style. Each `.pill` is the QR primitive's clickable target dressed in VEN's iconography. The pane uses VEN's grouped-card-with-hairline-dividers idiom. The two primitives integrate cleanly because VEN is essentially a Ventura skin on QR's IA.
- Banned patterns avoided: 10 — single 44 px top bar; depth used only for the destructive-confirm-pending state (decoration not control); no terminal echo. README explicitly refuses to "bolt on" SR or CWS because the direction names QR+VEN — that's the v7 brief's "compose, don't pile up" thesis taken seriously.
- Restraint: 10 — exactly 8 affordances, with the Help pill explicitly disclaimed via `pointer-events:none` + 0.45 opacity (so the rule is enforced in HTML, not just rhetoric). Most disciplined affordance count in the round so far.
- Iframe primacy: 9 — iframe gets ~960 px of 1440 when pane is open (67 %), 100 % when fullscreen. Lower iframe primacy than designs that drop the rail to 56 px, but the README discloses the 20 px Ventura tax for legibility.
- One-door / one-secondary: 8 — Settings has two doors (rail · Settings pill = quick popover, Manager → Settings tab = long form). README frames this as "same furniture, two depths — not three separate dialogs," which is more honest than design 010's similar duplication. Blueprints has two doors too (rail pill + opt-in launcher overlay), framed as "escalated, not duplicated." Acceptable but borderline.
- Direction adherence: 10 — the direction literally says "Ventura icon-and-label pills (not bare icons)" — the HTML has `<div class="pill"><div class="glyph"><svg…></svg></div><div class="label">Sites</div></div>` patterns. Each pill is literally a tinted-glyph + label structure. Meta line materialized verbatim.
- Visual fidelity: 9 — tinted SVG glyphs with proper Ventura-style gradient squares, real PHP log lines with level filters (3 errors / 7 warnings / 29 info counts), real Koinonia blueprint JSON, typed-phrase + diff + auto-snapshot triple-guard for Apply+Reset, 3 saved playgrounds, 5 demonstrated states, mobile panel with 3 detents.
- Synthesis quality: 9 — QR + VEN is the cleanest "IA + visual style" composition in the round. The rail pills aren't QR-styled-with-Ventura-paint; they're structural Ventura pills (glyph + label DOM siblings). The pane uses VEN's section-title-uppercase-tracking-wide + card-wrapped-grouped-rows.

## What works

- The `data-tint` attribute on each pill is the cleanest implementation of Ventura's per-section accent color in any v7 design — sites=blue, manager=purple, blueprints=orange, settings=graphite, each tint applied to the glyph background gradient.
- The Help pill at `opacity:.45;pointer-events:none` is the most candid "rule documented in HTML, not just README" I've seen in the round. The critic doesn't have to take the designer's word; the disabled state is in the markup.
- Apply+Reset uses all three escalations at once: typed-phrase confirm + visible JSON diff + auto-snapshot (timestamped before reset). The README explicitly calls this out as the differentiator vs. other QR+VEN designers who'd reach for the same pills.
- Card-wrapped row groups inside the pane (rather than Ventura's looser stacks) is a real refinement of Ventura — the README names this as a deliberate variation.
- README is unusually explicit about *not* composing other primitives ("I refused to bolt those on because the direction names QR+VEN") — restraint as a design choice, documented.
- The save-state pill embeds the inline Save button — single capsule, single source of truth for the saved/unsaved state. Lesson from v5 review canon applied.
- The Files pane widens to 420 px (vs 380 px default) when tree needs room — a small adaptive trade-off rather than forcing a constant 380 px even when content suffers.

## What's weak or missing

- Only **4 rail pills** (Sites / Manager / Blueprints / Settings) plus the disabled Help. Brief allows ≤ 5; this is comfortably under. But this means Manager (which contains all 5 Site-Manager tabs: Settings, Files, Blueprint, Database, Logs) has to be the home for the actual per-site operations. Settings has both a rail pill AND a Manager → Settings tab — one-door violation in README's own framing, even though the README softens it as "two depths of the same door."
- The Blueprints pill at the rail and the "Browse all 43 blueprints" overlay are two doors to the same content. README frames as "escalated, not duplicated" but a critic could call it dual-channel.
- Help is disabled at idle — but "Help & shortcuts" is listed as living inside the More kebab. So the rail Help pill is *also* a redundant door to the More menu's Help item. Three sites for help discovery (More menu, Help pill, and per-pane tooltips) is more than the one-door rule.
- The README claims "ventura tinted icon-pill" but the implementation uses uniform-shaped pills with internal tinted glyphs. macOS Ventura's Settings.app icons each have *unique* shapes (a globe for Network, a brain for Spotlight, etc.) — these pills are all the same rounded square just with different colors. Ventura-flavored but not Ventura-faithful at the glyph level.
- Mobile section is mentioned briefly (Panel F · 3 mobile screens with 3 detents) but I haven't verified the rail rotation behavior matches the desktop pill aesthetic. Quick scan shows mobile section exists.

## How to push harder

- Pick one canonical door for Settings: either the rail pill (quick) OR the Manager tab (long form), not both. The "two depths of the same door" framing is honest but fails strict reading.
- Give each rail pill a unique glyph silhouette (not just a unique tint inside a uniform rounded square). Real Ventura uses *shape* as the primary distinguisher; tint is secondary. Sites = grid of cards, Manager = folder-with-tab, Blueprints = blueprint roll, Settings = gear — each in its own shape.
- Reduce help to one door (the More kebab) and remove the disabled rail Help pill entirely; that frees one rail slot for a future primary action.
- The destructive-confirm-pending blur on iframe (3LD usage) is documented in README but only fires during Apply+Reset. Showing it in a panel with annotation would make the 3LD claim verifiable.
- Verify mobile rail rotates to a *Ventura-styled* bottom tab bar (not a generic bottom tab bar). Mobile fidelity is where Ventura designs often weaken.

## Lesson for the next v7 designer

If you name "icon-and-label pills" in your meta line, make each pill's icon a *unique silhouette* (not the same rounded square with a different tint) — the critic will diff your SVG paths, and tinted-circle differentiation reads as a skin, not as Ventura faithfully.
