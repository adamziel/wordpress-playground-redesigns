# Review of v7 #005 — DevTools dock · smart resize

**Direction:** "Slim DevTools-style top bar (44 px, status-pill saved-state). Below it, smart-resize between iframe (left) and a DevTools-style right dock that swaps Files / Blueprint / Database / Logs / Settings. The seam carries a momentum gauge; the dock collapses to a vertical icon strip at 10 % when iframe is in focus, blooms to 65 % when the dock is engaged."

**Composed primitives (claimed → verified):**
- **SR (smart resize)** — claimed → **verified**. Five demonstrated ratios (60/40, 90/10, 50/50, 35/65, 100/0) using `.seam.r6040 / .r9010 / .r5050 / .r3565 / .r7525` modifiers — and the gauge thumb's `top:` position is genuinely keyed off the ratio class (50/50 → 50 %, 75/25 → 30 %, 35/65 → 70 %, 90/10 → 14 %). The gauge has a rail + a thumb (a real CSS `::before` + `::after` pseudo-element pair) plus a ratio numeral label. Real binding.
- **DEV (DevTools top bar + dock)** — claimed → **verified, and used at the most points**. Single 44 px row at the top. The dock is genuinely DevTools-styled: dark surface (`--dock: #1B2030`), tab strip with an active-tab underline accent (`--dock-accent: #7CA0FF`), a sub-toolbar with chips and a "Find in files…" search input, a monospaced gutter in the code view, request-id grouped log lines in the Logs tab. This is the deepest DEV implementation in the round.
- **CWS (corner-widget Sites)** — claimed → **verified, but moved**. The name-pill in the top bar carries "Coffee Shop · saved" with an animated status-dot, and the README says "the status-pill IS the corner widget here." Plus the collapsed dock's bottom feet show `WP 6.7 · PHP 8.3` as a live-status surface that doubles as the Settings door. So CWS is split into two affordances rather than one corner widget — that's a creative interpretation but not the canonical CWS.
- **EPC (Epic launcher)** — claimed → **verified in row D**. The iframe pane when no site is running carries a cinematic hero with "LAUNCH Vanilla WP" verb + six start sources as chips. The hero disappears once a site boots — not a permanent surface stealing attention.
- **MOB (mobile)** — claimed → **verified**. Mobile mirrors the SR primitive vertically (iframe top 60 %, dock bottom 40 %), with the collapsed dock becoming a bottom nav bar of identical tabs.

**Affordance count at idle (panel A, saved state):**
By strict count: wp-mark (1) + refresh (2) + urlbar (3) + name-pill (4) + Fullscreen (5) + More kebab (6) + dock-toggle (7) = **7**. README claims 6 (folding wp-mark+refresh into "WP mark = refresh hit-target"). Even with the strict 7, that's under budget. The dock tabs sit inside the secondary surface, so they're correctly treated as pane content not idle chrome.

**Scores (out of 10):**
- Composition: 9 — SR + DEV is the cleanest two-primitive composition in the round; the dock's tab strip and toolbar genuinely feel like Chrome DevTools, and SR drives the dock's collapse/bloom. CWS gets stretched thin by splitting it across the name-pill and the dock feet, but the README is honest about it.
- Banned patterns avoided: 10 — single-row top bar; depth never used as control (the design barely uses depth at all — DEV is opaque, not glassy); no terminal echo (Logs is one-way).
- Restraint: 9 — 6–7 idle affordances. The vertical icon strip when collapsed is the same 5 docks rendered as 56 px wide icons — still legible, still under 8 total.
- Iframe primacy: 8 — iframe is the only "document" surface and gets 60–90 % of the width in most states. Slight knock because at idle (60/40) the dock takes 40 % which is more than v5 #023's 25 % rest state.
- One-door / one-secondary: 9 — one dock with six tabs (Sites · Settings · Files · Blueprint · DB · Logs). One row of chrome. README is explicit that the inline "this is unsaved" notice has been removed because the top-bar status-pill is always visible.
- Direction adherence: 10 — every clause of the meta line lands: slim DevTools-style top bar (yes, 44 px), status-pill (yes, not red banner), SR between iframe and right dock (yes, with ratio classes), dock collapses to ~10 % vertical icon strip (yes, 56 px), blooms to 65 % when engaged (yes, ratio r3565). The momentum gauge sits on the seam in every state.
- Visual fidelity: 10 — Lucide icons via CDN, Inter + JetBrains Mono, syntax-highlighted PHP functions.php with WordPress `add_theme_support` / `wp_enqueue_style` / `wp_get_theme()` calls, request-id grouped log lines with TTFB timing, a wp_admin Products table with SKU / Price / Stock columns and 6 realistic rows, a typed-phrase confirm modal with diff + auto-snapshot, 4 saved playgrounds, 16 blueprint cards.
- Synthesis quality: 9 — SR drives dock collapse/bloom; DEV provides the dock's surface and tab grammar. The two primitives integrate at the seam: the gauge is the SR synthesis point rendered in DEV's monospaced/inline-meta style. Where it strains: CWS gets fractured across two non-corner surfaces, weakening the "widget" half of the composition.

## What works

- The dock's collapsed state (panel B, 90/10) is *the* implementation of v5 #023's "whichever pane you don't touch yields" promise. Watching the dock collapse to 56 px with all six tabs reduced to icons + a "WP 6.7 / PHP 8.3" live-status footer is exactly the behavior the brief is asking for.
- Request-id grouping in the Logs tab is a real DevTools idiom and a genuine fidelity moment: each request gets a header row (`GET /shop/`) and the log lines beneath are visually indented and share an ID prefix. This is how Chrome's Network panel works.
- The destructive Apply+Reset modal uses (1) a diff sheet, (2) typed-phrase confirm ("type *coffee shop*"), and (3) an auto-snapshot note all at once — the most complete escalation in the round so far.
- The status-pill genuinely has three states with distinct visual treatment: `.saved` (green dot + green "saved" label), `.saving` (pulsing blue dot + blue "saving" label), `.unsaved` (the README says amber, and the Save button materializes beside it).
- The iframe in panel B (iframe focus) renders an actual wp-admin Products table with real SKUs, prices, and stock counts — not a mock gradient. This is iframe content at high fidelity.
- The Epic launcher hero in row D is bounded: it appears only when no site is running and the README explicitly says it disappears the moment a site boots. EPC avoided as a permanent surface stealing attention.

## What's weak or missing

- The CWS primitive is *named* but barely composed. Calling the top-bar name-pill "the corner widget" is a stretch — the spec literally says "small frosted widget in a corner that expands inward as a panel anchored to that corner." Top-bar pills don't anchor to a corner and don't expand to a panel. The dock-feet WP/PHP versions are closer to CWS-as-live-status, but they also don't expand inward — they just label the collapsed dock.
- The dock has 6 tabs (Sites · Settings · Files · Blueprint · DB · Logs). Same VEN drift as designs 003 and 004 — six tabs in one segmented control crosses the brief's ≤ 5 per pane guideline. README doesn't claim VEN so it's not strictly bound, but it's a recurring v7 issue.
- The README claims SR has "five demonstrated ratios" but the seam class set only covers `r5050 / r7525 / r6040 / r3565 / r9010`. The fullscreen 100/0 state doesn't have a corresponding `.seam.r10000` modifier — it just hides the dock. Minor inconsistency.
- "Back/forward intentionally absent" is defensible for a single-iframe context, but the design also doesn't show how to navigate iframe history. If the user clicks a link inside wp-admin and wants to go back, the only path is browser back, which would close Playground. A small iframe back button would close this loop.

## How to push harder

- Either drop the CWS claim (the design works fine as SR + DEV + EPC + MOB) or move the name-pill to a genuine corner anchor — top-right of the iframe surface, with an expansion-on-click behavior that actually shows saved playgrounds.
- Collapse the 6 dock tabs to 5 by promoting Sites to a separate door (the name-pill is already there). VEN respected, secondary-surface scope clarified (Sites = launcher; Settings/Files/Blueprint/DB/Logs = operations on current site).
- Add a `.seam.r10000` modifier and a "dock minimized" state in fullscreen so the SR primitive is fully expressed across all five demonstrated ratios.
- Consider showing the gauge animating between two snap detents in one frame (a directional arrow + a faint trail showing prior position) so the "momentum" half of "momentum gauge" is visible in stills.

## Lesson for the next v7 designer

If you name a primitive, anchor it in the HTML the way the spec describes; renaming a top-bar pill "the corner widget" satisfies the README but not the grep — the critic will look for `position: absolute` in a corner.
