# Review of v7 #030 — Depth-Decay Tabs

**Direction:** "A single browser-style tab strip is the only persistent chrome. Each tab is a saved Playground. The active tab is sharp at z0 (0 px blur); adjacent tabs recede to z1 (8 px blur); distant tabs to z2 (18 px blur). The URL field lives inside the active tab (browser-of-browsers metaphor). The Site Manager slides up as the single secondary surface."

**Composed primitives (claimed → verified):**
- **TAB (tabs-only nav)** — claimed → **verified, the load-bearing primitive**. `.tabstrip` is the entire navigation surface (no rail, no drawer, no bottom bar). Each `.tab` is a saved Playground; the `+ New` tab is the only launcher door. Five sample tabs in every desktop frame.
- **3LD (visual depth)** — claimed → **verified, applied uniquely**. CSS custom properties `--z0: 0px / --z1: 4px / --z2: 9px / --z3: 16px / --z4: 36px`. Applied as `filter: blur(var(--zN))` on `.tab.active / .tab.z1 / .tab.z2 / .tab.z3`. Critically: blurred tabs *remain clickable* (the README says "the blur is `filter`, not `pointer-events:none`"). Depth is decoration; the tab is still a target. Frame N audits the scale with a 5-tile legend.
- **DEV (slim top bar)** — claimed → **verified, with the cleanest single-row solution**. The URL pill is *inside* the active tab (`.tab.active .urlpill`), so there's no second row. Active tab grows to 42 px and reveals the inline URL field; inactive tabs keep just favicon + pip + title. This is a genuine refinement of the browser metaphor that avoids the v7-banned two-row chrome.
- **MOB** — claimed → **verified**. Frames O/P/Q render the strip at 390 px with tab widths collapsed to ~70 px, the active tab keeping its URL pill (shrunk), close × hidden on inactive tabs.

**Affordance count at idle (Frame A):**
README claims 7 saved / 8 unsaved. Enumerated: WP mark (1), Tab strip (2 as one nav region with multiple tabs), Refresh (3), Site Manager toggle (4), Fullscreen (5), More kebab (6), URL pill inside active tab (7). Save = 8 when unsaved. The README applies "the v5 #018 rail accounting" to the tab strip — defensible.

**Scores (out of 10):**
- Composition: 10 — TAB + 3LD is the cleanest two-primitive composition in the round. The browser-tab metaphor + quantified blur tier are both literal AND structurally bound via the `.tab.zN` class hierarchy. The synthesis point is the active tab's class transition: `.tab.active` removes blur, others get z1/z2/z3.
- Banned patterns avoided: 10 — single-row chrome (URL pill embedded *inside* active tab, the smartest no-two-rows solution in the round); depth never as interactive control (every blurred tab is still clickable, verified in CSS); no terminal echo.
- Restraint: 10 — 7 idle, 8 unsaved. Tab strip counted as one region per the established convention. Hover-revealed inside-tab × buttons explicitly excluded as not idle chrome. Most disciplined affordance accounting in the round.
- Iframe primacy: 10 — iframe occupies 100 % below the 48 px bar; ≈ 60 % when drawer is open; 100 % in fullscreen (frame L). Dedicated fullscreen button + F hotkey + Esc/F exit pill.
- One-door / one-secondary: 10 — Sites/launcher only via `+` tab; Settings only via Site Mgr → Settings sub-tab; Save only via the (conditional) Save button; switch site = tab click; close site = per-tab `×`; Apply+Reset only inside Site Mgr Settings. Export PR / Download .zip live *only* in the drawer head (README explicitly cites design 020's duplication critique as the reason for this discipline).
- Direction adherence: 10 — every clause materializes verbatim: single tab strip = only chrome (yes), each tab = saved Playground (yes, 5 tabs), active tab sharp (yes, `--z0: 0`), adjacent recede to z1 (yes, `.tab.z1`), distant to z2 (yes, `.tab.z2`), URL field inside the active tab (yes, `.tab.active .urlpill`), Site Manager slides up (yes, drawer with 5 sub-tabs).
- Visual fidelity: 10 — Inter + JetBrains Mono via CDN, Lucide via unpkg, real PHP log lines (HPOS deprecation, SameSite cookie warning, `wc_get_low_stock_amount()` fatal, REST 500 body), real wp_options table (14 rows), 12 tables in DB sidebar, real Gaming News blueprint JSON with 8 valid steps, 17 demonstrated frames (A–N + O/P/Q mobile), 16 blueprint cards, 4 saved playgrounds.
- Synthesis quality: 10 — the synthesis is the tab class system. `.tab.active` removes blur AND reveals URL pill AND increases height AND becomes the chrome's URL bar. One element class change drives TAB + 3LD + DEV simultaneously. Most elegant primitive composition in the round.

## What works

- The "URL pill inside the active tab" is the genuinely novel solution to "no two-row chrome." Other designs either omit the URL bar from the slim bar (007 seam-tabs) or include it in a separate slim capsule (020/027/029). This one *makes the URL bar part of the active tab*, so it's structurally impossible to have two rows.
- 5-tier depth scale (`--z0..--z4`) is more granular than other 3LD designs (010/029 use 4 tiers). The extra tier is for the modal overlay backdrop, separating tab-level depth from overlay-level depth.
- The README explicitly cites design 020's "Export PR / .zip duplication" critique and fixes it ("kebab covers Rename / Delete / etc. only"). v7's "designs reviewed by prior critics" thesis taken seriously.
- The README claims "blurred tabs remain clickable" and the CSS confirms: `.tab.z1/.z2/.z3` use `filter: blur(...)` not `pointer-events: none`. The brief's "depth as decoration, never interactive control" rule is verified at the CSS level.
- Frame M (first touch) is the cleanest empty state in the round: a single oversized `+ New Playground` tab as the only chrome, with a cinematic LAUNCH hero. Solves the discoverability problem by making the metaphor's primary action the only visible affordance.
- The tab's status pip (green/amber/grey) carries the saved/unsaved/booting state — same lesson as design 020's CWS pulse but applied per-tab.
- Real refinement of the browser metaphor: active tab is 42 px, inactive is 38 px, hibernated z3 is 32 px. Height encodes depth tier alongside blur.
- The mobile section (O/P/Q) preserves the same depth-decay rule at narrower widths.
- Lucide icons via unpkg explicitly named (no script tag forgotten).

## What's weak or missing

- The active tab's URL pill is tighter than a full-width URL bar would be — README explicitly acknowledges this ("less URL real estate in exchange for a single-row chrome"). For long paths like `/wp-admin/admin.php?page=woocommerce_orders&action=edit&post=14`, the path will truncate. README admits "longest paths get truncated to ~14 chars" on mobile.
- The metaphor "tab = saved Playground" requires user learning. README mitigates with three approaches (the `+` tab as obvious launcher, the status pip making tabs read as "sites with state", the first-touch frame M). All sensible, but a first-time visitor still needs to learn that tabs aren't browser tabs in the traditional sense.
- The tab strip with 5+ tabs at 1440 px width gets crowded. README notes "when the strip is very full (8+ tabs), the depth-decay only goes to z3 — there's no 'z4 tab', just visual squeeze." Acceptable fallback but the maximum capacity isn't explicit.
- The browser-tab metaphor implies that "close tab" loses the Playground. README mentions per-tab `×` for closing, but the consequence (does closing also delete from OPFS?) isn't surfaced. A user expectation gap.
- Inactive tab `×` buttons are "hover-revealed inside-element interactions, not counted as idle chrome." Defensible by the OS-window-close-button convention but worth flagging.
- 17 frames is the densest coverage in the round (tied with design 020). Some frames may overlap; could be tightened.

## How to push harder

- Show a frame with 10 tabs in the strip to demonstrate how the depth-decay handles a heavy-tab state. Currently only 5 tabs are shown — the high-density behavior is described but not verified.
- Add a "Confirm close" mini-modal when a user clicks the per-tab `×` on a saved Playground, surfacing the "OPFS delete" consequence. Real browsers don't need this because tabs are ephemeral; Playgrounds aren't.
- For the truncated URL pill, add a hover-state that expands the pill into a full-width readable strip while preserving the tab structure. Currently the pill is the active tab's full URL space.
- Consider showing one frame with the tab strip on mobile at 8+ tabs — proving the metaphor's limit case.
- The README's reference to fixing design 020's duplication (Export PR / .zip) is excellent v7-thesis discipline. Consider adding a small "fixes" annotation in the README's §1 listing other v7 critiques explicitly addressed.

## Lesson for the next v7 designer

If your single-row chrome thesis requires a URL bar, *embed it inside the active focus element* (here, the active tab) instead of allocating a separate slim capsule — the URL bar then becomes a state of the focus element, and the critic sees no second row by structural impossibility, not just by intent.
