# Review of #078 — wp-admin styled (faux-WP)

**Direction:** "wp-admin styled (faux-WP): makes the Playground chrome look like wp-admin itself (admin bar + left menu) — so the inner iframe feels nested but consistent."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 9/10 · Visual 10/10 · Originality 9/10

## What works
- The wp-admin mimicry is the most rigorous in the gallery: `#1d2327` admin bar, 32px tall with W mark, `#1d2327` left menu at 160px wide with `#2271b1` blue active state, Dashicons-style icons, fly-out submenus mirroring "Posts > Add New", `#f0f0f1` content well, `wp-list-table` for saved sites, `notice notice-success/notice-warning` strip notifications, "Screen Options" pull-down tab for quick edit.
- All 6 §2.2 sources rendered as Add-New entries (lines 880+): Vanilla WordPress / WordPress PR / Gutenberg PR — and presumably the rest. README explicitly enumerates Vanilla / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip in the "Add New" button group.
- Apply+Reset: `<button class="button button-primary">Apply Settings & Reset Playground</button>` (line 632) + "Apply wipes the current site." `danger-note` (line 634).
- Address bar `<input id="addrbar" type="text" value="/wp-admin/index.php" placeholder="/wp-admin or any path…" spellcheck="false">` (line 454) + a duplicate (line 482) — §2.1 item 2.
- "Persistent storage (OPFS)" toggle (line 629) — **brief violation? Let me check.** §2.3 lists Settings as WP version / PHP / Language / Network / Multisite. "Persistent storage" is the §2 Save action, not a toggle. If it's a save-state checkbox, that's an invented feature. Worth flagging.
- WP Admin / Homepage / Additional actions grouped under a "Navigate" item per README. Lines 502 / 581 / 595 show menu items.
- Unsaved-Playground notice: "**This is an Unsaved Playground.** Your changes will be lost on page refresh." + "Save site locally" button (lines 640–642).
- Database / Logs / Files / Blueprint as submenu items of "Tools" per README.
- The "scope:abc/wp-admin/index.php" URL in the iframe wrapper (line 651) — explicit Playground URL scoping, faithful to current Playground.
- README is honest about the confusion risk: "looking exactly like wp-admin — risks confusing users about *where* they are: is this the inner site or the Playground around it? I mitigate this by tinting the outer admin bar a slightly darker hue and labelling it 'WordPress Playground' with a small 'outer' badge, and by drawing a subtle blue inset border around the iframe area."
- "The wp-admin pattern was designed for content management, not for switching between disposable sandboxes" — admitted metaphor cost.

## What's missing or weak
- **Brief violation: "Persistent storage (OPFS)" toggle (line 629)** is an invented setting. §2.3 has WP / PHP / Language / Network / Multisite — there's no "should this persist?" toggle. The Save action is the only persistence verb. Same overreach as #003 and #007.
- **Apply & Reset has no confirm step rendered.** The `button-primary` fires; the danger-note is good but a wp-admin-style confirm modal would be on-metaphor.
- **The outer-vs-inner confusion** is the design's defining risk — README admits and mitigates with tint + badge + border. Verify the "outer" badge is actually visible in the artifact.
- **The Sites / Library mapping into a left menu that originally listed CPTs** — README admits the shoehorn.
- **The PR / GitHub / URL Add-New tiles** are visible (lines 880+) but verify whether they have inline paste inputs.
- **The address bar duplicates** (lines 454 + 482) — verify whether these are the same field or two separate inputs.
- **Originality is high** — overlaps with #077 (Block-editor styled) in the WordPress-self-reference lane. #078 commits to wp-admin's classic chrome; #077 commits to Gutenberg's block editor.
- **The wp-admin pattern dates** — README admits, future WP refresh would require following.
- **Screen Options as the quick-settings popover** — clever metaphor reuse, on-direction.

## How to push harder
- Remove the "Persistent storage (OPFS)" toggle — it's not in §2.3 and adds a setting the brief forbids. Either drop it or label it as a per-session indicator (read-only).
- Render the destructive Apply&Reset confirm as a wp-admin-style modal — WordPress's own "Are you sure you want to delete this?" pattern (with type-the-post-title confirm for plugins, type-DELETE for posts). Use the existing wp-admin destructive vocabulary.
