# Review of #031 — Trello / sticky notes (corkboard)

**Direction:** "Trello / sticky notes: each saved playground is a sticky on a corkboard, draggable. Click a sticky to expand it into the running site."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 7/10 · Visual 9/10 · Originality 9/10

## What works
- Full corkboard metaphor delivered: cork texture, tape-labelled chips, paperclip overflow on the iframe sticky, manilla-folder Site Manager slide-out with coloured paper tabs, post-it toast flags. The visual idiom is consistent and committed-to.
- All 6 §2.2 sources are torn-paper note-cards in the "Stack a sticky" tray: Vanilla WordPress / WordPress PR… / Gutenberg PR… / From GitHub repo… / Blueprint URL… / Import .zip (line 868–873). Each card has a `data-launch` attribute and an icon.
- The address bar is a real `<input id="addr" type="text" value="wp-admin/edit.php?post_type=page" aria-label="Address path">` — wide tape strip in the middle of the iframe sticky. Accessible label included.
- Apply+Reset appears in two places: the popover (line 855) and the inside-folder Settings tab (line 1033). Both labelled "Apply & reset".
- Database carries early-access notice verbatim. Logs has search input "Search PHP error log…". Blueprint has Copy link + Download bundle. Files has tree.
- Unsaved-Playground notice + "Save site locally" button inside the Settings folder (line 1011–1012).
- The README claims the duplicated "Edit settings popover is gone — the popover trigger now opens the Settings tab inside the folder" — but the artifact has both a popover at line 855 AND the inside-folder version at line 1033. Same data, two surfaces. Worth checking which is the canonical one.
- Drag-and-drop is supported visually; README admits "every sticky is also a real button" for keyboard users.
- 43-blueprint index-card binder at the bottom with category tabs + search (line 1207). On-metaphor: "pull a card onto the board" to boot it.

## What's missing or weak
- **Apply & Reset has no confirm step in either surface.** Both buttons fire. The corkboard metaphor invites a "tear this sticky off" interaction — show a peel/tear animation as the destructive confirm.
- **Settings appear in two places** — same critique as most designs. The popover (line 855) and the in-folder Settings tab (line 1033) duplicate WP/PHP/Language/Network/Multisite + Apply&Reset. The README claims the popover-trigger opens the folder; either the popover content shouldn't render OR the folder is the click target.
- **Stickies take more space than rows** — README admits, scrolling at 1440×900. The 43-card binder mitigates by being a pull-out drawer.
- **The "running iframe is the largest, brightest sticky"** is a clever orientation cue, but if a user pins multiple playgrounds, only one can be the running iframe — making the saved stickies just thumbnails. Acceptable but worth noting.
- **Drag-and-drop is essential to the metaphor** — README acknowledges and adds button-equivalent. But the artifact's static state can't show the drag interaction; a small animation or `:active` style would help.
- **The PR / GitHub / URL launcher cards** are note-cards with `data-launch` attributes (the JS at line 1440 even labels "Gutenberg PR (paste PR #)") but the actual paste input flow isn't rendered. §3 flows 11–12.
- **The Site Manager folder uses paper-tab styling** which is "heavy visually" per README — on mobile collapses to a flat bottom-sheet to avoid the craft-fair look.
- **Notifications as post-it flags stacking bottom-left** — fits §2.4 (bottom-left toast region).
- **WP Admin / Homepage jump buttons** — likely inside the iframe-sticky's overflow paperclip, but the artifact's grep doesn't surface them as discrete affordances. The breadcrumb at line 899 shows the URL but no jump buttons.
- **The folder title is editable** (`<input class="folder-title" value="Coffee Shop Demo">`) — playground rename satisfied.
- **The "save tape-strip" idiom for the title is creative**, but the URL bar tape and the title-tape might compete for attention. Two tape elements in close proximity.

## How to push harder
- Render the destructive Apply & Reset as a "tear off this sticky" interaction. Show the sticky lifting + a confirm "Are you sure? This will rip up the current playground" overlay before commit. The metaphor's strongest destructive UX is undelivered.
- Demonstrate the drag-to-pin interaction with a small CSS animation or `:hover` indicator on the index-card binder cards — a static viewer should see the metaphor's interactive promise.
