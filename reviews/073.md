# Review of #073 — Restaurant-menu style

**Direction:** "Restaurant-menu style: blueprints grouped as 'Starters / Mains / Desserts' with prose descriptions and a sommelier-style helper."

**Scores:** Feature completeness 10/10 · Direction 10/10 · IA 9/10 · Visual 10/10 · Originality 10/10

## What works
- Direction faithfully delivered: blueprint gallery as printed menu with Apéritifs / Entrées / Plats du Chef / Sides / Desserts / Digestifs sections. Playfair Display for menu copy, IBM Plex Mono for technical surfaces. Warm cream paper + deep claret + gold accent palette.
- **Every PR / GitHub / URL source has a real inline paste input in the menu** (lines 1334–1355):
  - WordPress PR: `<input value="#62841">` + Open button
  - Gutenberg PR: `<input placeholder="e.g. 56204">` + Open button
  - From GitHub: `<input value="https://github.com/WordPress/twentytwentyfive">` + Open button
  - Blueprint URL: `<input value="https://playground.wordpress.net/blueprints/coffee-shop.json">` + Open button
  - Import .zip: `<input value="coffee-shop-backup-2026-05-21.zip" disabled>` (file picker)
  
  This is among the most complete implementations of §3 flows 11–12 in the gallery.
- All 6 §2.2 sources rendered with poetic "wine list" sub-labels: "house pour" (Vanilla) / "cellar reserve" (PRs) / "à la carte" (From GitHub) / "sommelier's choice" (Blueprint URL) / "brought from home" (Import .zip).
- **Apply+Reset has the most original destructive copy in the gallery**: "**Send back to the kitchen?** Applying these will reset your dish — the running site is remade from scratch." (lines 1085–1089) + Apply & reset button. The metaphor turns destructive action into a familiar restaurant interaction.
- Address bar `<input value="wp-admin/edit.php?post_type=product">` (line 1050) — §2.1 item 2.
- Database (Cellar) / Logs (Service Log) / Files (Pantry) / Blueprint (Recipe) — playful but verifiably labeled per README ("underlying labels stay explicit in tooltips").
- Unsaved-Playground notice + name plate (line 1267) — playground name dot.
- "Sommelier" — rule-based filter helper per README, explicitly *not* AI-driven to stay inside §5 ("no new features"). Brief-aware restraint.
- "Power users who want to skim 43 blueprints quickly may chafe at prose-heavy listings, so the menu also includes a compact search bar and category chips" — mitigation acknowledged + rendered (search input at line 1300).
- README is honest: "The flavor metaphor risks feeling cute past the first visit — to mitigate, the menu copy is the *only* place the metaphor goes hard; technical surfaces stay flat, monospaced, and functional." Hybrid approach.
- The iframe gets a clean white inner mat to maintain boundary with WordPress's own chrome.

## What's missing or weak
- **Apply & Reset has no typed-confirm gate.** The "Send back to the kitchen?" copy is the strongest reframe in the gallery, but the button still fires on a single click. A "Type 'send back' to confirm" inline gate would match #066's bar.
- **Settings appear in a popover** (line 1082) + the Site Manager Settings tab (implied) — README claims "unified into one 'Edit my order' popover that the Site Manager Settings tab references rather than duplicates." Verify it's truly one component.
- **Power users may resent the prose-heavy listings** — README admits and mitigates with search + chips.
- **The "Cellar" / "Pantry" / "Service Log" labels for technical surfaces** — README says they stay explicit in tooltips, but the labels themselves might confuse a first-time power user. Verify hover-labels expose the literal "Database" / "Files" / "Logs" text.
- **The sommelier helper as "three questions"** — verify whether the sommelier UI is rendered as an interactive state.
- **Mobile collapse to folded card + bottom sheet** — described.
- **Originality is sky-high** — overlaps with #074 (Cookbook) in the food/menu metaphor lane, but #073 commits to the restaurant context (Sommelier, dining room, kitchen) while #074 commits to the recipe context (ingredients, prep time, method).

## How to push harder
- Implement the "Send back to the kitchen?" as a typed-confirm gate — type "send it back" to enable the destructive button. The metaphor's existing copy is the best in the gallery; gate it properly.
- Render the Sommelier as a real interactive 3-question state — "What are you hungry for? a) a quick taste b) something seasonal c) the chef's experiments" → 3 filtered blueprints. The README promises it; show it open.
