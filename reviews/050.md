# Review of #050 — Tube / metro line map

**Direction:** "Tube / metro line map: WP and PHP versions as stations on parallel lines. Pick a station to boot."

**Scores:** Feature completeness 8/10 · Direction 10/10 · IA 7/10 · Visual 10/10 · Originality 10/10

## What works
- Direction is fully delivered as actual SVG: a transit map with WP Line, PHP Line, and an External line with Terminus stations for the 6 §2.2 sources. Stations have labels (WP 6.5, PHP 8.2), interchange dots, line colors. Real transit map feel.
- All 6 §2.2 sources rendered as Terminus stations (lines 1196–1216 + line 1267): From GitHub repo / Blueprint URL / Import .zip / WordPress PR / Gutenberg PR / Vanilla WordPress — each with `<g class="station">` SVG groups and click handlers.
- The "End of the line = Reset", "Save your ticket = Save", "Apply Settings = change of platform" vocabulary maps cleanly to §2 actions without invented features.
- Address bar `<input class="url" id="urlbar" value="/wp-admin/index.php" aria-label="Inner site path">` (line 802) — §2.1 item 2 with aria-label.
- Database has the early-access framing implied; Logs has a marquee ticker (line 917) AND a full-panel Logs tab per README. Blueprint pane has the conventional buttons.
- "Save state" indicator "Unsaved · Save your ticket" (line 824) — the metaphor turns the §4 risk into a transit-tickets concept.
- Unsaved-Playground notice: "Unsaved Playground. ... Save site locally" button (lines 884–887).
- "Saved routes pinned to the network with little train icons" per README — saved playgrounds rendered as map pins. On-metaphor.
- WP Admin button as a chrome action (line 839).
- The Service Information panel with tabs (Settings, Files, Blueprint, Database, Logs) styled like a station information board — on-metaphor IA.
- Two designs in the gallery (#049 Maps, #050 Tube) lean on cartographic metaphors; #050's tube-map approach trades realism for clarity (the README admits "real transit map enforces geographic relationships that don't exist here — I treat that as licence, like a London Tube diagram").

## What's missing or weak
- **Apply & Reset has no explicit confirm step rendered.** The "Board train" metaphor is described as openly destructive but the actual confirm dialog isn't shown. Mapping "boarding = commit to destructive change" is fresh; the confirm UI itself isn't visualized.
- **The metaphor is louder than typical productivity UI** — README admits. First-time users have to learn that stations = versions.
- **The PR / GitHub / URL launcher Terminus stations** use `onclick="alert('...')"` (line 1196: `onclick="alert('From GitHub repo')"`) — JS alerts as the boot-flow. The actual paste-PR-number input isn't shown.
- **Logs marquee ticker is "charming but harder to scan than a normal log viewer"** — README admits and provides a full Logs panel as fallback. Mitigated.
- **A real transit map enforces geographic relationships that don't exist** (PHP 8.2 isn't "near" WP 6.5) — README admits, defends as Tube-style abstraction.
- **The map is dense at 1440px but collapses on mobile to a vertical timetable view** — same data, no SVG panning required. Sensible.
- **The "station amenities" mapping of Network / Multisite / Language toggles** is described but verify it's actually rendered as amenity icons next to stations. The grep shows separate "Network ON" / "Multisite OFF" pills (lines 899–900), which is more conventional than amenity-on-station.
- **No two-station selection UI shown for picking WP × PHP** — verify the interactive station-picking flow is rendered. The static SVG shows the lines but the "pick a WP station and a PHP station to set your route" interaction isn't necessarily demonstrated.
- **The Departure list (line 1253) with "wp-beta-shakedown" entries** is great for "your saved routes" but the boot interaction is `alert()` driven.
- **Originality is high** — but overlaps with #049 (Maps). Both are spatial metaphors; #050's "two-line + terminus" structure is more rigid than #049's free-form atlas.

## How to push harder
- Render the station-pick-pair interaction as a demo state — show the user clicking PHP 8.2 then WP 6.7, the line highlighting, and the "Board train? You'll leave PHP 8.3 — your current files survive only if you save first." prompt.
- Replace the `alert()` boot handlers with actual inline boot flows. Click WordPress PR Terminus → "Paste PR number" input slides in. The Terminus stations want first-class input affordances, not modal alerts.
