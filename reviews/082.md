# Review of #082 — Single-line top toolbar, everything else collapses

**Direction:** "Single-line top toolbar, everything else collapses: super minimal. Every panel is a slide-over from one of the four screen edges."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 8/10 · Originality 8/10

## What works
- Direction faithfully delivered: 44px top toolbar with refresh/URL/name/Save + four edge-direction handles. Four edges own roles: Top = quick Settings, Left = Site Manager, Right = Launcher + blueprints + Your Playgrounds, Bottom = Logs / DB / Files / Blueprint editor. Keyboard `[`, `]`, `,`, `\`, `Esc` map to edges.
- All 6 §2.2 sources rendered as new-card buttons (lines 1498–1525): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — with sub-line "Try a Gutenberg PR build" etc.
- Apply+Reset (line 1429): `<button class="btn primary"><i class="ic" data-lucide="play"></i> Apply & Reset</button>` and a second surface at line 1552 "Apply & reset the running site" + Network/Multisite toggles.
- Address bar `<input id="urlbar" value="/2026/05/hello-from-wordpress-playground/" spellcheck="false">` (line 1195) — §2.1 item 2.
- Unsaved-Playground notice with playground rename input (line 1362): "**This is an Unsaved Playground.**" + Save site locally button (lines 1382, 1384).
- WP Admin / Homepage in `qb` quick-buttons (line 1364).
- Download .zip with "Bundling playground.zip…" toast (lines 1325–1328).
- Blueprint search "Search 43 blueprints, your sites, plugins…" (line 1475).
- README is honest: "Collapsing everything to edges costs first-time discoverability — a brand-new user sees a near-empty page and might miss the four edge handles. I mitigate that with a subtle pulse on the left handle, visible toolbar icons mirroring each edge, an onboarding toast that names the shortcuts, and labels on every handle."
- Bottom drawer with tabs (Files / Blueprint / DB / Logs) — single source of truth for the four scrolling-tools, mitigates the §4 file-tree-duplication pain.

## What's missing or weak
- **Apply & Reset has no confirm step.** Both surfaces fire.
- **Settings appear in two surfaces** — quick popover (line 1416) AND Site Manager Settings (line 1572). README admits "kept but visually unified."
- **The PR / GitHub / URL new-cards** are tile cards with `onclick="boot(...)"` handlers; no inline paste inputs.
- **First-time user sees a near-empty page** — README admits and provides pulse + onboarding toast mitigation.
- **Single toolbar gets tight on narrow viewports** — README admits, name chip drops on mobile + panels become full-width sheets.
- **Originality is medium** — overlaps with #059 (Heads-up overlay) in the "chrome appears on edge gesture" lane. #082's distinguishing move is the more deliberate "each edge owns a role" assignment + keyboard shortcuts.
- **The four-edge handles peek out** — verify whether the handle peeks are rendered visibly.
- **No keyboard-shortcut legend rendered** — README mentions `[`, `]`, `,`, `\`, `Esc` but the artifact doesn't display them.

## How to push harder
- Render the four-edge handle peek state — show small visible nubs at each edge so a static viewer sees the affordance.
- Display a keyboard-shortcut legend in the chrome or as a `?` overlay — the keyboard-driven mental model is the design's main power-user claim; expose it.
