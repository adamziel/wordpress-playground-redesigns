# Review of #069 — Channel guide / TV grid

**Direction:** "Channel guide / TV grid: 2D grid where rows are categories and columns are blueprints. Select to 'tune in'."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 10/10 · Originality 9/10

## What works
- Direction faithfully delivered: CRT-style iframe screen, 2D EPG grid (rows = categories, columns = channels), channel numbers (CH 100, CH 101, CH 102…), DVR row for saved playgrounds, remote-control keys, phosphor-green-and-amber aesthetic. Highly committed.
- All 6 §2.2 sources rendered as Input Sources channels (lines 847–852): CH 100 Vanilla WordPress / CH 101 WordPress PR / CH 102 Gutenberg PR / CH 103 From GitHub / CH 104 Blueprint URL / CH 105 Import .zip — each with channel number + sub-line. Plus CH 107 Download .zip, CH 110 Download bundle, CH 111 Reset Apply settings.
- Apply+Reset has a **distinguished pair of buttons**:
  - `<button class="btn danger" title="Wipes the site and restarts">Apply settings & Reset Playground</button>` (line 1035)
  - `<button class="btn primary">Apply (no reset)</button>` (line 1036)
  
  This is the cleanest "destructive vs. non-destructive apply" split in the gallery alongside #052 and #053. The user can choose to apply settings without resetting if they don't change WP/PHP version.
- Address bar `<input value="/wp-admin/edit.php?post_type=post" aria-label="Address">` (line 598) — §2.1 item 2.
- Database labelled "SQLite-backed DB browser · early access" inline as CH 202 (line 939). Logs has filter "Search the log (regex ok)… /WC_/" (line 1195). Blueprint has Run Blueprint / Copy link / Download bundle (lines 1146–1148).
- Unsaved-Playground notice: "You're watching an **Unsaved Playground**. Changes will be lost on page refresh." + Save site locally (lines 1004–1005) — TV-watching metaphor naturally for live ephemerality.
- Playground rename `<input value="Coffee Shop">` (line 978).
- "All Channels" virtual row mitigates the "43 hidden behind horizontal scroll" risk per README.
- README is honest: "the metaphor is loud — it commits to a phosphor-green-and-amber CRT aesthetic that will polarize. Power-users will love the keyboard-channel-surfing; first-time designers may find the channel-number indirection cute but slow versus a plain card grid."
- Mobile: stacks rows vertically + fullscreen mode with GUIDE bottom-sheet — admitted EPG-doesn't-make-sense-on-phone downgrade.

## What's missing or weak
- **Apply & Reset has no typed-confirm gate.** The README claims "CRT-style confirmation dialog" but the artifact shows only the danger button. Verify whether the CRT confirm renders as a separate state.
- **The "channel-number indirection"** is the metaphor's real cost — users have to learn CH 102 = Gutenberg PR.
- **Settings appear in a "Settings menu" sliding from the right** + the §2.3 Settings tab via the EPG — verify the merger is delivered.
- **Database / Logs / Files as "engineering channels"** is a charming reskin but doesn't actually change the underlying UX.
- **The CRT aesthetic** will polarize. README admits.
- **Mobile downgrades the EPG to vertical stack + fullscreen** — admitted real cost.
- **Originality is high** — overlaps with #050 (Tube map) in the spatial-channel-selection lane, but the 2D EPG grid is its own move.
- **The Channel column listing PR / GitHub / URL inputs as channels** still doesn't expose inline paste fields — "tuning in" to CH 101 should reveal the PR number input.
- **The "remote control" buttons** — verify they're rendered as actual buttons or just described.

## How to push harder
- Render the CRT-style Apply&Reset confirm — a phosphor-green "TUNING…" overlay with "Confirm reset? Press OK to continue" + a typed-channel-number gate would land the metaphor's destructive moment.
- Make "tuning in" to a PR Input channel actually reveal an inline PR-number paste field on the EPG cell. The channel-number metaphor invites a "channel selector" that takes a number.
