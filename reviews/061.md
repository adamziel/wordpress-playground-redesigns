# Review of #061 — Time-machine

**Direction:** "Time-machine: visual rewind through the playground's state history. Stacked-pages-receding effect."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 9/10 · Visual 10/10 · Originality 10/10

## What works
- 3D receding stack of state frames behind the live iframe, bottom scrubber, time-machine vocabulary applied consistently. The stacked-pages-receding aesthetic is on-direction.
- **The destructive Apply+Reset reframe is the strongest in the gallery alongside #052**: "Apply spawns a new frame so this one is never destroyed — rewind anytime." (line 986) + a single "Apply & branch" button (lines 844, 983). Destructive in concept doesn't exist anymore — every "destructive" action just adds a new frame to the history.
- All 6 §2.2 sources rendered as Frame Archive entries (lines 1202–1207 in nav + lines 1280–1283 in cards): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each with descriptive sub-line ("Boot the build artifact of a wordpress-develop pull request").
- Address bar `<input value="/wp-admin/themes.php">` (line 769) — §2.1 item 2.
- **The §4 unsaved-state risk gets a brilliant treatment**: "Unsaved drift since last pin · 12 min" state-chip (line 794) + a callout "**Unsaved drift.** This frame hasn't been pinned to browser storage. Refresh now and frames #6–#7 are lost." (line 966) with a "Pin 'Coffee Shop · v2'" button. Drift-since-last-pin is a smart metaphor for unsaved changes that integrates with the time-machine model.
- Database has implicit early-access; Logs is a *temporal feed correlated with frame timestamps* (line 1113: `<em>· now</em>` annotation) — clicking jumps the scrubber, per README; Blueprint has Copy link button (line 1067).
- Save = "Pin this frame" — addresses §4 by reframing save as a snapshot action with a strong visual metaphor.
- README is honest: "Snapshotting every action implies storage cost we haven't accounted for; in real implementation, frames would be lazy diffs against the last save, not full OPFS clones." Technical caveat acknowledged.
- "First-time users may not immediately read the receding cards as 'older states,' so we lean on a subtle parallax animation and clear timestamp labels" — discoverability mitigation.
- File Browser + Blueprint tab merged into one "Source" view with `blueprint.json` highlighted as root manifest — fixes the §4 file-tree-duplication pain.

## What's missing or weak
- **The 3D stack is striking but consumes vertical pixels** — README admits, collapses to thin breadcrumb on mobile + behind a toggle on desktop.
- **Settings appear in an "Inspector drawer pinned to the right of the current frame"** — README claims merging the three entry points into one inspector. The artifact has multiple settings surfaces (lines 839, 976) — verify whether they're the same component in different states or two separate panels.
- **Treating PR boots / blueprints / zip imports uniformly as "foreign timelines"** is clarifying but buries advanced flows one click deeper — README admits.
- **Snapshotting every action implies storage cost** — README admits and proposes lazy diffs.
- **The time-machine metaphor is opinionated** — first-time users may not parse receding cards. Parallax + timestamps mitigate.
- **The "Apply & branch" pattern** is the strongest UX safety claim — but it implicitly relies on the scrubber being able to actually rewind. The README admits "in this mock it's a conceptual artefact rather than a true rewind" — same hedge as #060.
- **The PR / GitHub / URL "foreign timeline" cards** have sub-lines but no inline paste inputs.
- **Originality is high** — overlaps slightly with #051 (Git timeline) and #052 (Branching tree) in the "history-as-IA" lane. #061's distinguishing move is the 3D receding stack + time-as-scrubber, which is more visually committed than #051/52's vertical/horizontal node graphs.
- **The "Source" view merging Files + Blueprint** is a clean §4 fix but means power users editing both lose a side-by-side view they had today.

## How to push harder
- Demonstrate the scrubber actually rewinding — show the receding stack reorganizing as the user drags the scrubber. Even a couple of state-snapshots would prove the metaphor's central claim.
- Make the "Pin this frame" button persistent in the chrome (always visible, pulses when there's drift). Currently it's inside the unsaved callout — promoting it to the chrome the way #043 (children's book) promoted the Save heart would close §4 risk.
