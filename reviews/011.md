# Review of #011 — Newspaper / magazine layout

**Direction:** "Newspaper / magazine layout: serif headlines, multi-column. The active site is the 'lede' article; blueprints are sidebar features."

**Scores:** Feature completeness 8/10 · Direction 10/10 · IA 6/10 · Visual 10/10 · Originality 10/10

## What works
- Most committed and visually distinctive direction execution I've reviewed. Playfair Display serif headlines, IBM Plex Mono datelines, drop caps, hairline rules, column rules, kicker tags ("Stylebook § Settings", "Features Section § B1", "Copy Desk § C1"), and section flags exactly the way a broadsheet works. The metaphor is doing real IA work, not just decoration.
- Every §2 feature is mapped to an editorial role with explicit fallback labels: Stylebook = Settings, Syndication = additional actions (Export to GitHub PR, Download .zip, Copy link to Blueprint, Run Blueprint), Weather Bureau = toasts, Copy Desk = Files, Editorial Brief = Blueprint, Records Bureau = Database, Police Blotter = Logs, Classifieds / Local Edition / Features = Saved Playgrounds + blueprint gallery + start-new. README explicitly addresses the cuteness risk by noting "underlying labels stay explicit on hover and in mobile" — a thoughtful trade-off.
- Address bar is a real `<input value="/wp-admin/edit.php?post_type=post" aria-label="Address bar">` masquerading as a dateline — and the aria-label keeps it accessible. §2.1 item 2 satisfied with the strongest typographic disguise.
- The destructive Apply+Reset is rendered as a "Stop the presses" galley-proof callout with the copy "Applying these changes will reset the playground. Your iframe state will be lost. Save the current edition first if you want it on file." + a "Save site locally first →" button inline. That's exactly the dual safety pattern the brief asks for at §4.
- Settings live in only ONE Stylebook rail. The README claim ("merged into a single right-rail Stylebook column visible on the front page") is actually delivered — no popover duplicate. Cleanest collapse so far.
- All six §2.2 sources represented in the Classifieds dropdown (WordPress PR, Gutenberg PR, From GitHub, Blueprint URL — and presumably Vanilla + Import .zip in the same block; only WP PR / Gutenberg PR / GitHub / URL visible in lines 495–507, but the Features section also calls out 43 blueprints with search + 10 category chips matching §2.2 exactly).
- Logs section (Police Blotter) has search-mini input, level chips (info/warn/err), real example log rows with timestamps.
- Mobile section described as "tabloid-style stacked feed with a bottom paper-edge nav" — and line 913 has a mobile `<input value="/wp-admin">` plus an Unsaved Playground card. Mobile renders.

## What's missing or weak
- **Three states minimum is met by sections (Lede / Booting elsewhere / Features), but the booting state is only implied in step text (line 863).** A dedicated "edition is going to press" booting layout would be a missed opportunity given how rich the metaphor is.
- **The "Stylebook" rail is the settings popover AND the Site Manager Settings tab merged, but the artifact contains BOTH the Stylebook AND a `<select>` row inside the masthead for "Playground name" (`contenteditable` span at line 600).** Two name-edit affordances side by side.
- **The "Vanilla WordPress" entry is not visibly listed** in the Classifieds block I read (lines 495–507 only show WP PR / Gutenberg PR / From GitHub / Blueprint URL). Missing in the static HTML or buried lower; needs verification.
- **No confirmation step for Apply & Reset.** The warning callout is excellent, but the "Apply & Reset ↺" button still fires immediately. Compare to a real newspaper galley-proof: there's a sign-off step before the press runs. The metaphor invites a "Editor's signature required" modal that the design doesn't render.
- **Database / Records Bureau is described in the README but I see no early-access notice** in the static HTML grep. §2.3 tab 4 mandates the "Database management is an early access feature" notice with the GitHub link. Worth verifying.
- **The 43-blueprint grid is rendered by JavaScript (`#bp-grid` line 662).** Without JS the gallery is empty — same JS-dependency hazard as #002, #003, #009.
- **The "newspaper IA collapses what was previously hidden behind tabs into a long-scroll page"** trade-off is admitted, but a first-time user scrolling a broadsheet to find the Database section won't know it's labelled "Records Bureau" until hovering. Even with the front-page nav strip, the cognitive load to map "I need Database" → "scroll to Records Bureau" is non-trivial.
- **The "Reload Edition" / "Print This Issue" copy** is delightful but the §4 risk that users miss saving is *worsened* by replacing "Save" with "Print This Issue" — saving has a discoverability cost the metaphor introduces. The README's mitigation (explicit labels on hover) helps but it's a real friction tax.
- **No WP Admin / Homepage jump buttons in the masthead.** The dateline is the URL bar but §2.3 also requires two explicit jump buttons. They might be elsewhere in the long scroll but they're not adjacent to the lede.

## How to push harder
- Add the "sign-off" / "to press" confirmation step. The galley-proof callout invites a modal that says "Editor-in-Chief: sign here to run the presses ____ [type 'reset' to confirm]". You're one detail away from making this design's Apply+Reset the best in the gallery.
- Render the booting state as a real "edition going to press" layout — a typesetting frame skeleton, plate-making progress, "Edition is being typeset…" copy. The metaphor will sing.
