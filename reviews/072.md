# Review of #072 — Pricing-page-style for blueprints

**Direction:** "Pricing-page-style for blueprints: blueprints presented as feature-comparison columns. 'Choose your starter.'"

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 9/10

## What works
- Direction faithfully delivered: SaaS pricing-page columns with headline price (PHP/WP combo), value prop, vertical checklist of included plugins/themes/content, "Most Popular" / "For Developers" / "For Creators" recommended badges, big primary CTA. Compare-all-plans table for the long tail.
- All 6 §2.2 sources rendered as "Free tier strip" cards (lines 664–694): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — explicitly framed as "no recipe, just configuration".
- Apply+Reset in two surfaces:
  - In-flow notice + button (line 1097): "**Applying settings resets the playground.** All data, posts, and uploads in this session will be lost." + "Apply & Reset"
  - Site Manager (line 1170): `danger-btn` "Apply & Reset Playground"
- Address bar `<input type="text" value="/wp-admin/edit.php?post_type=post" aria-label="Address bar">` (line 624) with aria-label.
- Unsaved-Playground notice: "**Unsaved Playground.** Your changes will be lost on page refresh." + Save site (lines 1072–1073).
- Playground rename `<input value="Acme Storefront — staging">` (line 1053).
- Blueprint pane has Copy link / Download bundle / Run Blueprint (lines 1250–1252). Files tree present.
- Download .zip / Copy link to Blueprint / Download Blueprint bundle in additional-actions menu (lines 1176–1179).
- "Your Playgrounds → 'Resume your subscription' row" — clever metaphor reframe (saved sites = accounts you already own).
- README is honest: "The metaphor breaks if pushed too literally: there is no money, no checkout, no plans-that-lock-features, so the word 'Free' appears on every tier (which is honest but redundant)."
- Categories become column groupings / pricing pivots ("Show: Themes only / WooCommerce / Gutenberg / Experiments") + left-rail filter.
- README acknowledges the "commercial feel" risk: "we lean into Automattic-y warmth (rounded corners, soft shadows, plenty of whitespace) instead of cold enterprise SaaS."

## What's missing or weak
- **Apply & Reset has no confirm step** despite explicit destructive copy in both surfaces.
- **Settings appear in two surfaces** (in-flow + Site Manager). Two visible locations.
- **The pricing-page metaphor naturally suggests hierarchy and choice anxiety** — useful for newcomers, condescending for repeat developers. README mitigates with Compare table + keyboard search overlay.
- **The word "Free" on every tier is honest but redundant** — README admits.
- **The PR / GitHub / URL launcher cards** are tile faces; no inline paste inputs.
- **Big tier columns are real-estate hungry on mobile** — collapsed to swipeable carousel with sticky CTA per README.
- **Originality is high** — overlaps with #073 (Restaurant menu) in the "blueprint = menu item" lane, but pricing-page-as-blueprint-gallery is distinct enough.
- **The "Recommended" / "Most Popular" badges** — verify they're persona-aware or static (static would be on-brief; persona-aware would risk inventing personalization).
- **Logs and DB as "Advanced add-ons" tabs** — fits the SaaS metaphor (add-ons), but doesn't change underlying UX.

## How to push harder
- Implement the destructive confirm as a SaaS-style "Downgrade your plan?" prompt — pricing pages typically don't have destructive moments, but the metaphor invites a "this will downgrade your site to a fresh install — are you sure?" interrupt.
- Differentiate from #073 by leaning into pricing-comparison-specific affordances: side-by-side compare (drag two blueprints, see a diff table), savings calculators ("save 10 min by starting from this preset"), or a "Most-bought-with" cross-sell that suggests plugin add-ons compatible with the chosen blueprint.
