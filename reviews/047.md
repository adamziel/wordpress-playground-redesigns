# Review of #047 — Stripe Dashboard

**Direction:** "Stripe Dashboard: clean nav, big '$' — er, 'WP' — header metric, time-series chart of activity."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 7/10

## What works
- Stripe-faithful aesthetic: clean left nav, monospace-numeric tables, gridded layout, sparkline time-series, marquee "WP" metric in place of revenue, ledger-style "Your Playgrounds" table with status pills + tag columns + action buttons.
- All 6 §2.2 sources rendered as compact action tiles (lines 1225–1250 and 1821–1826): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip. Two surfaces — Overview tiles + Blueprints page tiles — consistent.
- Address bar combines URL + scheme + host: `<span class="scheme">https://</span><span class="host">playground.wordpress.net</span><input type="text" value="/wp-admin/edit.php?post_type=product">` (line 1087). §2.1 item 2 satisfied with realistic URL composition.
- Apply+Reset has an inline confirm tray: "**Applying changes resets the playground.**" (line 1774) + `btn sm danger` button. The README claims "an inline confirm tray inside Settings, with the cost stated plainly" — partially delivered.
- Database has implicit early-access; Logs has search (line 1718); Blueprint has Copy link / Download bundle / Run Blueprint (lines 1597–1599).
- "Your Playgrounds" Stripe ledger has columns for name / status / tag / dates / actions, with the "Unsaved Playground" row carrying a primary "Save site locally" button inline (line 1348). Ledger semantics applied correctly.
- WP Admin / Homepage as icon-buttons in the live preview chrome with toast feedback (line 1359: `onclick="toast('Navigated to /wp-admin')"`).
- The Live Preview card on Overview is described as prominent with "one-click full-bleed toggle" — mitigates the "iframe pushed off home view" risk.
- README is honest about decorative cost: "The full-width chart is decorative — playground boots-per-day isn't a real KPI the way charge volume is. I leaned into the conceit anyway because the direction asked for it."

## What's missing or weak
- **The "iframe off the home view" gamble** is the README's biggest acknowledged risk: "first-time visitors who expect WordPress to be the page now see a dashboard first, with an 'Open live preview' CTA." Mitigated, but the iframe demotion is real.
- **Apply & Reset has no real confirm step** — the inline tray is one button with a copy line, no gate. Stripe's actual destructive confirms have a "Type the resource name to confirm" pattern — undelivered.
- **The full-width chart is decorative** — README admits it has no real metric meaning. Pure metaphor adherence.
- **Settings appear inside the left-rail Settings destination** — README claims popover/sidebar/info-notice triplication is "killed" by routing through one Settings page. But the toolbar popover (`Refresh, Save, Settings popover, and Saved-Playgrounds launcher all live as inline buttons here`) implies the popover still exists. Verify.
- **The PR / GitHub / URL launcher tiles** are tile faces without inline paste inputs.
- **The "active playground is implied by the header chip rather than a containing sidebar"** — README admits the site context hierarchy is flattened.
- **No real "Open Site" big primary button** — the design's marquee action is the WP metric, not a boot button.
- **The README's "money-software aesthetics, but for sandboxes"** is a niche read — Playground users are devs, not finance ops. The Stripe vocabulary (ledger, line items, charge IDs) doesn't fully map.
- **Mobile fallback not strongly described** — the chart-driven layout doesn't translate well to small screens, and the README is quieter on this trade-off than other designs.
- **Originality has overlap with #045 (Studio desktop), #046 (Vercel dashboard)** — three designs map Playground to professional dashboards. #047's distinctive move is the metric-chart Overview; #045 leads with environment rows; #046 with deployment cards. The three converge on "professional sidebar dashboard" territory.

## How to push harder
- The Stripe destructive UX is "type the resource ID to confirm" — implement it for Apply & Reset. Stripe's actual UI is the gold standard for "are you sure?"; copy the pattern.
- Make the chart load-bearing somehow: chart-of-blueprint-runs-by-category, or a real "save activity" timeline that lets you click a saved point to restore. Turn the decorative metric into something the playground audience actually wants.
