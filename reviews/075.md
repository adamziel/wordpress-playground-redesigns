# Review of #075 — Catalog / IKEA

**Direction:** "Catalog / IKEA: numbered SKUs for each blueprint with cool product photography mockups."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 9/10

## What works
- Direction faithfully delivered: numbered SKUs like `PLG.204.871` / `PLG.101.044` / `PLG.305.117`, IKEA-yellow store header, "Departments" filter strip, "Featured Articles" rail, "Special Orders" sidebar for power-user paths.
- All 6 §2.2 sources rendered as Special Orders entries (lines 1747–1752): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each with two-line sub-copy ("Boot from a wordpress-develop pull request" etc.).
- **Apply+Reset has the IKEA reframe**: "Apply & Restock" button (line 1660) — the destructive action is metaphorically "restocking the article" (replacing your current playground with the new spec). On-metaphor but verify whether the destructive nature is sufficiently warned.
- Address bar `<input value="" placeholder="Find an aisle: /wp-admin, /shop, /?p=12 …">` (line 1568) — §2.1 item 2 with a playful "aisle finder" placeholder.
- Database = Inventory (counter); Logs = Service Log; Files = Stockroom; Blueprint = Assembly. Customer Service desks per README.
- Unsaved-Playground notice: "**This is an Unsaved Playground.**" + Save site locally button (lines 1636–1638).
- "Your Cart" with PLG SKUs for saved sites (lines 1672–1696) — saved playgrounds as IKEA cart items.
- Download .zip in Returns & Exchanges (line 1712): "Bundle the current playground (database + files) as a portable .zip you can re-import later."
- Article search by SKU: `<input placeholder="Search articles by name, tag, or SKU (e.g. PLG.204.871)">` (line 1766).
- README is honest: "The IKEA-yellow header is high-contrast on purpose — it's a brand commitment, not a neutral chrome — and may clash with WordPress blue inside the iframe."
- "The 'Special Orders' desk treats PR/GitHub/URL boots as side-rail forms rather than headline tiles, which de-emphasizes them versus the current launcher (intentional: those are power-user paths)."

## What's missing or weak
- **Apply & Restock has no explicit destructive warning rendered in the grep.** The metaphor reframes destruction as restocking, but the user might miss that "restocking" wipes their current data. Verify whether a warning callout sits next to the button.
- **Settings collapse into "Tag (Article Details) panel"** per README — verify it's truly one component vs. duplicated across surfaces.
- **The PR / GitHub / URL Special Orders entries** — verify whether they expose inline paste inputs (the "service-desk forms" framing suggests they should).
- **Mobile collapse to single column + "best viewed on desktop" hint for deep service tasks** — README admits the mobile downgrade.
- **The IKEA-yellow header may clash with WordPress blue** — README admits. Brand commitment cost.
- **Power users will find SKU cards larger than necessary** — README admits and defends.
- **Originality is high** — overlaps somewhat with #072 (Pricing page) and #073/74 (Restaurant/Cookbook) in the "blueprint = product/recipe" lane, but #075's distinguishing move is the SKU + IKEA-catalog visual language. The article numbering is its own contribution.
- **No mention of how Network / Multisite toggles inside the tag panel relate to the global Apply & Restock action** — line 1660 has the Apply button right after the toggle list, but no inline warning about destructive consequences.

## How to push harder
- Render the Apply & Restock destructive warning explicitly — "Restocking will discard your current article (Coffee Shop · saved 2 days ago) and replace it with a new spec. Confirm." A typed-confirm gate for the SKU number would close §4.
- Make the Special Orders service-desk forms inline-paste actually rendered — clicking "WordPress PR" should reveal the PR-number input on the same panel, mimicking IKEA's actual service-desk paperwork.
