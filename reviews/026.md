# Review of #026 — Wizard / Step-by-step

**Direction:** "Wizard / step-by-step: every action becomes a multi-step flow with progress bar. Heavy hand-holding for new WordPress users."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 8/10 · Originality 8/10

## What works
- The wizard scaffolding is fully delivered: a state-switcher at the top lets the viewer cycle through 9+ states (each with its own progress bar, step rail, "What is this?" helper pane). This is the rare design where the brief's "three states minimum" is exceeded by 3x.
- "Start a new playground" wizard step 1 has all 6 §2.2 sources as big tiles (Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip) — but importantly the wizard reframes each as a single question ("How do you want to start?"). That's the README's thesis honored.
- Settings Wizard genuinely collapses the three §4 settings entry points into one flow with explicit "One destructive thing" copy at line 2101 — explicit warning is one of the strongest in the gallery.
- Apply+Reset confirm step is finally rendered as an actual wizard step (per README: "the wizard's final step is always an explicit confirmation screen with a diff of what will change"). State 4+ shows the destructive-confirm pattern.
- Database has the early-access framing inside its wizard; Logs has search + level filters; Blueprint inspect wizard has Run Blueprint, Copy link, Download bundle as later-step verbs.
- The §4 "small text Save" pain is converted into "Save site wizard · Step 3 of 3" — Save becomes a destination, not a button (line 2811+). A user can't miss it.
- Address bar `<input value="https://playground.wordpress.net/">` / `<input value="/wp-admin/">` etc. across states — §2.1 item 2 satisfied.
- README is honest about trade-offs: "Switching PHP versions becomes 3 clicks instead of 2" — power users pay the tax explicitly. The bet is on first-time users, which is the right bet for Playground's mission.
- Each wizard ends with "What's next?" Download .zip / Export PR / Save locally — additional actions discovered in context rather than buried in a kebab.

## What's missing or weak
- **Power users hate this — README admits.** Switching PHP version = 3 clicks instead of 2. The blueprint gallery can no longer be browsed as a wall of 43; it's paginated/filtered inside step 2 of a wizard. That cost is real and not mitigated beyond an acknowledgment.
- **Visual chrome cost** — progress bar + step rail + helper pane all consume real estate the iframe used to own. The iframe is still present but smaller.
- **The wizard-shaped settings change** means a user who wants to "just toggle Multisite" has to enter a multi-step flow. Slight overkill for low-stakes toggles. The mitigation should be a "skip to end" shortcut for repeat users — not clear if it exists.
- **State 4's destructive-confirm "diff of what will change"** is referenced but I'd want to verify the actual diff UI renders (showing "PHP 8.2 → 8.3, will reset SQLite database, will lose 4 wp-content files"). The diff is the design's strongest claim about Apply+Reset; rendering it is critical.
- **The wizard for File browser ("pick file → preview → edit → save")** turns file editing into 4 steps. For multi-file blueprint authoring this becomes laborious. README admits the cost.
- **The "What is this?" helper pane** is great for first-time users but adds chrome cost. Toggleable? The artifact may not show the toggle state.
- **Address bar is the persistent toolbar** alongside refresh, homepage, admin shortcuts — that's enough chrome to access the iframe without entering a wizard, so power users have at least one escape hatch.
- **Mobile fallback** is not strongly emphasized in the README; the wizard format is wider-than-mobile-friendly, so its mobile translation would either be vertically stacked or just identical (which is fine for a thumb-only experience).
- **Pagination of 43 blueprints inside step 2** is described in the README as "paginated/filtered" — the design might lose discoverability of obscure blueprints that today's wall-of-tiles shows at a glance.

## How to push harder
- Add a "Skip the wizard / show me everything" toggle in the global chrome — for repeat users who want today's dense UI back. Honor the README's first-time-user thesis without permanently penalizing power users.
- Make the destructive Apply+Reset diff state render explicit "Before → After" tables (PHP 8.2 → 8.3, WP 6.6 → 6.7, blueprint will re-run 4 steps, you'll lose ~4 wp-content files). That diff is your most defensible UX claim; show it fully.
