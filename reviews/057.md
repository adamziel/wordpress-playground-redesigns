# Review of #057 — Three-pane Outlook/Mail

**Direction:** "Three-pane Outlook/Mail: nav | list | reading pane. Inbox of 'things you can do' on the left."

**Scores:** Feature completeness 9/10 · Direction 9/10 · IA 7/10 · Visual 8/10 · Originality 6/10

## What works
- Faithful three-pane Outlook composition: left folder rail (Your Playgrounds / Start new / Blueprints / Site Manager folders for Settings/Files/Blueprint/Database/Logs), center list pane (uniform item feed), right reading pane (iframe + ribbon).
- All 6 §2.2 sources rendered (lines 1204–1224): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip.
- Apply+Reset button "Apply settings & reset" (line 1166) with primary class.
- Address bar `<input value="/wp-admin/edit.php?post_type=post" spellcheck="false">` (line 1111) — §2.1 item 2.
- Universal search at the top: `<input placeholder="Search blueprints, playgrounds, files, log entries…">` (line 1093) — Outlook-style cross-folder search.
- Database tab as a list of tables. Logs as a chronological list of entries with `ts/lvl/msg` columns (lines 1407–1414).
- Files folder as a flat path-prefixed list per README (rather than a true tree) — admitted trade-off.
- "Unsaved Playground" tagged with yellow background + warning emoji (line 1522) — list-row visual flag for unsaved state.
- Settings popover preserved as a "compose"-style flyout per README — toolbar quick-access.
- Toasts dock bottom-right of reading pane per Outlook convention.
- README is honest: "Power users may miss the always-visible Site Manager tab strip; here it's one click deeper." Real cost named.

## What's missing or weak
- **Apply & Reset has no confirm step.** The primary button fires.
- **Settings appear in two places** — README explicitly admits: "The settings popover is preserved as a quick 'compose'-style flyout on the toolbar for users who don't want to navigate to the Settings folder." Same data, two surfaces.
- **The Files folder as flat path-prefixed list (not a tree)** — README admits the cost. File-tree depth is lost.
- **The Database folder as a list of tables** — clicking opens a list of rows, which is on-metaphor but feels less rich than a dedicated DB browser.
- **The PR / GitHub / URL launcher items are list rows** without inline paste inputs.
- **Originality is medium** — overlaps with #020 (Mail.app) in the gallery's three-column-mail-shaped lane. Both designs treat Playground as an email client; #020 leans further into the metaphor (Reply = Refresh, etc.), #057 keeps the metaphor pragmatic (folders are folders, items are items).
- **Site Manager tabs are now folders one click deeper** — power users now have to navigate Rail → Folder → List → Reading pane, which is more clicks than the current tabbed sidebar.
- **The README admits "Power users may miss the always-visible Site Manager tab strip"** — the metaphor's tradeoff for first-touch users (clearer mental model) penalizes power users.
- **No clear demo of a destructive confirm dialog** — Outlook itself has "Permanently delete this item?" modals. The metaphor invites them.
- **Mobile collapses reading pane to a sheet** — admitted.

## How to push harder
- Render the Outlook-style "Permanently delete?" modal for Apply & Reset. The metaphor's existing destructive UX is the right pattern; reuse it verbatim.
- Differentiate from #020 (Mail.app) by leaning into Outlook-specific affordances: Focused / Other tabs for blueprint Featured vs. All, conversation threading for blueprint variants, and Rules for auto-applying settings to certain blueprints. The current artifact's three-pane shell could come from either client.
