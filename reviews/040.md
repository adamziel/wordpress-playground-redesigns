# Review of #040 — High-contrast accessibility-first

**Direction:** "High-contrast accessibility-first: WCAG AAA contrast, very large hit targets, keyboard hints visible by default, focus rings prominent."

**Scores:** Feature completeness 10/10 · Direction 10/10 · IA 9/10 · Visual 8/10 · Originality 9/10

## What works
- Best-in-class direction adherence and the most usable design in the gallery for low-vision / keyboard users. Atkinson Hyperlegible typography, pure black on near-white, single accent (#1645AD) + single alarm (#A40000) — both AAA-checked. 4px solid amber focus rings that never fade.
- **The Apply & Reset destructive confirmation is fully delivered** (line 2273+):
  - Explicit data-loss copy: "Applying these settings will **permanently delete all data** in **'My Coffee Shop'**: 6 posts, 2 products, 0 saved drafts, and the database. Saved sites in your library are unaffected." (line 2280)
  - A "type 'reset' here" confirmation input (line 2282) — actual gate on the destructive action
  - The README mentions "hold-to-confirm pattern" as well
  
  This is the strongest destructive UX in the entire gallery.
- All 6 §2.2 sources rendered as labeled cards (lines 1924–1964) AND in the Import button (line 1782): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each with a bracketed kbd hint.
- Toolbar uses real text buttons with icons + labels — no icon-only controls. "Save site" + "⌘ S" badge inline (line 1561, 2026).
- Address bar is a real `<input id="url1" type="text" value="/2026/05/welcome-back" aria-label="Address bar: type a path inside WordPress">` with explicit aria-label (line 1538). Best aria coverage in the gallery.
- Boot state has disabled address bar `aria-label="Address bar (disabled while booting)"` (line 2017) + an aria-live status strip. Loading states announced.
- Settings collapses into ONE tabbed Site Manager — README explicitly says "the popover quick-settings is gone because a fully-keyboard-reachable tab list with persistent shortcut hints is faster than a hover popover." Only design in the gallery that aggressively kills the popover entirely.
- Network/Multisite checkboxes (lines 1685, 1694) with `<label>` + `.lbl` + `.sub` descriptive text — proper form semantics.
- Database (early access) at line 2236. Logs has filter input with placeholder "Filter logs — e.g. 'fatal' or 'wp-cron'" (line 2096). Blueprint has Run Blueprint / Copy link / Download bundle.
- Keyboard shortcut legend (line 2300) — `Cmd S`, `I` for Import, etc. visible as a reference card.
- Per README: "Every primary action carries a visible keyboard shortcut in a bracketed badge" — delivered.
- Mobile fallback: kbd hints collapse into a Shortcuts sheet to recover space — sensible.

## What's missing or weak
- **Visual density is intentionally low** — README admits "Designers who want a polished, brand-forward marketing surface will find this stark." It's a deliberate trade-off but worth flagging that this design is the least visually-distinctive in the gallery (which is the point).
- **No per-category color washes for blueprints** — categories rely on labels and icons. README admits the cost.
- **Persistent kbd badges add ~24px to every primary control** — taxes screen real estate.
- **The PR / GitHub / URL launcher cards** — verify whether they expose paste inputs inline. Without inline inputs the deep keyboard-first promise is partially undelivered for those flows.
- **The high-contrast design's "accent + alarm only" constraint** means visual hierarchy beyond text-weight relies on the 4px focus rings + amber bar markers. Powerful for focus, less expressive for non-interactive content.
- **The blueprint gallery has limited thumbnail differentiation** — the §4 "43 blueprints with no preview thumbnails" pain isn't addressed (and shouldn't be, given the direction's anti-decoration stance). Cost of direction.
- **The kbd badges and 56px hit targets** mean the design at 1440x900 has fewer features on screen than #002 or #035. Trade-off.
- **Mobile section** — the README mentions collapsing kbd hints into a Shortcuts sheet; verify mobile state is rendered.
- **Status announced via aria-live** — described, undeniable accessibility benefit, but a sighted reviewer can't see the announcement so its presence is implicit in the artifact.

## How to push harder
- This design's destructive confirm + accessibility focus + no-icon-only chrome is already the strongest-in-class. The next push is to make the WCAG-AAA story visible AS a feature in the marketing surface — a small "AAA verified" badge or contrast indicator that recursively communicates the design's value to anyone landing on it.
- Push originality further: a "Color-blind safe" toggle that flips the accent/alarm pair to a blue/yellow combination (deuteranopia-safer) without changing layout. Nobody else can claim accessibility this credibly; ship the proof.
