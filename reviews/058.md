# Review of #058 — Single dropdown

**Direction:** "Single dropdown: collapse everything except the iframe behind ONE menu in the top-right. Brutalist minimalism."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 9/10

## What works
- The direction is delivered with maximum restraint: ONE menu in the top-right, iframe at 100% canvas. No toolbar, no sidebar, no drawer, no popover, no toast region. Brutalist Helvetica + IBM Plex Mono on cream paper, 1px black borders, zero ornament.
- All 6 §2.2 sources rendered as menu rows (lines 872–902): Vanilla WordPress / WordPress PR `[PR#]` / Gutenberg PR `[PR#]` / From GitHub `[repo]` / Blueprint URL `[json]` / Import .zip `[upload]` — each tagged with the expected input type.
- Apply+Reset uses **typography for the warning** rather than color: "**Apply Settings & Reset Playground** Applying settings reboots the iframe and wipes the current site database. `[DESTRUCTIVE]`" (lines 1015–1016) — `tag destructive` class. The brutalist commitment to "typography does the warning" is honored.
- Address bar `<input id="addrInput" value="wp-admin" autocomplete="off" spellcheck="false">` (line 850) — §2.1 item 2. Lives at the top of the dropdown so navigation is a menu item.
- "Open /wp-admin" labelled row (line 855) — WP Admin jump as an explicit menu item.
- Rename row (line 832) — playground rename present.
- Database has implicit early-access (verify); Logs has search (line 1109); Blueprint pane probably has the conventional buttons.
- **Save lives as a 64px-tall row at the top of the menu** that owns the whole width — README claims and the §4 lose-on-refresh risk is solved by making Save unavoidable when the menu opens.
- The permanent 1px label-strip along the very top edge showing `<NAME> · <URL>` — the only persistent status outside the menu. Honest minimum.
- README is honest: "Brutalism in 2026 is itself a fashion — it isn't neutral — but the brief asked for it." Owns the position.
- "There are no shortcuts visible (a real ship would add `?` keymap; I left it out to stay honest to brutalism)" — the design's commitment to no-hidden-affordances is rigorous.

## What's missing or weak
- **Apply & Reset has no confirm step.** The `[DESTRUCTIVE]` tag is typographically loud but the button still fires. Brutalist would invite a "type RESET to confirm" inline input — the metaphor's existing minimalism would make a typed-confirmation feel native.
- **No shortcuts visible** — README deliberately omits per direction. Power users will resent the round-trip.
- **One menu means power users round-trip to the menu for every action** — README admits.
- **The iframe gets 100% canvas but loses persistent indicators** — README admits and adds the 1px label-strip. Minimum status only.
- **The "Apply Settings & Reset" button** within the menu — verify whether it's gated by the same brutalism (no confirm) or if there's a deeper interaction.
- **The PR / GitHub / URL rows have `[PR#]` / `[repo]` / `[json]` tags** but no inline paste inputs visible from the grep. Verify whether clicking expands an inline form vs. opening a deeper menu.
- **Mobile drops the strip entirely and the menu becomes the whole screen** — sensible.
- **The "Unsaved Playground" label appears in the top strip** (line 756) — but the 64px Save row that the README promises is inside the menu, not in the chrome. A user who doesn't open the menu sees only "Unsaved Playground" as a small label.
- **Categories in the blueprint browser became a sticky 1-row monospace filter strip; cards became a dense table** — typographic-table approach loses thumbnail scannability. README admits ("brutalism doesn't decorate, it lists").
- **Originality is high** — overlaps with #035 (Brutalist web 1.0). Both designs commit to brutalism + no decoration; #058's distinguishing move is "one menu" vs. #035's long-scroll document.

## How to push harder
- Render the destructive Apply & Reset as a brutalist inline confirm: an `<input>` styled flat that requires typing "RESET" to enable the submit. Brutalism + destructive UX = type-it-or-it-doesn't-fire. #035's review made the same suggestion; #058 has the metaphor budget to deliver it more cleanly.
- Demonstrate the 64px Save row's prominence — show how the menu looks when the playground is unsaved, with the Save row dominating. The §4 mitigation is the design's strongest claim; visualize it.
