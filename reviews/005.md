# Review of #005 — Browser-tab metaphor

**Direction:** "Browser-tab metaphor: each saved playground is a real tab strip at the top. New tab = new playground. URL bar is the WordPress address bar. Bookmarks bar = blueprints."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 9/10

## What works
- Among the strongest direction-adherence in the gallery. Chrome-y tab strip with real tabs ("wp-admin · Coffee Shop demo", "Gutenberg PR #62301 booting", "+"), URL bar that genuinely is the WordPress address (`<input class="url-input" value="/wp-admin/edit.php?post_type=product">`), a bookmarks bar of five pinned blueprints + chevron to "all 43", an overflow `⋮` menu, and a DevTools-style F12 drawer for files/blueprint/DB/logs. Every direction signal is wired to a §2 feature.
- The URL input is a real `<input>`, satisfying §2.1 item 2 — one of the few designs that nail this.
- New-tab page (state 2) carries the omnibox in "blueprint search" mode with placeholder "Search blueprints, paste a blueprint URL, GitHub repo, or PR number…" — collapses the six §2.2 launchers into a single typeable input AND the discrete cards. Both affordances coexist.
- Settings popover (drops from a chip near the URL bar) has the §2.3 unsaved notice ("Unsaved Playground. Your changes will be lost on page refresh. [Save site locally]") at the top, followed by WP / PHP / Language / Network / Multisite rows, with a "Apply & reset wipes this Playground's database and files" red-bordered footer — the cleanest collapse of the three settings entry points in the gallery so far.
- Overflow menu lists Save Playground (⌘S), Download as .zip, and (per the README) Export to GitHub PR — matching §2.3 additional actions.
- The blueprint editor in DevTools uses a real two-column layout with a file tree of the blueprint bundle (`gutenberg-pr-62301/blueprint.json`, `mu-plugin.php`, `starter-content.xml`, `hero.jpg`, `README.md`), Copy link + Download bundle + Run blueprint buttons.
- Mobile section commits to the metaphor: Safari-style tab switcher, bottom bar with Tabs / Blueprints / DevTools / Menu, real URL bar pill. The metaphor degrades gracefully.
- Logs has a real search field with example placeholder ("Filter logs… (e.g. WP_DEBUG, 'undefined function', /wp-admin)") and grouped request lines — addresses §4's no-filter complaint AND adds per-request grouping that §4 also mentions.

## What's missing or weak
- **Apply & Reset has no confirm step.** The popover button is labeled "Apply & reset" with explanatory text but no confirmation modal. §4 flagged this as the current pain; the design names the problem without solving it.
- **The bookmarks bar shows 5 blueprints pinned + "Import .zip"** but the bookmarks bar is missing the "All 43" chevron icon. The README claims one — the static HTML only shows the 5 + Import buttons. Quick visual: a `<div class="bm-import">` is there but no `<div class="bm-more">`.
- **Saved playgrounds vs. closed tabs vs. "Your Playgrounds" merger is ambiguous.** The README says they merge into a "recently-closed-tabs menu plus a 'Reopen Playground' tile on the New Tab page." The artifact only shows three open tabs and a tile per blueprint; the recently-closed menu is not demonstrated. §3 user flow 5 ("Resume a saved playground") loses its discoverability.
- **The "Search blueprints" omnibox in the new-tab page has no autocomplete / suggestion list rendered.** Power feature unbuilt.
- **The URL bar's "Settings chip"** that triggers the settings popover is described as "near the URL bar, the way browsers tuck site settings into the padlock area." In the artifact this chip is small and lives left of the URL — discoverable only with a hover/title attribute. Compare to the prominent Save button.
- **Three states are mostly delivered (working session w/ DevTools open, new tab launcher, booting tab), but the "blueprint authoring" pane shown in state 3 lacks a "Save blueprint" affordance.** Run blueprint is there; saving the JSON file itself is not.
- **No visible "rename playground" affordance** on the tab itself — clicking the tab label should let me edit "Coffee Shop demo". Mac Finder did this via input; this design renders the label as a span.
- **The PR-number / GitHub-repo / Blueprint-URL launcher cards are tiles, not inputs.** Tapping should reveal an input; the artifact only shows the tile face. §3 flows 11–12 incomplete here, just like in most designs.
- **"Tabs scale poorly past ~10 Playgrounds, so the strip overflows into a 'Tabs' search dropdown"** — claimed in the README but not rendered in the artifact.

## How to push harder
- Make Apply+Reset a real Chrome-style "Reload site? Unsaved changes will be lost" interrupt — leverage the browser metaphor for the confirm step, not just for the chrome. The URL bar already lives next to the refresh button; turn the refresh button red and demand confirmation when settings are dirty.
- Render the "9+ tabs" overflow popover. The omnibox-as-tab-search trick is the strongest scaling claim the README makes; show it in a fourth state.
