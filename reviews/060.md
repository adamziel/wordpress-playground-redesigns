# Review of #060 — Video player chrome

**Direction:** "Video player chrome: bottom bar with play/pause/scrubber metaphors — scrubber is a timeline of states; play = start, pause = freeze, etc."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 7/10 · Visual 9/10 · Originality 10/10

## What works
- Direction commitment is total: transport bar at the bottom, scrubber, Now Playing pill, Library panel (right), Equalizer drawer (slides up from transport), Quality popover, share menu. The video-player metaphor is consistently applied.
- **The state scrubber is the design's headline feature** — it gives users "a literal visualisation of 'where am I in this playground's history' which the current UI completely lacks." This is the most original interaction in the gallery — none of the other 60+ designs let you scrub time-of-playground-states.
- All 6 §2.2 sources rendered as Library "New playback" sources (lines 1139–1149): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each with descriptive sub-line.
- Apply+Reset uses `pill-btn danger` (line 1406): "Apply & reset playground" with alert-triangle icon. Distinct destructive styling.
- Address bar `<input value="/shop/?orderby=popularity">` (line 1023) — §2.1 item 2.
- Database has implicit early-access; Logs has search input "Search logs…" with example value "cart" (line 1311); Blueprint has Run Blueprint / Copy link / Download bundle (lines 1495–1497).
- Unsaved-Playground notice: "**Unsaved Playground.** Your changes will be lost on page refresh." + Save site locally (lines 1547–1548).
- "Save = bookmark/record button on the transport bar that pulses when there are unsaved changes" — addresses §4 via the metaphor's existing language.
- README is honest: "Some controls land where they don't naturally belong (Database browser inside a 'Channels' tab took some justification)" + "The scrubber-of-states is technically only as good as the underlying snapshot system; in this mock it's a conceptual artefact rather than a true rewind (matching the brief: don't invent new features, this just *visualises* the existing save points and applied blueprints)." Brief-aware framing.
- "The audio metaphor stops at the door of each panel; inside each panel labels are honest" — solves the "Quality = permalinks" metaphor-stretch risk by limiting metaphor depth.
- The "share popover" for Export PR / Download .zip — actions = sharing, on-metaphor.

## What's missing or weak
- **Apply & Reset has no confirm step rendered.** Despite the `pill-btn danger` styling, the button fires.
- **The scrubber is a conceptual artefact, not a true rewind** — README admits explicitly. The metaphor visualizes existing save points, doesn't add new ones. Fair framing; worth noting that the headline feature is partially mocked.
- **Settings appear in the "Quality" popover AND the Equalizer drawer's Settings tab** — two surfaces.
- **The transport bar adds 72px of vertical chrome** — README admits, condensed to mini-player on mobile.
- **The PR / GitHub / URL launcher sources** have sub-lines but no inline paste inputs.
- **The "audio-style tabs labelled honestly"** decision means the metaphor weakens at the panel level — internally, Settings is just Settings, not "Quality". Honest but slightly inconsistent.
- **The pulse on Save when unsaved** is a great §4 mitigation — but verify the artifact actually shows the pulse animation (vs. just a static state).
- **The "drag the scrubber and you're scrubbing through the life of this session"** — verify whether the scrubber is interactive or just a static visualization in the artifact.
- **WP Admin / Homepage jump buttons** — verify whether they're explicit transport-bar buttons or buried in a menu.

## How to push harder
- Make the scrubber genuinely interactive — even a static viewer should see "drag the dot to a past state" hover affordances. The scrubber is the design's signature; render it visibly draggable.
- Render the destructive Apply & Reset confirm as a video-player "Are you sure you want to discard your changes?" overlay — YouTube and Netflix both pause and overlay the video for important prompts. Use the metaphor's existing destructive UX.
