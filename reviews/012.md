# Review of #012 — Game launcher (Steam)

**Direction:** "Game launcher (Steam): library on the left, big art for the currently selected playground/blueprint, 'Play' button. Recently played / friends-playing sections."

**Scores:** Feature completeness 9/10 · Direction 9/10 · IA 7/10 · Visual 8/10 · Originality 7/10

## What works
- True Steam aesthetic: dark slate blues, left library with search ("Search your library"), big cinematic hero with a green "Play" / "Save to Library" verb, "Recommended" carousel for featured blueprints, "Recently played" rail for saved playgrounds. Hits the visual targets in the README.
- All six §2.2 sources mapped to "+ ADD A PLAYGROUND" menu (WordPress PR / Gutenberg PR / From GitHub repo / Blueprint URL / Import .zip / Vanilla implied). The cards also reappear in a deeper New Playground source list.
- Settings overlay (Shift+Tab style) carries WP version / PHP / Language / Network access / Multisite with the unsaved-state notice text quoted directly: "This is an Unsaved Playground. Your changes will be lost on page refresh." + Save site locally button.
- The destructive Apply+Reset has a `destructive` class with explicit copy: "Apply Settings & Reset Playground — this is destructive. The current playground will be wiped and rebuilt with the new settings." Better than baseline.
- Address bar is a real `<input value="/">` in the HUD strip, satisfying §2.1 item 2.
- DB / Logs / Files / Blueprint overlay panels all present with appropriate controls (Logs has search input + level filter; Blueprint has Copy link / Download bundle / Run Blueprint buttons; Files has tree).
- Mobile section is real (line 1493+), with m-cards consolidating "WordPress / Gutenberg PR" and "From GitHub / URL / .zip" — admits these are PR/URL inputs that need a follow-up screen.

## What's missing or weak
- **Apply & Reset has no confirm step.** Steam itself prompts "Are you sure you want to uninstall?" for destructive actions; this design didn't borrow that pattern. The "Apply & Reset" button on the overlay fires immediately despite the destructive copy.
- **The overlay is gated by Shift+Tab or a "controller pill"** — first-time users will not discover this. The README acknowledges the trade-off but doesn't compensate (no first-run hint, no pulse animation on the overlay trigger).
- **"Vanilla WordPress" not in the visible "+ ADD A PLAYGROUND" menu list** I read (lines 852–862 show Download .zip, Gutenberg PR, From GitHub, Blueprint URL, Import .zip, Copy link, Download blueprint bundle, but no Vanilla and no WordPress PR row in that block). The cards at lines 961–973 cover Gutenberg PR / GitHub / Blueprint URL / Import .zip — but Vanilla and WordPress PR might be missing from the artifact. Verify.
- **The PR / GitHub / URL "+ ADD" entries have no input affordance.** §3 flows 11–12.
- **Featured carousel renders as 5 cards but the full 43 gallery search overlay is shown elsewhere ("Search 43 blueprints…", line 1277).** Two gallery surfaces with no clear differentiation — is the carousel a Recommended subset, or the same gallery filtered?
- **Address bar lives in the HUD strip during a running session AND in the overlay's URL field** — same address shown twice. Steam doesn't show a URL bar inside the overlay.
- **Save state for the active playground is mostly invisible in the hero.** The hero shows the blueprint name + Play button, but a returning user can't see whether the current playground is saved or unsaved without invoking the overlay. §4 risk.
- **"Recently played" rail bleeds into "Your library" terminology** — the README maps "Your Playgrounds" to Recently Played, but in a real Steam app these are different lists (library = installed, recently played = subset). The artifact's labelling is ambiguous.
- **The `.destructive` class is a CSS warning style only**, not an interactive guard — there's no "type the name to confirm" gate the Steam metaphor invites.

## How to push harder
- Borrow Steam's "Uninstall" confirm modal pattern verbatim — a dark overlay with "Apply settings and reset playground? This will erase your current site, including any unsaved files. — [Cancel] [Apply & Reset]". Match the metaphor's existing destructive-action language.
- Surface Vanilla WordPress as the first row of "+ ADD A PLAYGROUND" — currently it appears nowhere in the dropdown I read.
