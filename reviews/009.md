# Review of #009 — Storybook-style component library

**Direction:** "Storybook-style component library: left rail = component categories (Site, Blueprint, Files, DB, Logs). The viewport is the 'canvas'. Tools like 'controls' and 'docs' tabs."

**Scores:** Feature completeness 9/10 · Direction 8/10 · IA 7/10 · Visual 8/10 · Originality 7/10

## What works
- Storybook vocabulary is mapped consistently: left story tree (Your Playgrounds with snapshots, Blueprint Library with categories, Sources for PR/GitHub/URL/zip), Canvas as the viewport, Canvas / Docs / Files / Blueprint / Database top tabs, and a bottom add-on panel with Controls / Actions / Logs / Outline.
- The URL bar is a real `<input class="url-input" value="/wp-admin/edit.php?post_type=page" placeholder="/wp-admin">` — §2.1 item 2 satisfied.
- The "Unsaved" warning is upgraded to a pulsing chip next to the URL bar (`<span class="unsaved-chip">`) — directly addresses §4's "lose a session on refresh" pain.
- All six §2.2 source entries appear twice (in the rail as "Sources" tree items and inside the Launcher modal as cards) and the Launcher renders a real "Start from a Blueprint" gallery toolbar with search + the 10 §2.2 category chips (All / Featured / Website / Personal / Content / Themes / Gutenberg / Experiments / WooCommerce / News).
- Database has the "early access" notice verbatim, and the Files / Blueprint / Database canvas tabs each split into a tree+content layout.
- Apply+Reset has a clear destructive sub-line: "Applying resets the site. All in-memory data is wiped." + an "Apply & reboot" button — with bold red `color:#B61E1E`.
- Settings rendered as a Storybook-style table with name/type/control/description columns — a real Controls add-on. Each row has a description like "Which WordPress release the iframe boots into. Changing this resets the site." — better doc per row than most.
- Actions add-on lists request/router/blueprint events with timing (`POST /wp-admin/admin-ajax.php?action=save-attachment · 182ms`) — Storybook Actions is one of the cleanest log replacements I've seen.
- Logs has search input "Filter logs — try 'error' or 'plugin'" and level pills. Toast region top-right with explicit OPFS-bytes copy.

## What's missing or weak
- **Three settings entry points still exist.** Quick popover (`popover-settings`) AND a full Controls add-on panel AND the Site Manager Settings tab. The README claims they collapse into a "dual surface" of popover + add-on panel; the artifact also shows a separate Canvas Settings tab elsewhere, breaking the merger. Same data, three surfaces — exactly §4.
- **Apply & Reset confirm step is missing.** Both the popover's "Apply & reboot" and the Controls panel's "Apply & reboot" fire immediately. The red warning text helps; a confirm modal would close the loop.
- **The README admits trade-off: "the file-tree component lives on the left side of three split panes — same component, different roots"** — but in practice this is the §4 "file browser + blueprint tab duplicate each other" pain quoted back at the user. The Storybook metaphor *requires* it, so the design picks one pain to keep.
- **The PR / GitHub / Blueprint-URL / Import .zip launcher cards are tile faces** — tapping should reveal an input. The artifact only shows the tile face. §3 flows 11–12.
- **Blueprint grid rendered by JS (`#bp-grid`).** The data array shows 22 of 43 blueprints (line 2075–2098); without JS the gallery is empty. §5 requires "at least 12" visible, which the JS does deliver — but the static HTML has zero cards. Designer flagged that, the artifact still depends on JS for the visible feature.
- **"Outline" add-on is a Storybook construct that doesn't map to anything in §2.** The README calls it "a11y / structure of the rendered site" — this is an invented feature (§5: "no AI, no marketplaces, no team sharing, no new features"). A11y inspection of the iframe contents is not in §2.
- **Mobile collapses the bottom add-on entirely.** The README admits "on mobile it collapses entirely in favor of a bottom tab bar" — but the bottom tab bar is not rendered in the artifact, only described. The brief requires 390×844 responsiveness.
- **WP Admin / Homepage jump buttons live in the actions overflow menu (`menu-actions`)** with shortcuts ⌘A / ⌘H, but they're not surfaced in the topbar where §2.3 puts them. The user has to click an overflow `⋮` first.
- **The "rename playground" affordance** is only a `<input>` value inside the title slot — no save/discard buttons or confirmation. Title-bar editing is implicit.

## How to push harder
- Remove "Outline" from the add-on panel — it's outside §2. Replace it with something §2 actually has, e.g., a "Database" add-on duplicating the SQLite browser as a docked drawer, fusing canvas tab + add-on into one surface and eliminating the file-tree-in-three-places trade-off.
- Render the Controls panel and the popover so it's visibly the SAME table, just collapsed to the popover's 5 quick rows. A shared schema source would close the "three settings entry points" complaint properly.
