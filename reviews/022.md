# Review of #022 — Music DAW / Ableton

**Direction:** "Music DAW / Ableton: track lanes = boot/install/configure steps of the blueprint, the iframe = the master output. Drag-and-drop steps to author a blueprint."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 7/10 · Visual 9/10 · Originality 10/10

## What works
- Best-in-class direction commitment. Browser left + Arrangement center + Detail bottom + Master Channel right is faithful Ableton Live shell with dark chrome, lane heads, clip rectangles, transport bar, and ping-toast feedback ("Bundle downloaded", "Copied blueprint URL").
- Genuine remix of blueprint authoring: clips on lanes representing install plugin / activate theme / run wp-cli / set option / login as editor steps. Spacebar to play, clips light up as they execute. This is the most authentic "drag clips" interpretation of the direction in the gallery.
- Transport bar metaphor: Refresh = Rewind, Run Blueprint = Play, Save = REC (red dot). The README's "Save button is the giant red Arm/Record dot" is delivered — addresses §4's lose-on-refresh by leveraging Ableton's "Are you recording?" muscle memory.
- All 6 §2.2 sources rendered as "Sources" in the Live Browser tree (line 1103–1108): Vanilla WordPress / WordPress PR… / Gutenberg PR… / From GitHub repo… / Blueprint URL… / Import .zip… with mode-hint chips (⏎ / PR / URL / ↑).
- Detail bottom has 5 tabs (Files / Code / Database / Logs / Settings) — though Code and Files might overlap with the README's claim about Files+Blueprint sharing. Database tab has the `early access` dot indicator.
- Apply & Reset destructive copy is strong: "'Apply settings & Reset' is destructive — it rebuilds the playground from scratch. Save first if you've made changes." plus a bright-yellow "Reload Project — Unsaved changes will be lost" treatment per the README (matching Ableton's sample-rate-change pattern).
- Address bar is a real `<input id="addr" value="/wp-admin/edit.php?post_type=product" spellcheck="false">` and there's a second mobile address-bar input.
- Mobile fallback to vertical step-list described; lane parallelism lost but features preserved.

## What's missing or weak
- **Settings appear in multiple places:** the Master Channel strip on the right (line 1478–1479), the detail-pane Settings tab (line 1471+), AND a separate popover (line 1687+). The README claims a single Master Channel consolidates them; the artifact has at least three surfaces.
- **Apply & Reset has no confirm step rendered.** The destructive warning is present in multiple copies; the confirm modal/inline gate is not. Compare to the "yellow Reload Project" Ableton actually shows — that screen could be the design's killer 4th state.
- **The DAW metaphor "overpromises for the 90% of users who just want to click 'Coffee Shop' and look at WordPress"** — README explicitly acknowledges and mitigates with a "Session view" toggle that flips to a clip-grid of featured blueprints. The session view exists in concept but I'd need to verify whether the artifact renders both Arrangement and Session views or only one.
- **The "PR / GitHub / URL" sources are tree rows that drop something onto a lane** — but the actual paste flow for PR number / URL / repo isn't visibly rendered. §3 flows 11–12.
- **The "drag-and-drop steps to author a blueprint"** is the strongest pitch of this direction, and the artifact shows clips on lanes — but the dragging interaction itself (a clip floating with cursor, drop zone highlight) isn't depicted. The static state shows the result, not the affordance.
- **"MIDI-map keys to clips"** is explicitly deferred in the README ("a deliberate 'this could grow here' affordance, not a new feature"). Good restraint; worth noting that the hint exists.
- **Visual density and dark chrome** — README acknowledges this as the cost of the metaphor.
- **Run Blueprint appears in three places**: the transport Play button, the detail pane's "Run Blueprint" primary button, and the lane-context Play. Coherent with the metaphor but redundant.
- **The Master Channel's "WP/PHP/language/network/multisite as faders"** is an excellent metaphor; in the artifact (line 1478–1479) I see toggles + a `v mono` value display rather than faders. The fader visual isn't fully delivered.
- **Logs filter is a basic input** (`<input placeholder="Filter logs…">`); the DAW metaphor invites a more aggressive remix (e.g., logs as a waveform / spectrum analyzer of severity over time). Missed opportunity.
- **The §2.4 toast region is implicit via ping toasts** but not anchored bottom-left per the brief; the design uses center-screen ping notifications that may compete with the DAW chrome.

## How to push harder
- Render the "Reload Project" Ableton-yellow confirmation as a fourth state — a clear "wait, you have unsaved clips" modal that requires Space or Enter to commit, just like Live's actual destructive prompts.
- Push the Logs metaphor into something only a DAW can express: severity rendered as track velocity / a vertical "PHP notice spectrum" with peak markers per second. Be the only design that can do this.
