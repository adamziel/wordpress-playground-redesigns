# Review of #021 — Spotify-like

**Direction:** "Spotify-like: left nav with 'Your Library' of saved playgrounds, big 'Made for you' grid of blueprints, bottom bar with playback-style controls (refresh, back/forward, save, settings)."

**Scores:** Feature completeness 9/10 · Direction 9/10 · IA 6/10 · Visual 9/10 · Originality 7/10

## What works
- Spotify shell faithfully delivered: dark/green theme, Your Library on the left with quick-starts (Vanilla, WP PR, Gutenberg PR, From GitHub, Blueprint URL, Import .zip), home shelves on the center pane, bottom transport bar with refresh/back/forward/save/settings. Right pane = Site Manager as "Now Queue" with 5 tabs.
- All 6 §2.2 sources rendered TWICE — once in the Library left-rail (line 499–521) and once in the home-page "new-card" shelves (line 680–702). Discoverability priority over minimal.
- "Save site locally" reuses the Spotify "Save to your library" heart icon (line 811: `<i data-lucide="heart">Save site locally`) — clever vocabulary borrow that helps the §4 "lose-on-refresh" pain by leveraging muscle memory.
- TWO Apply+Reset destructive banners ("Apply Settings will reset the playground. Unsaved changes will be lost." and "'Apply' performs a destructive reset of the playground."). Both have explicit copy + apply button.
- Address bar is a real `<input id="urlInput" value="/">` with aria-label.
- Site Manager Now-Queue has all 5 tabs: Settings (with unsaved notice + WP/PHP/Language/Network/Multisite + apply button), Files (tree), Blueprint (Run Blueprint + Copy link + Download bundle), Database (early access banner), Logs (search input + level chips).
- Mobile section referenced in README ("collapses entirely under 900px replaced by a bottom playback bar plus a one-tap Site-Manager toggle") — though I'd need to verify the mobile state actually exists in the artifact.
- The "refresh button is also the giant green Play disc" — a delightful dual-purpose that the README admits is decorative but feels right for the metaphor.

## What's missing or weak
- **The README explicitly admits keeping the three settings entry points** ("each now plays its native role: the popover is a quick transport-bar tweak, the Site Manager tab is the deep dial-in, and the unsaved-banner with its save button"). This is the §4 pain quoted back as a feature — the design just rebrands the duplication.
- **The bottom transport bar holding refresh/back/forward/save/settings creates redundancy with the toolbar — by design** (README admits). For new users, having Save in two places + a heart icon (3 save surfaces) increases choice paralysis.
- **Apply & Reset has no confirm step.** The destructive banner is good copy; no UI gate.
- **Calling the active session "Now playing" is friendly but slightly misleading — a playground isn't time-bounded — and the scrub bar is decorative (shows session age, not progress)** — the README admits this metaphor leak. A user reading "1:42 / ∞" or whatever the scrub renders won't know what it means.
- **The PR / GitHub / URL launcher entries on the home page and the library** are pure cards with no input affordance.
- **The "Made for you" shelves invite a recommendation algorithm**, but §5 forbids new features. The shelves are static; the metaphor implies personalization that doesn't exist.
- **"Loud, dark, green-accented, deliberately consumer-facing"** can read as off-brand for a WordPress dev tool — README acknowledges and counters with "the deep tabs stay serious", but the home view is hard to take seriously when it's labelled "Made for you" and "Browse all" like a streaming service.
- **The launcher drawer is dissolved into home shelves**, which is good for discoverability but the full 43-blueprint gallery still opens as a modal — so the "no drawer" claim only partially holds.
- **Heart icon for save is great, but "Save to library" in Spotify is non-destructive idle action; "Save site locally" in Playground is necessary for survival.** The metaphor minimizes the urgency the §4 pain demands.
- **Two destructive banners exist** (lines 574 and 838) — both surface the same warning text in slightly different copy. Pick one canonical message.

## How to push harder
- Make the heart save indicator a louder, persistent state — Spotify dims unfilled hearts in your library; the dim/filled state should map to unsaved/saved playground and pulse on the bottom bar when a refresh-risk is detected.
- Replace the static "Made for you" shelves with shelves that genuinely vary based on something in §2 (recently used blueprints, ones tagged Featured, ones tagged matching your last-loaded blueprint's category) — without inventing user personalization, just based on §2 data the design already has.
