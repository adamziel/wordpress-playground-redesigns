# Review of #020 — Email client (Mail.app)

**Direction:** "Email client (Mail.app): 3-column. Mailboxes = blueprints. Message list = saved playgrounds. Reading pane = the live site."

**Scores:** Feature completeness 8/10 · Direction 9/10 · IA 7/10 · Visual 8/10 · Originality 8/10

## What works
- Three-column composition delivered crisply: Mailboxes (left) with blueprint categories + Your Playgrounds + "From WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip" rows as smart-mailboxes; Message list (middle) with sender/subject/preview/timestamp/unread-dot per blueprint row; Reading pane (right) with iframe + Mail-style action bar.
- Mail's vocabulary genuinely reframes the IA: "Unsaved (this tab)" appears as a special Inbox; saved playgrounds = inbox messages; blueprints = newsletter subscriptions; the "from" field becomes the blueprint author. Cohesive metaphor.
- All 6 §2.2 sources present: WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip as mailbox rows + Vanilla button in a separate spot (line 949).
- Apply & Reset has a `danger` class with the destructive button visible in the inspector/sp-row settings strip.
- Address bar is a real `<input value="/wp-admin/edit.php?post_type=product">` in the reading-pane header — §2.1 item 2.
- Database has the early-access notice. Logs has search input "Search 142 log lines (e.g. PHP Warning, woocommerce)" — usefully shows the log count. Blueprint pane has Run Blueprint / Copy link / Download bundle. All five §2.3 tabs surface as a detail-view tab strip at the bottom of the reading pane.
- Mobile section (line 1513+) translates to a single-column list with chevron-right rows, like Mail iOS. Good fallback.
- Two settings surfaces (popover at line 1253 + inspector inline at line 1334) — the README's "merged into single Flag/Inspector drawer" claim is mostly honored: the second surface is actually a smaller embedded copy in the Site Manager strip.

## What's missing or weak
- **The "Reply = Refresh / Forward = wp-admin / Archive = Save / Move = Export PR / Junk = Download .zip" mapping** is cute but README admits it's "discoverable only through tooltips." A first-time user clicking "Junk" to download their playground will not parse it. The metaphor cost is real and the design accepts it without compensating beyond tooltips.
- **Apply & Reset has no confirm step.** The destructive button fires; no "Are you sure?" modal even though Mail's own UI has "Delete X messages?" patterns. The metaphor invites it.
- **The "sender" column** invents WordPress contributors as authors per blueprint (README explicitly says "I cast WordPress contributors as the 'sender' of each blueprint, which adds charm but may confuse"). This is invented metadata bordering on §5 ("don't extend, reshape") — though it's a labelling choice not a feature.
- **The unread-dot = "not booted yet" semantics** is a clever reframe but conflicts with `Unsaved (this tab)` semantics elsewhere. Two state systems collide: unread dots for booting, "Unsaved" flag for save-state.
- **The PR / GitHub / URL launcher rows are mailbox entries** — clicking should reveal a paste field, but the artifact shows them as static rows that select different "inboxes". The input flow §3 (11–12) is implicit.
- **WP Admin / Homepage jump buttons exist as Forward/Reply All gestures** per the README, but with the cryptic semantic mapping, a power user knows where they are; a first-time user doesn't. Less direct than #014 or #015's explicit chrome.
- **Mobile collapse to bottom-sheet message list → full-screen reading pane** is described as "like Mail iOS" but the artifact shows a single-column list with row-arrows — the full-screen reading pane state on mobile isn't demonstrated.
- **The "Flag = Settings" mapping** lives in the action bar; a first-time user would not know the flag icon opens settings.
- **Three-column at 1440px is dense; the README admits "demands an aggressive collapse on mobile"** — the design accepts this but doesn't fall back gracefully on tablets / narrow desktops.

## How to push harder
- Pick one or two action-bar mappings that *aren't* gimmicks. Reply/Forward/Archive can stay metaphorical; Junk → Download .zip and Move → Export PR are stretches that hurt usability. Replace them with named buttons or move them to the "More actions" overflow that Mail itself uses.
- Render the Apply&Reset as a Mail-style "Delete X items?" confirm sheet — Mail's own destructive pattern, applied to the rest-of-playground destructive action.
