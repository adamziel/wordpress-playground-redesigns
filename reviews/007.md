# Review of #007 — Dark cyberpunk hacker terminal

**Direction:** "Dark cyberpunk hacker terminal: green-on-black, monospace, ASCII separators. The iframe sits inside a faux CRT bezel. Settings shown as wp-cli commands you can copy."

**Scores:** Feature completeness 8/10 · Direction 10/10 · IA 6/10 · Visual 9/10 · Originality 9/10

## What works
- The most stylistically committed entry I've reviewed. JetBrains Mono + VT323, real CRT scanlines (a `repeating-linear-gradient` on `body::before`), vignette via `radial-gradient` on `body::after`, ASCII tree markers (`▾ ▸ ◆ ◇ ≡`), `dmesg`-style toasts, kbd-styled keycaps. Pure phosphor green `#00ff9c` on `#050b07`. This actually feels like the direction.
- The address bar is a real `<input class="cmd-input" value="/wp-admin/edit.php?post_type=product">` prefixed with `wp://` — satisfies §2.1 item 2 with the terminal-prompt metaphor wrapping it.
- The CRT bezel around the iframe shows `RENDER:// playground.wordpress.net/wp-admin/edit.php` plus telemetry (`200 OK · 132ms · 9 queries`) — the WP site really feels nested inside the terminal.
- Apply+Reset gets a `!! DESTRUCTIVE` warning callout in red with explicit `Confirm with y / N` text below the popover. Strongest destructive-confirmation copy I've seen so far, in keeping with the direction.
- All five §2.3 tabs present as a pane rail `[1] settings / [2] files / [3] blueprint / [4] db / [5] logs` selected by digit keys. Database carries the "early access" label (line 1059: `[C] db browser (:4) — early access`).
- Site Manager has Save site locally (`:save site locally`), wp-admin and home buttons, rename ("[edit]"), actions overflow with "export to github as PR (`:gh push`)" and "download as .zip (`:zip`)".
- Logs pane has a `grep` input filter plus level filters in line; toasts render as `dmesg` lines with timestamps (`+0ms`, `+1.2s`, `+4.4s`) — distinctive and brand-perfect.
- Mobile collapses to a TUI-themed phone with a 5-tab bottom bar (site/files/bp/db/more); the CRT metaphor degrades to a small `.ph-crt-content` block — acknowledged trade-off in the README.

## What's missing or weak
- **Brief violation: invented feature.** The Site Manager env block shows `storage: browser/opfs` AND an OPFS quota meter (`142.4 MB / 2 GB`) AND the Settings popover has a `storage: browser/opfs ▾` selectable row. §5 forbids new features; the storage selector is the same overreach #003 made. There's only one OPFS storage in §2.
- **Settings still appear in three places** (the Site Manager pane #1, the quick-edit popover triggered by toolbar gear, AND the Site Manager `[D] settings popover`). The README claims "the three settings entry points all collapse into one quick-settings popover" but the artifact renders both the popover AND the in-pane env+quick-actions row. Same data, three surfaces.
- **The "type `:` and the command you want" claim is partially unsupported.** The command palette `[F]` is rendered but the input shows "apply blueprint coffee" as a value — there's no input for `:save`, `:zip` etc. in the address bar to make typing a verb the primary verb. Most of the time the user clicks chip-buttons.
- **All 43 blueprints not enumerated in the visible artifact.** A bp data array exists (line 1399), but the launcher overlay shows ~12 cards on screen (above the §5 minimum, but for a "tmux 3-column" layout the README promises, the artifact only shows two real columns rendered).
- **No `prefers-reduced-motion` for the blink/spinner animations or scanlines.** The CRT effect is potentially nausea-inducing; some accessibility token would be a sign the designer thought about the trade-off they admitted.
- **WP Admin / Homepage buttons live inside the Site Manager pane as chips (`⌘ wp-admin`, `⌂ home`)** — discoverable but the §2.3 header buttons (which the brief paints as primary) are pushed into a side pane. The CRT bezel doesn't have a homepage button.
- **"Apply & reset" appears with the destructive notice and `y/N` confirmation copy, but the actual confirmation step is described, not rendered.** Show the y/N modal as a fourth state — turn that gorgeous warning into actual gameplay.
- **`network: offline` in the env table conflicts with the `network=offline` toggle in the Settings popover** — but the toggle is shown with no on-state, yet the env says offline. The settings UI shows a state but the toggle position contradicts it. Minor consistency miss.

## How to push harder
- Remove the storage selector and the OPFS quota meter; restore those to "info-only" telemetry not a setting. The brief is explicit: don't extend, reshape.
- Render the `y/N` reset confirmation as a real fifth state — a centered green-on-black modal with the wp-cli command preview (`$ wp-playground reset --confirm=y` / `$ wp-playground reset --confirm=N`) and a real input field that requires typing `y` to commit. That's the moment the direction unlocks something no other design can.
