# Review of #013 — Game launcher (Epic-style)

**Direction:** "Game launcher (Epic-style): vertical nav, huge hero card for the featured blueprint. Animated cover thumbnails."

**Scores:** Feature completeness 9/10 · Direction 8/10 · IA 7/10 · Visual 8/10 · Originality 6/10

## What works
- Epic Games Store look-and-feel: narrow vertical icon rail, colossal auto-rotating hero, ken-burns shelves, "Featured / Continue Playing / New Releases / Browse All 43" rows. Storefront-as-launcher is committed-to.
- All six §2.2 sources are labelled tiles in the "+ NEW" launcher row: Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip. Complete inventory.
- Address bar is a real `<input type="text" value="/wp-admin/edit.php?post_type=page" aria-label="Site URL">` — §2.1 item 2 satisfied with accessibility label.
- Two Apply+Reset surfaces exist: the in-game overlay AND a settings popover. Both carry destructive copy ("Applying will erase the running playground" / "This cannot be undone"). The cannot-be-undone version is stronger than baseline.
- Settings popover has WP / PHP / Language / Network / Multisite; the overlay's Settings tab has the same plus an "Unsaved Playground" notice with "Save site locally" inline.
- DB tab carries "Database management is in early access" notice; Files has a tree; Blueprint has Run Blueprint, Download bundle, Copy link buttons; Logs has search.
- Boot state rendered as a step list ("Downloading WordPress core" → "Activating theme" etc.). Toast "Blueprint URL copied to clipboard" shown.

## What's missing or weak
- **Two settings surfaces exist** despite the README claiming a single source of truth. The quick popover (line 1145–1158) and the in-overlay Settings tab (line 1247–1260) duplicate the exact same five rows + Apply&Reset. The README admits "the duplicate quick-settings popover is kept" — explicit acknowledgement of the §4 duplication.
- **No confirm step for Apply & Reset.** Both surfaces have a single red button. "Cannot be undone" is good copy, no UI gate.
- **The hero's "LAUNCH" verb replaces "Save / Apply / Run"** per the README, but in practice the artifact still uses Save / Apply & Reset / Run Blueprint as labels in multiple places. The unification claim is not delivered.
- **The PR / GitHub / URL launcher tiles** are pure tiles without input. §3 flows 11–12.
- **Originality is weak.** This is "Steam reskinned with Epic's specific accent color" — design #012 already did the launcher-store metaphor. The README admits the trade-off ("pushes power-user work behind a single overlay gesture") which is exactly Steam's trade-off. The two designs do similar work; #013's specific Epic gestures (narrow icon rail, hero auto-rotation, ken-burns) are decorative differences, not IA differences.
- **Vanilla WordPress and the saved-mock data live at the bottom in a JS array (line 1523+)** — the static HTML's library shelves probably depend on JS to render.
- **The "Unsaved Playground" label is shown in the top HUD (`#pg-name-label`)** but it's small text, not a louder warning. §4 risk underaddressed.
- **The 43-blueprint search overlay has TWO `<input placeholder="Search blueprints…">` rendered** (lines 1091 and 1109) — identical inputs in nested overlays. Duplicate search bars.
- **The Logs tab and Files tab are inside the right-side in-game overlay** — but the README didn't show a transition state from "WordPress playing" (chrome dimmed) to "overlay open" (chrome visible). The dim-when-playing claim is unverified in the artifact.
- **The "Continue Playing" / saved sites rail mixes "Unsaved Playground"** with saved entries, blurring the distinction the brief enforces.
- **WP Admin / Homepage jump buttons** — I don't see them in the HUD. They're not in §2.1 (top toolbar) but §2.3 (Site Manager) demands them; the overlay's tab list may have them but they're not surfaced.

## How to push harder
- Actually deliver the "one source of truth" settings claim — wire the toolbar gear to focus the overlay's Settings tab instead of opening a separate popover. Same UI, one surface.
- Differentiate from #012 by leaning into a feature Epic does that Steam doesn't — weekly free blueprint of the week, achievement-style "blueprints completed" indicators, or a Mature Content gate for the "early access" DB tab. The current design's only Epic-vs-Steam axis is decorative.
