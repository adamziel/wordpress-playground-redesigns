# Review of #046 — Vercel/Netlify Dashboard

**Direction:** "Vercel/Netlify dashboard: deploy-style cards for each playground. 'Build logs' panel slides up. 'Promote to production' doesn't exist — that becomes 'Save'."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 8/10

## What works
- Vercel/Netlify shell delivered: All Playgrounds index with deploy-style cards (commit-like IDs, status pills, sub-line meta), single-project view with tabs (Overview / Files / Blueprint / Database / Logs / Settings), sticky build-status topbar, sliding build-logs drawer.
- **"Save" = "Promote to production"** rebrand is consistently applied: line 1408 has `<button class="promote-cta">Save site locally</button>` — the design's central thesis lands.
- All 6 §2.2 sources rendered as deployment "Git provider" tiles (lines 1172–1192): Vanilla WordPress / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each with sub-line like "Boot from any gutenberg PR #", "Paste a remote blueprint.json", "Restore an exported playground". (WordPress PR is presumably nearby; verify visibility.)
- Apply+Reset has explicit destructive copy: "**Apply & Reset Playground** — destructive. Wipes the current site and re-boots." (line 1027) with `btn-danger` class.
- Address bar `<input value="/shop/">` (line 908) — §2.1 item 2.
- Database has the early-access notice verbatim with GitHub-tracker link (line 1325).
- Logs has filter input (line 970) + a separate "Search deployments" input (line 1359). Search across deploy history is the Vercel/Netlify killer feature being borrowed.
- Blueprint pane has Copy link / Download bundle (lines 1239–1240). Files tree present.
- "Preview (Unsaved)" status pill (line 752) — the metaphor maps Unsaved Playground to a Preview deployment, which is a brilliant reframe that the §4 lose-on-refresh risk benefits from.
- The README is honest about the metaphor stretch: "Playground doesn't actually build or deploy anything, so words like 'Deployment', 'Preview', 'Production' are aspirational. To stop confusion, status pills always reflect Playground reality ('Booting', 'Ready', 'Saved')." Brief-aware.
- Mobile collapses dashboard to single stack with tab bar — admitted loss of "scan a fleet" pleasure.

## What's missing or weak
- **Apply & Reset has no confirm step.** Vercel itself has a "Type the project name to confirm deletion" pattern for destructive actions; this design's `btn-danger` fires immediately despite the explicit destructive copy.
- **The README claims "the three settings entry points are collapsed into a single Settings tab"** but the quick popover still keeps "the most common quick-edit (WP & PHP version)" — that's two surfaces.
- **The metaphor stretch is real** — "Deployment / Preview / Production" doesn't quite fit a single-instance WASM playground. README admits.
- **The PR / GitHub / URL launcher tiles** are tile faces without inline paste inputs visible in the artifact's main flow.
- **The blueprint gallery as "templates page with framework-style tags"** is on-metaphor but the §2.2 categories (Featured / Website / Personal / Content / Themes / Gutenberg / Experiments / WooCommerce / News) may not map cleanly to "framework tags". Verify the category chips are present.
- **The dashboard chrome eats pixels** — README admits the iframe loses room.
- **Mobile loses the "scan a fleet" pleasure** — README admits. The multi-pane dashboard's strength is dense overview.
- **No big "Open Site" button** like #045 — instead the "promote-cta" Save button dominates. Different choice; aligns with the "Save = Promote" thesis.
- **Build-logs drawer that slides up** — described, may not be rendered as a separate state. The grep doesn't surface a `slide-up` build-logs animation.
- **Status pill says "Network: domain mediadl.cdn.failed"** (line 805) — a specific Network-failure surface, which is great realism for the deploy metaphor.

## How to push harder
- Render the "type the project name to confirm" destructive confirm Vercel actually uses. The deploy metaphor's strongest destructive UX is the named-confirmation pattern; this design's `btn-danger` fires without it.
- Demonstrate the build-logs slide-up drawer as a separate state. Vercel's slide-up build-log is one of its signature interactions; without it, the "deploy-platform" framing is partially undelivered.
