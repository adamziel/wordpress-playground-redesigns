# Review of #002 — IDE-like

**Direction:** "IDE-like: VS Code aesthetic. Activity bar on the left (icons), tabbed editor in the middle for the WordPress iframe + blueprint/file editors, status bar at the bottom (PHP version, WP version, logs counter, save status)."

**Scores:** Feature completeness 10/10 · Direction 9/10 · IA 7/10 · Visual 9/10 · Originality 6/10

## What works
- All five direction elements are present and crisp: 48px activity bar (Explorer / Search / Source Control / Run / Blueprints / Settings), tabbed center editor with the iframe as one tab next to `blueprint.json`, `functions.php`, `wp-config.php`, Logs, DB, Welcome, and a bottom status bar showing unsaved-dot, Save, WP 6.7, php 8.3, en_US, Network · on, "7 logs", SQLite · 1.2 MB, ⌘K.
- Every §2 feature is reachable as a named, labelled affordance:
  - §2.1: refresh button, back/forward, URL breadcrumb (`playground://twenty-twenty-five-demo/wp-admin`), name pill with unsaved dot, Save site button, WP Admin and Homepage jump buttons.
  - §2.2: dedicated Welcome tab with 6 "Start new" cards (Vanilla / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip) + Featured blueprints + Your Playgrounds list + 10 category filter chips matching §2.2.
  - §2.3: Settings pane with all five rows (WP version / PHP / Language / Networking / Multisite) + a Source Control pane that promotes "Export to GitHub PR" and "Download .zip" as first-class actions next to Import.
  - §2.4: Toast region (top-right) demonstrated with two real toasts including the "Unsaved changes" warning about refresh-loss.
- The Logs pane carries severity pill filters (Error · 1 / Warning · 3 / Info · 2 / Notice · 1) AND a regex search field — addresses §4's "unstyled text, no filter" pain.
- DB pane has the early-access notice text verbatim, a tables sidebar, and a SQL bar — directly fulfills §2.3 tab 4.
- Status bar pills are interactive surrogates (clicking WP/PHP pills should open the version popover) and a quick-settings popover IS rendered with all five settings + Apply&Reset.
- "Working tree changes" inside Source Control is genuinely novel — modified files highlighted before export, a real "git-like" diff staging surface that turns the Site Manager's File-tree+Blueprint duplication into a single working tree.

## What's missing or weak
- **Settings duplication is exactly what §4 complained about.** Settings live in (a) the Settings pane (full page), (b) the popover, AND (c) clickable status-bar version pills. The README claims they "merge into one visual surface" — they don't; three different layouts render the same rows.
- **Apply Settings & Reset has no confirm step rendered.** The button has a red `danger` class and "Resetting wipes the current site — files, database, everything" caption, but tapping it would fire immediately. The popover version has the same problem. §4 explicitly flags this as a current pain point.
- **The URL breadcrumb is not editable.** `playground://twenty-twenty-five-demo/wp-admin` is rendered as a series of `<span>`s. §2.1 item 2 mandates a typeable path with submit. The breadcrumb is decorative.
- **The Welcome tab's "Featured" grid is rendered by JS (`id="featGrid"`) and the full 43 grid is also JS-rendered (`id="bpGrid"`).** Without running the script the cards are invisible — the static HTML only shows the Twenty Twenty-Five Demo and a few placeholders. If JS fails, the gallery is empty.
- **Activity bar has a "Source Control as Export" entry** which is a clever metaphor, but it lives next to a Run icon and a Blueprints icon — the user has no hint that "export to GitHub PR" lives under a git-merge icon, not under a github icon.
- **Mobile fallback collapses the activity bar to a bottom tab bar** per the README, but I don't see a `@media` mobile state demonstrated in the artifact — only the IDE shell at desktop. The brief asks for 390×844 responsive; this might break the §5 hard constraint.
- **The "Network access" and "Multisite" toggles in the Settings pane** show "Enabled · wp_remote_get works" — accurate but the toggle UI doesn't have a focus state or destructive-warning that toggling Multisite requires a reset (only the global "Apply settings & reset" surfaces it).
- **"Address bar" pain point unresolved.** The breadcrumb shows current path but the URL is split across three spans (`scheme`, `host`, `path`); a user typing `/wp-admin` needs to click which one? The README admits the address bar is "contextual" and moved out of the global toolbar, but the replacement is unclear about what's clickable.

## How to push harder
- Cut Settings to one surface. The popover + status pill + Settings pane triad recreates the very problem this design claims to solve. Make the version pills *jump* to the Settings pane (pinning the row) rather than opening their own popover, and remove the duplicated quick-popover.
- Render the Apply+Reset confirm modal as one of the visible states. A VS Code-style "save before reset?" dialog would let you both honor the destructive warning AND keep the IDE metaphor.
