# Review of #024 — Tile dashboard (Windows 8 / live tiles)

**Direction:** "Tile dashboard (Windows 8 / live tiles): blueprints and saved sites as resizable tiles with live status (running, idle, error)."

**Scores:** Feature completeness 9/10 · Direction 9/10 · IA 7/10 · Visual 8/10 · Originality 8/10

## What works
- Real Metro / Windows 8 aesthetic: flat color blocks, big tile groups, semantic-zoom-style category pills, flipping live-tile content with status pills (Running / Idle / Booting / Error / Unsaved). The tile-as-everything thesis is honestly delivered.
- All §2 features mapped to tile groups: "Site Tools" group with Settings / Files / Blueprint / DB / Logs tiles (each showing telemetry like "Avg. response 38ms" or "Database (early access)"), "Sources" group with Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip + Download .zip + Export to GitHub PR alongside, blueprint group with 43-search input.
- Apply & Reset tile shows "Apply & reset" as a tile face (line 601) AND a full Settings panel has a `danger-btn` with explicit `(destructive)` suffix (line 1244): "Apply settings & reset (destructive)".
- Address bar `<input id="urlInput" value="/wp-admin/edit.php?post_type=product" spellcheck="false">` in a thin charm-bar on the right edge — §2.1 item 2 satisfied with the metaphor's right-edge convention.
- Database tile carries `early access feature` notice with GitHub-issue-tracker link verbatim. Logs has search input "Search PHP error log" and level pills. Files tile has tree. Blueprint has Run Blueprint + Copy link + Download bundle.
- Unsaved Playground tile uses `corner-status idle` with a pulse animation + "Unsaved" label — the live-tile metaphor makes the §4 risk visible at a glance.
- README claims the design "removed duplicate Settings entry points: one Quick Settings tile flips between WP version / PHP / Language; clicking it expands into the full popover inline" — the tile is at line 596 and the Settings panel at line 1231+. Both surfaces exist.
- Failed-boot tile ("From GitHub · failed boot") demonstrates the Error live-status — a clever way to make the §4 "no preview" pain less harmful for failed boots.

## What's missing or weak
- **Settings still appear in multiple places.** The Quick Settings tile (line 596+), the full Settings panel (line 1231+), AND an apply-reset tile (line 601). The "demoted to the same visual weight" claim is honest but it doesn't reduce the duplication count.
- **Apply & Reset has no confirm step rendered.** The `danger-btn` fires. The README claims "the live tile flips to a confirm state" but the artifact only shows the button.
- **The PR / GitHub / URL launcher tiles** are tile faces without inline paste inputs (line 1466: `showToast('Paste a Gutenberg PR number')` — so tapping toasts a hint rather than showing an input). §3 flows 11–12 mostly punted to a toast.
- **Resizable tiles imply per-user state** — README admits "state is per-user, so first-touch users see a designer-curated default." Fair, but the artifact doesn't demonstrate resizing affordances.
- **Flipping tile content can hide info behind a flip cycle** — README admits and mitigates with "hover pauses flip" + "status pill always visible". The animations themselves aren't fully shown.
- **Win key / charm bar metaphor** is a niche reference; users under 30 may not have used Windows 8. The README acknowledges Metro "reads as toy-ish to people who associate it with Windows 8's commercial failure" — and the gallery view inherits that risk.
- **The 43-blueprint gallery is two `<input id="bpSearch">` + `<input id="bpSearch2">` in different sections** (lines 914, 1394) — duplicate search inputs.
- **The "Site Tools" group tiles each show telemetry** (last error, file count, last log line), which is great for live-tile semantics — but the §4 pain about logs having no level filter is still the responsibility of the Logs tile interior, which the artifact shows has search + level pills, so this part works.
- **Mobile collapses to a single column of 2x-wide tiles** — described, not necessarily fully rendered in artifact.
- **WP Admin / Homepage** appear as part of iframe context but I don't see them as dedicated tiles in the Site Tools group. They live behind the address bar / charm bar.

## How to push harder
- Render the resizing affordance — let one tile expand to "wide" or "large" (Metro convention). Show a "pinned" state on the iframe tile to demonstrate why it's giant.
- Implement the destructive confirm as a live-tile flip — the Apply & Reset tile flips from "Apply & reset" face to a "Tap again to confirm" face, with the WP/PHP/language change diff shown. The metaphor invites this; the artifact doesn't deliver.
