# Review of #028 — Two-up split view

**Direction:** "Two-up split view: iframe on top, controls on bottom — like a video editor's preview + timeline."

**Scores:** Feature completeness 9/10 · Direction 9/10 · IA 7/10 · Visual 8/10 · Originality 7/10

## What works
- Authentic video-editor metaphor: full-bleed program monitor (iframe) on top, transport strip directly beneath (refresh, address bar, save, name, settings, site-manager), and a 3-pane bottom bay (Browser left / Timeline center / Inspector right). Resizable + collapsible bay per README.
- Blueprint boot steps and logs render in a center "timeline" — clever remix that takes a video-editor's strongest affordance (timeline) and uses it for what blueprint authoring needs: ordered step visualization.
- All 6 §2.2 sources rendered twice — as start tiles in the Browser rail (lines 726–731) AND as mobile menu items (lines 1275–1280): Vanilla WP / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip. Each tile has a sub-label hinting at the input ("core/wordpress PR #", "Paste any blueprint JSON URL", "Previously exported playground"). Better PR-input hinting than most.
- Apply+Reset destructive copy is in two places — popover ("Applying these will reset the playground. Unsaved changes are lost.") and the inspector Settings tab ("Applying changes will reset the playground. Your unsaved files, posts, and DB will be wiped."). The second is more specific.
- Address bar `<input value="/wp-admin/edit.php?post_type=post">` in the transport strip — §2.1 item 2.
- Database carries the early-access notice. Logs has filter input with example "PHP Notice, woocommerce, /wp-admin". Blueprint pane has Run Blueprint + Copy link + Download bundle. Files tree visible.
- Mobile section rendered (line 1240+) collapsing to vertical stack with bottom-sheet bay — admitted loss of side-by-side feel.
- Inspector hot-swap (Settings / Files / Blueprint / DB / Logs) keeps each tool first-class without sidebar bloat.
- Save/Unsaved label visible in transport strip; pulled into a persistent "Unsaved Playground" pill (line 754).

## What's missing or weak
- **Apply & Reset has no confirm step.** Despite explicit destructive copy in both surfaces, neither has an "Are you sure?" gate. A video-editor metaphor invites the "Save before quitting?" prompt every NLE shows.
- **The README admits "the Inspector hot-swapping behavior means you can't see Files and Logs at the same time"** and explicitly punts on the "pop out" feature ("it would have been a new feature, which the brief forbids"). Honest constraint, real cost for power users.
- **The popover-vs-inspector duplication of Settings** — README claims "three settings entry points collapse into one popover triggered by a gear icon" but the artifact has settings in both the popover AND the inspector's Settings tab (lines 654–659 and 893–901). Same data, two surfaces. §4.
- **The PR / GitHub launcher tiles** show "Paste PR #" sub-copy but don't expose an inline paste input — clicking the tile must open a deeper input form, which isn't rendered.
- **The timeline-style step viewer is more legible for blueprint authors than for someone who just wants 'Vanilla WordPress, please'** — README admits and points users to the "+" button in the rail. Good honesty, real cost.
- **Three-pane bottom bay at 1440x900 leaves the iframe top-half** — that's ~450px of vertical for the iframe, comparable to today's chrome but less than the brief's intent of "iframe dominates". A vertically-half iframe makes scrolling /wp-admin more cramped.
- **The "+" big new-playground button is described as not the loudest thing on the screen** — README admits the first-touch user might miss it. Could be louder.
- **Collapsing the bay gives the iframe "the entire viewport" — the 'full-screen monitor' mode** is a great escape hatch, but it isn't rendered as a state in the artifact. Show the collapsed view.
- **The transport strip's "Site Manager toggle"** is described but it's unclear what it actually toggles — the inspector is one tab of the bay, so does the toggle hide the entire bay or just the inspector?
- **The Logs / Timeline merging** — both live in the center timeline, but Logs are real-time and Blueprint steps are pre-defined. The metaphor might conflate two different timelines.

## How to push harder
- Render the collapsed "full-screen monitor" state — the README's strongest claim about giving the iframe space. Without it, the metaphor cost (chrome eats half the viewport) is the design's defining cost without any escape hatch demonstrated.
- Implement the Apply & Reset confirm as the video-editor's "Save before closing?" pattern — a modal that mimics Premiere's "Discard changes? You'll lose your edits to: [list]" with explicit file count.
