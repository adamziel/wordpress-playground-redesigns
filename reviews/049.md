# Review of #049 — Maps

**Direction:** "Maps: imagine WordPress versions, PHP versions, blueprints as locations on a map you can pan around. Click a pin → boot that combo."

**Scores:** Feature completeness 8/10 · Direction 10/10 · IA 6/10 · Visual 10/10 · Originality 10/10

## What works
- Most original direction interpretation in the gallery so far. The Playground rendered as a hand-drawn cartographic atlas: islands for blueprint categories (Themes Archipelago, Gutenberg Volcanic Range, WooCommerce Coast, News Cape, Personal Bay, Content Mainland, Experiments Outer Reach), compass-rose dial for WP/PHP versions (latitude/longitude), bookmark pins for saved playgrounds, Captain's Log drawer for Site Manager, "More routes" sextant for additional actions.
- All 6 §2.2 sources rendered as Harbor pins on the map (lines 951–971 SVG text labels + lines 1268–1273 harbor-items list): ⚓ Vanilla Harbor / 🔧 WordPress PR / 🧊 Gutenberg PR / 🐙 From GitHub / 🔗 Blueprint URL / 📦 Import .zip — each with sub-line ("Just sail.", "Boot from a core PR.", "Paste any JSON URL.", "Re-sail a saved bundle.").
- Apply+Reset has on-metaphor copy at line 1071: "**Apply & reset** wipes the site and re-sails to your new coordinates. There is no undo." + a rose-colored button.
- Address bar is real: `<input value="wp-admin/edit.php?post_type=post">` wrapped as a "scroll" (line 617).
- "Save site locally" as an anchor button (line 1053) — anchoring metaphor for save.
- Database has early-access notice + GitHub-tracker (line 1146). Logs has search input (line 1173). Blueprint pane has Copy link / Download bundle (lines 1117–1118).
- Atlas search input: `<input value="woo" placeholder="Search the atlas…">` (line 1208) — search highlights matching pins, dims rest.
- The README is honest about the unfamiliarity cost: "A map is unfamiliar for a dev tool — first-time users may not immediately read 'pin = blueprint.' I mitigate with explicit pin labels and category headings on each island, but it's a learning curve cost in exchange for memorability."
- Mobile turns map into a "spyglass" vertical stack — sensible.
- The map keeps as one SVG + CSS-gradient ocean to control paint weight per README.
- Captain's Log with tabs (Settings, Files, Blueprint, Database, Logs) inside one drawer (line 1017+) — clever consolidation.

## What's missing or weak
- **Apply & Reset has no confirm step rendered.** The "There is no undo" copy is strong; no UI gate.
- **The cartographic metaphor flattens hierarchy** — README admits "a 43-pin gallery sprawls wide and demands panning, while a grid sorts faster." Real cost for power users.
- **At small zoom levels the labels collide** — README admits. Cartographic legibility is the metaphor's main risk.
- **Two settings surfaces** — the "Captain's Log" Settings tab (line 1066–1074) and a separate Settings popover (line 671–674). Despite the README's unification claim, two surfaces exist.
- **The PR / GitHub / URL launcher rows in the harbor list** are tile-style without inline paste inputs. §3 flows 11–12.
- **The "latitude = WP, longitude = PHP" compass dial** is brilliant in concept — but verify whether the actual interactive dial allows the user to pick coordinates and see the resulting WP×PHP combination. The artifact suggests it's mostly visual.
- **Painting weight** — README claims SVG + CSS gradient keeps it snappy, but a hand-drawn atlas with 43 pins is busier than most designs in the gallery.
- **The "marketing landing page is replaced by the map itself"** is bold — first-touch users on `playground.wordpress.net` see a fantasy atlas, not WordPress. Strongest version of the iframe-demotion bet. The README's strength is acknowledging it; the design's strength is committing to it.
- **The Captain's Log drawer swinging from the right** — verify the swing animation is described; static viewers see only the open state.
- **The Save = anchor metaphor** is delightful but a first-time user won't parse "anchor icon = save".
- **WP Admin / Homepage jump buttons** — likely inside the Heading bar or the Captain's Log overlay, but I don't see them surfaced as discrete chrome buttons.

## How to push harder
- Render the destructive Apply & Reset as a "Set sail for new coordinates? You'll lose your current cargo (5 posts, 3 plugins)." prompt — the map metaphor's destructive UX language is literally there to use.
- Make the compass-rose WP×PHP dial interactive in the artifact — even a static state showing "you've moved the compass from 6.6×8.2 to 6.7×8.3, here's what changes when you set sail" would be the strongest demo of the design's strongest novel interaction.
