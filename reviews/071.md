# Review of #071 — Persona-tabbed top nav

**Direction:** "Persona-tabbed top nav: top tabs for the four personas; each surfaces a curated set of blueprints + tools."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 9/10 · Visual 8/10 · Originality 9/10

## What works
- Direction fully delivered: four persona tabs at top (Theme Author / Plugin Author / Site Owner / Learner) + a fifth "All tools" tab (lines 1399–1419). Switching personas re-tunes the entire chrome.
- **The persona-tuned chrome is genuinely behavioral, not just labeled**:
  - Address bar suggested-jumps swap per persona (line 1450: `data-targets="theme:/wp-admin/themes.php plugin:/wp-admin/plugins.php owner:/wp-admin/ learner:/wp-admin/edit.php"`)
  - Site Manager default tab varies per persona (Files for Theme Author, Logs for Plugin Author, Settings for Site Owner, Blueprint for Learner) per README
  - Launcher curates featured blueprints per persona ("Picked for Theme Authors", line 1865)
  - "Theme Author workspace" titled section (line 1710)
- All 6 §2.2 sources rendered (lines 1815–1855): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip.
- Apply+Reset: `<button class="btn-danger">Apply & Reset</button>` (line 1598).
- Address bar `<input id="addr" type="text" value="/" aria-label="Path">` (line 1448) with aria-label.
- Database has the early-access notice with GitHub-tracker link (line 1668). Logs has filter input "Search log lines…" (line 1685). Blueprint has Run Blueprint / Copy link / Download bundle (lines 1629–1631).
- Unsaved-Playground notice: "**Unsaved Playground.** Your changes will be lost on page refresh." + Save site locally button (lines 1564–1565). README claims a persistent amber pill in the toolbar that turns amber when unsaved — addresses §4 with a persistent indicator.
- Playground title editor `<input class="title-input" value="Unsaved Playground">` (line 1498).
- **The "All tools" fifth tab** is the power-user escape hatch — exposes the full unfiltered surface. README's mitigation for "I just want everything" users.
- README is honest: "Persona tabs add a layer of IA that veteran users may resent ('just give me everything'). I mitigate that with a 'All tools' fifth tab." + "The persona choice is sticky (localStorage) but never blocks — switching is one click and non-destructive."

## What's missing or weak
- **Apply & Reset has no confirm step.** The danger button fires.
- **Settings appear in the popover AND the Site Manager Settings tab** — README admits "merged into one persona-aware quick-settings popover *plus* the Site Manager Settings tab." Two surfaces openly preserved.
- **The PR / GitHub / URL launcher tiles** have lbl text but no inline paste inputs.
- **First-time misidentification cost** — if someone picks "Site Owner" but they're actually a Plugin Author, the curated featured shortlist will be off. README mitigates: "the curation only affects featured shortlists, not the underlying gallery, so the cost of a wrong guess is a few extra clicks, not lost features."
- **The persona tabs at the top consume vertical space** — but they're a single row, so the cost is small.
- **Originality is high** — overlaps with #070 (Onboarding-driven) in the persona-targeting lane (direction 70 is "first screen asks 'Are you a theme author / plugin author / site owner / learner?'"). #071's distinguishing move is making personas a persistent top nav rather than an onboarding gate.
- **"Persona-aware" address-bar quick-jumps** are clever but a non-Theme-Author landing on /wp-admin/themes.php will be confused. The persona-tuning happens silently — labels should make it visible.
- **The "default Site Manager tab varies per persona"** — verify whether this is rendered with different default-tab states per persona.

## How to push harder
- Render the destructive Apply & Reset as a persona-aware confirm — e.g., "Apply & Reset will wipe 5 themes (Theme Author) / 3 plugin builds (Plugin Author) / your post drafts (Site Owner)". The destructive UX should leverage the persona context for specificity.
- Make the persona-tuned behavior visibly explained — when a Theme Author lands and sees `/wp-admin/themes.php` as a suggested jump, a small "(tuned to Theme Author)" annotation would close the cognitive gap.
