# Review of #032 — Concept-album minimal

**Direction:** "Concept-album minimal: just the iframe + a single floating '...' button that expands into a radial pie menu of actions."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 6/10 · Visual 9/10 · Originality 9/10

## What works
- The single-black-dot thesis is delivered without compromise. The iframe is 100% of the viewport until the user taps; everything else lives in a radial pie with 8 petals (Refresh / URL / Save / Sites / Blueprints / Settings / Tools / More). Stable petal positions (12 = Refresh, 3 = Settings) per README.
- "The dot pulses softly. Unsaved." (line 554) — the dot itself becomes the §4 unsaved indicator. Single pixel doing double duty. Minimal is mostly delivered.
- All 6 §2.2 sources rendered as the Sites petal's "new from" row list: Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip (lines 682–687) with keyboard hints (`P`, `G`, `H`, `U`, `I`) — keyboard-driven.
- URL petal pops a centered single-input bar `<input type="text" value="/wp-admin/edit.php?post_type=page">` (line 656) — §2.1 item 2 satisfied via the URL petal.
- Apply & Reset has a `btn bad` class (line 815) with explicit "Apply settings & reset playground" copy + the Settings sheet "This is an Unsaved Playground" notice.
- Database has early-access notice with GitHub-issue-tracker reference. Logs petal has filter input "Filter logs (e.g. fatal, deprecated, query)". Blueprint pane has Copy link / Download bundle / Run Blueprint.
- The README's "URL petal as search-everywhere prompt" is clever and on-metaphor.
- Mobile inverts the radial to a bottom-anchored half-wheel — sensible.
- The legend / discoverability mitigation: README admits the dot hosts "a small breathing label on first load ('tap me')" + unsaved-state subtly pulses the ring.

## What's missing or weak
- **Discoverability cost is the design's main weakness** — README admits "first-time users will see only a dot and the iframe — they have to commit one tap to see what Playground can do." The "tap me" label is a single-string mitigation; a more thorough first-time guide is undelivered.
- **Apply & Reset has no confirm step.** The `btn bad` button fires. The concept-album minimal aesthetic invites a hush-modal that says only "Apply & reset?" with two single-character keys — render it.
- **Power users lose the always-on Save button** — README admits and provides petal positions to memorize. The cost is real.
- **The "More" petal is a kitchen sink for Export PR / Download .zip / Import .zip / WP PR / Gutenberg PR / From GitHub / Blueprint URL / WP Admin / Homepage** — 9 actions in one petal. That's not minimal, that's a hidden full menu. The petal opens a sheet that's just a normal list — the concept-album promise is broken at that layer.
- **Settings appears in two places** — the Settings petal's overlay AND the "save your work" nudge inside that same overlay AND a separate Settings sheet rendered at line 791+. Verify there's only one canonical surface.
- **8 petals on a small dot** — radial menus historically struggle past 6 items. With 8 petals, the user must hover-precision more than a typical mouse interaction.
- **The "petal expands into its own sheet"** means once you tap a petal, the radial menu disappears and you're back in a rectangular UI. The radial is just the launcher; the actual work happens in conventional overlays. Worth flagging that the metaphor is thinner than first claim.
- **The PR / GitHub / URL launcher rows** show "Boot from a WordPress core PR number" etc. but no inline paste input — same §3 flow 11–12 problem.
- **No visible WP Admin / Homepage** in the radial except buried in More — §2.3's primary jump buttons are 2 levels deep.

## How to push harder
- Render the destructive Apply & Reset as a concept-album-minimal confirm: just the question "Apply & reset?" centered on a dimmed iframe, with `Y` and `N` keys as the only interaction. Match the radial's "speak only when asked" aesthetic.
- Reduce the "More" petal to 3 truly-rare actions (Export PR / Download .zip / Import .zip) and promote WP Admin + Homepage to the URL petal — the address bar is the right home for "go to /wp-admin", and putting it 2 levels deep contradicts §2.3.
