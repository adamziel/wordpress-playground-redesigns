# Review of #068 — Apple TV grid

**Direction:** "Apple TV grid: featured row, 'Recently Used' row, 'Recommended for theme authors' row, 'Recommended for plugin authors' row."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 7/10

## What works
- Apple TV streaming aesthetic: cinematic dark backdrop, slow-cycling "Now Showing" hero, horizontal poster shelves (Featured / Continue Watching / Recommended for Theme Authors / Recommended for Plugin Authors / Gutenberg & Experiments / WooCommerce). Chrome melts away during a working session, edge-hover reveals.
- All 6 §2.2 sources rendered as "Start New" shelf tiles (lines 1140–1219): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip.
- Apply+Reset: `<button class="reset">Apply & Reset</button>` (line 1773) + "Apply & Reset wipes the running site." copy (line 1777). A second surface at line 2450.
- Address bar `<input value="/wp-admin">` (line 1040) — §2.1 item 2.
- Unsaved-Playground notice (line 1049): "Unsaved Playground."
- "More" overflow with Export PR + Download .zip (line 1633: "Export PR, Download .zip").
- Bottom-docked transport drawer (Files / Blueprint / Database / Logs / Settings) — slides up like Apple TV info pane.
- README is honest: "Hiding chrome during a working session is a delight but bets that returning users will discover the edge-hover reveal."

## What's missing or weak
- **Apply & Reset has no confirm step rendered.** Two surfaces (line 1773, 2450) both fire.
- **The README admits "the three settings entry points collapse to one (the Settings panel inside that drawer)"** — yet the artifact has two settings surfaces (lines 1765–1777 and 2445–2450). Same data, two locations.
- **Edge-hover-reveal of chrome** is undiscoverable to first-touch users — README admits, no first-launch tutorial described.
- **The "Recommended for Theme Authors / Plugin Authors" shelves** imply persona recommendations — the labels are static (not personalized), which is on-brief but the metaphor invites personalization.
- **The PR / GitHub / URL tiles** are streaming-service tiles without inline paste inputs.
- **Database / Logs / Files content** — verify they're rendered as content inside the bottom-docked drawer.
- **A streaming aesthetic is heavy on imagery** — README admits and uses gradient SVG posters as substitute.
- **Originality has overlap with #012 (Steam) / #013 (Epic) / #048 (Apple Music)** — four media-launcher metaphors. #068's distinguishing move is the "chrome melts away during work" + edge-reveal, more aggressive than the others.
- **Horizontal-scrolling shelves on mobile rely on swipe** — README admits "slower than a filterable list when you know what you want", mitigated by search affordance in the hero.

## How to push harder
- Render the destructive Apply & Reset as an Apple TV-style overlay — chrome fades, a centered "Reset Brewroom?" modal with explicit destructive copy. Use the metaphor's existing fullscreen modal pattern.
- Differentiate from #012/#013/#048 by demonstrating the edge-reveal interaction explicitly — show a "Hover edge to reveal chrome" hint on first visit, and a separate state where chrome is fully hidden during work. The hide-on-work claim is the design's distinguishing move; render it visibly.
