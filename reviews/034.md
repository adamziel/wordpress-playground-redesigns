# Review of #034 — Sketch / Excalidraw hand-drawn

**Direction:** "Sketch / Excalidraw hand-drawn: hand-drawn fonts, dashed borders, wonky strokes. Playful."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 7/10 · Visual 9/10 · Originality 8/10

## What works
- Direction commitment is complete: Caveat/Kalam/Patrick Hand handwriting fonts, wonky ±0.5° rotated stickers + sticky-note saved playgrounds, color-coded cardstock (yellow/pink/blue), red marker stamps for destructive actions, graph-rule background, taped feature labels. The metaphor is consistent without being oppressive.
- Saved vs Unsaved playgrounds are visually distinguished by solid stickies (saved) vs dashed dotted-line ghost stickies (unsaved) — the §4 risk is rendered as a different texture, not just a label. Strong.
- All 6 §2.2 sources rendered as cards in the Launcher (lines 1146–1176): Vanilla WP / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each with a sub-line ("Boot any core PR #", "Paste a JSON URL", "Repo → theme/plugin").
- Apply+Reset gets its destructive treatment in three places:
  - Settings popover: "**Heads up:** Apply & Reset wipes the current site. No undo." (line 1120)
  - "Heads up. Apply & Reset will wipe the current site. [Apply anyway] · [Cancel]" (line 1486) — actual cancel/confirm choice rendered as a sketch annotation
  - Inline "Tweak the engine, then Apply & Reset" framing
  The middle one is the closest the gallery has come to a real visual confirm step that fits the metaphor.
- Database has the early-access notice with GitHub-tracker link. Logs has filter input + colored marker level (red ERR / orange WARN / blue INFO / green OK) per README.
- Address bar `<input value="/?p=hello-from-wp-playground">` — §2.1 item 2.
- "Save site locally" with a 💾 emoji button — the unsaved-Playground notice is "Refresh = poof. Save it locally to keep it." (line 1019). The copy is delightful AND clear.
- Blueprint pane has Run Blueprint / Copy link / Download bundle. Files tree present.
- README is honest about real costs: handwriting fonts hurt readability at small sizes; JSON in Kalam laid out monospace-style is "a deliberate compromise"; CJK fonts fall back; an accessibility "level the page" toggle would be added in a real ship. Best-in-class self-awareness.

## What's missing or weak
- **Apply & Reset has TWO surfaces with destructive copy (line 1039 and 1123)** — duplication, and the "[Apply anyway] · [Cancel]" link pair at line 1486 looks like the canonical confirm but it's a tertiary state, not gated behind the initial button click. Verify the confirm state is the actual destination of the buttons.
- **Settings appear in two places** — README explicitly admits ("The three duplicated settings entry points all stay, but I unify their visual language") — the popover at line 1117–1123 and the Site Manager Settings tab. Two surfaces, same data.
- **Handwriting fonts hurt readability** — README admits and mitigates with Patrick Hand for body. Still slower to read than Inter. The DB table at the "doodle" frame is admittedly awkward.
- **The 43-blueprint corkboard with doodled cards + thumb-pins** is great for personality but loses the §4 thumbnail-readability pain point — README acknowledges "purists will say a code editor should never use a handwriting font, and they're right".
- **Wonky rotation gimmick is tasteful at ±0.5°** — README admits would feel oppressive if every element wobbled. Most actually sit straight.
- **The PR / GitHub / URL launcher cards** are scribbled tiles without inline paste inputs.
- **Logs as handwritten lines with marker color for level** is on-metaphor but illegible at high log volume. The artifact may compensate with filter input.
- **Mobile collapses to a hamburger + bottom sheet on a hand-drawn phone frame** — described, sensible.
- **Accessibility cost**: hand-drawn fonts score worse on dyslexia-friendliness. README acknowledges and proposes a "level the page" toggle — the toggle is not rendered in the artifact.
- **Internationalization cost**: Caveat doesn't cover CJK / Cyrillic. README acknowledges — major real-ship constraint.
- **The "red stamp = destructive" pattern (line 1498)** is a great legend, but turning destructive UX into a stamp doesn't add a confirmation gate — it just colors the button.

## How to push harder
- Promote the [Apply anyway] · [Cancel] confirm at line 1486 to the actual destination of the Apply & Reset buttons. Right now the destructive button at 1039/1123 fires; only this tertiary state shows the confirm. Wire them together.
- Render the "level the page" accessibility toggle. The README promised it; show it as a chrome control that swaps to system-ui at the same weights. Best-in-class self-awareness deserves best-in-class follow-through.
