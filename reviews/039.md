# Review of #039 — Neumorphism

**Direction:** "Neumorphism: soft pillowy buttons, inset/outset shadows, monochrome palette."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 7/10 · Visual 9/10 · Originality 7/10

## What works
- Direction commitment is total: a single warm sand/clay base, no borders, no dividers, hierarchy entirely through depth (convex = afford, concave = active/pressed, flush = container). Visual restraint is the whole point.
- **A single accent ink (coral `#c75b48`) is reserved for destructive confirmation** — applied to the Apply & Reset button (lines 808, 1093 with `pill danger` class) and nothing else. This is the cleanest application of "color = importance" in the gallery. Coral pillow signals "the one thing that's not safe to touch."
- All 6 §2.2 sources rendered (lines 604–629): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each as a soft pillow.
- Apply+Reset destructive copy: "Applying changes resets the playground — your data will be wiped." (line 806) + the coral pillow as the only colored thing in the entire UI.
- Address bar `<input value="/wp-admin/edit.php?post_type=post" aria-label="Address"/>` (line 569) — §2.1 item 2 with accessible label.
- Playground title editor `<input class="field" value="Coffee Shop — local copy" aria-label="Playground title">` (line 684) — rename present.
- Unsaved-Playground notice: "This is an Unsaved Playground. Changes will be lost on page refresh." + "Save site locally" pill (lines 761–763).
- Database has SQL input "SELECT * FROM wp_posts WHERE post_status='publish' LIMIT 5" (line 934). Logs has search input (line 954). Blueprint pane has Run Blueprint + Copy link.
- README is honest: "Neumorphism is famously low-contrast — I've leaned warm and bumped text to near-black to keep WCAG AA on body text, but the soft buttons themselves rely on shape rather than contrast." Production would need "a high contrast mode toggle."
- Mobile rendered (line 1009+) with addr-mini, yp-row at smaller size — mobile scale considered.
- Tags/categories rely on shape-coded subtle icon glyphs rather than color — the monochrome constraint is fully honored.

## What's missing or weak
- **Apply & Reset has no confirm step.** The coral pillow visually marks the action but the button fires. The README implies the coral is the confirm; it's actually just a color-coded action. A "press to confirm, hold to apply" depth-based interaction would be on-metaphor.
- **Settings appear in two places** — README admits explicitly: "the popover is the same pressed control surface as the Site Manager Settings tab, so users learn it once." Same data, two homes, openly preserved.
- **Accessibility cost is significant**: "the soft buttons themselves rely on shape rather than contrast, which is a known accessibility weakness. I accept that here as a stylistic constraint." Honest, but for low-vision users this design fails by direction.
- **High-contrast mode toggle not rendered** — README acknowledges it's needed, doesn't include.
- **The monochrome palette means tags/categories can't rely on color** — shape-coded icons help but recognition is slower than color-coded tags. README admits.
- **Shadows expensive on low-end GPUs** — README admits, would compose to SVG filter images in production.
- **The PR / GitHub / URL launcher tiles** are soft pillows without inline paste inputs. §3 flows 11–12.
- **The "left drawer" segmented selector (Launcher vs Workspace)** is a clever IA fold — Site Manager and Saved Playgrounds become two modes of the same drawer. But verify it actually toggles in the artifact.
- **Originality overlaps with #015 (Windows 11 Settings) and #014 (macOS Settings)** — all three lean on subtle depth + monochrome palette. Neumorphism's distinct identity is the pillowed buttons; #015 has pill-shaped categories that are visually close. Originality not maximum.
- **First-time user discoverability** — without color cues, scanning for "where's the Save button?" relies entirely on shape memory. The "Save site locally" pill is labeled, but tertiary actions buried in the chrome will be hard to find.

## How to push harder
- Render the destructive Apply & Reset as a depth-based interaction: press the coral pillow → it sinks deep → "Hold to confirm — 2 seconds" with a depth-fill animation. The metaphor's strongest interaction language (convex/concave/flush) maps directly onto progressive commitment.
- Add the high-contrast mode toggle the README acknowledges. It's the difference between a stylistic constraint and an inclusive design.
