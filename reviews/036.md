# Review of #036 — Y2K / Aero Glass

**Direction:** "Y2K / aero glass: glossy buttons, gradients, glass-like panels with chromatic aberration accents."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 7/10 · Visual 9/10 · Originality 8/10

## What works
- Direction commitment is uncompromising: Aero glass strip, beveled chrome, chromatic-aberration title bars (`aberr` class with `data-text`), jellybean buttons, frosted-glass desktop, draggable windows, iridescent gradients. Vista/Aqua/iMac G4 nostalgia is rendered faithfully.
- All 6 §2.2 sources appear as iridescent jellybean buttons (line 1294–1299): Vanilla WP / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each with color-tinted icons.
- Apply+Reset appears in two surfaces:
  - Dock-attached settings popover (line 945) with "Applying settings resets the running playground."
  - Site Manager Settings tab (line 1034) with longer copy "Apply Settings & Reset wipes the running site and re-boots from your blueprint."
- Address bar `<input type="text" id="addrInput" value="/wp-admin/edit.php?post_type=post" spellcheck="false">` — labelled "Address Bar" per Y2K convention.
- Unsaved-Playground notice with "Save site locally" pink heart button (line 1015–1017). The chromatic aberration on "Unsaved Playground" (line 914) makes the status visually distinct without losing the metaphor.
- Database has implicit early-access framing. Logs has search "Search logs (e.g. fatal, deprecated, query)" + level pills. Blueprint pane has Run Blueprint / Copy link / Download bundle.
- Title editor input `<input value="Unsaved Playground" id="playgroundName">` — playground rename present.
- Mobile section rendered (line 1230+) with same Y2K vocabulary at small scale.
- README is honest about real costs: "Aero is beautiful but heavy: gradients, bevels, blurs and chromatic accents eat readability and accessibility." + "In production I'd composite to a few raster textures" for `backdrop-filter` performance.
- **Explicit acknowledgment that settings duplication is preserved** by design: "three settings entry points (popover, Site Manager tab, info-notice button) are intentionally kept to honor the brief's 'no feature drops' rule, but unified visually." Honesty about §4.

## What's missing or weak
- **The "intentional preservation of three settings entry points"** is an arguable misreading of the brief. §4 lists the popover/sidebar tab/info-notice trio as a *pain point*, not a feature. The brief asks designers to "merge screens, remove modes/screens" while preserving features — the three entry points aren't features, they're surface duplicates of the same five-row setting set. Most other designs collapsed them; this design keeps them as a feature.
- **Apply & Reset has no confirm step.** Two destructive surfaces (line 945 and 1034); both fire immediately.
- **Aero's blur + bevels + gradients eat readability** — README admits and bumps body type sizes. The cost is real for long sessions.
- **Performance cost**: many `backdrop-filter: blur()` + radial gradients = GPU-heavy. README admits production would composite to rasters.
- **The PR / GitHub / URL / Import .zip jellybean buttons** are tile-shaped without inline paste fields. §3 flows 11–12 punted to a follow-up step that isn't shown.
- **Modern users' "flat" muscle memory** — README admits, mitigates with "slightly louder" affordances.
- **Chromatic aberration accents (red/blue split borders) are decorative rather than functional** — README admits, defends as Y2K thesis. Fair.
- **No clear demo of the destructive Apply+Reset as a Vista-style "Are you sure?" UAC interrupt** — the metaphor invites this and the design doesn't deliver.
- **The dock-attached settings popover + Site Manager Settings tab + the unsaved-info-notice "Save site locally" button** all surface settings/save controls in 3+ places. Maximum duplication, justified by direction adherence.
- **Memory/RAM aesthetic + modern broswer performance** — the design specifically risks running poorly on machines that the brief expects to support.

## How to push harder
- Read §4 again: the three settings entry points are a *pain*, not a feature. The brief says "Reshape, don't extend" — preserving the duplication is the wrong reading. Collapse them while preserving the data they expose; the metaphor will be cleaner.
- Render a Vista-style UAC "Are you sure?" overlay for Apply & Reset — chromatic aberration on the warning text, glass-panel shake animation, two jellybean buttons (`Reset` / `Cancel`). That's the strongest Aero destructive UX possible.
