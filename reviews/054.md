# Review of #054 — Floating Action Button (Material You)

**Direction:** "Floating action button (Material You): single FAB bottom-right that expands into the major actions. Material-3 dynamic color."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 7/10 · Visual 9/10 · Originality 7/10

## What works
- Material You aesthetic delivered: M3 tonal palette generated from a single Playground purple seed, radial speed-dial FAB, bottom sheet with M3 segmented selector, Material chip/button/icon system. Visual identity coherent.
- All 6 §2.2 sources rendered in the launcher dialog (lines 623–628): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each with sub-line ("Boot from core PR number", "Paste a blueprint .json URL").
- Apply+Reset has a `destructive-card` CSS class with red border and `var(--md-error)` color — proper M3 error semantics applied.
- Address bar inputs present: `<input value="/shop/?orderby=popularity">` (line 373), `<input value="/wp-admin/post-new.php">` (line 443) — §2.1 item 2.
- Save site locally as a primary button + a Download .zip secondary in the same row (lines 482–483).
- Database has implicit early-access (verify); Logs presumably has search; Blueprint pane has Run Blueprint / Copy link to blueprint / Download bundle (lines 575–577).
- Unsaved-Playground notice: "**Unsaved Playground** — your changes will be lost on page refresh." + "Save site locally" filled button (lines 724–725).
- The "Refresh / Save / Saved Playgrounds / Site Manager / Settings / New" FAB radial petals — per README — collapses the 7 toolbar items into 6 petals + 2 docked chips. Iframe gets the entire stage.
- Mobile uses the same FAB unchanged — sensible scaling.
- README is honest: "A FAB-only chrome is delightful but discoverable only after the first tap; first-time users get a short pulsing hint." Mitigation acknowledged + provided.
- "The expanded radial menu has 6 petals — close to the M3 sweet-spot of 3-5 — so it pushes the pattern but stays legible." Self-aware about the pattern's limits.

## What's missing or weak
- **Apply & Reset has no confirm step rendered.** Despite the M3 destructive-card styling, the actual confirm dialog isn't demonstrated. M3 itself has destructive-action dialog patterns; render one.
- **Settings appear in the bottom sheet's Settings tab** AND potentially in the FAB's Settings petal opening it directly — README claims they're "one Settings tab in the bottom sheet." Verify whether the FAB petal also opens a popover or just routes to the sheet.
- **Bottom sheets on desktop steal vertical real estate from the iframe** — README admits, mitigates with resizable sheet (collapses to 56px handle).
- **The PR / GitHub / URL launcher items** are sub-lined tiles without inline paste inputs visible.
- **Discoverability cost**: FAB-only chrome means a first-time user sees only a purple bubble + iframe. The "tap me" ripple hint is described but undelivered in static artifact.
- **Originality overlaps with #032 (Concept-album minimal) and #033 (Radial pie menu)** — three designs land on "hide everything, expand from a center point". #054's distinguishing feature is the strict Material You aesthetic + 6 petals + bottom sheet (instead of overlays).
- **6 petals "pushes the M3 sweet-spot"** — README admits. Radial menus get hostile past 5 items.
- **The FAB at bottom-right + URL/name chips somewhere** — verify the URL chip placement. The artifact has multiple `<input>` URL fields but their placement relative to the FAB isn't clear from grep.
- **WP Admin / Homepage** — likely behind a FAB petal or sub-petal. Not surfaced as direct chrome.
- **Mobile: the bottom sheet covers the iframe** — README admits and defends ("acceptable since on mobile the iframe is already cramped").

## How to push harder
- Render the destructive Apply & Reset as an M3 dialog — modal, with title "Apply settings and reset playground?", supporting text listing what'll be wiped, and a "Cancel" + "Reset" pair where the destructive option uses `var(--md-error)`. M3 has the pattern; use it.
- Demonstrate the FAB pulsing "tap me" ripple as a state — the discoverability mitigation is the design's main first-touch claim.
