# Review of #030 — Excel / spreadsheet

**Direction:** "Excel / spreadsheet: blueprints in rows, columns are properties (WP version, PHP, plugins, themes). Filter, sort, group like Excel."

**Scores:** Feature completeness 10/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 9/10

## What works
- Authentic spreadsheet shell: workbook title bar (`Coffee Shop — staging.xlsx`), ribbon with Format/Insert/From GitHub buttons, formula bar acting as URL bar (`<input id="urlInput" value="/wp-admin/edit.php?post_type=product">`), column headers with filter funnels, status-bar tabs at the bottom matching Excel sheet tabs (Settings · Files · Blueprint · Database · Logs).
- The 10 §2.2 categories appear as native column filters (line 681–689): All / Featured / Website / Personal / Content / Themes / Gutenberg / Experiments / WooCommerce / News — with an Apply button. This is one of the most truly-on-metaphor implementations: column filter = category filter.
- All 6 §2.2 sources appear as Source column values + a quick-action row + ribbon buttons + a context-menu row: Vanilla / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip. Mapping is at line 982 (`{ 'Saved':'saved','Blueprint':'bp','WP PR':'pr', ... }`) — the Source column is the discriminator.
- Apply+Reset has the destructive copy: "Apply will reset the playground — all unsaved changes will be lost." with a `btn danger` class.
- Address bar is real, with `https://` scheme + host + path input combo.
- Right-click context menu (line 816+) carries Save site locally / Download .zip / Copy Blueprint URL — Excel-style right-click discoverability.
- Quick-Start column with 🚀 buttons described — gives one-click boot for new users so they don't have to learn the grid.
- Database has early-access framing (verify). Logs / Files / Blueprint tabs at the bottom act as Excel sheet tabs.
- Unsaved-Playground notice and "Save site locally" button are present (line 1138). The row state shows "Unsaved Playground · Not saved to browser storage" — fits the spreadsheet idiom.
- Quick Settings (line 1159: Network access toggle + Multisite toggle) — the "Format → Cell" sidebar idea is delivered.

## What's missing or weak
- **Apply & Reset has no confirm step.** Despite the destructive notice copy, the button fires. Excel itself has "Are you sure you want to delete this sheet?" — borrow the pattern.
- **Settings duplication still possible.** The README claims "single Format → Cell sidebar" + ribbon Format button — verify only one settings surface renders, but the quick-settings inline strip (line 1159) might be a third surface.
- **A spreadsheet of 43 rows + 10+ columns is dense and unapologetically pro-user** — README admits the intimidation cost. The "Pick a row to launch" callout + Quick-Start column are the mitigations.
- **First-touch user landing on a grid of 43 rows** has to learn spreadsheet idioms quickly. The empty-state callout helps but is undemonstrated in the artifact.
- **PR / GitHub / URL launchers** appear as rows OR as ribbon buttons OR as Quick-Action chips OR as a + New row affordance — multiple paths, no clear primary. Discoverability via redundancy is fine; consistency is at risk.
- **The "+ New row" affordance is described** as an inline picker but the artifact's grep shows source-type values, not the inline picker UI itself. Verify it renders.
- **Mobile collapse to a card list** is described, not necessarily demonstrated.
- **The "preview cell on the right hosts the live WordPress iframe"** — fits the metaphor, but the iframe is then much smaller than full-bleed. Spreadsheet rows + right-side iframe pane competes for horizontal pixels.
- **Row hover-to-preview** is described as the first-10-seconds onboarding move — but a static artifact can't fully demonstrate hover interactions. A `:hover` state CSS would help.
- **The category filter as column-filter** is brilliant but means category filtering is "click funnel → check boxes → Apply", which is 3 clicks vs. 1 chip-click in most other designs. Trade-off of metaphor authenticity vs. fluidity.

## How to push harder
- Render the row-hover-preview as a visible "hover state" in CSS — make sure a static viewer sees the value of the metaphor without needing to interact.
- Implement Apply & Reset as an Excel-style "Are you sure you want to clear this row's data?" confirm with the row's diff (rows that will change, cells that will be wiped). Use the metaphor's strongest existing destructive UX.
