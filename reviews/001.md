# Review of #001 — Command-palette-first

**Direction:** "Command-palette-first: every action triggered by Cmd/Ctrl+K. The toolbar shrinks to almost nothing. Settings/blueprints/files all accessed through a fuzzy-search palette."

**Scores:** Feature completeness 9/10 · Direction 9/10 · IA 6/10 · Visual 8/10 · Originality 8/10

## What works
- Commits hard to the direction. The toolbar collapses to a single pill (refresh, URL `/wp-admin/themes.php`, save dot, save button, ⌘K hint), and every §2 verb is reachable from the palette: blueprints, settings, WP Admin, Homepage, Rename, Export to GitHub PR, Download .zip, Import .zip, Apply settings & Reset.
- Five distinct desktop states (palette open with blueprints, panes stacked with Files+Blueprint, Settings palette + Logs pane, Booting, full 43-blueprint gallery grid) — exceeds §5's "three states minimum" requirement.
- Palette filter chips encode the exact §2.2 inventory: All / Blueprints / Settings / Files / Your Playgrounds / Logs / Actions, and Apply&Reset carries a "destructive" tag plus a "Requires confirm step" subtitle, with the keyboard shortcut ⌘⇧⌫.
- The "Quick row" of five visible quick-pills (Blueprints, Files, Settings, Logs, More) and the first-run tip ("press / or ⌘K to do anything") directly addresses the discoverability trade-off the README acknowledges — better than just declaring it.
- WordPress PR / Gutenberg PR / From GitHub / Blueprint URL entries each show a "paste PR #" or "owner/repo" or "JSON" tag — at least gesturing at the input flow §3 (11–12) requires.
- Inline notice inside Settings palette literally quotes the §2.3 unsaved copy: "This is an Unsaved Playground. Applying settings resets the site … Save site locally first →".
- 18 Lucide icons used, real typography (Inter + JetBrains Mono), thoughtful color tokens, dark theme committed to a Raycast aesthetic.

## What's missing or weak
- **The pane-tabs row at the right edge (Settings / Files / Blueprint / Database / Logs) is a tabbed sidebar that contradicts the thesis.** The README claims "no menus, no tabs, no popovers" — but State 02 and State 03 render exactly tabs at the top of the right pane. The design is *both* a palette UI *and* a Site Manager tabbed sidebar. If the palette is the only verb, the right pane should not expose tabs — invoking each pane should go through the palette.
- **Address bar is read-only.** The URL slot in the pill is a `<span>` (`/wp-admin/themes.php`, `/locations`, etc.), never an `<input>`. §2.1 item 2 mandates "type a path … and Submit". The palette's URL command isn't actually demonstrated either.
- **The unsaved-state warning sits inside a settings palette section that opens only if a user types `⌘,`.** A first-time visitor refreshing the tab and losing work would not see this notice. The header pill's "dot-unsaved" indicator is a single pixel.
- **"Apply Settings & Reset" requires a confirm step "you'll be asked to confirm"** — but the confirm step is described, not rendered. Show the confirm modal/palette stage.
- **No visible blueprint→preview interaction.** The "Coff" search result lists `↵ Boot` and `⌥↵ Preview`, but the preview pane never appears in any of the five states.
- **Database tab carries the "early access" tag in the Settings palette command but the actual DB pane is never opened.** The fifth Site Manager tab from §2.3 is missing as a rendered surface — we see Files+Blueprint and Logs panes, never DB content (only a tab label).
- **Mobile compresses to a single FAB-and-pane** but the bottom of the mobile section only shows 12 of 43 blueprints and no mobile path to WordPress PR or Gutenberg PR input.
- **Saved playground rename action exists** ("Rename Playground… Currently: Coffee Shop") but there's no inline editable title in any rendered state — the rename is a verb only.
- **"Network access" toggle and "Multisite" toggle live in the Settings palette as command rows but the on/off state would be hard to scan in a list.** A toggle widget can't fit naturally inside a fuzzy-list paradigm — this design fights itself here.

## How to push harder
- Resolve the palette-vs-tabs contradiction. Either commit to "every pane is summoned via ⌘K and exits via Esc, no persistent tab strip", OR rebrand the right-pane tab strip as "open panes" with a clear chord (e.g., ⌘1–5 to swap) rather than the same look as a Site Manager sidebar.
- Replace the static URL `<span>` in the pill with a real input affordance — even a placeholder `Type a path…` would satisfy §2.1's "Submit to navigate". And show the Apply+Reset confirm step as its own palette state.
