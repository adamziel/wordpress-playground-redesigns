# Review of #077 — Block-editor styled (Gutenberg-inside-Gutenberg)

**Direction:** "Block-editor styled (Gutenberg-inside-Gutenberg): the Playground's own chrome uses Gutenberg blocks visually — block toolbar, inserter, sidebar inspector."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 9/10 · Visual 10/10 · Originality 9/10

## What works
- Direction faithfully delivered: floating block toolbar above the selected Site Block, "Inserter" slide-out on the left (with Blocks / Patterns / Saved tabs mirroring Gutenberg), tabbed Inspector panel on the right (Settings / Files / Blueprint / Database / Logs as block sidebar tabs), top bar mirroring the editor's (app icon, undo/redo replaced by Refresh, document title, view options, save status, Publish-equivalent Save).
- All 6 §2.2 sources rendered in the Inserter Patterns tab (lines 701–736): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each as a Pattern card.
- Apply+Reset has Gutenberg-aware safety annotations:
  - Settings panel: "Apply & Reset" with `alert('A confirmation dialog would warn this resets the playground.')` (line 621) — the design explicitly stubs out the confirm.
  - Full Settings tab: "Applying settings reboots the playground and erases all local changes." + `btn btn-danger` "Apply & Reset Playground" with `alert('Confirm dialog would appear.')` (lines 950–952) — same pattern. The design **acknowledges** the confirm is required but doesn't render it.
- Address bar `<input id="addr-bar" value="/hello-from-playground" onkeydown="if(event.key==='Enter')navTo(this.value)">` (line 815) — real input with submit-on-Enter handler.
- Save-state badge in top bar with idle/unsaved/saving/saved states (line 586: "Unsaved — local only"). Replaces the duplicate "Save site locally" CTA per README.
- WP Admin / Homepage as Block Navigation pills (lines 568, 819).
- "Download .zip / Export to GitHub PR" in More-options kebab on Block Toolbar (line 645, 834).
- Database / Logs / Files / Blueprint Inspector tabs match §2.3 verbatim.
- Inserter search input "Search blueprints, themes, plugins…" (line 677).
- Playground rename: `<input class="control" value="Unsaved Playground">` + "Inferred URL slug" `<input class="control" value="unsaved" readonly>` (lines 943–944). Two-field naming with auto-slug.
- The README is honest: "This metaphor will delight Gutenberg-fluent users and disorient anyone who has never seen the block editor — it leans hard on a single mental model."
- "Because a real block editor has only one selected block at a time, multi-playground workflows (open two side-by-side) are not expressible — but the current Playground doesn't support that anyway, so the metaphor doesn't actually cost a feature." Brief-aware constraint analysis.

## What's missing or weak
- **Apply & Reset's confirm dialog is stubbed with `alert()`** — the README acknowledges it should exist, the artifact doesn't render it. Promote the alert to an actual confirm modal.
- **Two Settings surfaces** exist (lines 617–621 inspector AND 935–952 full tab) — README claims "popover scrolls/opens the Inspector to that panel, so there is one source of truth." Verify whether they're truly synced.
- **The PR / GitHub / URL Pattern cards in the Inserter** are tile faces; no inline paste inputs.
- **Inserter "Blocks" tab for blueprints + "Patterns" tab for sources + "Saved" tab for Your Playgrounds** is a clever 3-tab structure that fits Gutenberg's actual UX. Verify all three tabs render.
- **Originality is high** — overlaps with #078 (wp-admin styled) in the WordPress-aesthetic-self-reference lane. #077 commits to Gutenberg's chrome; #078 commits to wp-admin's. Both are legitimate.
- **The "block toolbar floats above the iframe with refresh/address/transform/more"** — verify the floating toolbar is rendered in the artifact (vs. a fixed top bar).
- **First-time users who haven't used Gutenberg may not parse the floating toolbar / inserter / inspector mental model** — README admits.
- **Theming chrome to look like Gutenberg means visual updates to real Gutenberg may require following along** — fair maintenance cost noted by README.

## How to push harder
- Render the destructive confirm dialog the README promises — make it a Gutenberg-style modal (with the editor's "Are you sure?" / "Discard changes?" pattern) instead of an alert(). The metaphor's existing destructive UX is right there.
- Demonstrate the "Saved" Inserter tab in a separate state — show how Your Playgrounds renders as draggable patterns the user can re-insert.
