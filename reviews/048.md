# Review of #048 — Apple Music Now Playing

**Direction:** "Apple Music Now Playing: iframe is the album art, blueprint name is the song title, controls beneath. 'Up next' is recently used blueprints."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 7/10 · Visual 9/10 · Originality 9/10

## What works
- Direction commitment is full: Apple Music vocabulary maps cleanly — Library / Browse / Radio / Up Next / Lyrics drawer / Now Playing bar / album-art treatment. The metaphor is rich and the README articulates each mapping.
- All 6 §2.2 sources as Stations in the nav (lines 980–985) AND as detail-row cards (lines 1227–1247): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — each with descriptive sub-line.
- Apply+Reset has the strongest two-surface coverage:
  - Quick popover: `<button class="btn primary">Apply & Reset Playground</button>` (line 1041)
  - Settings detail-drawer: `<div class="txt"><b>Apply Settings & Reset Playground</b><span>Destructive · wipes the running site. Saved playgrounds are unaffected.</span></div><button class="btn">Apply & Reset</button>` (line 1452–1453) — clear copy about *what is unaffected*, which is a refreshing precision few other designs deliver.
- Address bar `<input value="/wp-admin/edit.php?post_type=product" aria-label="Inner site URL">` (line 1003) — §2.1 item 2 with aria-label.
- Heart icon for Save (per README) — "add to Library" metaphor. Maps to §4 risk via Apple Music's "add to library" muscle memory.
- Database has the early-access notice with GitHub-tracker link (line 1566). Logs has search "Search logs (e.g. WC_, fatal, deprecated)…" (line 1607). Blueprint has Copy link / Download bundle / Run Blueprint (lines 1515–1517).
- Unsaved-Playground notice: "**This is an Unsaved Playground.**" (line 1421) + "Save site locally" primary button (line 1424).
- Procedurally generated gradient covers for the 43-blueprint library — README explicitly: "every blueprint gets a procedurally generated gradient 'cover' derived from a hash of its title, so the 43-blueprint browse grid actually looks like a music library." Solves the §4 "no preview thumbnails" pain without faking screenshots.
- Up Next = recently used blueprints, Library = Your Playgrounds, Browse = full 43 with genres = categories. Genuine 1-to-1 metaphor mapping.
- "Album art" iframe with "Full Screen" toggle mitigates the first-touch-iframe-shrink risk.
- README is honest: "Aestheticising the iframe as album art means at small viewports the iframe shrinks to a stylized thumbnail and the WordPress site is no longer the literal hero — that's a real loss for first-touch anonymous users."

## What's missing or weak
- **Apply & Reset has no confirm step.** Two surfaces with copy; both fire immediately. Apple Music has "Remove from Library" confirms; the metaphor invites it.
- **Settings appear in three places** — quick popover (line 1041), Settings detail-drawer (line 1440), AND a "More menu" with Apply (line 1473). The README claims they "collapse into a single ... More menu" but the artifact has three settings surfaces.
- **The iframe-shrunk-to-album-art** is the metaphor's biggest cost. README admits and mitigates with Full Screen toggle. First-touch user sees a music-app-y dashboard, not WordPress.
- **The PR / GitHub / URL launcher tiles** are tile faces with descriptive sub-lines; no inline paste input visible.
- **Database tab is honestly just a tab inside the drawer, not a clever musical pun** — README admits the metaphor has limits.
- **The heart-as-save** is delightful but opaque — README mitigates with "tooltip on first hover", undelivered in the artifact.
- **The "Lyrics-style detail drawer with Settings/Files/Blueprint/DB/Logs pills"** — verify the pills actually swap content as separate panes; the artifact's grep suggests they do.
- **Originality has overlap with #021 (Spotify-like)** — both lean on music-player metaphors. #021 uses Spotify's heart save + bottom transport bar; #048 uses Apple Music's library/up-next/album-art mapping. The two designs diverge in metaphor depth (Apple Music's "Up Next" maps directly to "recently used blueprints", which is a tighter fit) but stylistically converge on dark + accent.
- **Mobile fallback** — the player-bar metaphor scales to mobile but the album-art iframe at phone width is even smaller. Verify mobile state demonstrated.

## How to push harder
- Render the destructive Apply & Reset as an Apple Music "Are you sure you want to remove this from your library?" prompt with the precise data-loss diff. The metaphor's existing destructive UX is right there.
- Implement the tooltip on first hover for the heart icon — without it, "save = heart" is a guessing game for non-music-app users. The README promised it; render it.
