# Review of #023 — Kanban

**Direction:** "Kanban: blueprints organized in cards across columns by category. Drag a card to 'Launch' column to boot it."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 8/10 · Originality 9/10

## What works
- Faithful kanban execution that does real IA work: a "Sources" column with all 6 §2.2 entries (Vanilla / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip), category columns acting as the §2.2 filter taxonomy (Featured / Website / Personal / Content / Themes / Gutenberg / Experiments / WooCommerce / News), a "Your Playgrounds" column, and a "Launch" column that's the running iframe. The drag-to-launch metaphor is the discovery → routing → execution arc unified.
- **Inline input affordances for sources.** Lines 701 (`<input placeholder="user/repo[/path]" value="WordPress/twentytwentyfive">`) and 714 (`<input placeholder="https://…/blueprint.json">`) — this is one of the few designs where PR / GitHub / Blueprint URL launchers expose actual paste fields directly on the card face. §3 flows 11–12 substantially better than peers.
- Category-as-column means category filter chips dissolve into column visibility toggles (line 741–749), which is a clever IA simplification — exactly what the README claimed.
- Apply+Reset is staged as a "Move card back to backlog" confirmation per the README; the artifact has `<button class="icon-btn danger" onclick="toastIt('Settings applied — playground reset')">Apply & Reset</button>` plus a Settings-sync inline notice ("Settings here mirror the quick popover and the toolbar pill. Changes apply on demand; the bottom row stages the destructive reset.") — explicit acknowledgement that the design knows the destructive footgun risk.
- Database tab has the early-access banner verbatim. Logs has search input. Blueprint pane has Run Blueprint / Copy link / Download bundle. Files tree present.
- Address bar `<input value="/wp-admin/edit.php?post_type=post" aria-label="Path">` — §2.1 item 2 satisfied with accessible label.
- Mobile collapses columns to a vertical accordion + Launch button — admitted loss of drag choreography but features preserved.
- Toast feedback `toastIt('Resuming Gutenberg PR #74321')` and "Jumped to /wp-admin" — explicit user feedback on every action.

## What's missing or weak
- **Settings appear in multiple places** — the README admits "Settings here mirror the quick popover and the toolbar pill" so it's openly synced across three surfaces. Same §4 pain rebranded as "sync" rather than "duplication".
- **Apply & Reset has no actual confirm modal rendered.** The README claims the destructive action becomes "Move card back to backlog" with a preview, but the artifact only has the danger button that fires `toastIt`. The card-move metaphor would be a great place to render a "drop card here to reset" inline confirm.
- **Drag-and-drop discoverability** — README admits this and adds "or click any card" persistent hint + play-icon on every card. Good mitigation.
- **The iframe lives in one lane = narrower than full-bleed** — README admits the metaphor tax. A user inspecting WordPress will have less horizontal room than today.
- **The horizontal-scroll board on desktop** is great when there are 10+ columns; on tablet (~1024px) the board becomes harder to navigate. The mobile fallback (vertical accordion) is fine but the tablet middle-ground is unaddressed.
- **No `wp-admin` / Homepage explicit chrome buttons** in the iframe lane title — there's a "Jumped to /wp-admin" toast on a settings icon click (line 929), but the icon is the wrong glyph (`settings` instead of `dashboard`). UX bug.
- **Featured tagging via "star ribbon"** is described but I don't see the ribbon SVG/decoration in the artifact's visible portion — verify it's actually rendered on featured cards.
- **The Apply & Reset destructive copy is weak.** It's just the `danger` class + alert-triangle icon and a toast. Compare to #007 (cyberpunk) or #011 (newspaper) which deliver real warning panels. Kanban's metaphor invites a "drop this card on the trash column" — render that.
- **The "Your Playgrounds" column blurs with saved blueprints** — both are cards; the distinction comes from a `yp-card` class. Visually subtle. A user scanning the board for "what's mine" may not pick out the saved playgrounds column.

## How to push harder
- Render the destructive "move card to backlog" as an actual drag interaction: when the user clicks Apply & Reset, the current Launch card animates out and a confirm modal appears as a "you are about to clear this lane" message. The kanban metaphor *is* the destructive UX; deliver it visually.
- Add the WP Admin / Homepage chrome buttons as discrete icon-buttons in the Launch card title bar — not buried behind a settings glyph. Use a `layout-dashboard` icon and a `home` icon to match §2.3 explicitly.
