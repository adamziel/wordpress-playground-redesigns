# Review of #065 — Twitter-like timeline

**Direction:** "Twitter-like timeline: vertical feed where each item is a saved-playground card with avatar (favicon), title, WP/PHP badges, last-used time, 'Boot' button."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 9/10

## What works
- Direction delivered as a real X/Twitter timeline: three-column shell (left rail / feed / right "What's happening" rail), posts with avatar + slug + WP/PHP badges + thumbnail + last-used + Boot button, thread-detail view for individual playgrounds, floating Compose button.
- All 6 §2.2 sources rendered (lines 517–520 + composer modal): WordPress PRs / Gutenberg PRs / From GitHub / Import .zip — plus presumably Vanilla and Blueprint URL in the composer.
- Apply+Reset: "**Apply Settings & Reset Playground** Destructive — wipes the running site and replays the blueprint." (line 1117) + a `pillbtn danger` button (line 1118).
- Address bar `<input name="q" value="/shop/?orderby=price" aria-label="Address bar">` (line 832) — §2.1 item 2 with aria-label.
- Database has the early-access notice with GitHub-tracker reference (line 1045). Logs has filter input "Search logs (regex ok)… e.g. WC_Tax_Calc" (line 1078). Blueprint has Copy link / Download bundle / Run Blueprint (lines 1004–1006).
- Unsaved-Playground notice prominently in feed AND thread:
  - Pinned at top of feed (line 571): "This is an Unsaved Playground. Your changes will be lost on page refresh. Save site locally to keep it."
  - In thread Settings (line 1106): same copy + Save site locally link.
- WP Admin button (line 854: `onclick="toast('Opening /wp-admin','info')"`).
- "Logs → replies" — each PHP log line is a threaded reply under the post; severity = colored avatar dot per README.
- README is honest: "The thread-detail tabs (Site / Files / Blueprint / DB / Logs) reintroduce some of the tabbed Site Manager I claimed to dissolve, but they live inside the thread context, not as a global sidebar." Honest about the partial retreat.
- "Saved Playgrounds drawer → the home timeline" — README's central thesis is delivered.
- Mobile collapses to single column with bottom tab bar (Home / Explore / Compose / Notifications / You) — recognizable X-mobile pattern.

## What's missing or weak
- **Apply & Reset confirm step is wrapped in `onclick="toast('Reset cancelled (demo)','info')"`** (line 1118) — the demo toast cancels, but the actual production behavior would fire. No typed-confirmation gate.
- **The PR / GitHub / URL launcher options are nav items + composer modal tabs** — verify the composer's "six tabs" with paste inputs are rendered. The grep doesn't show inline paste-PR-number inputs.
- **The thread-detail tabs (Site / Files / Blueprint / DB / Logs)** — README admits "reintroduce some of the tabbed Site Manager I claimed to dissolve." Partial metaphor retreat.
- **Power users may find scrolling through a feed slower than a tree** — README mitigates with "pin a thread" (pinned tweet equivalent).
- **The social metaphor risks looking unprofessional** — README mitigates with badges + monospace slugs + serif-free typography.
- **The right rail "What's happening" with trending blueprints and PR feeds** — verify whether these are static featured content or implies a recommendation algorithm (which would violate §5).
- **The "@playground" account reposting featured blueprints** — clever framing, but the metaphor of an automated account reposting could imply a system that doesn't exist.
- **The Settings sheet from the left rail + per-thread Settings tab** — verify they're the same component.
- **Compose button with six modes** — verify all six PR/GitHub/URL/zip/Vanilla/Blueprint paste fields are rendered.

## How to push harder
- Implement the typed-confirm pattern (per playground name) for Apply & Reset — the demo `toast('Reset cancelled')` shows the design knows the danger; render the real gate.
- Render the Compose modal's "six tabs" state showing actual paste fields for PR # / GitHub repo URL / Blueprint JSON URL / .zip upload. The X-Compose-with-media metaphor is the design's strongest unified-launcher claim; show it open.
