# Review of #084 — Shapeshifter Sidebar

**Direction:** "Sidebar that swaps content (single sidebar, multiple personalities — Files mode, Blueprint mode, Settings mode, Logs mode, DB mode — selected by a column of icons)."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 8/10 · Visual 9/10 · Originality 7/10

## What works
- Direction faithfully delivered: 60px activity bar with six icons (Site / Files / Blueprint / Database / Logs / Settings), each transforming the 340px sidebar into a different application. iframe stays as constant stage.
- All 6 §2.2 sources rendered in the Launcher mode (lines 1006–1011): Vanilla WordPress (with `is-primary` class) / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip.
- Apply+Reset: `<button class="apply">Apply & Reset Playground</button>` (line 858) + warn callout "Apply wipes current site." (line 860) + secondary surface at line 993.
- Address bar `<input type="text" value="2025/05/welcome-to-playground/">` (line 647) + secondary at line 1101.
- Database has implicit early-access; Logs has filter input "Filter logs…" (line 906) + download button; Blueprint has Copy link (line 815); Files tree present.
- Unsaved-Playground notice: "**This is an Unsaved Playground.**" + "Save site locally →" (lines 829–831).
- Download .zip + Import .zip in Site mode (lines 708, 710).
- Playground rename `<input class="name" value="My Coffee Shop demo">` (line 687).
- Booting state with step list (line 945: "Apply site options").
- README is honest: "The single-sidebar model means you can't *simultaneously* watch logs while editing a file — you have to switch modes. The activity bar mitigates this with a red dot indicator on Logs when new errors arrive."
- "Removed the redundant Blueprint-tab file tree column — Files mode already shows the tree; Blueprint mode shows the editor + a 'Files' pivot button to jump modes." Addresses §4 file-tree-duplication pain.

## What's missing or weak
- **Apply & Reset has no confirm step.** Both surfaces (lines 858, 993) fire.
- **Settings appear in two surfaces** — README admits "two: a Site-Manager-style Settings *mode* in the sidebar (with the inline 'save locally' notice on Unsaved Playgrounds) and a quick popover still anchored to the top bar for one-click WP/PHP changes." Honest about retaining one duplicate.
- **Mode-switching means no simultaneous file-editing + log-tailing** — README admits and provides red-dot indicator mitigation.
- **The PR / GitHub / URL launcher tiles** are `<a>` links without inline paste inputs.
- **Permanently-visible 340px sidebar costs horizontal iframe real estate** — README admits and proposes collapsible (not implemented).
- **Discoverability of modes depends on icon vocabulary legibility** — README admits and uses Lucide's domain-familiar metaphors + hover labels.
- **Originality is medium** — overlaps with #002 (IDE-like activity bar) and #009 (Storybook left rail). #084's distinguishing move is the explicit "single sidebar with six modes" framing vs. always-visible tabs.
- **No demo of the collapsible-sidebar mitigation** the README mentions.

## How to push harder
- Wire the activity-icon re-click to actually collapse the sidebar to 60px — show the collapsed state as a fourth or fifth demo. The horizontal-real-estate-cost mitigation is described, undelivered.
- Render the destructive Apply&Reset confirm — even a simple "Are you sure?" overlay would close §4. The mode-switching metaphor invites a "you're leaving Settings mode" framing for the destructive moment.
