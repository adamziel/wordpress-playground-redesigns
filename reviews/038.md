# Review of #038 — Glassmorphism + heavy blur

**Direction:** "Glassmorphism + heavy blur: the iframe is the background; controls float in frosted-glass cards above."

**Scores:** Feature completeness 9/10 · Direction 9/10 · IA 7/10 · Visual 9/10 · Originality 6/10

## What works
- Direction faithfully delivered: heavy backdrop-blur (40–60px per README), 8% white tint, 0.5px white inner stroke, soft drop-shadow, no opaque chrome anywhere. iOS Control Center / Big Sur menu bar aesthetic captured.
- All 6 §2.2 sources rendered as cards inside the bottom-anchored Launcher sheet (lines 1175–1210): Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip, with sub-copy.
- Apply+Reset is in two places: Site Manager Settings tab (line 1143 with "Apply will wipe the current site.") and a popover (line 1243).
- Database has the early-access notice (line 1649) with the GitHub-tracker link. Logs has filter input. Blueprint pane has Copy link / Download bundle / Run Blueprint. Files has tree.
- "Unsaved Playground" notice with "Save site locally" button (line 1108–1110). The pulsing/Unsaved indicator is in the topbar pill.
- Mobile section rendered (line 1776+) with mobile-start-tile rows for sources, addressing §5 mobile constraint.
- Database has an SQL input `value="SELECT * FROM wp_posts WHERE post_status = 'publish' LIMIT 6"` — power-user feature beyond just browsing tables.
- README is honest: "Glassmorphism punishes low-contrast WordPress sites — when the iframe is mostly white, blurred glass on white reads weakly." Mitigations described (+6% tint, inner stroke, drop-shadow). "Accessibility-strict reviewers will still want a 'reduce transparency' toggle (not implemented)" — acknowledges the gap explicitly.

## What's missing or weak
- **Apply & Reset has no confirm step.** Both surfaces fire immediately.
- **Settings appear in two surfaces** — the README claims "the toolbar gear opens the same frosted popover the Site Manager Settings tab embeds — one component, two homes." The single-component-two-homes claim is honest, but it's still two visible surfaces with the same data.
- **Reduce-transparency toggle is acknowledged as missing** — explicitly punted. For accessibility, this is a meaningful unimplemented affordance the design knows it needs.
- **Glassmorphism punishes white WordPress sites** — README admits. The default WP marketing landing page is mostly white, so the default first-touch viewer sees the design at its weakest contrast moment.
- **`backdrop-filter` GPU cost** — README admits and "keep the blurred layers small". The current artifact has many glass panels stacked; rendering on a low-end laptop is uncertain.
- **The PR / GitHub / URL launcher cards** are tile faces without inline paste inputs. §3 flows 11–12.
- **Originality is weak** — glassmorphism overlaps significantly with #008 (Apple Vision / spatial). Both designs lean on backdrop-blur + glass panels; #008 adds 3D z-depth, #038 keeps it 2D. The two designs occupy similar visual territory; the originality budget is limited.
- **Heavy blur hides micro-typography hierarchy** — README admits, leans on weight contrast (600 vs 450) + uppercase eyebrow labels.
- **The "playground.wordpress.net/scope:cafe-9k2/wp-admin/edit.php?post_type=page" address text** is rendered as a `<span>`, not an editable input. Verify the actual address input exists elsewhere — the toolbar has a glass capsule but the chunkable URL text isn't clearly an `<input>` in the grep. **Potential §2.1 item 2 miss** — verify whether the address can actually be typed.
- **The "remove opaque chrome entirely" claim** is mostly delivered, but the bottom Launcher sheet at line 1108+ has settings rows that are functional inputs — they look glassy but the design's coherence depends on tinted backgrounds doing the work that normal forms would.

## How to push harder
- Make the address bar a real `<input>` — the current span at line 1062 / 1613 may be read-only. §2.1 item 2 demands typing.
- Implement the "reduce transparency" toggle. The README admits it should exist; rendering it as a chrome control elevates the design from "I know about this risk" to "I solved it".
