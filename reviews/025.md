# Review of #025 — Card stack (Tinder-style)

**Direction:** "Card stack (Tinder-style): swipe through blueprints; swipe right to launch, left to dismiss. Mobile-native vibe."

**Scores:** Feature completeness 8/10 · Direction 9/10 · IA 6/10 · Visual 8/10 · Originality 9/10

## What works
- Genuine commitment to swipe IA: a single tilted card center-stage, fan-of-cards behind, bottom action FABs (skip/launch/save) per dating-app convention. Tab bar at the bottom (Discover / Matches / Live / Tools / Me) holds the rest of §2 IA.
- Maps every §2 feature to a dating-app verb without inventing new ones: Matches = Your Playgrounds, Likes/Super Save = save site, Profile = settings, Logs as chat under each match.
- All 6 §2.2 sources appear as bottom-sheet rows ("Import .zip", "Blueprint URL", "From GitHub repo", "WordPress PR", "Gutenberg PR") — each with a paste/upload verb in the value slot (line 958–962). This is one of the few designs that explicitly shows "Paste #" as the affordance for PR launchers.
- The README explicitly addresses the "muted WordPress-blue gradient rather than hot pink to keep it credible" — visual restraint despite the cheeky vocabulary.
- Apply+Reset has its own row in the Tools sheet with `<h4>Apply settings & reset playground</h4>` + button (line 968–970).
- Live-session screen has a real `<input value="/wp-admin">` URL bar (line 607) and a persistent Unsaved-match banner with Save site locally button — addresses §4 with a louder treatment than baseline.
- Database tab carries `early access` tag inline. Logs has "Search 312 log entries…" input. File tree present.
- Tools tab acts as Site Manager replacement with rows for WP Admin / File browser / Database / Download .zip / Copy link / etc. Each row is a "card" the user "matches with" — coherent metaphor.
- Mobile-native unchanged on desktop per the README — same deck shape on both, with the iframe expanding to full chrome "the instant you match".

## What's missing or weak
- **Settings dispersed across multiple bottom sheets.** The Profile preferences sheet (line 944+) has Network / Multisite toggles AND the Apply & Reset section AND the import/url/github/pr/gut-pr rows — that's a "Site Manager Settings tab + popover + launcher" combined into one sheet, which is the design's strength, but the README also describes settings on the Me tab, and other settings surfaces on per-match Site Manager drawers. Multiple bottom sheets surfacing settings.
- **Apply & Reset has no confirm step.** The "Apply & reset" button fires.
- **Power-user efficiency cost** — README admits "lousy for precision: power users who already know exactly which blueprint they want would rather click than swipe through 43 cards." The search field above the deck and "jump to" grid behind the small grid icon mitigate, but the design honestly bears this cost.
- **The dating-app vocabulary** is delightful AND credibility-risking — README acknowledges and counters with muted blues + button equivalents for every gesture.
- **Per-card metadata is limited** by the single-card frame — README admits and mitigates with swipe-up "peek". The peek state isn't necessarily rendered in the static artifact.
- **"Logs become a chat-stream feed under each match"** — described, but I'd need to verify whether the chat-style log UI is actually rendered in the Matches view.
- **The Site Manager dissolving into per-match drawers** is a strong remix but it means each match needs its own Files / Blueprint / DB / Logs surface duplicated — the §4 "file browser + blueprint tab partially duplicate each other" problem just got worse, multiplied by saved sites.
- **The "Boost: WordPress PR / Gutenberg PR" FAB** (line 534) — using dating-app monetization vocabulary (Boost) for PR launches is the kind of metaphor stretch the README warned about. A power user reading "Boost" won't know it means "boot from PR".
- **Refresh + address bar + name + save** live on a thin floating bar over the iframe — solid, but the Save heart is the §4 mitigation and it's only visible during a live session, not in Discover/Matches/Tools/Me tabs.
- **Swipe-up gesture to open Site Manager** on the live card — undiscoverable to first-time desktop users.

## How to push harder
- Render the swipe-up "peek" state as a real fourth view. The peek is the only place blueprint JSON + WP/PHP + Apply+Reset live, per the README — that's a load-bearing surface and it deserves a static demonstration.
- Replace "Boost" with explicit "Paste PR #" copy on the FAB. The dating-app metaphor doesn't need monetization vocabulary; the design's credibility budget is better spent elsewhere.
