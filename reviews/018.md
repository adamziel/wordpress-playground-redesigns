# Review of #018 — Discord layout

**Direction:** "Discord layout: server list on the far left (each saved playground = a server icon), channel list (Files/Blueprint/Logs as channels), main pane is the iframe or chat-like log stream."

**Scores:** Feature completeness 9/10 · Direction 10/10 · IA 7/10 · Visual 9/10 · Originality 10/10

## What works
- Most original direction execution so far: server rail with circular pill icons (Unsaved Playground server has dashed border + amber dot, exactly Discord's "new server" state), channel list with `# settings / # files / # blueprint / # database / # logs` channels + a "RUN" voice-channel-style block for Refresh / WP Admin / Homepage / Address bar. This is genuinely Discord-shaped while doing real §2 work.
- **Logs-as-chat is the killer remix.** Each PHP message is a chat row with avatar (PHP, MySQL, WP-Cron), timestamp, severity reaction emoji, threaded stack traces. The chat search input "Search logs… (e.g. wp_die, 500, plugin slug)" maps Discord's channel search onto §4's logs pain. The "filter logs OR paste a wp-cli command" input is a clever reframe that doesn't violate §5 (no new feature, just an alternative input mode for the same logs).
- All 6 §2.2 sources rendered in the "Add a Server" launcher modal: Vanilla WordPress / WordPress PR / Gutenberg PR / From GitHub / Blueprint URL / Import .zip — with descriptions like "43 templates · Art Gallery, Coffee Shop, Gutenberg PRs, …" for blueprints.
- Apply+Reset gets the strongest contextual wrap: a Discord-style "danger" notice — "Apply & Reset is destructive. Re-applying these settings wipes the running site and re-boots from the chosen versions. Your unsaved files will be lost." + `btn btn-danger` with the zap icon. Per README also wrapped in a "Discord-style danger confirm".
- Site Manager Settings tab has the Unsaved-Playground notice text quoted: "This is an Unsaved Playground. Your changes will be lost on page refresh." with a Save site locally button.
- Three states evident: working session, booting (Step 2/6 · Downloading coffee-shop/theme.zip), and launcher modal. Mobile (line 1720+) shows a phone-shaped layout with the same channel metaphor compressed.
- DB has the early-access notice via the topic line "SQLite-backed MySQL emulation · 14 tables · early access" and a SQL search input.
- Member-list-as-blueprint-browser on the right is a real reframe; the 43 cards become 43 "online members".
- A floating Quick Settings panel (line 1433+) AND the in-channel Settings page share the same data — and the README claims the popover trigger is gone. The quick panel may be a toast or a floating overlay; needs verification.

## What's missing or weak
- **Quick settings panel exists (line 1433–1437) alongside the `# settings` channel** — so settings appear in two places, not "one channel". The README claims popover is gone. The artifact has a non-popover overlay that quacks like one.
- **The PR / GitHub / URL launcher cards** in the server-add modal don't expose paste inputs; tapping a card supposedly opens the next step. §3 flows 11–12 incomplete as in many designs.
- **Apply & Reset still has no confirm step rendered.** The destructive copy is excellent, but the README claims "wrapped in a Discord-style 'danger' confirm" — that confirm modal is not in the artifact, only the warning.
- **The "Maximize iframe" affordance for narrow-iframe complaint** is mentioned in the README but the artifact doesn't show the maximized state — only the chat-shaped column with the iframe.
- **Members-as-blueprints metaphor stretches** when there are 43 "members" — Discord's online list is fine at 100s but a blueprint isn't really a member, it's a template. The README acknowledges metaphor strain but commits to it.
- **"Voice-channel-style RUN block holds Refresh / WP Admin / Homepage / Address bar"** — that's four affordances stuffed into one persistent block. The metaphor is cute but the URL bar inside a voice-channel feels semantically off. The README admits it's "a deliberate metaphor stretch".
- **The 43-blueprint card list in the member panel is likely JS-generated** (`#` rows pattern), so static HTML may have just the chrome.
- **WP-PR / Gutenberg-PR / Blueprint URL launcher entries map to entire "smart-mailbox"-style sub-views** within the Add a Server modal — but tapping should reveal a number/URL paste field, which the artifact doesn't render.
- **Mobile collapses to a single column with bottom-tab bar (Servers/Channels/Live/Members)** per the README, but the artifact's mobile section shows a more conventional iOS phone shell. The Discord-mobile-app shape is implied but not committed-to.
- **The Empty-state replacing the marketing landing page** is described but I don't see a clear "welcome to your server" channel content. The booting state has step text; the post-boot empty channel is unclear.

## How to push harder
- Render the Discord-style danger confirm modal as one of the states — "Apply settings & Reset Playground?" with a "Type the server name to confirm" input field (Discord's actual destructive pattern for server deletion). The metaphor will sing.
- Promote Logs-as-Chat to be the design's flagship — give it threaded "spam mode" filters, slow-mode for noisy chatty plugins, and a "Pin this error" function. Nobody else can do this remix; lean into it.
