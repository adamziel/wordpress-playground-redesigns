v5 · refinement · slot 006

Pull-down for chrome

The WordPress site is the screen. Everything else lives behind a single Safari-style gesture: drag the top handle to reveal toolbar (peek), launcher (half) or full library (open). Release to retract. One door per feature, one secondary surface, one verb.

Parent: v1 #06 Mobile-first · Axis pushed: iframe primacy + restraint via single gesture · Idle affordances: 3
One gesture, three detents

Pull 12% → toolbar peeks (48px). Pull 60% → launcher swings down. Pull all the way → full blueprint library. Release halfway and it snaps.

One secondary surface

Settings, Files, Blueprint, Database, Logs are tabs inside the bottom sheet — never their own sidebars. Only one is open at a time.

Iframe-primary always

Pull-down overlays the WordPress site — never reflows it. Press Esc from anywhere to retract everything and enter true fullscreen.

1 Desktop · working session · chrome retracted 1440 × 900

Idle state. Three always-visible affordances: (1) the pull-down handle, (2) the saved-state pill — green = saved, (3) the fullscreen toggle. Saved playgrounds sit at the bottom-left in their last spatial position. Nothing else.
