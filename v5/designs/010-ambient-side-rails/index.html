Idle affordance ledger (target ≤ 8)
  • 1 · Left rail — one door to "Launcher" (Your sites + Start new + 43 Blueprints)
  • 2 · Right rail — one door to "Site Manager" (Settings · Files · Blueprint · DB · Logs)
  • 3 · Top bar identity — name + saved-state dot + version chip (single static cluster)
  • 4 · URL pill — address field with embedded refresh + saved-state
  • 5 · Refresh — embedded in URL pill
  • 6 · Save state — embedded in URL pill (clickable; becomes "Save now" when unsaved)
  • 7 · Fullscreen iframe — F or icon top-right
  • 8 · More menu — kebab top-right (one home for Export GitHub PR, Download .zip, etc.)
One door per feature
  • Settings → only Site Manager · Settings tab (popover + info-notice button removed)
  • Apply & Reset → confirm-by-typing the playground name. No more "tiny red banner"
  • Launcher → only the left rail. No second nav for blueprints
  • Export GitHub PR / Download .zip → only the kebab menu
  • WP-admin / Homepage → only the Site Manager header (also reachable via URL pill)
Ambient color — what it earns
  • Rails sample the iframe's dominant color (live). The chrome belongs to the site rather than over-printing it.
  • The seam between rail and iframe is a 1px theme-color line — a "lit edge" the eye reads as boundary, not chrome.
  • Demonstrated cool (Vanilla WP) and warm (Art Gallery) tints on mobile B.
  • No glass-for-glass-sake: the only blurred panels are the secondary pane and the menu, where they overlap the iframe.
