How the IA collapses · why Smart Resize earns its keep

Old chrome: top toolbar (7 affordances) + left Site-Manager sidebar (5 tabs + 3 header buttons + modals) + right Saved-Playgrounds drawer (3 sections + 43-blueprint sub-screen) + popovers. Settings has three doors — popover, sidebar tab, info-notice button.

New chrome: a 44px top bar with 5 chrome chips (WP/Refresh · URL · Name pill · Fullscreen · More) and a single secondary pane with 6 swap-tabs (Sites · Settings · Files · Blueprint · DB · Logs). One door per feature. The launcher is the Sites tab. WP Admin / Homepage live as right-anchored chips inside the pane head (contextual to the current playground, not global housekeeping). Apply & Reset requires a typed phrase. Save state is a status pill; Save the action is a separate dark button that only appears when unsaved (see State G/J).

The momentum-based split is the whole IA. The pane is never closed. Whichever surface you interact with grows; the other shrinks. No drag handle to hunt for — focus IS the layout. The seam between panes carries a tiny gauge so users see the ratio shift before they have to think about it. F takes the iframe to 100% (fullscreen, Esc to exit) — this is the explicit toggle the brief asks for. Both 1440 and 390 share the same primitive: at 390 the seam is horizontal (top/bottom) instead of vertical, but every other element is identical. Mobile is a wider mobile of itself.
