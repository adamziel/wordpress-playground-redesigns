01

One affordance at idle.

The name pill is the entire chrome. A new user knows where to click; a returning user can forget the UI exists. The save-state LED makes the only persistent fact (am I saved?) legible.

02

One door per feature.

Settings live only in the drawer's Settings tab. Save lives in the menu and in ⌘S. Launcher lives only behind the name. The old triple-entry-points are gone.

03

One secondary surface.

The right drawer is the only secondary pane. Its five tabs swap content within the same frame. No second sidebar, no overlapping panels.

04

Iframe is never reflowed.

The drawer overlays — it does not push. The launcher overlays. Fullscreen (F) hides even the corner. Everything you reveal is reversible without rearranging the site.

Affordance budget

Idle affordances visible: 1.

The brief asks for ≤ 8 always-visible affordances at idle. We have one: the name pill in the top-right corner. The save-state LED inside the pill is a status indicator, not a clickable affordance. Every other control — refresh, address bar, save, launcher, site manager, blueprint, files, logs, db, settings, github export, .zip download, apply & reset, fullscreen — is summoned by hovering or clicking that single pill.
