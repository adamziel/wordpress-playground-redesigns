# 031 — Left gutter

## Thesis
The entire UI is a 48-pixel monochrome icon gutter pinned to the left edge. No top bar. No right rail. No floating windows. Eight icons live in the gutter — Refresh, Save, Sites, Settings, Files, Blueprint, Database, Logs — and a 28-pixel address strip sits along the bottom of the iframe to carry the URL, the playground name, and the save-state. One secondary pane swings out from behind the gutter when you click any icon; the iframe still owns 99% of every other pixel. That is the whole IA: gutter as nav, pane as detail, address strip as the only navigable URL door, iframe as the world.

## What I merged / removed and why
The triple-entry-point sin is dead. The Settings popover, the Site Manager Settings tab, and the "Save site locally" info-notice all collapse to gutter icon #4 (Settings). The Saved Playgrounds drawer, the "saved playgrounds" dropdown, and the full 43-blueprint browser collapse to gutter icon #3 (Sites) with a 3-way segmented control (Yours · New · Blueprints) inside the pane — applying lesson 002 (don't ship three drawers stacked in one pane). The top toolbar doesn't exist: Refresh and Save become gutter icons, the address bar moves to the bottom of the iframe as a single composite element (not a row of buttons), and the playground name lives inside that bar as an editable pill. The "Open / Close Site Manager" button is gone because the Site Manager *is* the gutter — five of its tabs are gutter icons #4-8 directly, so there's no parent control needed. The five Site Manager tabs render in the same pane (same x, same y, same width-class) with no nested tab bar inside the sheet — applying lesson 005 (no tabs inside tabs). Export to GitHub PR + Download .zip both live in the pane head's ••• menu, in one place, only inside Settings — applying lesson 016 (don't ship two doors for the same destination). Apply+Reset becomes a typed-phrase confirm sheet (you must type the playground name to delete) instead of a small red banner, per the v5 brief's principle 8.

## Trade-offs
A 48px gutter can hold ≤9 comfortable icons before density bites. To fit at exactly 8 doors + 1 fullscreen mode-toggle, I had to merge the WP Admin button and the Homepage button into pane-head icons inside the Settings pane (they're context-dependent on this site, not global), and I had to make the fullscreen toggle look intentionally dim and small — labelled in the README as a mode toggle, not a door. Power users who flip frequently between Files and Database now pay one click each (one gutter button), but the spatial position of each tab on the rail is fixed so muscle memory locks in fast — closer to VS Code's activity bar than to a generic tab strip. The pane swaps width between 380px (Settings, Logs, Sites) and 460px (Files, Blueprint, Database) which is a small deviation from "one fixed pane width" — I accept this because the alternative is a cramped Files editor at 380px or a wasteful 460px Settings form. Glass appears in exactly one place: the fullscreen Esc pill in state K, where it earns its keep by being readable over arbitrary iframe content; the rest of the chrome is solid (gutter is ink, pane is paper).

## Lessons applied from earlier v5 reviews
- **Review 001** ("show the expansion gesture as a first-class state"): the Sites pane is shown both *closed* (state A) and *open* (state B) on the same desktop frame, plus the same icon is rendered in its `is-active` state in both the gutter and the pane title — the morph is legible without animation.
- **Review 002** ("don't stack three drawers in one pane"): Sites pane uses a real 3-way segmented control (Yours · New · Blueprints), not stacked sections. State C demonstrates both Yours and Blueprints segments rendered explicitly.
- **Review 006** ("don't import a clever idea that undermines restraint" + "show every Site Manager tab"): no spatial dock, no Cmd+K palette, no glass widget Christmas tree — just gutter + pane + address. All five Site Manager tabs (Settings, Files, Blueprint, Database, Logs) are rendered at full fidelity in states D-G.
- **Review 010** ("at-idle iconography, not just labels"): every gutter button has a Lucide icon at idle. Active icons get a *paper* background (inverted ink) so the active state is visible without expansion.
- **Review 018** ("eliminate third door + cite reviews by number"): I started with this design citing 018 as a peer (also a left-rail direction), but the structural difference is clean: 018 keeps a top strip with URL + name + save; mine removes the strip entirely and moves URL/name/save into the gutter + bottom address. Two distinct interpretations of "rail-first."
- **Review 022** ("exclusive-open IA, not a social-pressure policy"): the secondary pane is *single-instance* by design — opening Files closes Settings, etc. No multi-window option exists in this direction; you cannot have two panes at once.
