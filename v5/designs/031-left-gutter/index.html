v5 · Design #031 · Direction: Left gutter

The 48-pixel slab.

No top bar. No right rail. A single 48px monochrome icon gutter on the left is the entire UI. Eight doors live in it — Refresh, Save, Sites, Settings, Files, Blueprint, Database, Logs — and a tiny address strip lives at the bottom of the iframe to hold the URL, name, and save-state. One secondary pane swings out from behind the gutter; the iframe still owns 99% of every other pixel.

Idle affordances: 1. Refresh 2. Save 3. Sites 4. Settings 5. Files 6. Blueprint 7. Database 8. Logs F = fullscreen (bottom of gutter, decorative-weighted — mode toggle, not a door)
Gutter — ink, always visible, 48px Pane — one secondary surface, swings from behind gutter Accent — used only for active state & URL scheme One door per feature: Settings popover = Settings tab. Period.

Read these frames

Mobile is literally the desktop rotated 90° — same 8 icons, same single secondary pane (sheet here), same address strip. No bespoke mobile-only IA, no "hamburger" hiding the gutter. The brief's wider-mobile principle is met by construction, not by adaptation.

One door per feature: Settings = gutter icon #4 (popover removed). Files = #5. Blueprint = #6 (with own multi-file tree, distinct from Files). Database = #7. Logs = #8. Sites/launcher = #3 (with sub-segments Yours / New / Blueprints, replacing the right drawer + the duplicate "saved playgrounds" popover). Export to GitHub PR + Download .zip = pane head ••• menu, only inside Settings. URL navigation = address strip at the bottom of the iframe (only URL-shaped element in the design).
