v5 · #011 · Auto-hide HUD

The chrome is a HUD. The iframe is everything.

WordPress Playground rendered like a video player. At idle the iframe owns 100% of the screen. Move the mouse near an edge and the HUD for that edge fades in: top edge = URL + identity + save; bottom edge = saved-sites dock + fullscreen; left edge = launcher rail; right edge = Site Manager rail. Two seconds of mouse stillness and every surface dissolves. Press F for absolutely-no-chrome fullscreen; Esc exits.

1 Desktop · idle · 2 s of no mouse motion → all chrome dissolved HUD HIDDEN

A working session: the Coffee Shop playground rendering at full bleed. The only visible artifact is a tiny status pill in the upper-right — three words and a heartbeat dot — so the user knows the HUD exists and the site is saved. Mouse anywhere and the appropriate edge fades in. No chrome means no decisions to make and no pixels stolen from the iframe.

Affordance budget · idle (revealed state)

  • TOP edge (5): refresh · address bar (URL + saved-state chip is the chip, not a button) · name chip · version chip (passive text) · kebab menu — 4 interactive + 1 passive
  • LEFT rail (1): single rail surface (4 icons collapse to ONE "Launcher" door — they are deep-links, not separate features)
  • RIGHT rail (1): single rail surface (5 icons = the 5 Site Manager tabs · ONE door)
  • BOTTOM (2): dock cluster (one element) · fullscreen pill

Total: 8 always-visible affordances when revealed (4 top buttons · 1 left rail · 1 right rail · 1 dock · 1 fullscreen). At true idle: 1 affordance (the corner heartbeat hint).

One door per feature: Settings only in Site Manager · Settings; Save only in URL pill chip; Launcher only via left rail; Site Manager only via right rail; Export/Download only in top-bar kebab; WP-admin/Homepage only in Site Manager header.
