# Review of v5 #012 — Corner FAB pie

**Parent direction:** "Corner FAB pie. Iframe fullscreen. Bottom-right has a single small dot. Click → 8-slice pie menu radiates."
**Affordance count at idle:** 1 (the dot). Saved-state lives *inside the dot's core color* — green/amber/blue — so save state costs zero extra affordances. Tied with #014 and #015 for tightest idle restraint.

**Scores (out of 10):**
- Feature completeness: 10 — 12 states (idle, pie open, URL pop, booting, sites/launcher, blueprint gallery, settings + reset-confirm, files, blueprint tab, database, logs + more menu, saved+fullscreen pill) plus mobile suite. Every SM tab gets its own dedicated state. Export + Download in More menu (state 11). Apply&Reset typed-phrase confirm in state 7.
- Restraint: 10 — single dot at idle. Strongest restraint in v5.
- Iframe primacy: 10 — iframe is 100% of viewport at idle (per README); overlays only; no reflow. Fullscreen via F/Esc and slice 7.
- One-door: 7 — README admits "slices 4 & 6 open the same Inspector with a different default tab — same room, two doormen." Same critique as #014 but more honestly disclaimed. The defense ("users searching for 'settings' expect a one-tap path that doesn't require knowing about a containing 'Manager' concept") is reasonable but it's still two doors.
- One-secondary: 10 — one Inspector pane; Launcher / Site Manager / URL omnibox / More menu all overlay from the same FAB.
- Direction adherence: 10 — entire IA is one dot + one pie. Two concentric rings (instant actions inner, pane openers outer) is a *refinement* of the basic 8-slice idea, mapping precisely to the v1 feature inventory.
- Visual fidelity: 9 — Inter font, real Lucide icons, two-ring pie geometry, dot color cycle (green saved / amber unsaved / blue booting), conic-gradient halo around the dot for draw-the-eye, real WordPress Coffee Shop iframe behind, near-opaque pie (#0d1220 94%) for label legibility, glass Inspector for color bleed.
- Synthesis with v1 lesson: 10 — closes v1 #15 fullscreen gap with slice 7 + F/Esc; collapses the v1 triple-entry sin; merges Saved Playgrounds with Launcher into one Inspector. Cites the brief's "muscle memory can't dismiss it" wording for the typed-phrase confirm.

## What works (specific)
- **Two concentric rings** is a structural refinement of the basic pie metaphor: inner ring = instant actions (Refresh / Save / Fullscreen / More) that complete on click; outer ring = pane openers (URL / Sites / Site Manager / Settings). The user can learn "inner = do it now, outer = open something." Best radial-menu IA in v5.
- **Saved-state inside the dot's core** is the cleanest save-state implementation in v5. Zero extra affordances, full visibility, color-coded. #003's slider tile and #013's name pill both cost more pixels.
- **Number-key shortcuts 1–8** mapped to slices; `Cmd-K` toggles the pie; `F` direct-fullscreens; `Esc` cancels. The keyboard fluency layer is complete.
- **Floating hint bubble for first sessions** + soft conic halo around the dot + hovering "Unsaved Playground · ⌘K" pill on cursor proximity — three discoverability mitigations layered. The README acknowledges this is "the design's biggest risk" and the mitigations are concrete, not hand-waved.
- The **Inspector is glass + the pie is opaque** distinction is a real glass-discipline: glass where it serves (iframe color bleeds through Inspector for depth), opacity where it serves (pie's slice labels need legibility). Compliance with v5 §2 rule 6.
- **Mobile Inspector goes full-bleed** (README admits, -1 in mobile rubric) — defended as "at 390 px every split would shrink both panes below readable." Honest.
- README explicitly notes "slice 6 is 'Settings (quick)' which is just slice 4 → Settings tab; same room, two doormen" — pre-emptively scoring -1 for itself before the critic does.

## What's weak or missing (specific)
- **Slices 4 & 6 are the documented one-door violation.** README defends but the defense is utility-based ("users searching for 'settings' expect a one-tap path") not principle-based. Could be solved by renaming slice 6 to "Quick Settings" (versions only, popover) vs slice 4 "Site Manager" (full sheet) — making the two doors lead to *different* surfaces.
- **The dot has zero label at idle.** README mitigates with hover halo + proximity pill + first-session hint — but a returning user three months later may still need to discover. Compare to #014's single-word-pill which embeds the playground name + `⌘K` hint visibly. #012's dot is more austere.
- **Eight slices is a lot for muscle memory.** #007's quarter-pie capped at 6 spokes with fixed angles/colors as the muscle-memory engine. #012's 8-slice arc is wider but each slice is narrower — angle estimation is harder. Number keys 1–8 compensate but the visual gesture is harder than 6 slices.
- **Inspector with three sub-tabs for Sites (Start new / Blueprints / Your saved)** is the same tabs-inside-sheet pattern flagged in #005, #008, #016. Defensible because the Inspector is one surface, but cognitively still a two-click depth (slice → tab).
- The **URL slice** (slice 5, opens an omnibox) is described in README but is essentially a separate input modality. Is it part of the Inspector or a popover? State 3 shows a "URL pop" — different from the right Inspector. Might be a *second* secondary surface.
- The "More" menu (slice 8) houses Export PR + Download .zip — same junk-drawer risk as #009 and #013. README claims "keeping both required exports together rather than buried" — fair.

## How to push harder (1–2 pointed suggestions)
1. **Differentiate slice 6 from slice 4 by surface type, not just default tab.** Slice 6 ("Quick Settings") = a *small Settings popover* (versions + language + multisite toggles + Apply&Reset link) anchored to the dot. Slice 4 ("Site Manager") = the full Inspector with five tabs. The user now has *one quick door* (popover, ephemeral) and *one deep door* (Inspector, persistent). Two doors, two purposes — eliminates the soft duplication.
2. **Add a "currently active" indicator on the pie when reopened.** If the user was last in /db, the next time they open the pie, slice 4 has a tiny dot indicating "this is where you were." Builds spatial memory across sessions without adding chrome.

## Lesson for the next designer
An 8-slice pie strains muscle memory more than you think — the angle between slices 3 and 5 is invisibly small in a 90° arc. If you push beyond 6 slices, lean on *consistent visual differentiation* (icon shape, color, ring) and *keyboard shortcuts* — angle alone won't carry it.
