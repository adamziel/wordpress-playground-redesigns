**Affordance count at idle:** 8 (7 chrome + 1 ambient depth gauge): capsule's 7 controls + the right-edge depth gauge (status-only, not a door). At cap.

## What works (specific)
- **Glass quantified as IA** is the design's conceptual breakthrough. Three z-levels (L0/L1/L2/L3 + L4 modal shade) with named blur strengths (0/8/18/36) — depth isn't aesthetic, it's *information hierarchy made visible*. The cleanest answer to v5 §2 rule 6 in the entire round.
- **Depth gauge with L0/L1/L2/L3 labels** is the direct application of review #010's lesson: passive status indicators need *labeled* glyphs, not abstract dots.
- **Glass-on-glass legibility countered** by making L1 the *least* blurred and tinted whitest, so it stays sharp on top of L2/L3 without competing. README explicitly addresses this — design-as-engineering disclosure.
- **Save collapsed to name pill, visible only when unsaved** — saved state hides the verb entirely, leaving only the green dot + name. Cleanest save-state-as-action in v5 (tied with #014, #017, #023, #024, #027).
- **L4 modal shade for destructive Apply&Reset** — the destructive action gets its own depth layer, *deeper than anything else*. Forcing the user through a modal-darkening + typed phrase confirms the action is destructively-different from normal flow.
- **Six review citations with specific implementing element** including the *anatomy diagram in Panel I* that labels the depth layers visually. Cumulative critic feedback rendered as a teaching layer.
- **Library nav rail (Library / Categories) instead of stacked sections** — review #002 lesson applied. The 43-blueprint gallery is a section inside L3 Library with a categories nav rail.
- **CSS feature query fallback** explicitly mentioned for `backdrop-filter` unsupported environments — engineering honesty.

## What's weak or missing (specific)
- **Heavy backdrop blur costs GPU** (README admits). On slow devices the depth metaphor degrades to flat tinted surfaces — the design's signature *disappears* on the failure path. Compare to #018 (zero glass) which has no degraded mode.
- **The depth gauge is decorative-status, not interactive** (README's word). It tells you which layers are stacked but you can't *click* it to recede a layer. A tappable gauge would close the loop.
- **The Inspector pane is fixed-width (1100 px on desktop) and not resizable** (README admits). Power users who want Files at full width pay friction.
- **Library's depth (L3) is "the deepest at rest, but rises to the foreground when summoned"** — that's *not* a stable z-relationship. README says depth = z-depth, but L3 swaps from backdrop to foreground on summon. The "where you came from" metaphor breaks because L3 *isn't* always behind L2; it can be in front.
- **The depth gauge being the 8th idle element** is a slightly inflated count — README defends as "status-only, not a door." Strict reading: it's status, fine, but it occupies a screen position that adds to chrome density. Compare to #018 which doesn't surface a status gauge.
- **Mobile rotates the metaphor 90°** (capsule bottom, inspector top sheet, library overlay) — preserves the seven affordances and gauge. README rates 4/5 honestly. The visual *depth perception* on mobile may not match desktop because the eye reads horizontal depth differently than vertical.

## How to push harder (1–2 pointed suggestions)
1. **Make the depth gauge interactive — tap an Lx pip to *return* to that layer.** Currently it's status-only; making it a navigation primitive (tap L0 → all higher layers dismiss; tap L1 → only the highest above dismisses) turns the design's most-novel visual element into a *control*. Net affordance count unchanged (still 8) but the gauge earns its slot.
2. **Anchor L3 (Library) as a stable backdrop always** — never let it swap to foreground. When summoned, Library rises *to L2 position* and the Inspector recedes to L3 (or L4). This preserves "heavier blur = further away = where you came from" as a *constant rule*, not a contextual flip. Otherwise the depth metaphor leaks.

## Lesson for the next designer
If your direction makes depth the navigation primitive, the z-relationship between layers must be *constant*, not contextual. A layer that's at the "backdrop" by default but rises to foreground when summoned breaks the metaphor — instead, recede the formerly-foreground layer when summoning a new one, keeping the *original* z-stack stable.
