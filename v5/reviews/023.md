# Review of v5 #023 — Smart resize

**Parent direction:** "Smart resize. Two panes (iframe + secondary). The one you interact with grows; the other shrinks. No manual divider — momentum-based."
**Affordance count at idle:** 7 (README-counted): WP/Refresh glyph · URL bar · Name pill (status only) · Save (appears only when unsaved) · Fullscreen icon · More kebab · Pane segmented control. WP Admin + Home are pane-internal contextual chips.

**Scores (out of 10):**
- Feature completeness: 10 — 10 demonstrated states (A–J). Idle 50/50, iframe focus 75/25, pane focus 35/65 with typed-phrase reset, Blueprint pane with 18 visible blueprints + 10 categories, Files with functions.php editor, Logs with level filters, Booting state, Database with early-access notice, Fullscreen, save-state transition. Comprehensive.
- Restraint: 9 — 7 at idle, with Save effectively 0 in saved state (appears only when unsaved). The pane is always present (25% min), which is intentional restraint — the chrome is *the layout itself*.
- Iframe primacy: 9 — README admits "the iframe is *never* fully maximised in idle — even at 75% iframe focus, the pane occupies 25%." Defended as "the pane is the chrome that proves you're in Playground." F+Esc provides escape hatch to 100%. The 75% iframe focus state is genuinely iframe-primary; the 50/50 rest state is the design's deliberate symmetry choice.
- One-door: 9 — settings collapsed; WP Admin/Homepage moved into pane head (not duplicated); More menu carries Export/.zip/Rename/Delete/Docs (legitimate IO drawer). Clean.
- One-secondary: 10 — single pane with 6 swap-tabs; *never* two panes at once. Most disciplined two-pane IA in v5.
- Direction adherence: 9 — gauge + ratio shift + inset blue bar + four-snap-detents (35/65, 50/50, 60/40, 75/25) all make the metaphor visible. README rates 4/5 honestly because "discoverability of 'click → grow' is the soft spot."
- Visual fidelity: 9 — Inter font, real Lucide icons, real WP iframe (Coffee Shop), 18 blueprint cards, real PHP error log with level filters, typed-phrase confirm, four-detent gauge in the seam. No glass at all — solid white chrome, principled restraint per v5 §2 rule 6.
- Synthesis with v1 lesson: 10 — extends #55 iPad multitasking "pushed further: the divider is not a thing you grab, it's a thing that *follows your eye*." **Explicitly cites three v5 reviews:** "lesson from review #001, #003, #005: never mix status and action on the same pill" → Save button only appears when unsaved; name pill is status only. This is a *direct* application of the cumulative critic feedback across multiple prior reviews.

## What works (specific)
- **Focus IS the layout.** The simplicity of this premise is its strength: no drag handle to discover, no resize gesture to learn. Click the iframe, it grows. Click the pane, it grows. The gauge in the seam *teaches the user* what just happened.
- **Save as a *separately appearing* button, not a perma-affordance**, is the cleanest implementation of "never mix status and action" in v5. The README explicitly cites the lesson by review number — best critic-feedback synthesis alongside #018.
- **Four snap detents (35/65, 50/50, 60/40, 75/25)** is intentional: no fiddling, predictable layout. Compare to #017's 25/50/75 detents which felt cramped at 75%; here the iframe is always at least 35%.
- **No glass anywhere** — solid white chrome, principled restraint per v5 §2 rule 6. The README scores 5/5 honestly; only #018 also nailed zero-glass.
- **Apply & Reset typed phrase ("reset coffee shop")** — the brief's destructive-flow requirement met.
- **Mobile uses the same primitive with a horizontal seam** (README) — same gauge, same six tabs, just rotated 90°. "Wider-mobile" promise honored.
- **More menu's contents named explicitly** (Export PR / Download .zip / Rename / Delete / Docs) — README transparency.
- **No drag-handle to grab** is the IA's love-it-or-hate-it choice. README defends with the gauge + inset blue bar feedback layer, which is the right safety net.

## What's weak or missing (specific)
- **Discoverability of "click to grow" is the design's real soft spot.** README acknowledges (-1) and lists three mitigations (gauge arrows, inset blue bar, keyboard shortcuts) but admits "a first-time user may need a tooltip — I considered an onboarding pulse animation but left it out to keep affordance count honest." Adding a one-shot tutorial pulse wouldn't increase the *idle* affordance count (it disappears after first use); the conservative call here costs UX.
- **The pane is never closed.** A user who wants the iframe full-bleed must press F (fullscreen). For users who just want a peek, that's a heavy gesture. Consider: a fourth detent at 90/10 (almost-iframe-only, pane shrinks to a 100 px strip with just the segmented control) — between 75/25 and fullscreen.
- **6 tabs in the segmented control** is on the verge of cramped at 35% pane width. With each tab needing ~50 px of label, 6 tabs need 300 px of width — only fits at pane width ≥ 480 px (i.e., when the pane is ≥ 35% of a 1440-viewport, fine on desktop). At 35% of a 960 px viewport (small desktop), it's 336 px — tight. README says mobile collapses to icons; could specify the breakpoint.
- **No manual override** means a user who *wants* 50/50 always must explicitly click both panes alternately. The four detents are momentum-based per README — but the explicit ratio control lives in a tooltip or keyboard map.
- **The contextual WP Admin / Homepage chips** are only "visible on Settings/Files/etc." — not always. A user wanting to jump to WP Admin from any tab must first navigate to Settings or Files. Promote them to the pane head's permanent right side regardless of tab.
- The "WP/Refresh combined glyph" (idle affordance 1) is *clicking the WordPress mark refreshes the iframe*. Slightly novel — but new users may not realize the WP mark is interactive (it looks like a logo). Add a refresh icon underlay or a hover tooltip.

## How to push harder (1–2 pointed suggestions)
1. **Add a fifth detent at 90/10 (almost-iframe).** Between 75/25 and fullscreen (F → 100/0), have a "peek" mode where the pane shrinks to a vertical icon strip showing just the 6 tab glyphs + the gauge. Clicking any glyph instantly returns to 35/65 or 50/50. This gives users an iframe-dominant state that doesn't require entering fullscreen.
2. **One-shot teaching state on first visit.** The first time the iframe appears at 50/50, briefly pulse both panes (1 s total) with the gauge animating from 50 → 75 → 35 → 50 — demonstrating the resize behavior visually. After first interaction, never again. This is *not* a new affordance (it's a transient onboarding) and it solves the discoverability gap cleanly.

## Lesson for the next designer
When your direction's main interaction is *implicit* (click to focus = layout shifts), build the *teaching layer* into the design itself — a gauge in the seam, an inset blue bar, a one-shot pulse on first visit. Don't worry about "increasing idle affordance count" with onboarding animations; transient teaching states aren't counted in the idle budget.
