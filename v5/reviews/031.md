# Review of v5 #031 — Left gutter

**Parent direction:** "Left gutter. 48px vertical icon gutter on the left. No top bar at all. The gutter holds Refresh, Save, Sites, Site-Manager-tabs, Settings."
**Affordance count at idle:** 8 (count-strip): Refresh · Save · Sites · Settings · Files · Blueprint · Database · Logs. Fullscreen icon is "intentionally dim and labelled as a mode toggle, not a door." Brand mark is decorative.

**Scores (out of 10):**
- Feature completeness: 10 — states A–K cover idle, Sites pane open, Sites pane with Blueprints segment, all 5 SM tabs at full fidelity (Settings, Files, Blueprint, Database, Logs in states D-G), typed-phrase Apply&Reset (mentioned), fullscreen with Esc pill (state K). Three-segmented control in Sites (Yours · New · Blueprints). All five SM tabs rendered.
- Restraint: 10 — exactly 8 doors. Fullscreen explicitly disclaimed as mode toggle. Single-instance pane (no multi-window).
- Iframe primacy: 9 — iframe owns 1376/1440 px at idle (96%), 1390/1440 in fullscreen. README rates 4/5 honestly; F key + gutter icon both work.
- One-door: 10 — Settings popover removed; "Open Site Manager" button removed; WP Admin/Homepage in Settings pane head; Export/Download only in Settings pane-head menu. URL bar is the only URL-shaped surface.
- One-secondary: 10 — *single-instance* pane by design (citing review #022 lesson) — opening Files closes Settings. No multi-window. Same anchor (left=48 px), only width varies (380 px for Settings/Logs/Sites, 460 px for Files/Blueprint/Database).
- Direction adherence: 10 — *no top bar at all*. Address strip is bottom-anchored. Name and save state live as pill + gutter dot. Every "Left gutter" lever is pulled hard.
- Visual fidelity: 9 — Lucide icons in gutter, solid ink/paper color scheme (no glass except fullscreen Esc pill), VS Code activity bar muscle-memory. The active gutter icon gets a "paper" background (inverted ink) — review #010 lesson applied.
- Synthesis with v1 lesson: 10 — extends #14 Ventura clean rail + #64 DevTools-style chrome. **Cites *six* prior v5 reviews with the specific implementing element**:
  - Review 001 → Sites pane shown both closed (state A) and open (state B); morph is legible.
  - Review 002 → 3-way segmented control (Yours · New · Blueprints), not stacked sections.
  - Review 006 → "no spatial dock, no Cmd+K palette, no glass widget Christmas tree — just gutter + pane + address."
  - Review 010 → every gutter button has Lucide icon at idle; active icon gets paper background.
  - Review 018 → "I started with this design citing 018 as a peer (also a left-rail direction), but the structural difference is clean: 018 keeps a top strip with URL + name + save; mine removes the strip entirely." **Sibling-awareness with explicit differentiation** — first design in v5 to *cite a sibling design's choice and choose differently while crediting it*.
  - Review 022 → "exclusive-open IA, not a social-pressure policy" → "the secondary pane is *single-instance* by design."

## What works (specific)
- **Single-instance pane** — direct application of review #022's critique. Opening Files closes Settings. No multi-window. This is the *most-honored* prior-review lesson in v5 so far.
- **Sibling-awareness with #018**: README acknowledges #018 is also a left-rail direction and explicitly distinguishes. "018 keeps a top strip with URL + name + save; mine removes the strip entirely and moves URL/name/save into the gutter + bottom address." This is critic-grade differentiation — usually two siblings ignore each other; here #031 leans into the contrast.
- **VS Code activity bar muscle memory** — fixed gutter positions mean Files is always at slot 5, Blueprint at slot 6, etc. Spatial memory without the cognitive cost of dropdowns.
- **Address strip at the *bottom* of the iframe** (28 px) is the design's biggest IA departure. URL editing is anchored to the iframe's bottom edge, not the top. Unusual but principled — the iframe stays unbroken at the top.
- **Only ONE translucent surface in the entire design** (fullscreen Esc pill). 5/5 glass critique earned.
- **Pane width swap (380 px vs 460 px)** — README admits this is "a small deviation from 'one fixed pane width'" — defended with the editor-needs-room argument. Honest disclosure of trade.
- **The fullscreen icon is intentionally dim and labelled as a mode toggle, not a door** — the design *names* the affordance class so the count is honest.

## What's weak or missing (specific)
- **8 doors in the gutter, no slack**. README admits "to fit at exactly 8 doors + 1 fullscreen mode-toggle, I had to merge the WP Admin button and the Homepage button into pane-head icons inside the Settings pane." This is the right call but means WP Admin / Homepage are *not* one-click on idle — they're two clicks (Settings icon → pane head WP Admin button). Power users who jump to /wp-admin frequently pay.
- **Address strip at the bottom** is non-standard. Users expect URLs at the top. README defends ("iframe stays unbroken at the top") but a returning user may scan for the URL where it isn't. Worth a brief teaching state.
- **Pane width swap (380 vs 460)** is a real deviation from "one fixed pane." Compare to #024 (fixed 30% always) and #017 (snap at 25/50/75). The width-swap makes the pane location stable but not the geometry.
- **Sites is one gutter icon, opens to 3-segment pane** — but #018 has 3 separate rail doors (Sites, Site Manager, Blueprints). Both choices are defensible; #031 chose one door + segmented; #018 chose three doors + flat. README explicitly notes this divergence — good awareness.
- The **address strip carries URL + name + save state as a composite** — but it's at the bottom. If the user is working in the iframe (scrolling, clicking), they'd lose track of the URL/save-state because it's *below* the content they're focused on. Top-anchored chrome is conventional for a reason.
- The **gutter's fullscreen icon** is "intentionally dim and small" — but if it's a *mode toggle* (not a door), is it still interactive? README counts it as the 9th item (after 8 doors). Visually dim could read as disabled.

## How to push harder (1–2 pointed suggestions)
1. **Add a "go to address" gesture from the gutter** — when the cursor hovers any gutter icon, a 8 px chevron appears at the bottom right of the gutter pointing toward the address strip, teaching users *where the URL lives*. This costs nothing at idle and solves the bottom-anchored URL discoverability problem.
2. **Make the WP Admin / Homepage one-click via long-press on the Settings gutter icon.** Currently they're 2-click (Settings → pane head). A long-press gesture on the Settings icon → opens a small popover with "WP Admin" and "Homepage" as direct deep-links. This is the same restraint #007 used (long-press on Sites → Settings) but inverted for the common case.

## Lesson for the next designer
If your direction is "left rail / left gutter," cite the sibling left-rail designs by number and explicitly differentiate — don't pretend they don't exist. #031's README distinguishes itself from #018 in two sentences and earns the reader's trust by acknowledging "we both chose the rail; here's why we differ." This is the cleanest sibling-aware design copy in v5.
