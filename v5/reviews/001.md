# Review of v5 #001 — Iframe Supreme

**Parent direction:** "Mobile-first, iframe ≥ 65%. Strict rule: iframe never shrinks below 65% of screen height even when sheets are open. Sheets overlay the bottom 35%."
**Affordance count at idle:** 5 (desktop state A): Refresh · URL pill · Fullscreen · More (kebab) · Workbench pill. The Workbench pill expands to 6 dock tabs on hover/click. Mobile idle drops to 4 (Fullscreen moves into More). Well under cap.

**Scores (out of 10):**
- Feature completeness: 9 — 20 blueprint cards visible (line 832–852), all 10 category chips, all 6 start sources, 3 saved + Unsaved Playground, full Site Manager tabs reachable, blueprint JSON with valid schema URL, typed-phrase Apply&Reset with a real before→after diff (line 1020–1024), mobile booting state with step-by-step list.
- Restraint: 10 — only one visible "Workbench" pill at idle, with the 6-tab dock collapsed behind it. The other v5 entries fanned the dock out as default; this one earns the discipline.
- Iframe primacy: 10 — the 65% floor is a *cited measurement* ("iframe takes 800 px of 900 px — 89%" at line 656), sheet is capped at 30%, ambient color-bleed strip lets WP theme color glow into the topbar.
- One-door: 8 — settings, files, blueprint, db, logs each have a single dock tab; save lives only in the URL pill; Apply&Reset only in the confirm sheet. Self-flagged caveat: Files tab + Blueprint tab both expose a file-tree column (file tree in Files at line 1300+ and Bundle files in Blueprint at line 963–971) — defended as intentional for multi-file blueprint bundles.
- One-secondary: 10 — exactly one sheet, six tabs swap content; no second sidebar or popover.
- Direction adherence: 10 — the 65% rule is mathematically enforced in every state and called out in the chrome label.
- Visual fidelity: 9 — Inter + JetBrains Mono, real Lucide icons, mac titlebar with traffic lights, ambient-bleed gradient that picks up the WP theme's hue (sage/coffee/etc), realistic blueprint JSON with $schema, valid blueprint steps, typed-phrase confirm with cursor blink.
- Synthesis with v1 lesson: 10 — directly cites #06 mobile-first ("desktop is literally the mobile composition stretched"); also fixes the v1 #15 fullscreen gap with an explicit FS icon plus Esc; the Workbench collapse to one pill at idle is a learned restraint from "too many visible buttons right away" feedback on #10.

## What works (specific)
- The "Workbench pill" collapse (line 712–717) is the single best idea in the design — it shrinks the dock from 6 tabs to 1 affordance at idle, restoring screen real-estate to the iframe without losing functionality. Counter to almost every other entry where the dock is always-visible.
- The 65% floor is *measured*, not hand-waved: line 656 declares "800 px of 900 px = 89%". This invites the critic to do math, and the math holds.
- The Apply & Reset confirm sheet (line 1011–1032) shows a before→after diff *and* a typed phrase (line 1026: "reset coffee-sho" with a cursor), not just a red banner. The required phrase relates to the playground name, so muscle memory can't defeat it.
- Ambient-bleed strip (line 686, 1212) is a 1px-tall gradient that picks up the iframe's dominant color and bleeds *upward* into the topbar — glass earns its keep by carrying theme color rather than decorating.
- "Run Blueprint" is named as *the* primary verb in the Blueprint editor (line 994), addressing the brief's "pick one primary verb" call.
- Mobile-C "Booting" state (line 1188–1238) shows blueprint steps with `done`/`now`/upcoming markers — better than a generic spinner.

## What's weak or missing (specific)
- The dock-collapse-to-pill (idle) → dock-expanded (sheet open) transition adds a *hidden* affordance: a user must click "Workbench" once to see the six tabs. The README acknowledges this as a deliberate trade, but in state B (line 779–787) the dock is already fanned to 6 tabs without showing the gesture that took it there. Make the expansion gesture visible (e.g. a "Workbench" tab still highlighted, or an annotation).
- The launcher sheet on desktop (state B) compresses three columns (Start new / 43 blueprints / Saved) into a 30%-tall sheet — that's ~270 px vertical at 1440×900. The blueprint cards (line 832–852) are 38 px tall each: in a 30% sheet, you see ~5 cards above the scroll. The gallery becomes scroll-driven discovery only. Compare to the desktop 1440px width: there's room to use a different IA at this width, not just stretch the mobile layout. The "desktop is wider mobile" principle is admirable but the gallery suffers.
- Save chip in the URL pill (line 678, 754) reads "Saved" or "Unsaved · Save" — but tapping the chip itself does what? The brief says one primary verb (Save) — that's the chip — but it also doubles as the *state indicator*. Mixed action/state can confuse: am I clicking to save, or just seeing status?
- "Fullscreen icon in topbar (desktop) and More menu (mobile)" per README — but the mobile More menu's contents aren't actually rendered in the design. The fullscreen state on mobile is implicit.
- The dock at line 778 (state B) shows tabs with text labels on desktop ("Sites", "Settings", etc.), but on mobile (line 1150–1157, state B) they're icon-only. That's the right call for mobile, but the design's premise is *no difference* between desktop and mobile — be explicit about the icon-vs-icon+label deviation in the README rather than letting the discrepancy ride.

## How to push harder (1–2 pointed suggestions)
1. **Show the Workbench expansion transition explicitly.** Add a sub-state ("State A2") between idle (one pill) and sheet-open (6 tabs) where the pill is just morphing into the dock — annotated. This converts a hidden affordance into a teachable gesture, and it's the single clearest visual proof that this design's idle restraint is real, not a trick.
2. **Differentiate "save state" from "save action" in the URL pill.** Either split into two micro-elements (a non-interactive dot + an interactive "Save" button that appears only when unsaved), or commit fully to the saved-state-as-button model and remove the "Save" verb everywhere else.

## Lesson for the next designer
If you commit to an idle-affordance floor (≤ 8 means ≤ 8), make the *expansion gesture* a first-class visible element — collapsing a 6-tab dock into a 1-pill mystery saves count but creates a discoverability cliff; show the morph in one of your states.
