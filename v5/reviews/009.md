# Review of v5 #009 — 32px Micro-toolbar Only

**Parent direction:** "32px micro-toolbar only. That's the entire chrome at idle. A 32px-tall top bar holds URL + name + save + menu. Click menu → contextual sheets fly in."
**Affordance count at idle:** 7 visible at scene 1 (line 745–768): Refresh · URL pill · Save/name chip · Sites · Site Manager toggle · Fullscreen · Menu (kebab). The WP logo (line 746) is decorative. 7 of 8 budget.

**Scores (out of 10):**
- Feature completeness: 9 — Site Manager segmented control with all 5 tabs (scenes show Settings, Files, Blueprint, DB, Logs each), Sites sheet with three sub-tabs (Saved/Start new/Blueprints), 43-blueprint gallery overlay, booting scene, fullscreen scene (chrome hidden entirely). The address bar accepts URL schemes (`playground:`, `blueprint:`, `pr:`, `gh:`, `sqlite:`, `log:`) — folding multiple boot sources into one combobox is a brilliant compression but it borders on a *new* feature. Each scheme corresponds to an existing source, so it's IA-only.
- Restraint: 10 — 32 px is the strictest chrome budget in v5 so far. The direction is literally enforced.
- Iframe primacy: 10 — iframe at 96% of frame height at idle (the README's claim, confirmed by the layout). Scene 8 hides chrome entirely with F + Esc. Sheets fly in over the iframe rather than reflowing it.
- One-door: 7 — README admits "menu kebab is a junk drawer but sectioned" — rename / duplicate / export GitHub PR / download .zip / copy blueprint URL / palette / docs / delete all in one menu. The sectioning (Quick / This playground / Export / App) helps but it's many doors collapsed into one menu, not one door per feature. Settings has one canonical home, which is correct.
- One-secondary: 9 — Site Manager sheet uses a segmented control to swap content across 5 tabs. Sites is a *different* sheet (three sub-tabs). Two distinct sheet types; one for the current playground (manage), one for switching sites. Defensible split.
- Direction adherence: 10 — 32 px is literal, measurable, and the entire IA hinges on it. The URL scheme combobox is a direction-amplifying flourish (Chrome omnibox style).
- Visual fidelity: 9 — Inter font, custom WP logo SVG (line 746), Lucide icons, color-coded saved/unsaved name chip, real WP iframe content (Coffee Shop pour-over copy), URL scheme rendering with colored scheme prefix, segmented control inside the sheet.
- Synthesis with v1 lesson: 10 — README explicitly cites #64 (DevTools, slim top bar; "halves the chrome budget") and #15 (Win11 Settings, fullscreen gap fixed). The URL scheme idea is a #29-Console-hybrid descendent done right.

## What works (specific)
- **The URL scheme combobox** (`playground:` / `blueprint:` / `pr:` / `gh:` / `sqlite:` / `log:`) at line 749 is the single most powerful idea in this design: it collapses six start-source sub-screens (WordPress PR, Gutenberg PR, From GitHub, Blueprint URL, Vanilla, etc.) into one input. Brave, useful, expert-friendly, and beginners still have the Sites sheet sub-tabs as a visual fallback.
- The name chip (line 753) flips between `bookmark-check` + saved state (saved) and `circle-dot` + Save button (unsaved). Two states, one element, no duplicate door — exactly the right pattern for the save-state v5 concern.
- The Save info-notice (line 850–854) at the top of the Settings tab still says "This is an Unsaved Playground. Your changes will be lost on page refresh." — but it lives in *one* place only (the Settings tab), not three. README's "merged the triple-entry-point sin" claim holds up under inspection.
- The "Apply & Reset" typed-phrase confirm is promised by README and corresponds to v1 #15's destructive-flow fix.
- Mobile design at line 1583+ keeps the same toolbar with one concession: the URL scheme drops at 390 px width to gain pixels. Honest, documented in README.
- Glass only on the topbar (the one ambient surface where it makes sense), with all sheets solid white and strongly bordered — direct compliance with v5 §2 rule 6 ("glass earns its keep").

## What's weak or missing (specific)
- The Menu kebab is a "junk drawer" (README's word). Eight items (rename, duplicate, export PR, download .zip, copy blueprint URL, command palette, docs, delete) — sectioned, but still a lot. Could the command palette (Cmd+K) absorb most of those one-shot actions, leaving the kebab for *only* destructive/global app commands? It's the spot where the one-door rule frays.
- "Sites" chip with count "4" (line 759–763) opens a sheet; "Site Manager toggle" (line 764) also opens a sheet. Two icon buttons next to each other, each opening a different sheet — clear if you read labels but visually they're peers. A new user might tap one expecting the other.
- The URL scheme combobox is powerful but it's *also* a single point of failure: if a user mistypes `blueprint:` they get nothing visible. Add a hover affordance (small chip showing the active scheme's icon + description) — currently it's a 32 px-tall pill with the scheme inline.
- The Sites sheet's three sub-tabs (Saved / Start new / Blueprints) is *another* tabs-inside-sheet pattern, same critique as #005 and #008. But here it's defensible because the three sub-tabs are *peers*, not nested categories, and the SM sheet has its own segmented control distinct from this. Borderline acceptable.
- The 32 px height pushes the URL bar text to 11–12 px (README admits). At small-screen widths that's a real legibility cost; mobile drops the scheme prefix to recover.
- "WP mark decorative" (README): if the WP logo is purely decorative, the count is honest. If it's clickable (e.g., to go home), it's an 8th affordance and the count is at cap.

## How to push harder (1–2 pointed suggestions)
1. **Move six of the kebab items into Cmd+K.** Rename, duplicate, copy blueprint URL, docs — these are all *one-shot text commands* that the command palette is built for. Leave only export to GitHub PR, download .zip, and delete in the kebab (the three destructive/IO actions). The kebab then becomes a true "danger drawer" rather than a junk drawer, and Cmd+K becomes the one-door home for non-destructive actions.
2. **Add a one-line "scheme legend" under the URL bar on first focus.** When the user clicks into the URL pill, briefly show a 24 px row of scheme chips (`playground:` `blueprint:` `pr:` `gh:` `sqlite:` `log:`) with one-word labels. Disappears on first keystroke. Solves the "I didn't know about schemes" discoverability problem without permanently spending toolbar pixels.

## Lesson for the next designer
A 32 px chrome budget is glorious — but you must decide what the *Menu kebab* is for. If it's a junk drawer, you've moved the IA's complexity from "visible chrome" to "hidden chrome", not eliminated it; sort items into either Cmd+K (text commands) or a destructive-only drawer.
