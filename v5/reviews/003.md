# Review of v5 #003 — Control-Center tiles

**Parent direction:** "Control-Center tiles. iOS Control Center metaphor: a grid of frosted tiles in a top-right quadrant overlay. Each tile is one feature; long-press for detail."
**Affordance count at idle:** 8 exactly, listed verbatim onscreen at line 1170: Refresh · Address bar · Saved-state pill · Fullscreen toggle · Control button · Sites dock · "+" new · the iframe itself. Hits the cap precisely.

**Scores (out of 10):**
- Feature completeness: 9 — 9 states cover all five Site Manager tabs (Files state 5, Blueprint state 9, Database in the tile grid, Logs state 6, Settings state 7), all six New sources (state 6), 43-blueprint browser with search and 10 chips (state 3), typed-phrase Apply&Reset with diff (state 7), booting state with blueprint progress (state 4), explicit fullscreen state (state 8). README's mapping table at line 2360+ enumerates feature → state — well done.
- Restraint: 9 — eight exactly is brave; the tile grid only appears on Control press. The grid contains 16 tiles in 4 rows but lives behind one button.
- Iframe primacy: 9 — iframe is full-bleed at idle; tile overlay sits in top-right quadrant, not full-screen; explicit Fullscreen tile + F shortcut + state 8 shows the dedicated mode.
- One-door: 7 — Save is in *three* places: the slider tile inside Control, the ⌘S shortcut, *and* the saved-state pill. README admits this — "save lives in three places only by reflection… but there is only one canonical door." A strict reading still counts duplicate doors. Apply&Reset has its own Reset tile in row 4 *and* lives inside the Settings tile sheet — that's two doors.
- One-secondary: 9 — the Control quadrant is the only swappable secondary surface; sites dock at bottom-left is technically a second surface but it's a *list* not a swappable pane.
- Direction adherence: 10 — slider tile, toggle tiles, badge tiles, frosted top-right quadrant, long-press to expand: every detail reads iOS Control Center. Even the typography hierarchy of name/sub on each tile is faithful.
- Visual fidelity: 9 — Inter font, gradient icon backgrounds (blue/green/orange/purple/red/slate/yellow), real Lucide icons, mac-style chrome, glass tiles with proper layering. Save slider is a particularly successful affordance.
- Synthesis with v1 lesson: 9 — synthesizes #38 (glass) + #14 (Ventura clean sidebar) + #15 (fullscreen toggle, Win11 fix). The direction's iOS-Control-Center premise pushes both #38's glass and #6 mobile-first further. Mobile mirror at line 1180 keeps identical chrome and dock.

## What works (specific)
- The Save *slider* (line 1294–1302) is genuinely novel: "Slide to save" requires a committed gesture, far better than a tiny button — and the slider fill (38%) visualizes how-much-progress-to-saving. This single element re-frames the saved-state problem.
- 8 affordances enumerated verbatim onscreen (line 1170). No designer can be more honest than that.
- Long-press as the secondary gesture is a real interaction: state 3 (Blueprints), state 5 (Files), state 6 (New), state 7 (Settings), state 9 (Blueprint JSON) all show the same overlay expanding in place. Six demonstrated expansions in one design — excellent state coverage.
- The "Reset" tile (line 1387–1392) is bright red and labeled "destructive" — visually warns *before* commit. Then state 7 shows the typed-phrase confirm sheet inside the same overlay (not a modal dialog).
- Sites dock at bottom-left (line 1155–1162) offers fast site-switching with avatar tiles, complementing the Sites *tile* in the Control overlay — sort of one door per feature, since the dock is for *switching* and the tile is for *managing*.
- Feature-to-state mapping table at line 2360+ — a thoughtful affordance for the critic.

## What's weak or missing (specific)
- "Save in three places" is a real one-door violation. The README defends it by saying only the slider is the canonical *door* but in user-research terms, three visible save mechanisms is exactly the failure mode v5 warns against. Pick one: either remove the saved-state pill from the chrome (and let the slider tile be the only save surface) or remove the slider and let ⌘S + the pill carry the load.
- The "Reset" tile in row 4 of the Control grid (line 1387) is a *third* door alongside the Settings tile's "Apply & Reset" subsection (state 7) and the Settings popover. Three entry points to a destructive action that should have one.
- The Control button reads "Control · ⌘K" — but "Control" is generic and unclear what it controls. Even "Panels" or "Workbench" would tell the user *what they'll see*. iOS just shows an icon, no label; here the word "Control" without context confuses.
- The "Dark chrome" toggle (line 1309–1314) is a *new feature*, not in the brief inventory. v5 §5 forbids new features. Cut it.
- The iframe is full-bleed but the top-chrome is solid (not glass) — that's the right call for restraint, but the README claims "glass earns its keep" because it "lets the WP site glow through behind every chrome surface." At idle the chrome is solid; glass only applies to the overlay. Either commit (make the topbar glass) or update the README to match reality.
- Mobile (line 1180–1227): the Control button is icon-only ("layout-grid" icon) — the label "Control" goes away. So discoverability drops further on mobile, where the gesture matters most.

## How to push harder (1–2 pointed suggestions)
1. **Collapse Save to the slider tile only — remove the saved-state pill from the chrome.** The Control overlay is intentionally summoned, so the slider only appears on demand — and the chrome becomes 7 affordances at idle (one tighter). Then add a single subtle dot to the address bar that turns green when saved (status only, not action). The pill's mixed action/status nature is the root of the violation.
2. **Replace the row-4 Reset tile with a contextual one inside the expanded Settings sheet.** Reset is destructive and contextual to the *current* settings — having it as a top-level tile is dangerous and duplicate. The Control grid should hold *navigational* tiles only; destructive actions belong inside the expanded sheet they apply to.

## Lesson for the next designer
If your direction allows a primary verb to appear in multiple forms (pill, slider, shortcut), don't ship all three — pick the one that *embodies* the direction. iOS doesn't put a "Save" button in the chrome and on the slider both; it commits to the slider because the gesture is the metaphor.
