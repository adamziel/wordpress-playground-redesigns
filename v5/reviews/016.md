# Review of v5 #016 — Museum frame

**Parent direction:** "Museum frame. The iframe sits inside a thin gilded chrome (5px border, real micro-typography of 'WORDPRESS PLAYGROUND' at the bottom of the frame). All action via top menu."
**Affordance count at idle:** 8 (enumerated in README): playground name engraving · address bar engraving · refresh · save-state pill · fullscreen toggle · overflow menu · placard plate · sites dock badge. At cap exactly.

**Scores (out of 10):**
- Feature completeness: 9 — 8 states (I working, II booting, III launcher, IV settings + Apply&Reset confirm, V Files, VI Blueprint + Database + Logs triptych, VII fullscreen, VIII mobile 3-view). Composite frames pack DB + Logs into state VI for budget. README explicitly mentions WP Admin + Homepage live in Settings tab (the critic's lesson from #002 referenced!).
- Restraint: 9 — exactly 8 at idle, defended. Brass micro-engraving is decorative not interactive (correctly disclaimed).
- Iframe primacy: 10 — 5px gilt border around the iframe; the canvas (iframe) gets all visual emphasis; fullscreen pill labeled "View artwork only" + F + Esc.
- One-door: 9 — placard is *the* single door; WP Admin/Homepage live in Settings (no duplicate); save-state pill carries the save-info-notice text in one place. The "more-vertical" overflow button at line 1219 is the second door for Additional actions — but README defends placard as the canonical home; this is a minor seam.
- One-secondary: 10 — literally the placard sheet, six tabs swap inside (Settings/Files/Blueprint/DB/Logs/Launcher). No second sidebar.
- Direction adherence: 10 — gilt frame, brass strip with engraved typography, scrolling micro-engraving ("WORDPRESS PLAYGROUND · EST · 2022 · WEBASSEMBLY · PHP IN THE BROWSER · NO SERVER REQUIRED" at line 1262), placard at the bottom, Cormorant Garamond typography per README. The metaphor is unmistakable and committed.
- Visual fidelity: 10 — among the highest in v5 so far. Real "Roast & Bean" coffee site iframe content, brass-gradient styled elements (frame corners, brass strip, placard), engraved typography rendering, placard with WP version + PHP version + plugin count + save timestamp as engraved sub-text. The "Save → Saving" pill state with `<i data-lucide="loader">` animation hint (line 1294–1296).
- Synthesis with v1 lesson: 9 — extends #14 (Ventura Settings clean category sidebar = one placard with 6 tabs) and #15 (Win11 fullscreen toggle = "View artwork only"). Cites #002's critique lesson explicitly: "I moved WP Admin and Homepage into the Settings tab as primary buttons (the critic's lesson from #002), so the Site Manager header actions don't become a hidden duplicate door." Direct citation of a v5 review — best synthesis evidence in v5 so far.

## What works (specific)
- **"Recast this work"** as the rename of "Apply & Reset" (README) is a beautiful, museum-aware verb that *reduces* the danger by changing the vocabulary. The user must consciously translate "Recast" → "Reset," which is its own friction layer. Combined with a typed-phrase confirm sheet, this is the most thoughtful destructive-flow in v5.
- The micro-typography scroll ("WORDPRESS PLAYGROUND · EST · 2022 · WEBASSEMBLY · PHP IN THE BROWSER · NO SERVER REQUIRED" at line 1262) carries the *brand and the explanation* in a decoration band — solving the "marketing page inside the iframe shouts at returning users" pain point by moving the brand whisper to the frame.
- The placard at idle shows engraved metadata: "WordPress 6.7 · PHP 8.3 · SQLite · 14 plugins · Saved 2 min ago" (line 1253). This is *status* on the placard, not buttons — the user sees version + storage + plugin count at a glance.
- "Open ⌃" affordance on the placard (line 1259) is the only invitation to interact. Subtle but present.
- The README explicitly cites the v5 #002 critic lesson and applies it ("WP Admin/Homepage live in Settings, not in menu, the critic's lesson from #002"). This is the *first* v5 design that demonstrably learned from a prior review — exactly what v5 is supposed to do.
- Composite state VI (Blueprint + Database + Logs as three smaller frames in a row, line 1881) — efficient state packaging that still gives each tab visible content.
- Save-pill states: "Save" → "Saving" with loader icon → "Saved" via color class change. Three distinct visual states demonstrated across states I and II.

## What's weak or missing (specific)
- **The "more-vertical" overflow button** (line 1219) on the brass strip's right side is a second door alongside the placard. README claims the placard is the only door, but state I clearly shows an overflow button. What's inside it? Either it's empty (cut it) or it duplicates the placard's tabs (one-door violation). README needs to spell out what overflow holds.
- **Placard discoverability rests on one hover.** README admits "the brass micro-typography glows, the plate animates a 2px lift" and "a one-line hint that fades after first interaction." This is the same first-touch problem as #011 and #014 — and the museum aesthetic actively *hides* affordances by being subtle. A returning user three months later may not remember the placard is interactive.
- The micro-typography is engraved decoration but it *runs across the placard's bottom*. New users may try to click it; it's *not* interactive. This is the same trap #010's version chip has — visually peer to other interactive elements but inert.
- **The aesthetic is unapologetically formal** (README's word). Cormorant Garamond + brass gradients + ivory wall is a strong opinion that may not align with WordPress's developer brand. The design is gorgeous but feels like an art-gallery app, not a dev sandbox. This is the *direction's fault*, not the designer's, but worth noting.
- Mobile uses a "full-height drawer rather than a 45% sheet because phone heights don't tolerate a smaller placard." That's a real but small divergence from the desktop's 45% sheet — README rates 4/5 honestly.
- The fullscreen pill is *labeled* "View artwork only" with the F shortcut — clear and on-brand. But the *overflow more-vertical* button is unlabeled. Either label it or remove it.

## How to push harder (1–2 pointed suggestions)
1. **Cut the brass-strip overflow button (more-vertical at line 1219).** If the placard is the canonical door, the brass strip should have zero menu doors — just refresh, URL, name, save, fullscreen. The overflow is a second door undermining the README's claim. If you need a "rarely-used commands" surface, it belongs *inside* the placard's Settings tab as a sub-section, not on the brass strip.
2. **Add a subtle "click to open" tooltip on first session only** — under the placard, in 9 px gold sans-serif: "click the plate ⌃ to manage this site." Fades after first interaction (README mentions this; ensure it's actually rendered in state I, not just promised). Match the gallery-card-tooltip pattern museums use.

## Lesson for the next designer
A museum aesthetic forces *discoverability via decoration* — but every decorative element risks being mistaken for an interactive one. Be ruthless about labeling: anything *touchable* must have a hover state or tooltip; anything *engraved* should be in a typeface so dignified that no user would try to click it.
