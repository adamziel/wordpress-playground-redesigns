# Review of v5 #015 — Cmd+K is the UI

**Parent direction:** "Cmd+K is the UI. No visible chrome at all except a 'Press ⌘K' hint. Every action lives in the palette."
**Affordance count at idle:** 3 (counted in README state 1 row): saved-state cue · fullscreen toggle · `⌘ K` hint pill. Very low — tied with #006 and #014 for tightest restraint.

**Scores (out of 10):**
- Feature completeness: 10 — 13 states cover idle, palette root with 15 commands, /blueprints with 18 of 43 + 10 filter chips, /settings as a sentence editor (referencing direction #50), /confirm typed-phrase, /files with tree + functions.php, /blueprint with real Coffee Shop JSON, /db with 12 SQLite tables + early-access notice, /logs grouped-by-request with stack trace, /new with 6 sources + featured + saved, booting (amber pill only), fullscreen (zero chrome), three mobile states. Most thorough state coverage in v5 so far.
- Restraint: 10 — 3 affordances at idle, named cost. The README is honest about "pushes restraint past comfort."
- Iframe primacy: 10 — at idle iframe is full bleed minus 3 small pills. Fullscreen state has zero chrome. Nothing reflows.
- One-door: 8 — settings/files/blueprint/db/logs each have one canonical mode home. README admits "/new and /blueprints both surface blueprints, which is a soft overlap" — but /new is the *launcher* (sources + featured) while /blueprints is the *gallery* (all 43 + filters). Different surfaces, related content. Soft seam but defended.
- One-secondary: 10 — one palette container; only the body swaps. Same React component for every mode.
- Direction adherence: 10 — "the direction IS the design — no chrome, palette is everything." Slash-modes carry the metaphor (CLI sub-shells, `⌫` to go back). No tabs.
- Visual fidelity: 9 — Inter font (presumably), Lucide icons, real PHP fatal stack trace in /logs, real Coffee Shop blueprint JSON in /blueprint, 12 SQLite tables in /db, 18 of 43 blueprints in /blueprints. The sentence editor for /settings ("This playground runs **WordPress 6.7** on **PHP 8.3**…") is a direct nod to direction #50.
- Synthesis with v1 lesson: 10 — the README explicitly addresses v1 #15's "needs a good way to handle the full-sized Playground" gap and closes it; references the brief's "small red banner is not enough" by escalating to typed-phrase confirm + auto-zip safety net.

## What works (specific)
- **Slash-modes as CLI sub-shells.** `/blueprints` → palette body becomes the gallery. `⌫` returns to root. This is a real metaphor, not just "we have tabs in our palette" — it's CLI thinking applied to UI navigation. The fact that going back costs a backspace is the *exact* echo a Vim user would recognize.
- **Bare shortcuts learnable through use**: `G A` for WP Admin, `G H` for Homepage, `⌘ S` for save, `F` for fullscreen. README states "new users *learn* the shortcuts by reading them next to each row" — this is exactly how Slack and Linear train muscle memory.
- **/settings as a sentence editor** ("This playground runs **WordPress 6.7**…") is the cleverest implementation of the v5 §50 direction. Settings stop being forms; they become English with inline-editable fragments.
- **Apply & Reset confirm with diff + typed phrase + auto-zip safety net** in state 5 is the strongest destructive flow in v5 so far.
- **Booting state (state 11) keeps only the amber state pill** — context-aware chrome reduction, not blanket dissolve.
- README is honest about the cost: "this design is honest that it pushes restraint past comfort. The user explicitly asked for ≤8 idle affordances; I delivered 3 and named the cost." Pre-emptive acknowledgment of the discoverability tax.
- **The pill is tappable on mobile** — so even without ⌘K on a soft keyboard, the user has a real button. The palette docks to bottom 78% on phones.

## What's weak or missing (specific)
- **Discoverability tax is real and only mitigated by *one pill*.** A first-touch visitor sees three pills (saved-state, fullscreen, ⌘K hint). Of these only the ⌘K hint suggests "there's more here." Compare to #014's single-word-pill which embeds the hint inline. Here, three pills compete: a returning user might tap the saved-state pill expecting save, but README says "the state pill clicking focuses palette to save row but does *not* duplicate save." That's still a *click* that produces an unexpected output (opening the palette).
- **No always-visible refresh button.** README admits this is "the biggest leap of faith" for first-time visitors. Refresh is `⌘ R` (which is browser-refresh by default — the design must intercept it or use a different binding). This conflicts with browser conventions in a problematic way.
- **Mobile keyboard burden.** Phones don't have ⌘. The README says "slash-commands still work because the input field is the same — the slash key is on every soft keyboard." True, but mobile users *do not type slashes naturally*; they tap buttons. The palette UI must surface slash-modes as *tappable chips* at the top of the body for mobile parity.
- The "soft overlap" between /new and /blueprints is real — both surface blueprints. A user looking for the Coffee Shop blueprint might go to /new, see it as featured, then realize they wanted to filter blueprints, and have to switch to /blueprints. README defends this as launcher vs gallery — but a single /sites mode with sub-views (Sources / Featured / All 43 / Saved) would be tighter.
- **Three pills at idle is the absolute minimum** but the *placement* matters: the README says "mid-bottom (eye-line for chrome)" — fine for the ⌘K hint, but what about the saved-state pill and fullscreen pill? Are all three mid-bottom? Or are they spread to corners?
- **Cmd+R conflict** with browser refresh is unaddressed. The user *expects* Cmd+R to refresh — if the design intercepts it for "refresh iframe", browser-level refresh becomes unreachable without keyboard shortcuts the user doesn't know. Either accept browser-refresh (and lose the in-app shortcut) or make `R` (no Cmd) the in-app shortcut and document loudly.

## How to push harder (1–2 pointed suggestions)
1. **Surface slash-modes as a tappable chip strip *inside the palette body header*** — `[/blueprints] [/files] [/blueprint] [/db] [/logs] [/settings] [/new]` chips below the input. This gives mobile users the same fluency as desktop slash-typers, and gives new desktop users a visual menu they didn't have to discover. The CLI metaphor survives because chips reflect *active* state and typing still works.
2. **Merge /new and /blueprints into a single /sites mode with a 3-segment header** (Sources / Featured / All 43). Eliminates the soft overlap; gives users a single root for "I want a new site," and the segmented header is one swap, not a slash-command change.

## Lesson for the next designer
If your direction is "everything in the palette," surface the slash-mode names as a *tappable chip strip* inside the palette — mobile users can't type slashes naturally, and new desktop users need a visual menu before they trust the keyboard. CLI restraint doesn't justify keyboard-only mobile.
