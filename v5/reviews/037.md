# Review of v5 #037 — HUD glass

**Parent direction:** "HUD glass. All chrome is glass that materializes on mouse-move and dissolves 2s after idle. The iframe is the only solid thing."
**Affordance count at idle:** 1 (corner pulse). At revealed: 8 (4 top pill + 1 launcher + 1 Site Manager + 2 dock keys). Tied with #014/#015/#026 for tightest true-idle restraint.

**Scores (out of 10):**
- Feature completeness: 10 — 12+ states including idle (1), materialize teaching layer (2), revealed (3), all 5 SM tabs as separate frames (5–9), booting with HUD locked open (10), Launcher pane, absolute fullscreen (12). Typed-phrase Apply&Reset with auto-zip recovery.
- Restraint: 10 — *one* affordance at true idle (the corner pulse). The persistent pulse doubles as the saved-state indicator — zero-cost status.
- Iframe primacy: 10 — "the iframe is the ONLY solid thing in the entire IA — even chrome's glass shows through to it." Absolute fullscreen with F dissolves chrome AND pulse; Esc exits.
- One-door: 9 — Settings/Save/Launcher/Site Manager each have one home. README admits "-1 for Export/.zip living in both SM header AND kebab menu — intentional duplicate (contextual vs global)" — flagged in trade-offs honestly.
- One-secondary: 10 — single `.pane` component, exclusive-open (opening Launcher closes Site Manager pane and vice versa). README explicitly cites review #022's lesson.
- Direction adherence: 10 — materialize animation (blur 0→22 px, scale .96→1) on mouse-move, dissolve (blur 0→8 px, opacity 1→0) 2s after idle. Glass blur values declared. The iframe being the *only* solid thing is enforced literally.
- Visual fidelity: 9 — content-aware tint as a *property of the medium* (iframe colors bleed up through capsules), four floating glass capsules at natural cursor reading positions, animated heartbeat pulse, one-shot teaching layer.
- Synthesis with v1 lesson: 10 — extends #38 glass + #11 auto-hide HUD (closest cousin). **Cites *six* prior v5 reviews with specific implementing element**:
  - From review of #011 (closest cousin): "you cannot rely on a single hint pill" → explicit one-shot teaching layer (state 2) labels all four capsules for 1.5 s on first reveal, plus persistent corner pulse that never dissolves.
  - From review of #001: "make the expansion gesture a first-class visible element" → state 2 renders materialize animation mid-fade with edge labels.
  - From review of #003: "pick one form for the primary verb" → Save only in the name chip.
  - From review of #002: "a 'one surface' sheet is only one surface if you handle internal IA" → Launcher pane's three tabs are first-class top-of-pane tabs, not stacked scroll.
  - From review of #006: "show every Site Manager tab" → states 5–9 demonstrate Settings, Files, Blueprint, Database, Logs each.
  - From review of #022: "multi-surface exclusive by IA" → exclusive open Launcher/Site Manager.

## What works (specific)
- **Directly addressing #011's critic feedback**: "If your direction is auto-hide, you cannot rely on a single hint pill" → #037 builds *two* persistent hints (corner pulse + one-shot teaching layer that labels all four capsules at once). The review-#011 critique becomes the design's structural foundation.
- **Differentiation from #011 (the closest cousin sibling)**: README explicitly notes "Where #011 anchors chrome to the four edges, **037 floats it**: four detached glass capsules ... hover above the iframe at the cursor's natural reading positions." Best sibling-aware differentiation in v5 alongside #021 vs #004 and #031 vs #018.
- **Content-aware tint as a property of the medium**, not a separate feature — "a roast-brown coffee theme tints the chrome warmly and a dark-mode Gutenberg PR tints it cool." Compare to #010 (ambient rails sample iframe color explicitly) — here it's *automatic* because glass + iframe = tint.
- **HUD locks open during boot, URL focus, and pane-open** — context-aware dissolve. The 2-second timer respects when the user is actively working.
- **Persistent corner pulse outside chrome that never dissolves** — even in absolute fullscreen the pulse is hidden, but at any non-fullscreen state, the pulse is permanent.
- **Materialize animation has explicit values** (blur 0→22 px, scale .96→1 over the reveal). The dissolve is named with values too (blur 0→8, opacity 1→0). Engineering specificity.
- **Single `.pane` with exclusive-open** (review #022 lesson) — Launcher and Site Manager cannot both be open at once. Hard-IA, not social-pressure policy.

## What's weak or missing (specific)
- **Export/.zip lives in both SM header AND kebab menu** — README flags as "intentional duplicate (contextual vs global)" and self-deducts -1. The defense ("same code path; dual surface is for users who think contextually vs globally") is the same argument #033 and others use. Strict reading is duplicate door.
- **The 2-second dissolve timer** is *fixed*. Users who pause to read text mid-task watch the HUD melt — README mitigates with the URL-field-focus lock, but a returning user reading an SM panel's text content (not in a focus state) loses chrome mid-read.
- **Mobile hover-to-reveal becomes tap-only** (-1 honest). The materialize gesture's mouse-move trigger doesn't exist on touch; the IA is "documented but not fully equivalent."
- **22 px blur on panes is a touch heavy** — README admits could pull to 18 px. Minor calibration.
- The **4 detached glass capsules** mean at revealed state there are *four glass surfaces visible at once* (top pill, left cluster, right cap, bottom dock). README counts as one IA but it's four distinct geometries. Compare to #035's three quantified depth layers — both have multiple glass surfaces but #035 names them as a *system* (L0/L1/L2/L3) while #037 names them as *capsules* (with implicit hierarchy).
- The **left cluster doubles as both "+" launcher entry and saved-site avatars** with right-edge add-affordances. Each avatar is a deep-link into the Launcher pane with the matching site pre-selected. Defensible — but the cluster has *multiple* affordances visually compressed.

## How to push harder (1–2 pointed suggestions)
1. **Drop Export/.zip from the kebab menu** — keep it only in the SM header (contextual). The kebab loses the redundancy; the contextual placement is where users find it when they're managing a site. Recovers the -1 on one-door.
2. **Make the 2-second timer adapt to *cursor distance from chrome*** — if the cursor is *near* (within 200 px of) a capsule, extend the timer; if it's *far* (deep in the iframe), shorten it. This is one parameter change that solves the "I'm reading, don't dissolve" problem without breaking the metaphor.

## Lesson for the next designer
When your direction has a near-identical sibling (here: #011 auto-hide HUD), use the README's *first paragraph* to differentiate. "Where X does Y, mine does Z" with the specific structural difference clearly stated earns the critic's trust before they read further. #037 differentiates "edge-anchored vs floating capsules at cursor reading positions" in one sentence.
