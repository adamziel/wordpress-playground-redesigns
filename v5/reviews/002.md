# Review of v5 #002 — Single drag-sheet, three detents

**Parent direction:** "Single drag-sheet, three detents. One bottom sheet. Drag to peek (40px), half (45%), or full. Every panel (Settings, Files, Blueprint, DB, Logs, Launcher) is a tab inside that sheet."
**Affordance count at idle:** 7 (counted onscreen by the designer's overlay at line 749–760): Refresh · URL bar · Name pill · Fullscreen · More · Sheet handle · Peek-meta chip. Matches the brief's ≤8 cap.

**Scores (out of 10):**
- Feature completeness: 10 — 18 visible blueprint cards, all 10 category chips (with counts), six start-sources as distinct tiles, four saved playgrounds, real WP-table SQLite Database view with 12 tables, PHP debug.log with a real fatal stack trace, blueprint JSON with valid schema URL, all five Site Manager tabs *plus* Sites as the 6th, Export GitHub PR + Download .zip + Import .zip in the More menu (line 1135–1138).
- Restraint: 9 — chrome is 44px top bar + sheet handle only; everything else lives inside the sheet.
- Iframe primacy: 9 — iframe is always the page; sheet at peek is 56 px; fullscreen state B explicitly shown with FS hint, F/Esc shortcuts.
- One-door: 10 — settings is only sheet tab, launcher is only sheet tab, files is only sheet tab. Old triple-entry is genuinely collapsed.
- One-secondary: 10 — literally one sheet; the State D 2×2 grid is a *teaching* layout (four mini-frames) showing the same sheet swapping content.
- Direction adherence: 10 — the direction *is* the design.
- Visual fidelity: 10 — real Inter + JetBrains Mono, color-coded save-state pill (amber/saving/saved), Lucide icons, syntax-highlighted CSS/JSON, real PHP fatal stack trace lines, color-graded saved-playground avatars (CS/GN/AG/NP), realistic SQLite row data.
- Synthesis with v1 lesson: 10 — explicitly inherits #06 mobile-first ("desktop is literally a wider phone"), one-secondary from #55, save-state pill from #64, and closes the #15 fullscreen gap.

## What works (specific)
- Save-state pill modes (line 178–183) cycle amber/unsaved → green-pulse/saving → green/saved with the *name* embedded. The pill *is* the saved-state primary visual cue — no info notice needed.
- Apply & Reset confirm sheet at line 839–858 shows a real *diff* (`+ wordpress_version: 6.7 / - 6.6 / + php_version: 8.3 / - 8.2`) plus a typed-phrase verification with a Send button that stays disabled until the name matches. Far better than the small red banner the brief calls out.
- The peek-meta chip (line 1087, "Sites · 3") tells the user *what tab is loaded* even when the sheet is closed — solves the discoverability complaint that 002's drag interaction would otherwise have.
- Mobile sheet at full (line 1407+) collapses the tab labels to icons only — clean implementation of "mobile is wider mobile."
- Working draggable JS at line 1773–1791 actually cycles detents on click and respects drag direction. Most v5 entries will hand-wave this.
- The "State D" 2×2 grid is a thoughtful teaching device: four mini-shells side by side showing Files, Blueprint, DB, Logs in the same surface. Hard requirement that every tab has visible content — met thoroughly.

## What's weak or missing (specific)
- The "More" menu (line 1133–1145) holds Homepage and WP Admin alongside Export/Download — but those two are Site-Manager-header actions in the v1 brief. Right now they're in two places (More menu + presumably also reachable via the Sites/Settings tabs), which risks the one-door violation.
- The State C blueprint gallery has **18 cards visible** out of 43 — meets the ≥16 requirement, but the gallery container is inside the *Sites* tab whose first section is also six start-source tiles and *also* a "Your playgrounds" list. The Sites tab is doing the work of three discrete sections. This is correct per the design's logic, but the tab is dense (over 950 px tall on desktop) — easy to miss the gallery below the fold without scrolling.
- The toggle row labels in Settings (lines 818–830) are short — "Network access" / "Multisite" — but the same field block is missing dedicated *WP Admin* / *Homepage* jump buttons (they live only in the overflow More menu).
- The mobile Sheet at FULL (state C) collapses tab labels to icons (good), but only shows 6 blueprint thumbs (line 1455–1462) — the desktop showed 18; the brief said ≥16. The phone's gallery is thin. Could be triaged better with the same 18 in a 2-column grid.
- Glass is *heavy*: sheet body, sheet header, top bar, every mini-frame in State D — all backdrop-blur. The README claims "two glass surfaces" but the count is higher. Defensible (each layer serves depth hierarchy), but it pushes principle #6 ("glass earns its keep").

## How to push harder (1–2 pointed suggestions)
1. **Restructure the Sites tab into peek-level vertical scrolling with sticky sub-tabs.** Right now Sources / Your playgrounds / Gallery are stacked sections; at half detent the user only sees Sources. Add a 3-segment sub-control ("Start · Yours · Gallery") at the top of the Sites tab so the gallery is reachable in one click at half detent — otherwise users must drag to full to even see the 43 blueprints.
2. **Move WP Admin / Homepage into the Settings tab as primary buttons** (next to the name field), not the More menu. They're contextual to *this* playground, not global housekeeping like Import/Export. Doing so eliminates the duplicate-door risk and frees the More menu to be a true global commands list.

## Lesson for the next designer
A "one surface" sheet is only one *surface* — but if you stack three different sections inside its main tab, you've shipped three drawers in disguise. Make the tab's internal scroll position a first-class part of the IA (sticky sub-tabs or detent-tied sections), not a tall stack.
