# Review of v5 #013 — One Floating Capsule

**Parent direction:** "One floating capsule. A single horizontal capsule near the top contains: URL field, name pill, save, menu. Everything else is summoned from menu."
**Affordance count at idle:** 5 (state A header at line 923): Refresh · URL field · Go button · Name pill (saved-state + Save) · Menu kebab. Mobile drops one. Below cap.

**Scores (out of 10):**
- Feature completeness: 9 — 11 states (A–L) cover idle, menu open, Site Manager Settings, Files, Blueprint, Database + Logs combined, Launcher, Blueprint Gallery, Booting + Fullscreen side-by-side, Apply&Reset typed confirm, and 3 mobile views. Database + Logs combined into one state (F) for budget — slight efficiency over dedicated states but acceptable.
- Restraint: 9 — 5 affordances at idle is well under cap. Menu dropdown carries everything else but is sectioned (This playground / Sites & blueprints / Export / App).
- Iframe primacy: 10 — capsule has 720 px max width on 1440 viewport, leaving the iframe genuinely dominant. State I demonstrates fullscreen with F/Esc.
- One-door: 8 — settings unified in Settings tab; save state unified in name pill (status + action combined). README admits WP Admin/Homepage live in Stage header *and* menu (keyboard shortcuts A/H) — but it's the same destination; one-door rule holds if you interpret shortcut + button as one canonical door with two paths. Borderline.
- One-secondary: 10 — single Stage element; segmented control swaps Launcher / Site-Manager / Blueprints; SM sub-tabs are a *vertical rail* inside the same Stage (line 1102) rather than another nested tab strip — clever.
- Direction adherence: 10 — "the capsule is literal — one rounded pill is the *entire* persistent chrome; no compromises like a status bar or peek dock." README's resistance to importing other v1 winners ("no Figma spatial dock, no DevTools status row, no Stage Manager thumbnails") is principled.
- Visual fidelity: 9 — Inter font, real Lucide icons, lockable URL with Go button (line 961), color-coded save-state dot (green/blue/amber), real iframe content (Roast & Bean coffee), Stage sheets are solid white per README (compliance with v5 §2 rule 6), capsule is the only frosted surface.
- Synthesis with v1 lesson: 9 — extends #64 DevTools (slim, browser-style top bar) and #38 glass (capsule only; rest solid). The "capsule" geometry is a refinement of DevTools' "iframe is the document" by going further: the chrome is even thinner.

## What works (specific)
- **The capsule has a Go button** (line 961) — submit-on-Enter is implicit, but a visible Go button confirms the URL is editable and submittable. Some other v5 entries hide this affordance.
- **SM sub-tabs as a vertical rail inside the Stage** (line 1102) instead of a horizontal tab strip is a structural choice that avoids the tabs-inside-tabs problem #005/#008 had. The Settings/Files/Blueprint/DB/Logs become *side-rail entries*, not nested tabs.
- **Database + Logs combined into one state** (state F) is a budget trade-off but the README defends it: both panels are tabular/log-like, and showing them side-by-side proves they're both *content-swappable* in the Stage.
- README explicitly resists v1-winner-borrowing: "Resisting the temptation to import other v1 winners (no Figma spatial dock, no DevTools status row, no Stage Manager thumbnails) — the direction is 'one capsule,' and one capsule is what shipped." This restraint is itself the v5 brief's #2 principle internalized.
- Save state is *only* in the name pill, with green/blue/amber colors. No separate Save button anywhere — including no kebab "Save" item. The pill *is* the save door. Cleanest save-state implementation in v5.
- The kebab menu is sectioned (This playground / Sites & blueprints / Export / App) — same pattern as #009, but here it's not a "junk drawer" because the capsule has only 4 other affordances; the menu *legitimately* needs to carry more weight.
- Apply & Reset is typed-phrase confirm + *auto-snapshot* (README: "irreversible-but-recoverable") — matches the brief's "destructive ≠ small red banner" demand.

## What's weak or missing (specific)
- **The kebab is the *only* door to most features.** Open Site Manager, Quick settings, Launcher, Blueprints gallery — all behind the same kebab. For experienced users this is fine; for new ones the kebab becomes a 10-item menu they have to scan. Compare to #007 where 6 spokes have fixed positions with muscle memory; here the kebab is just a vertical list.
- **No visible fullscreen icon.** The state I shows fullscreen with F/Esc, but the capsule itself has no fullscreen toggle button (line 953–970). Fullscreen is *only* keyboard-driven (F) or accessible via the kebab. v5 §2 rule 1 says "explicit *fullscreen iframe* mode that hides chrome with a one-key/one-click toggle" — keyboard-only fails the "one-click" half. Add a fullscreen icon to the capsule (would bring count to 6 — still well under cap).
- **WP Admin / Homepage live in Stage header *and* menu** is a one-door violation by strict reading. README rates 4/5 honestly. Pick: either keep them only in Stage header (the SM view), or only in the kebab. Two doors is two doors.
- **Mobile drops the name pill into the Stage header** at 390 px (README admits). That means on mobile the *idle save state is invisible* — a returning mobile user can't tell at a glance if their work is saved. Some persistent save-state badge (corner dot like #014) would help.
- The "Go" button (line 961) at the right of the URL field is a slightly redundant affordance since Enter is the default. Cutting it would reduce visual clutter and free the capsule to focus on the URL input.
- The capsule glass *only on the capsule and the menu dropdown* is consistent with v5 §2 rule 6. But the Stage sheets are solid white — and the brief never required all chrome to be glass, so that's a valid restraint choice. Worth noting that two glass surfaces is fewer than #005's three-glass target.

## How to push harder (1–2 pointed suggestions)
1. **Add a fullscreen pill to the capsule itself** — a 6th affordance — so the brief's "one-click toggle" is met. The capsule has room (it's 720 px wide); a single `maximize-2` icon between the name pill and the kebab would be unmissable. Idle count goes from 5 → 6, well under cap.
2. **Sort the kebab menu by *frequency*, not by category.** Right now it's sectioned (This playground / Sites & blueprints / Export / App). For a junk-drawer-style kebab, frequency-ordered is more useful: Site Manager > Launcher > Quick settings > Blueprints > Export > Download > rare commands. The category split is correct in spirit but at 10+ items, top-of-list muscle memory is more valuable than logical grouping.

## Lesson for the next designer
"One floating capsule" requires you to *visibly* expose the fullscreen toggle on the capsule — keyboard-only F doesn't satisfy the brief's "one-key/one-click" pair. Don't trade visible affordances for a tighter capsule width; spend the pixels on the one feature that matters most for iframe primacy.
