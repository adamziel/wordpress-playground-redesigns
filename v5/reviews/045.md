# Review of v5 #045 — Spatial × Restraint

**Parent direction:** "Spatial × Restraint. Take Figma's spatial-memory but show ≤ 6 affordances at idle. Pan/zoom is the only navigation. Sites are positioned in 2D space."
**Affordance count at idle:** 6 — omnibox · status capsule · keyboard hint · + new · minimap · zoom widget. **Direction's ≤6 cap met exactly.**

**Scores (out of 10):**
- Feature completeness: 10 — full canvas with positioned saved playgrounds, Blueprint Sky region (43 cards + 10 chips + 6 sources reachable by panning), single secondary pane with 5 SM tabs, frame chrome scoped to focused frame at zoom ≥80%, minimap, zoom widget, typed-phrase Apply&Reset, Save state via name-pill, fullscreen with Esc pill.
- Restraint: 10 — 6 idle affordances exactly (direction-specific cap). Frame chrome is contextual to focus + zoom, not counted in idle budget.
- Iframe primacy: 10 — iframe is the focused frame's content; chrome scales with attention. Panel H demonstrates F + Esc + single pill fullscreen.
- One-door: 10 — sources only in Sky rail; settings only in pane tab; export only in kebab. Best one-door discipline in the direction's spatial premise.
- One-secondary: 10 — one pull-tab pane, 5 tabs swap content. No second sidebar, no second drawer.
- Direction adherence: 10 — pan/zoom is the *only* navigation; library is a *place*, not a drawer; six affordances enforced. The direction's ≤6 cap is the *tighter* cap than v5 §2's ≤8 — and #045 meets it.
- Visual fidelity: 9 — labeled minimap (`CANVAS · N SITES`), zoom widget showing %, omnibox, 5-layer fixed z-stack documented in Panel N, three glass surfaces named (omnibox / secondary pane / minimap).
- Synthesis with v1 lesson: 10 — extends #10 Figma-canvas with the brief's specific ≤6 cap. **Cites *six* prior v5 reviews with the specific implementing element**:
  - #010 → minimap labeled, zoom widget shows %, status capsule labels its dot.
  - #014 → each of 6 idle pills points to a distinct destination; sources have one door.
  - #018 → cited above and in Panel O.
  - #023 → keyboard hint pill + labeled minimap + visible zoom % teach pan/zoom on first use.
  - #029 → pinch-zoom is native, not invented; mobile sheet uses iOS 3-detent standard.
  - #035 → 5-layer fixed z-stack documented; Library never rises above the pane — you *pan* to it.
- **Plus sibling-aware differentiation from v1 #10 (parent), v5 #004/#021 (Stage Manager / v2), v5 #044 (Epic launcher × Recede)** — README itemizes how this design differs from each spatial-flavored sibling. Best multi-sibling-awareness in v5 so far.

## What works (specific)
- **The Blueprint Sky is the design's most novel idea**: zoom out past 25% and the canvas reveals a second region where 43 blueprints float as cards you drag onto empty space to instantiate. The library *is* a place, not a drawer. This is the v1 #10 spatial metaphor pushed to its logical end.
- **Six-affordance cap met exactly** — the direction-specific tighter cap (≤6) is honored, not just the v5 default ≤8. The README quantifies this in the affordance count section.
- **Three teaching layers for the implicit pan/zoom gesture** (per #023 lesson): keyboard hint pill (dismissible), labeled minimap, always-visible zoom widget. The implicit interaction has *three* counter-cues.
- **Frame chrome scaled with attention**: refresh, URL, name pill, kebab, maximize are visible *only on the focused frame at zoom ≥ 80%*. Chrome appears when you commit to a frame, disappears when you pan away. Best context-aware chrome in v5.
- **Library never rises above the pane — you pan to it** (per #035 lesson). The z-stack is constant; navigation is *spatial*, not depth-based.
- **Sibling-awareness with v1 #10, v5 #004/#021, v5 #044**: README explicitly differentiates from each. This is the most-detailed sibling-awareness in v5 (alongside #021 and #031).
- **One bit of persistent state per saved site (x/y coordinates)** — the only storage shape change, deliberately scoped (matches the brief's principle 9: "Spatial memory is a feature… No new persistent storage shapes beyond x/y").
- **No "instant launcher" key but ⌘K opens omnibox with search + boot-PR + URL nav** — the impatient power-user case is covered.

## What's weak or missing (specific)
- **First-touch discoverability is the design's biggest risk**, as with every spatial UI. README addresses with three teaching layers but a brand-new user may still not realize the canvas extends beyond the viewport. The teaching layers help only after the user starts to interact.
- **Pan/zoom on touch is pinch-zoom + drag** (native gestures, per #029 lesson) — but on a mobile device, pinch-zoom typically zooms the *browser viewport*, not the in-app canvas. The design needs to intercept pinch with `touch-action: none` and re-implement zoom semantics. Engineering challenge, not addressed in README.
- **The secondary pane "could be solid"** (README -1 on glass) — the see-through-to-iframe is a feedback loop but losing it would simplify.
- **Mobile sheet for source rail is the mobile concession** (README -1) — same kind of concession as other spatial designs.
- **Sites positioned in 2D space** means a user's "spatial memory" is *literally their layout choice* — but if a user never moves a frame, all frames sit at default positions and the spatial memory advantage is moot. The design assumes users will arrange.
- **The "+ new" affordance** opens... what? A new playground at the cursor position? The README implies it but doesn't fully describe the gesture. Compare to #008's "+" tab which clearly opens the New Tab page.

## How to push harder (1–2 pointed suggestions)
1. **Auto-arrange new playgrounds in a grid initially.** When a user creates their first playground, it appears at (0,0); the second at (480,0); the third at (960,0); the fourth at (0,400). After the user moves *any* frame manually, auto-arrangement disables. This gives non-arranging users a default spatial layout that still teaches the canvas's extent.
2. **Add a "zoom to fit" gesture (double-tap empty canvas) that frames all saved sites + the Blueprint Sky in one view.** This teaches the canvas's extent *immediately* on second tap, dramatically improving first-touch discoverability without spending an affordance.

## Lesson for the next designer
When your direction has a tighter cap than v5's default (≤6 vs ≤8), *meet the tighter cap exactly* and call it out in the affordance count section. Don't hide behind v5's looser default — the direction's discipline is the whole point.
