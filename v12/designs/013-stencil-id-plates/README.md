# 013 — Stencil ID Plates

## Thesis

A military shipping yard at night. The Playground browser is the manifest: each saved site is a steel crate plate with a single stencil-cut title, and the only metadata that survives is the stock-number compound stenciled into the corner. Industrial typography, no decoration, the iframe is the inside-of-the-crate. The dark is oil-stained tarmac; the only chromatic relief is a signal orange used once per surface.

## List pattern

Stencil ID plate. Each saved playground appears as a flat dark plate, the title set in Saira Stencil One (`COFFEE SHOP`) centred or left-aligned across the plate's face, with a small stencil "stock number" tucked at the corner (`WP67/PHP82/14M · 22V`). The stock number is a single terse compound — **never** a row of separate meta. No icon, no second line, no card-within-card. Start routes follow the same dialect — the plate face reads `VANILLA WORDPRESS`, the corner reads `WP67/PHP82`. The library overlay extends the yard with 16+ crate plates labeled by blueprint name.

## Focus test

The iframe is the loudest object because the chrome is matte-dark plate steel. The rail is six stencil marks total (one logo + five docks). The pane is a column of crate plates without inner borders. The library overlay is the yard at full scope. Once the user sees one plate they understand all plates. The signal orange appears exactly twice on the desktop frame — saving pill, active plate notch — and once on mobile.

## Trade-offs

The stock-number compound is dense — `WP67/PHP82/14M·22V` is information-rich but reads slowly. We trade legibility for compression. Hover reveals an expanded version. The Saira Stencil One title face is intentionally narrow; on small screens the plate face is more legible than a typical card row, but at very long site titles we let the title scale down rather than wrap.
