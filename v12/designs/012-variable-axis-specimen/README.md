# 012 — Variable-Axis Specimen

## Thesis

A type-specimen book translated into a working tool. Every visible entry uses one typeface — Recursive Sans, the only typeface variable along weight, casual, slant, and mono axes. We treat those axes as the data carriers. Where v11 designs added meta rows beside titles, this design pushes the meta INTO the titles. A heavier title means a larger Playground. A more casual title means the current session. A slanted title is actively running. There is no second line.

## List pattern

Variable-axis specimen. Each saved site is its title at a unique (wght, CASL, slnt) coordinate. `Coffee Shop` at wght 800 casl 1.0 slnt -8 = large, currently active, casual register because it's the one you're inside. `Vueo theme dev` at wght 550 casl 0.4 slnt 0 = medium, quiet. `Gaming News` at wght 350 casl 0 slnt 0 = small, dormant. `Unsaved Playground` at wght 250 casl 0 slnt 0 = the lightest, dustiest entry. Start routes use the same dialect: each route's typographic weight encodes the perceived "newness" of the path (Vanilla heaviest, Import .zip lightest). The library overlay continues — 16+ blueprint titles set at axis points that encode their popularity and category.

## Focus test

The pane is a specimen sheet — one column, generous leading, no chrome between entries. The reader's eye reads the WEIGHTS as a small bar-chart, the SLANTS as a status indicator. The iframe stays loudest because the pane intentionally has zero borders and zero icons — it is pure type.

## Trade-offs

The metaphor demands that the user learns to read weight-as-size. First-time users may not realize that the heaviest title means anything other than "important." A legend at the pane foot is provided. We resist adding labels under the titles — that would collapse this back into a two-row card.
