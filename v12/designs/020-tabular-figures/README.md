# v12 #020 — Tabular Figures Column

## Thesis
A treasurer's annual report rendered in dark mode. The whole design is a thesis about *tabular numerals* — when a proportional typeface like Source Sans 3 or Source Serif 4 turns on its `tnum` and `lnum` OpenType features (we set them at the root via `font-feature-settings` and individually via `font-variant-numeric: tabular-nums lining-nums`), the figures line up like spreadsheet columns even though the alphabetic letters keep their natural proportions. This solves the v11 round's failure cleanly: the meta-row that v11 needed in order to render WP/PHP/size/date can instead become *aligned columns of figures* on the same line as the title, and the alignment is enough — no second row needed, no icon needed.

## List pattern
Tabular figures column. Each entry is a CSS-grid row: `[title] [WP] [PHP] [size] [date]`. The title is Source Serif 4 500 at 16px, the figures are Source Sans 3 at 13px with tabular-lining numerals. The columns align across rows because the digits are tabular-width. The active row has its title shift to drafting-blue (#5C8AA6) and its hairline divider darken to a blue-soft tint. Start routes use the same grid template — `[Vanilla WordPress] [6.7] [8.2] [stable] [en_US]` — with the version columns degrading gracefully to em-dashes where they don't apply. The library overlay uses the same primitive at the page scale: a 5-column table with `[Blueprint] [WP] [PHP] [Size] [Steps]`, with a single header row at the top of the body (the design's only header-row, since the catalogue benefits from labels at scale; in the smaller pane, the saved-sites list omits the row-header to avoid label-redundancy — the figures speak for themselves).

## Focus-test answer
**Q: Which version of WordPress does Coffee Shop run?** A: Read across — the second column says 6.7. No hover, no expand, no second row. The vertical alignment of figures across rows means you can scan one column down to compare without re-reading each row's narrative.

## Trade-offs
Tabular figures require the typeface to *have* a true tabular figure set, which Source Sans 3 and Source Serif 4 both do — but if the user's browser fails to load Google Fonts, the system fallback won't preserve the alignment. We mitigate by setting `font-variant-numeric: tabular-nums lining-nums` everywhere figures appear so even an OS fallback that supports the feature will keep the alignment. The pane width must be ≥ 460px to comfortably hold five columns; we set it to 480px to leave breathing room. Accent rule respected: drafting-blue appears only in the saving pill, active title, active divider, active dock indicator, identity mark, and the "Apply & Reset" primary affordance. No nested borders: the ledger uses hairline dividers between rows, with no card border around the pane body.
