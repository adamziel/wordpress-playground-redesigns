# 014 — Hierarchy by Size

## Thesis

A typographic experiment that removes every datum except one: recency. The newest playground is set at 56px, the next at 36px, the next at 24px, and the oldest at 18px. Size is the only signal. There is no meta line. There is no icon. The reader's eye reads the list as a typographic mountain — the largest peak is what you opened last, and the foothills are what you have forgotten. The metaphor is the opposite of a sortable spreadsheet: it doesn't let you sort, it imposes a single editorial gradient.

## List pattern

Hierarchy by size only. Saved sites are stacked in one column. Each title's font-size is proportional to how recently you used the playground; the newest is the loudest line on the screen after the iframe itself. Start routes follow the same dialect — `Vanilla WordPress` is the largest route because it is the default, `Import .zip` the smallest because it is the least-trodden path. The library overlay continues the same typographic mountain across 16+ blueprint titles, where size encodes popularity rather than recency.

## Focus test

The iframe is the loudest object. The pane is a quiet column where the only chromatic accent is the saving pill. The rail is five DM Serif Display letters at different sizes — no icons, no borders, just letterforms at scale. Removing borders and icons reveals the design's whole bet: typography can carry data alone. If the user understands the rule once, they read the whole pane without conscious effort.

## Trade-offs

The visual gradient is steep — the smallest title is genuinely small (18px) and may be hard to click on touch. We accept that constraint and provide hover affordance to lift each entry into a more comfortable scale. The pane scrolls more than a flat-list pane would; that is the cost of the metaphor.
