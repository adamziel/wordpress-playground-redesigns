# Interactive Product Demo Shell

## Thesis

This prototype turns WordPress Playground into a guided product demo without making it a landing page. The iframe remains the main product surface, while a left step rail explains where the user is in the launch-to-running-to-export workflow.

## Skill Material Used

Applied UI Pro Max recommendations for Interactive Product Demo, step progression animations, touch-friendly controls, confirmation messages, and the Tech Startup typography pairing. The design uses clear demo states, a visible progress path, large controls, interface-colored highlights, and persistent feedback for saving, booting, exporting, and destructive reset.

## Flow Model

The step rail is the canonical door to workflow phases: Launch, Running Site, Manager, Export, and Reset. Selecting a step swaps the single stage panel; it does not create multiple sidebars. This extends v7 by making the launcher/library/manager state visible as a guided path while retaining iframe primacy and a single slim top command row.

## Trade-offs

The guided shell is more approachable for first-time users than the IDE model, but repeated expert use may feel more directed than necessary. The explicit steps make mobile behavior easier because the panel can stack below the iframe while preserving the same order.
