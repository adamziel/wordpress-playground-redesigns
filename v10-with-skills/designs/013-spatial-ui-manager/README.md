# Spatial UI Manager

## Thesis

This prototype uses spatial UI as a memory aid: the WordPress iframe is the fixed canvas, while a single floating lens shifts between launcher, manager, data, and export views. The interface feels dimensional, but feature ownership remains conventional and predictable.

## Skill Material Used

Applied UI Pro Max recommendations for Spatial UI (VisionOS), Modern Dark Cinema, blur-purpose, contrast maintenance, focus states, and adaptive navigation. The frosted panels use clear borders and solid text surfaces so glass does not undermine readability. Depth, parallax, and scale are used as hierarchy only.

## Flow Model

The top command strip handles live iframe controls and fullscreen/focus. The floating lens is the only secondary surface and contains all start routes, saved playgrounds, blueprint gallery, settings, files, blueprint JSON, database, logs, exports, and reset confirmation. This keeps the v7 one-surface lesson while giving saved sites spatial position.

## Trade-offs

The spatial style is memorable and gives the product a polished future-facing feel, but frosted interfaces need careful contrast. This prototype compensates with dark text plates, larger labels, and a mobile layout that removes overlap by stacking the lens below the iframe.
