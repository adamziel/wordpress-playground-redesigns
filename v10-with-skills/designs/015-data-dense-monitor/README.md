# Data-Dense Monitor

## Thesis

This prototype frames WordPress Playground as an operations monitor for a live browser runtime. The iframe is still the largest surface, but the surrounding manager emphasizes tables, logs, counters, and explicit state so developers can scan what is happening without opening several drawers.

## Skill Material Used

Applied UI Pro Max recommendations for Real-Time Monitoring, Data-Dense Dashboard, Developer Tool / IDE, Developer Mono, accessible chart/data rules, and touch/feedback guidelines. The design uses tabular figures, dense rows, persistent legends, high-contrast status labels, and clear keyboard focus.

## Flow Model

The top bar contains the only live iframe controls. The monitor panel is the single secondary surface with tabs for Overview, Launch, Settings, Files, Blueprint, Database, Logs, and Export/Reset. It extends v7 by making manager data glanceable while still avoiding a second persistent toolbar.

## Trade-offs

The density is powerful for debugging and export workflows but less relaxed than a first-run shell. Mobile parity is handled by stacking the monitor below the iframe and preserving all tabs, though scanning dense tables on a phone is inherently more demanding.
