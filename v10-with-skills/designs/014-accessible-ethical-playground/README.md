# Accessible Ethical Playground

## Thesis

This prototype optimizes WordPress Playground for clarity, keyboard use, and recoverability. It avoids decorative ambiguity: the iframe is first, every control has a visible label, and destructive changes require an explicit typed phrase with a nearby explanation.

## Skill Material Used

Applied UI Pro Max recommendations for Accessible & Ethical interfaces, focus states, keyboard navigation, skip links, touch target size, form labels, confirmation dialogs, and color-not-only status. The visual system uses high contrast, minimum 44px controls, persistent labels, aria-live feedback, and no reliance on hover-only interactions.

## Flow Model

The design keeps one command bar and one secondary surface. The secondary surface is a tablist with Launch, Settings, Files, Blueprint, Database, Logs, and Export/Reset. This differs from v7 mainly by treating accessibility rules as the visual language rather than as a checklist layered on top.

## Trade-offs

The interface is less atmospheric than the spatial or OLED versions, but it is more predictable and easier to audit. It uses more text labels, which increases visual density, yet that is deliberate because feature names, state, and recovery steps stay visible.
