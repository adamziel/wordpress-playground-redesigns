# Vercel Precision Runner

## Thesis
Vercel Precision Runner turns WordPress Playground into a product surface generated from a concise DESIGN.md-style system: named tokens, deliberate typography, consistent surfaces, and a small visible source annotation. The iframe remains the main object; launcher, saved Playgrounds, Site Manager tabs, exports, and destructive reset all sit in one secondary system rather than competing sidebars.

## Skill material used
Skill: awesome-design-md. Source material: awesome-design-md README: Vercel description plus Vercel DESIGN.md monochrome precision, Geist-like type, and hairline controls. I used the collection as reading material only, translating its descriptions into tokens, density, surface rules, and component behavior without copying brand assets.

## Flow model
Single command strip and deployment-style activity rail; manager content swaps in the lower operations panel. The prototype visibly covers running session, booting/starting, and launcher/library/manager states. It includes saved Playgrounds, all six start routes, a searchable and filterable blueprint gallery with at least 16 cards, Settings, Files, Blueprint, Database, Logs, Export to GitHub PR, Download .zip, Import .zip, and typed Apply+Reset confirmation with auto-snapshot language.

## Trade-offs
The design favors visible coverage over full production behavior. The iframe is real and the local controls update state, but the WordPress internals are mock preview content. Some long data surfaces are compressed to keep the iframe dominant and avoid the two-row persistent top bar rejected in v7.
