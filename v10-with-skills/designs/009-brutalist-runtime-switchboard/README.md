# Brutalist Runtime Switchboard

## Thesis

This prototype turns WordPress Playground into a mechanical runtime switchboard. It rejects soft SaaS cards in favor of hard grid lines, command strips, oversized type, small monospace telemetry, and one red accent. The iframe is the main bay. The secondary surface is a dense utility grid that can show launcher, manager, or state panels.

## Skill material used

Used `taste-skill/SKILL.md` for anti-slop rules, single accent discipline, tactile states, loading/error/empty states, and a stronger asymmetric layout. Used `brutalist-skill/SKILL.md` for Swiss industrial print, sharp 90-degree geometry, visible compartmentalization, command strips, dense telemetry, and typed reset. Used `redesign-skill/SKILL.md` to avoid dashboard defaults and duplicate entry points.

## Flow model

The switchboard preserves v7 lessons by keeping one toolbar strip, one utility surface, explicit focus mode, and no two-row chrome. Start routes and saved playgrounds belong to Launch. Settings, Files, Blueprint, Database, Logs, WP Admin, Homepage, Export to GitHub PR, and Download .zip belong to Manager. Runtime state owns booting, running, loading, empty, and error states.

## Trade-offs

The brutalist density makes every feature visible and inspectable, but it is intentionally less comfortable than the soft variants. Mobile keeps the same command order in a single column and sacrifices some visual drama to prevent horizontal scrolling at about 390x844.
