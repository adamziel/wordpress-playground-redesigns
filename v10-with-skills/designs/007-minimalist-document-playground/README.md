# Minimalist Document Playground

## Thesis

This design makes Playground feel like a working technical document with a live WordPress page embedded in the center. The secondary surface is a document index and inspector, not a dashboard sidebar. It keeps the iframe primary while showing launch routes, saved playgrounds, settings, files, blueprint JSON, database tables, logs, exports, and typed reset as inline document blocks.

## Skill material used

Used `taste-skill/SKILL.md` for single accent restraint, tactile states, loading/error/empty states, and avoidance of generic AI UI. Used `minimalist-skill/SKILL.md` for warm monochrome, editorial type contrast, flat structure, crisp 1px dividers, and document-like spacing. Used `redesign-skill/SKILL.md` to remove duplicate feature doors and keep the UI from becoming a pile of cards.

## Flow model

The v7 iframe primacy lesson is preserved by keeping the live site in the widest column. The one secondary surface is the document index: Launch, Manager, and State anchors swap the inline document content. Start routes and saved playgrounds share the Launch section, while Site Manager owns Settings, Files, Blueprint, Database, Logs, WP Admin, Homepage, Export to GitHub PR, and Download .zip.

## Trade-offs

The document metaphor is quiet and easy to scan, but less immediately theatrical than other Taste variants. It works best for developers authoring blueprints and reading logs. Mobile keeps the same order as desktop: toolbar, iframe, then the document inspector.
