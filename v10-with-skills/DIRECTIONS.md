# v10 With Skills Directions

Designs 001 to 005 use `awesome-design-md`.

1. **DESIGN.md Console Atlas**: Treat the interface as a formal design-system specimen. Use an explicit token sheet feel inspired by developer-brand DESIGN.md sources, with iframe first and a compact specimen drawer for Playground tools.
2. **Vercel Precision Runner**: Monochrome precision, Geist-like type, tight command surface, and a single deployment-style activity rail for launches, saves, exports, and logs.
3. **Linear Issue Board Recast**: Use Linear-like calm, dense issue workflow: saved Playgrounds are tasks, the active iframe is the selected issue, and the manager is a right-side property inspector.
4. **Miro Canvas for Sites**: Saved Playgrounds live on a spatial canvas with yellow-accented collaboration cues, but focusing a site expands the iframe into the dominant object.
5. **IBM Carbon Lab Bench**: Structured enterprise blue system, clear data tables, accessible tab logic, and a lab-bench manager with strong states and no decorative excess.

Designs 006 to 010 use `taste-skill`.

6. **High-Variance Focus Split**: Taste baseline with asymmetry, a single accent, tactile controls, and smart resize between iframe and a tasteful command studio.
7. **Minimalist Document Playground**: Use the minimalist Taste variant: document-like workspace, warm monochrome, editorial type contrast, inline blocks for settings and blueprint data.
8. **Soft Agency Control Room**: Use the soft high-end variant with double-bezel surfaces, fluid island controls, generous spacing, and a premium but restrained product feel.
9. **Brutalist Runtime Switchboard**: Use the Taste anti-slop/brutalist direction: sharp mechanical layout, no default cards, command strips, typed reset, and developer-grade density.
10. **Motion Bento Without Marketing**: Use Taste motion rules for subtle state transitions and alive panels, but keep the product surface operational instead of turning it into a landing page.

Designs 011 to 015 use `ui-ux-pro-max-skill`.

11. **Developer IDE OLED**: Use UI Pro Max Developer Tool / IDE recommendations: dark syntax palette, green run accent, real-time monitor plus minimal IDE structure.
12. **Interactive Product Demo Shell**: Use UI Pro Max Interactive Product Demo style: step progression, clear demo states, launcher-to-running continuity, and touch-friendly controls.
13. **Spatial UI Manager**: Use UI Pro Max Spatial UI guidance for depth and focus, with frosted windows and explicit contrast safeguards.
14. **Accessible Ethical Playground**: Use UI Pro Max accessibility-first rules: high contrast, visible focus, large touch targets, labels, clear errors, and predictable navigation.
15. **Data-Dense Monitor**: Use UI Pro Max dashboard/real-time monitor guidance: logs, settings, database, and exports live in dense tables while iframe still owns the screen.
